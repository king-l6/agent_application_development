# AI 工程术语表

## A

### Agent
- **人们说：** "能自主思考和行动的 AI"
- **实际含义：** 就是一个 while 循环——LLM 决定调用哪个工具，执行它，观察结果，然后重复这个过程
- **为什么这么叫：** 借用了哲学概念——"agent" 指任何能在世界中行动的事物。在 AI 里，就是 "LLM + 工具 + 循环"

### Attention
- **人们说：** "AI 如何关注重要部分"
- **实际含义：** 一种机制，每个 token 计算所有其他 token 值的加权和，权重由它们之间的相关性决定（通过 query 和 key 向量的点积计算）
- **为什么这么叫：** 2017 年的论文《Attention Is All You Need》类比人类的选择性注意力来命名

### Alignment
- **人们说：** "让 AI 变得安全"
- **实际含义：** 让 AI 系统的行为符合人类意图、价值观和偏好的技术挑战，包括设计者未曾预料到的边缘情况

### Autoregressive
- **人们说：** "AI 一次生成一个词"
- **实际含义：** 一种模型，根据之前所有的 token 预测下一个 token，然后将预测结果作为输入反馈给下一步。GPT、LLaMA 和 Claude 都是自回归模型

### Activation Function
- **人们说：** "层之间的非线性东西"
- **实际含义：** 在每个线性层之后应用的函数，用于引入非线性。没有它，堆叠任意多个线性层都会坍缩为单个线性变换。ReLU、GELU 和 SiLU 是最常见的。选择直接影响训练时梯度是否能流动

### Adam (Optimizer)
- **人们说：** "默认优化器"
- **实际含义：** 自适应矩估计（Adaptive Moment Estimation）。结合了动量（一阶矩）和每个参数的自适应学习率（二阶矩）。对早期步骤有偏差修正。在大多数任务上无需太多调参就能表现良好

### AdamW
- **人们说：** "更好用的 Adam"
- **实际含义：** 带解耦权重衰减的 Adam。在标准 Adam 中，L2 正则化会被每个参数的自适应学习率缩放，这不是我们想要的。AdamW 直接将权重衰减应用到权重上，独立于梯度统计。训练 Transformer 的默认优化器

### Autograd
- **人们说：** "自动求梯度"
- **实际含义：** 一种记录张量运算并通过反向模式微分自动计算梯度的系统。PyTorch 的 autograd 动态构建计算图（动态图），而 JAX 使用函数变换（grad）。这就是反向传播变得实用的原因——你只需编写前向传播，框架会自动计算所有导数

## B

### Batch Size
- **人们说：** "一次处理多少样本"
- **实际含义：** 在更新权重之前，一次前向/反向传播中处理的训练样本数量。更大的批次给出更稳定的梯度估计，但消耗更多内存。典型值：训练时 32-512，推理时更大。批次大小与学习率相互影响——批次翻倍，学习率也应翻倍（线性缩放规则）

### Backpropagation
- **人们说：** "神经网络如何学习"
- **实际含义：** 一种算法，通过链式法则从输出层向后计算每个权重对误差的贡献，然后按比例调整权重
- **为什么这么叫：** 误差从输出层逐层向后传播到输入层

## C

### Context Window
- **人们说：** "AI 能记住多少内容"
- **实际含义：** 单次 API 调用能容纳的最大 token 数（输入 + 输出）。不是记忆——它是一个固定大小的缓冲区，每次调用都会重置

### Chain of Thought (CoT)
- **人们说：** "让 AI 一步一步思考"
- **实际含义：** 一种提示技巧，要求模型展示推理步骤。这能提升多步问题的准确率，因为每一步都为下一步的 token 生成提供了条件

### CNN (Convolutional Neural Network)
- **人们说：** "图像 AI"
- **实际含义：** 使用卷积操作（滑动滤波器扫描输入）来检测局部模式的神经网络。堆叠卷积层可以检测越来越复杂的特征：边缘、纹理、物体

### CUDA
- **人们说：** "GPU 编程"
- **实际含义：** NVIDIA 的并行计算平台。让你能在数千个 GPU 核心上同时运行矩阵运算。PyTorch 和 TensorFlow 底层都使用 CUDA

### Chunking
- **人们说：** "把文档切成小块"
- **实际含义：** 在嵌入检索之前将文本分成片段。块大小决定了搜索结果的粒度。太小：丢失上下文。太大：稀释相关性。常用策略：固定大小加重叠、基于句子、或语义分割。典型块大小：256-512 token，10-20% 重叠

### Contrastive Learning
- **人们说：** "通过比较来学习"
- **实际含义：** 训练时把相似的样本对拉近，把不相似的样本对推远（在嵌入空间中）。CLIP 就使用这种方法：匹配图像-文本对 vs 不匹配的图像-文本对

### Cosine Similarity
- **人们说：** "两个向量有多相似"
- **实际含义：** 两个向量夹角的余弦值：dot(a, b) / (||a|| * ||b||)。范围从 -1（相反方向）到 1（完全相同方向）。忽略大小，只关心方向。嵌入和语义搜索的标准相似度指标

### Cross-Entropy
- **人们说：** "分类损失"
- **实际含义：** 衡量两个概率分布之间的差异。对于分类：-sum(y_true * log(y_pred))。对于语言模型：正确下一个 token 的负对数概率。越低越好。困惑度就是 exp(交叉熵)

## D

### Data Augmentation
- **人们说：** "制造更多训练数据"
- **实际含义：** 对现有数据创建修改后的副本（旋转图像、添加噪声、改写文本），在不收集新数据的情况下增加训练集的多样性。减少过拟合

### Decoder
- **人们说：** "输出部分"
- **实际含义：** 在 Transformer 中，解码器使用因果（掩码）自注意力，使得每个位置只能关注到前面的位置。GPT 是纯解码器。BERT 是纯编码器。T5 是编码器-解码器

### Diffusion Model
- **人们说：** "从噪声生成图像的 AI"
- **实际含义：** 一种被训练来逆转逐步加噪过程的模型——它学习预测并去除噪声，在生成时从纯噪声开始迭代去噪

### DPO (Direct Preference Optimization)
- **人们说：** "更简单的 RLHF"
- **实际含义：** 一种跳过奖励模型的训练方法——它直接优化语言模型，使其在人类偏好对中倾向于更好的回答

### Dropout
- **人们说：** "随机关闭神经元"
- **实际含义：** 在训练期间，随机将一部分激活值设为零。迫使网络不依赖任何单个神经元。推理时关闭。简单但有效的正则化方法

## E

### Eigenvalue
- **人们说：** "PCA 用的数学概念"
- **实际含义：** 对于矩阵 A，特征值 lambda 满足 Av = lambda*v（v 是某个向量）。它告诉你矩阵在对应方向上缩放向量的程度。大的特征值 = 数据中方差大的方向

### Embedding
- **人们说：** "把词变成数字的 AI 魔法"
- **实际含义：** 一种学习到的映射，将离散项（词、图像、用户）映射到连续空间中的稠密向量，相似的项在空间中彼此靠近
- **为什么这么叫：** 这些项被"嵌入"到一个距离有意义的几何空间中

### Encoder
- **人们说：** "输入部分"
- **实际含义：** 在 Transformer 中，编码器使用双向自注意力，每个位置可以关注所有位置。BERT 是纯编码器。适合理解型任务（分类、命名实体识别），但不适合生成

### Epoch
- **人们说：** "遍历一遍数据"
- **实际含义：** 就是一次完整地遍历训练集中的每个样本。多个 epoch = 多次看到数据。更多 epoch 可以提高学习效果，但也有过拟合的风险

## F

### Feature
- **人们说：** "数据中的一列"
- **实际含义：** 数据的一个可测量的属性。在经典机器学习中，你需要手动设计特征。在深度学习中，网络会自动从原始数据中学习特征

### Few-Shot
- **人们说：** "先给 AI 一些例子"
- **实际含义：** 在要求模型执行任务之前，在提示中放入少量输入-输出示例。通常是 3-5 个例子。模型通过模式匹配这些例子来理解期望的格式和行为。与 zero-shot（无示例）和 fine-tuning（成千上万个示例融入权重）形成对比

### Fine-tuning
- **人们说：** "用你的数据训练 AI"
- **实际含义：** 以预训练模型的权重为起点，在更小、任务特定的数据集上继续训练。只更新现有权重，不会从零开始添加新知识

### Function Calling
- **人们说：** "能使用工具的 AI"
- **实际含义：** 一种结构化方式，让 LLM 请求执行外部函数。你用 JSON Schema 描述定义工具，模型输出结构化的 JSON 对象指定要调用哪个函数及参数，你的代码执行它，结果返回给模型。与 agent 不同——function calling 是机制，agent 是循环

## G

### Guardrails
- **人们说：** "AI 的安全过滤器"
- **实际含义：** LLM 周围的输入/输出验证层，用于检测和阻止有害内容、提示注入攻击、PII 泄露或离题回答。通常是管道形式：输入过滤器 -> LLM -> 输出过滤器。可以是基于规则的（正则、关键词列表）或基于模型的（安全评分分类器）

### GPT
- **人们说：** "ChatGPT" 或 "那个 AI"
- **实际含义：** 生成式预训练 Transformer（Generative Pre-trained Transformer）——一种特定的架构，使用纯解码器 Transformer，在大型文本语料库上训练，预测下一个 token
- **为什么这么叫：** Generative（生成文本）、Pre-trained（在大数据上预先训练一次，之后适配）、Transformer（架构）

### GAN (Generative Adversarial Network)
- **人们说：** "两个 AI 互相打架"
- **实际含义：** 生成器网络试图创建逼真的数据，判别器网络试图区分真假。它们一起训练：生成器越来越擅长欺骗判别器，判别器越来越擅长检测伪造品

### Gradient
- **人们说：** "斜率"
- **实际含义：** 一个偏导数向量，指向最陡峭上升的方向。在机器学习中，你朝梯度的反方向走（梯度下降）来最小化损失

### Gradient Descent
- **人们说：** "AI 如何进步"
- **实际含义：** 一种优化算法，沿着使损失函数下降最陡的方向调整参数，就像在高维景观中走下山坡

## H

### Hyperparameter
- **人们说：** "你要调的设置"
- **实际含义：** 在训练之前设置的数值，控制训练过程本身：学习率、批次大小、层数、丢弃率。与模型参数（权重）不同，这些不是从数据中学到的

### Hallucination
- **人们说：** "AI 在撒谎" 或 "在编造"
- **实际含义：** 模型生成听起来合理但没有基于其训练数据或给定上下文的文本——它只是在补全模式，不是在检索事实

## I

### Inference
- **人们说：** "运行 AI"
- **实际含义：** 使用训练好的模型对新数据进行预测。不涉及权重更新。这就是你在生产环境中所做的：发送输入，得到输出

### Inductive Bias
- **人们说：** 没听说过
- **实际含义：** 内置于模型架构中的假设。CNN 假设局部模式重要（卷积）。RNN 假设顺序重要（序列处理）。Transformer 假设一切可能和一切相关（注意力）。正确的归纳偏置帮助模型用更少的数据学得更快

### JAX
- **人们说：** "谷歌的机器学习框架"
- **实际含义：** 一个兼容 NumPy 的库，增加了自动微分（grad）、即时编译（jit）、自动向量化（vmap）和多设备并行（pmap）。与 PyTorch 的面向对象风格不同，JAX 是纯函数式的——没有隐藏状态，没有就地修改。Google DeepMind 用它开发 AlphaFold、Gemini 和大规模研究

## K

### KV Cache
- **人们说：** "让推理更快"
- **实际含义：** 在自回归生成过程中，缓存之前 token 的 key 和 value 矩阵，这样每一步就不需要重新计算它们。用内存换速度。对快速 LLM 推理至关重要

## L

### Latent Space
- **人们说：** "隐藏表示"
- **实际含义：** 一种压缩的、学习到的表示空间，相似的输入映射到相近的点。自编码器、VAE 和扩散模型都在潜空间中工作。它比输入维度更低，但捕捉了重要的结构

### Learning Rate
- **人们说：** "AI 学习的速度"
- **实际含义：** 控制梯度下降步长的标量。太大：越过最小值，发散。太小：收敛太慢或陷入局部最优。最重要的超参数，没有之一

### LLM (Large Language Model)
- **人们说：** "AI" 或 "大脑"
- **实际含义：** 基于 Transformer 的神经网络，训练目标是预测序列中的下一个 token，拥有数十亿参数，在互联网规模的文本数据上训练

### LoRA (Low-Rank Adaptation)
- **人们说：** "高效的微调"
- **实际含义：** 不是更新所有权重，而是在原始权重旁边插入小型低秩矩阵。只训练这些小矩阵，内存减少 10-100 倍

### Loss Function
- **人们说：** "AI 错得有多离谱"
- **实际含义：** 衡量预测输出与实际输出差距的函数。训练就是最小化这个函数。均方误差用于回归，交叉熵用于分类，对比损失用于嵌入。损失函数的选择定义了模型眼中"好"的标准

## M

### Mixed Precision
- **人们说：** "训练加速技巧"
- **实际含义：** 前向传播和大多数操作使用 float16（更快、更省内存），但梯度累积和权重更新保留 float32（更精确）。获得约 2 倍加速，精度损失可忽略不计

### MoE (Mixture of Experts)
- **人们说：** "只有部分模型在运行"
- **实际含义：** 一种包含许多"专家"子网络的模型，通过路由机制将每个输入只发送给少数几个专家。完整模型很大，但每次前向传播很便宜，因为大多数专家被跳过了。Mixtral 和 GPT-4 使用这种架构

### MCP (Model Context Protocol)
- **人们说：** "AI 使用工具的一种方式"
- **实际含义：** 一个开放协议（基于 stdio/HTTP 的 JSON-RPC），标准化了 AI 应用连接外部数据源和工具的方式，包含工具、资源和提示的类型化模式定义

## N

### NaN (Not a Number)
- **人们说：** "训练崩溃了"
- **实际含义：** 表示未定义结果的浮点值（0/0，无穷大减无穷大）。在训练中，NaN 损失通常意味着：学习率太高、梯度爆炸、对零取对数、或除以零。训练失败时首先要检查的东西

### Normalization
- **人们说：** "缩放数据"
- **实际含义：** 将数值调整到标准范围。批归一化跨批次归一化。层归一化跨特征归一化。两者都能稳定训练并允许更高的学习率

## O

### Overfitting
- **人们说：** "模型记住了数据"
- **实际含义：** 模型在训练数据上表现良好但在未见过的数据上表现差。它学到了噪声，不是信号。解决方法：更多数据、正则化（丢弃法、权重衰减）、早停、数据增强、更简单的模型

### Optimizer
- **人们说：** "更新权重的东西"
- **实际含义：** 使用梯度更新模型参数的算法。SGD 最简单。Adam 最常见。每个优化器有不同的特性：收敛速度、内存使用、对超参数的敏感性

## P

### Parameter
- **人们说：** "模型大小"
- **实际含义：** 模型中的可学习值，通常是权重或偏置。"70 亿参数"意味着 70 亿个可学习的数字。每个 float32 参数占 4 字节，所以 70 亿参数 = 仅权重就需要 28GB 内存

### Perplexity
- **人们说：** "模型有多困惑"
- **实际含义：** 平均交叉熵损失的指数值。越低越好。困惑度为 10 意味着模型在每个步骤的确定性相当于从 10 个 token 中均匀随机选择

### Precision & Recall
- **人们说：** "准确率指标"
- **实际含义：** 精确率 = 你标记的项目中，有多少是正确的。召回率 = 所有正确项目中，你找到了多少。它们互相权衡：抓住每封垃圾邮件（高召回率）意味着更多误报（低精确率）。F1 分数是它们的调和平均。假阳性代价高时用精确率，假阴性代价高时用召回率

### Prompt Engineering
- **人们说：** "用正确的方式和 AI 对话"
- **实际含义：** 设计输入文本以可靠地产生期望的输出——包括系统提示、少样本示例、格式指令和思维链触发词

### Prompt Injection
- **人们说：** "用文字黑掉 AI"
- **实际含义：** 一种攻击方式，输入中的恶意文本覆盖了系统提示或指令。直接注入：用户输入"忽略之前的指示"。间接注入：检索到的文档包含隐藏指令。这是 LLM 版的 SQL 注入。目前没有完整的解决方案——防御靠多层输入验证、输出过滤和权限分离

## Q

### QLoRA
- **人们说：** "比 LoRA 更便宜"
- **实际含义：** 量化 LoRA（Quantized LoRA）。冻结的基础模型权重保持 4-bit 精度（NF4 格式），同时用 16-bit 训练 LoRA 适配器。相比标准 LoRA，内存再减少 3-4 倍。需要 14GB 的 7B 模型用 LoRA 后只需 4-6GB。质量在大多数基准测试上接近完整微调的 99%

## R

### RAG (Retrieval-Augmented Generation)
- **人们说：** "能搜索的 AI"
- **实际含义：** 一种模式：从知识库中检索相关文档（使用嵌入相似度），将它们塞入提示中，让 LLM 基于这些上下文来回答
- **为什么这么叫：** 检索（Retrieval）+ 增强（Augmented）+ 生成（Generation）

### RLHF (Reinforcement Learning from Human Feedback)
- **人们说：** "他们如何让 AI 变得有用"
- **实际含义：** 一个训练流水线：(1) 收集人类对模型输出的偏好，(2) 基于这些偏好训练一个奖励模型，(3) 使用 PPO 优化 LLM 以产生更高奖励的输出

### Quantization
- **人们说：** "让模型变小"
- **实际含义：** 将模型权重从 float32（4 字节）降低到 int8（1 字节）或 int4（0.5 字节）。用少量精度换取 4-8 倍的内存减少和更快的推理。GPTQ、AWQ 和 GGUF 是常见格式

### ReLU
- **人们说：** "激活函数"
- **实际含义：** 修正线性单元（Rectified Linear Unit）：f(x) = max(0, x)。最简单的非线性激活函数。计算快，正值不会饱和。到处都在用，因为它有效且便宜。变体：LeakyReLU、GELU、SiLU

### ROUGE
- **人们说：** "摘要评估指标"
- **实际含义：** 面向召回率的摘要评估替补（Recall-Oriented Understudy for Gisting Evaluation）。衡量生成文本和参考文本之间的重叠。ROUGE-1 统计单字匹配，ROUGE-2 统计双字匹配，ROUGE-L 找到最长公共子序列。计算成本低但只衡量表面相似性——两个意思相同但用词不同的句子得分会很低

## S

### Semantic Search
- **人们说：** "理解含义的智能搜索"
- **实际含义：** 通过含义而不是关键词匹配来查找文档。将查询和所有文档嵌入到同一个向量空间中，然后返回嵌入与查询嵌入最接近的文档。"支付失败"能找到"交易被拒绝"，即使它们没有共享任何词。由嵌入模型 + 向量数据库驱动

### Streaming
- **人们说：** "看到回答逐字出现"
- **实际含义：** LLM 在生成 token 时逐个发送，而不是等待完整响应。使用服务器发送事件（SSE）或 WebSocket 协议。将首个 token 的感知延迟从秒级降到毫秒级。对生产环境的聊天界面至关重要。每个块包含一个增量（部分 token 或词）

### Self-Attention
- **人们说：** "模型如何决定关注什么"
- **实际含义：** 每个 token 计算 query、key 和 value 向量。两个 token 之间的注意力权重 = 它们的 query 和 key 的点积，经过缩放和 softmax。输出 = value 向量的加权和。让每个 token 都能看到其他所有 token

### SFT (Supervised Fine-Tuning)
- **人们说：** "教模型遵循指令"
- **实际含义：** 在（指令，回答）对上微调预训练模型。模型学习根据指令生成回答。这就是把基础模型变成聊天模型的方法

### Softmax
- **人们说：** "把数字变成概率"
- **实际含义：** softmax(x_i) = exp(x_i) / sum(exp(x_j))。将任意实数的向量转换为概率分布（全部为正，总和为 1）。用于分类头、注意力权重，以及任何需要概率的地方

### Swarm
- **人们说：** "一群像蜜蜂一样协作的 AI agent"
- **实际含义：** 多个 agent 通过消息传递共享状态和协调，涌现行为来自简单的个体规则而非中央控制

## T

### System Prompt
- **人们说：** "AI 的指令"
- **实际含义：** 对话开始时的一条特殊消息，用于设定模型的行为、角色和约束。在用户消息之前处理。在大多数 UI 中用户不可见。定义模型能做什么和不能做什么、语气、格式偏好和领域焦点。与用户提示不同——系统提示由开发者设置

### Tensor
- **人们说：** "多维数组"
- **实际含义：** 深度学习框架中的基础数据结构。0 维张量是标量，1 维是向量，2 维是矩阵，3 维及以上是张量。在 PyTorch 和 JAX 中，张量跟踪其计算历史以进行自动微分，可以在 CPU 或 GPU 上运行。所有神经网络的输入、输出、权重和梯度都是张量

### Token
- **人们说：** "一个词"
- **实际含义：** 由分词器（如 BPE）生成的子词单元（英语中通常 3-4 个字符）。"unbelievable" 可能被分成 3 个 token："un" + "believ" + "able"

### Temperature
- **人们说：** "创造力设置"
- **实际含义：** 在 softmax 之前对 logits 进行缩放的标量。Temperature=1 是默认值。越高 = 分布越平坦 = 输出越随机。越低 = 分布越尖锐 = 输出越确定。Temperature=0 就是 argmax（总是选择最可能的 token）

### Transfer Learning
- **人们说：** "使用预训练模型"
- **实际含义：** 将一个任务上训练好的模型适配到不同任务上。早期的层学习通用特征（边缘、句法模式），这些特征可以迁移。只有后面的层需要任务特定的训练。这就是为什么你可以为任何 NLP 任务微调 BERT

### Transformer
- **人们说：** "现代 AI 背后的架构"
- **实际含义：** 一种使用自注意力处理序列的神经网络架构（让每个位置都能关注到其他所有位置），而不是使用循环，从而实现大规模并行化
- **为什么这么叫：** 它通过注意力层将输入表示转换为输出表示

## U

### Underfitting
- **人们说：** "模型没学到东西"
- **实际含义：** 模型太简单，无法捕捉数据中的模式。训练损失居高不下。解决方法：更多参数、更多层、更久训练、更少正则化、更好的特征

## V

### VAE (Variational Autoencoder)
- **人们说：** "一种生成模型"
- **实际含义：** 一种自编码器，通过强制编码器输出服从高斯分布来学习平滑的潜空间。你可以从这个分布中采样并解码生成新数据。重参数化技巧使其可以通过反向传播训练

### Vector Database
- **人们说：** "AI 专用的特殊数据库"
- **实际含义：** 一种优化用于存储向量（浮点数的稠密数组）并执行快速近似最近邻搜索的数据库。相似度搜索、RAG 和推荐系统的核心操作

## W

### Weight
- **人们说：** "模型学到的东西"
- **实际含义：** 模型参数矩阵中的一个数字。输入大小 768、输出大小 3072 的线性层有 768*3072 = 2,359,296 个权重。训练调整每个权重以最小化损失函数

### Weight Decay
- **人们说：** "正则化"
- **实际含义：** 在损失函数中加入与权重大小成正比的惩罚项。等同于 L2 正则化。防止权重变得过大。典型值：0.01-0.1

## Z

### Zero-Shot
- **人们说：** "不需要训练"
- **实际含义：** 在模型没有专门训练过的任务上使用它，提示中也没有任务特定的示例。模型从预训练中泛化。之所以有效，是因为大模型见过的数据足够多，可以处理新的任务格式
