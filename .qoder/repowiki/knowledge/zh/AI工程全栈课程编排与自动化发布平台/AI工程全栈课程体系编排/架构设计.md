- **线性依赖拓扑**：课程遵循严格的先修关系（如数学基础→深度学习→Transformer→LLM），通过 `ROADMAP.md` 和 Mermaid 流程图定义全局学习依赖。
- **标准化单元契约**：所有503个课程单元强制采用统一的目录结构（`code/`, `docs/`, `outputs/`）和六步教学法（Motto → Problem → Concept → Build It → Use It → Ship It），确保跨模块知识迁移的一致性。
- **双轨实现范式**：核心架构要求在每个技术节点同时提供“从零实现”（纯Python/NumPy/Math）与“框架应用”（PyTorch/Sklearn）两套代码，以打通底层原理与工程应用的认知隔阂。
- ** artifacts 驱动的输出**：每个单元必须产出可复用的工程制品（Prompt模板、Agent Skills、MCP服务器或独立Agent），并通过统一的 `scripts/install_skills.py` 脚本注入到主流AI编程助手环境中。
