- **分层结构**：模块按“基础概念-核心协议-高级特性-安全与部署”逻辑递进，从通用的四步工具循环（描述-决策-执行-观察）延伸至标准化的 MCP 和 A2A 协议。
- **核心抽象**：以 `Tool`（名称/Schema/执行器）为基本单元，通过 MCP 的六原语（Tools/Resources/Prompts + Roots/Sampling/Elicitation）和 A2A 的 Task/Artifact 模型实现跨框架互操作。
- **传输层设计**：支持本地 stdio（JSON-RPC 换行分隔）与远程 Streamable HTTP（单端点 POST/GET/SSE），并通过 `Mcp-Session-Id` 和 Origin 校验保障会话安全。
- **证据文件**：`01-the-tool-interface/code/main.py` 展示了无 LLM 依赖的工具循环骨架；`06-mcp-fundamentals/docs/en.md` 定义了 MCP 生命周期与能力协商机制；`19-a2a-protocol/docs/en.md` 阐述了 Agent Card 发现与不透明任务委托模型。
