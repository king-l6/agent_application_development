# 范数与距离

> 你的距离函数定义了"相似"的含义。选错了，下游的一切都会崩溃。

**类型：** 构建
**语言：** Python
**前置条件：** 阶段 1，课程 01（线性代数直觉）、02（向量、矩阵与运算）
**时间：** ~90 分钟

## 学习目标

- 从头实现 L1、L2、余弦、马氏、Jaccard 和编辑距离函数
- 为给定的 ML 任务选择合适的距离度量，并解释为什么替代方案会失败
- 将 L1 和 L2 范数与 LASSO 和 Ridge 正则化及其几何约束区域联系起来
- 展示同一数据集在不同度量下如何产生不同的最近邻

## 问题

你有两个向量。也许是词嵌入，也许是用户画像，也许是像素数组。你需要知道：它们有多接近？

答案完全取决于你选择哪个距离函数。两个数据点在一个度量下可能是最近邻，在另一个度量下可能相距甚远。你的 KNN 分类器、推荐引擎、向量数据库、聚类算法、损失函数——它们都依赖于这个选择。选错了，你的模型就在为错误的目标优化。

没有通用的最佳距离。L2 适用于空间数据。余弦相似度主导 NLP。Jaccard 处理集合。编辑距离处理字符串。马氏距离考虑相关性。Wasserstein 移动概率质量。每一种都编码了对"相似"含义的不同假设。

本课程从头构建每一个主要的距离函数，展示每种方法何时是合适的工具，并演示相同的数据如何根据你使用的度量产生完全不同的最近邻。

## 概念

### 范数：衡量向量的大小

范数衡量向量的"大小"。两个向量之间的每个距离函数都可以写成它们差值的范数：d(a, b) = ||a - b||。所以理解范数就是理解距离。

### L1 范数（曼哈顿距离）

L1 范数是所有分量绝对值的和。

```
||x||_1 = |x_1| + |x_2| + ... + |x_n|
```

它被称为曼哈顿距离，因为它衡量的是你在城市网格中只能沿坐标轴移动时走过的距离。没有对角线。

```
点 A = (1, 1)
点 B = (4, 5)

L1 距离 = |4-1| + |5-1| = 3 + 4 = 7

在网格上，你向东走 3 个街区，向北走 4 个街区。
```

何时使用 L1：
- 高维稀疏数据（文本特征、独热编码）
- 当你需要对异常值具有鲁棒性时（单个巨大差异不会主导结果）
- 特征选择问题（L1 正则化促进稀疏性）

与 L1 正则化（Lasso）的联系：将 ||w||_1 添加到损失函数中会惩罚权重绝对值之和。这会将小权重推到恰好为零，实现自动特征选择。L1 惩罚在权重空间中创建菱形约束区域，而菱形的角落在坐标轴上，其中某些权重为零。

与损失函数的联系：平均绝对误差（MAE）是预测值与目标值之间的平均 L1 距离。它对所有误差进行线性惩罚，使其相比 MSE 对异常值更加鲁棒。

### L2 范数（欧氏距离）

L2 范数是直线距离。分量平方和的平方根。

```
||x||_2 = sqrt(x_1^2 + x_2^2 + ... + x_n^2)
```

这就是你在几何课上学到的距离。n 维空间中的勾股定理。

```
点 A = (1, 1)
点 B = (4, 5)

L2 距离 = sqrt((4-1)^2 + (5-1)^2) = sqrt(9 + 16) = sqrt(25) = 5.0

直线，斜穿过网格。
```

何时使用 L2：
- 低到中等维度的连续数据
- 当特征尺度可比时
- 物理距离（空间数据、传感器读数）
- 像素级别的图像相似度

与 L2 正则化（Ridge）的联系：将 ||w||_2^2 添加到损失函数中会惩罚大权重。与 L1 不同，它不会将权重推向零。它会按比例将所有权重向零收缩。L2 惩罚创建圆形约束区域，因此在坐标轴上没有角落。权重变得很小，但很少恰好为零。

与损失函数的联系：均方误差（MSE）是 L2 距离平方的平均值。平方对大误差的惩罚比小误差更重。

```
MAE（L1 损失）：  |y - y_hat|         线性惩罚。对异常值鲁棒。
MSE（L2 损失）：  (y - y_hat)^2       二次惩罚。对异常值敏感。
```

### Lp 范数：通用的家族

L1 和 L2 是 Lp 范数的特例：

```
||x||_p = (|x_1|^p + |x_2|^p + ... + |x_n|^p)^(1/p)
```

不同的 p 值产生不同形状的"单位球"（所有到原点距离为 1 的点的集合）：

```
p=1:    菱形               （角在坐标轴上）
p=2:    圆形/球体           （通常的圆球）
p=3:    超椭圆             （圆角方形）
p=inf:  方形/超立方体       （沿坐标轴的平边）
```

### L-无穷范数（切比雪夫距离）

当 p 趋近于无穷时，Lp 范数收敛到最大绝对值分量。

```
||x||_inf = max(|x_1|, |x_2|, ..., |x_n|)
```

两点之间的距离由它们差异最大的单一维度决定。所有其他维度都被忽略。

```
点 A = (1, 1)
点 B = (4, 5)

L-inf 距离 = max(|4-1|, |5-1|) = max(3, 4) = 4
```

何时使用 L-无穷：
- 当任何单一维度上的最坏情况偏差很重要时
- 棋盘（国际象棋中的王以 L-无穷方式移动：向任何方向走一步都算 1）
- 制造公差（每个维度都必须在规格范围内）

### 余弦相似度和余弦距离

余弦相似度衡量两个向量之间的夹角，忽略它们的模长。

```
cos_sim(a, b) = (a . b) / (||a||_2 * ||b||_2)
```

它的范围从 -1（方向相反）到 +1（方向相同）。垂直向量的余弦相似度为 0。

余弦距离将其转换为距离：cosine_distance = 1 - cosine_similarity。范围从 0（方向相同）到 2（方向相反）。

```
a = (1, 0)    b = (1, 1)

cos_sim = (1*1 + 0*1) / (1 * sqrt(2)) = 1/sqrt(2) = 0.707
cos_dist = 1 - 0.707 = 0.293
```

为什么余弦在 NLP 和嵌入中占主导地位：在文本中，文档长度不应该影响相似度。一篇关于猫的文档，如果长度是另一篇关于猫的文档的两倍，仍然应该被认为是"相似的"。余弦相似度忽略模长（长度），只关心方向。两篇词分布相同但长度不同的文档指向相同的方向，余弦相似度为 1.0。

何时使用余弦相似度：
- 文本相似度（TF-IDF 向量、词嵌入、句子嵌入）
- 任何模长是噪声、方向是信号的领域
- 推荐系统（用户偏好向量）
- 嵌入搜索（向量数据库几乎总是使用余弦或点积）

### 点积相似度 vs 余弦相似度

两个向量的点积是：

```
a . b = a_1*b_1 + a_2*b_2 + ... + a_n*b_n
      = ||a|| * ||b|| * cos(夹角)
```

余弦相似度是用两个模长归一化后的点积。当两个向量都已经单位归一化（模长 = 1）时，点积和余弦相似度相同。

```
如果 ||a|| = 1 且 ||b|| = 1：
    a . b = cos(a 和 b 的夹角)
```

它们何时不同：点积包含模长信息。模长较大的向量获得更高的点积分数。这在某些检索系统中很重要，因为你希望"热门"项排名更高。模长充当隐含的质量或重要性信号。

```
a = (3, 0)    b = (1, 0)    c = (0, 1)

dot(a, b) = 3     dot(a, c) = 0
cos(a, b) = 1.0   cos(a, c) = 0.0

两者在方向上一致，但点积还反映了模长。
```

在实践中：
- 当你想要纯粹的方向相似度时使用余弦相似度
- 当模长包含有意义的信息时使用点积
- 许多向量数据库（Pinecone、Weaviate、Qdrant）让你在两者之间选择
- 如果你的嵌入已经 L2 归一化，选择无关紧要

### 马氏距离

欧氏距离平等对待所有维度。但如果你的特征相关或具有不同尺度，L2 会给出误导性结果。

马氏距离考虑了数据的协方差结构。

```
d_M(x, y) = sqrt((x - y)^T * S^(-1) * (x - y))
```

其中 S 是数据的协方差矩阵。

直观理解：马氏距离首先对数据进行去相关和归一化（白化），然后在变换后的空间中计算 L2 距离。如果 S 是单位矩阵（不相关、单位方差特征），马氏距离退化为欧氏距离。

```
示例：身高和体重是相关的。
身高 6'2"、体重 180 磅的人并不异常。
身高 5'0"、体重 180 磅的人则很异常。

欧氏距离可能认为它们离均值同样远。
马氏距离正确地将第二个识别为异常值，
因为它考虑了身高-体重的相关性。
```

何时使用马氏距离：
- 异常值检测（离均值马氏距离大的点是异常值）
- 当特征具有不同尺度和相关性时的分类
- 当你有足够的数据来估计可靠的协方差矩阵时
- 制造业中的质量控制（多变量过程监控）

### Jaccard 相似度（用于集合）

Jaccard 相似度衡量两个集合之间的重叠程度。

```
J(A, B) = |A 交 B| / |A 并 B|
```

范围从 0（无重叠）到 1（相同集合）。Jaccard 距离 = 1 - Jaccard 相似度。

```
A = {猫, 狗, 鱼}
B = {猫, 鸟, 鱼, 蛇}

交集 = {猫, 鱼}         大小 = 2
并集 = {猫, 狗, 鱼, 鸟, 蛇}  大小 = 5

Jaccard 相似度 = 2/5 = 0.4
Jaccard 距离 = 0.6
```

何时使用 Jaccard：
- 比较标签、类别或特征的集合
- 基于词存在性（而非词频）的文档相似度
- 近似重复检测（Jaccard 的 MinHash 近似）
- 比较二值特征向量（存在/缺失数据）
- 评估分割模型（交并比 = Jaccard）

### 编辑距离（莱文斯坦距离）

编辑距离计算将一个字符串转换为另一个字符串所需的最小单字符操作数。操作包括：插入、删除或替换。

```
"kitten" -> "sitting"

kitten -> sitten  （替换 k -> s）
sitten -> sittin  （替换 e -> i）
sittin -> sitting （插入 g）

编辑距离 = 3
```

使用动态规划计算。填充一个矩阵，其中 (i, j) 位置是字符串 A 的前 i 个字符与字符串 B 的前 j 个字符之间的编辑距离。

```
        ""  s  i  t  t  i  n  g
    ""   0  1  2  3  4  5  6  7
    k    1  1  2  3  4  5  6  7
    i    2  2  1  2  3  4  5  6
    t    3  3  2  1  2  3  4  5
    t    4  4  3  2  1  2  3  4
    e    5  5  4  3  2  2  3  4
    n    6  6  5  4  3  3  2  3
```

何时使用编辑距离：
- 拼写检查和纠错
- DNA 序列比对（带加权操作）
- 模糊字符串匹配
- 杂乱文本数据的去重

### KL 散度（不是距离，但被当作距离使用）

KL 散度衡量一个概率分布与另一个概率分布之间的差异。在课程 09 中已经介绍过，但它属于这个讨论范畴，因为人们尽管知道它不是距离，却仍将其用作"距离"。

```
D_KL(P || Q) = sum(p(x) * log(p(x) / q(x)))
```

关键性质：KL 散度不是对称的。

```
D_KL(P || Q) != D_KL(Q || P)
```

这意味着它不满足距离度量的基本要求。它也不满足三角不等式。它是一个散度，而不是一个距离。

前向 KL（D_KL(P || Q)）是"均值寻求"的：Q 试图覆盖 P 的所有模态。
反向 KL（D_KL(Q || P)）是"模式寻求"的：Q 聚焦于 P 的单一模态。

KL 散度的应用场景：
- VAE（ELBO 中的 KL 项将潜变量分布推向先验）
- 知识蒸馏（学生试图匹配教师的分布）
- RLHF（KL 惩罚使微调后的模型保持接近基础模型）
- 策略梯度方法（约束策略更新）

### Wasserstein 距离（推土机距离）

Wasserstein 距离衡量将一个概率分布变换为另一个概率分布所需的最小"功"。可以这样理解：如果一个分布是一堆土，另一个是一个坑，你需要移动多少土以及移动多远？

```
W(P, Q) = 对所有传输方案 gamma 取下确界 E[d(x, y)]
```

对于一维分布，它简化为累积分布函数的绝对差值的积分：

```
W_1(P, Q) = integral |CDF_P(x) - CDF_Q(x)| dx
```

为什么 Wasserstein 重要：
- 它是一个真正的度量（对称，满足三角不等式）
- 即使分布不重叠时也能提供梯度（KL 散度会趋于无穷大）
- 这一性质使其成为 Wasserstein GAN（WGAN）的核心，后者解决了原始 GAN 的训练不稳定性

```
不重叠的分布：

P: [1, 0, 0, 0, 0]    Q: [0, 0, 0, 0, 1]

KL 散度：无穷大（log 中的零）
Wasserstein：4（将所有质量移动 4 个格子）

Wasserstein 提供了有意义的梯度。KL 则不能。
```

何时使用 Wasserstein：
- GAN 训练（WGAN、WGAN-GP）
- 比较可能不重叠的分布
- 最优传输问题
- 图像检索（比较颜色直方图）

### 为什么不同任务需要不同的距离

| 任务 | 最佳距离 | 原因 |
|------|---------|------|
| 文本相似度 | 余弦 | 模长是噪声，方向是含义 |
| 图像像素比较 | L2 | 空间关系重要，特征尺度可比 |
| 稀疏高维特征 | L1 | 鲁棒，不会放大罕见的大差异 |
| 集合重叠（标签、类别） | Jaccard | 数据本质上是集合值，而非向量 |
| 字符串匹配 | 编辑距离 | 操作映射到人类编辑直觉 |
| 异常值检测 | 马氏 | 考虑特征相关性和尺度 |
| 比较分布 | KL 散度 | 衡量使用 Q 代替 P 所损失的信息 |
| GAN 训练 | Wasserstein | 即使分布不重叠也提供梯度 |
| 嵌入（向量数据库） | 余弦或点积 | 嵌入训练将含义编码在方向上 |
| 推荐 | 点积 | 模长可以编码流行度或置信度 |
| DNA 序列 | 加权编辑距离 | 替换代价因核苷酸对而异 |
| 制造质量控制 | L-无穷 | 任何维度的最坏偏差都重要 |

### 与损失函数的联系

损失函数是应用于预测值与目标值的距离函数。

```
损失函数       所使用的距离        行为
MSE            L2 平方             严重惩罚大误差
MAE            L1                  平等惩罚所有误差
Huber 损失     大误差用 L1，        两者优点：对异常值鲁棒，
              小误差用 L2          在零附近梯度平滑
交叉熵         KL 散度             衡量分布不匹配
合页损失       max(0, margin - d)  只惩罚低于边界的值
三元组损失     L2（通常）           拉近正样本，推远负样本
对比损失       L2                  相似对靠近，不相似对超过边界
```

### 与正则化的联系

正则化在损失函数中添加权重的范数惩罚。

```
L1 正则化（Lasso）：   loss + lambda * ||w||_1
  -> 稀疏权重。某些权重变为恰好为零。
  -> 自动特征选择。
  -> 解有角落（在零点不可微）。

L2 正则化（Ridge）：   loss + lambda * ||w||_2^2
  -> 小权重。所有权重向零收缩。
  -> 无特征选择（没有值变为恰好为零）。
  -> 处处光滑可微。

弹性网络：                  loss + lambda_1 * ||w||_1 + lambda_2 * ||w||_2^2
  -> 结合了 L1 的稀疏性和 L2 的稳定性。
  -> 相关特征组一起保留或一起丢弃。
```

为什么 L1 产生稀疏性而 L2 不产生：想象二维权重空间中的约束区域。L1 是菱形，L2 是圆形。损失函数的等高线（椭圆）最有可能在某个角接触菱形，此时一个权重为零。它们与圆形在光滑点接触，此时两个权重都不为零。

### 最近邻搜索

每个距离函数都隐含一个最近邻搜索问题：给定一个查询点，在数据集中找到最近的点。

在有 n 个点、d 维的数据集中，精确最近邻搜索每次查询的复杂度是 O(n * d)。对于大型数据集，这太慢了。

近似最近邻（ANN）算法以微小的精度换取巨大的速度提升：

```
算法            方法                      使用者
KD-树           轴对齐空间划分              scikit-learn（低维）
球树            嵌套超球体                  scikit-learn（中维）
LSH             随机哈希投影                近似重复检测
HNSW            分层可导航小世界图           FAISS、Qdrant、Weaviate
IVF             基于聚类的倒排文件索引       FAISS（十亿级）
乘积量化        压缩向量，在压缩空间中搜索    FAISS（内存受限）
```

HNSW（分层可导航小世界图）是现代向量数据库中的主导算法。它构建了一个多层图，其中每个节点连接到其近似最近邻。搜索从顶层（稀疏，长跳跃）开始，下降到底层（密集，短跳跃）。

## 应用

最常见的实际用途：在向量数据库中查找相似项。

```python
import numpy as np

def cosine_similarity_matrix(X):
    norms = np.linalg.norm(X, axis=1, keepdims=True)
    norms = np.where(norms == 0, 1, norms)
    X_normalized = X / norms
    return X_normalized @ X_normalized.T

embeddings = np.random.randn(1000, 768)

sim_matrix = cosine_similarity_matrix(embeddings)

query_idx = 0
similarities = sim_matrix[query_idx]
top_k = np.argsort(similarities)[::-1][1:6]
print(f"与项 0 最相似的 Top 5：{top_k}")
print(f"相似度：{similarities[top_k]}")
```

当你调用 `model.encode(text)` 然后搜索向量数据库时，底层就是这样运行的。嵌入模型将文本映射到向量。向量数据库计算你的查询向量与每个存储向量之间的余弦相似度（或点积），使用 ANN 算法避免检查所有向量。

## 练习题

1. 计算 (1, 2, 3) 和 (4, 0, 6) 之间的 L1、L2 和 L-无穷距离。验证对于任意一对点，总有 L-inf <= L2 <= L1。证明为什么这种顺序是必然的。

2. 创建两个向量，使余弦相似度很高（> 0.9）但 L2 距离很大（> 10）。从几何角度解释发生了什么。然后创建两个向量，使余弦相似度很低（< 0.3）但 L2 距离很小（< 0.5）。

3. 实现一个函数，接受数据集和查询点，返回在 L1、L2、余弦和马氏距离下的最近邻。找一个数据集，使所有四个度量在哪个点是最近的上存在分歧。

4. 使用 CDF 方法手算 [0.5, 0.5, 0, 0] 和 [0, 0, 0.5, 0.5] 之间的 Wasserstein 距离。然后计算 [0.25, 0.25, 0.25, 0.25] 和 [0, 0, 0.5, 0.5] 之间的 Wasserstein 距离。哪个更大，为什么？

5. 实现用于近似 Jaccard 相似度的 MinHash。生成 100 个随机集合，计算所有对的精确 Jaccard，并与使用 50、100 和 200 个哈希函数的 MinHash 近似进行比较。绘制近似误差。

## 关键术语

| 术语 | 人们说的 | 实际含义 |
|------|---------|---------|
| 范数 | "向量的大小" | 将向量映射到非负标量的函数，满足三角不等式、绝对齐次性，且仅零向量为零 |
| L1 范数 | "曼哈顿距离" | 分量绝对值之和。在优化中产生稀疏性。对异常值鲁棒 |
| L2 范数 | "欧氏距离" | 分量平方和的平方根。欧氏空间中的直线距离 |
| Lp 范数 | "广义范数" | 分量绝对值的 p 次幂之和的 p 次根。L1 和 L2 是特例 |
| L-无穷范数 | "最大范数"或"切比雪夫距离" | 最大绝对值分量。Lp 在 p 趋近无穷时的极限 |
| 余弦相似度 | "向量之间的夹角" | 用两个模长归一化的点积。范围从 -1 到 +1。忽略向量长度 |
| 余弦距离 | "1 减余弦相似度" | 将余弦相似度转换为距离。范围从 0 到 2 |
| 点积 | "未归一化的余弦" | 分量逐项乘积之和。等于余弦相似度乘以两个模长 |
| 马氏距离 | "考虑相关性的距离" | 在使用数据协方差矩阵进行白化（去相关和归一化）的空间中的 L2 距离 |
| Jaccard 相似度 | "集合重叠" | 交集大小除以并集大小。用于集合，而非向量 |
| 编辑距离 | "莱文斯坦距离" | 将一个字符串转换成另一个字符串所需的最少插入、删除和替换操作 |
| KL 散度 | "分布之间的距离" | 不是真正的距离（不对称）。衡量使用 Q 编码 P 所产生的额外比特数 |
| Wasserstein 距离 | "推土机距离" | 将质量从一个分布传输到另一个分布所需的最小功。一个真正的度量 |
| 近似最近邻 | "ANN 搜索" | 比精确搜索快得多地找到近似最近点的算法（HNSW、LSH、IVF） |
| HNSW | "向量数据库算法" | 分层可导航小世界图。用于快速近似最近邻搜索的多层图 |
| L1 正则化 | "Lasso" | 在损失中添加权重的 L1 范数。将权重推向零（稀疏性） |
| L2 正则化 | "Ridge"或"权重衰减" | 在损失中添加权重的平方 L2 范数。将权重向零收缩而不产生稀疏性 |
| 弹性网络 | "L1 + L2" | 结合 L1 和 L2 正则化。处理相关特征组的效果优于单独使用任意一种 |
