我的深度学习代码中有一个张量形状错误。帮我修复它。

**错误消息：** [在此粘贴错误]

**我的张量形状：**
- [名称]: [形状]
- [名称]: [形状]

**我正在尝试做的操作：** [描述它]

---

调试时，遵循此精确流程：

**步骤 1：确定操作类型。**
产生错误的操作是什么？将其映射到以下之一：
- 矩阵乘法 / Linear 层（内层维度必须匹配）
- 广播（从右侧对齐，每个维度必须相等或为 1）
- 连接（除 cat 维度外所有维度匹配）
- 卷积（期望特定的秩和通道位置）
- Reshape（总元素数必须保持不变）

**步骤 2：写出形状约束。**
对于确定的操作，显式写出预期的形状：
```
matmul(A, B): A 是 (..., m, k), B 是 (..., k, n) -> (..., m, n)
broadcast(A, B): 右对齐，每对必须（相等）或（其中一个为 1）
cat([A, B], dim=d): 除 dim d 外所有维度匹配
Linear(in_f, out_f): 输入最后一个维度必须等于 in_f
Conv2d(in_c, out_c, k): 输入必须是 (B, in_c, H, W)
```

**步骤 3：找到不匹配。**
将实际形状与约束进行比较。识别违反规则的精确维度。

**步骤 4：选择最小修复。**
从此表中选择：

| 症状 | 修复 |
|------|------|
| 缺少批次维度 | `.unsqueeze(0)` |
| 缺少通道维度 | `.unsqueeze(1)` |
| 多余的 size-1 维度 | `.squeeze(dim)` |
| matmul 内层维度错误 | `.transpose(-1, -2)` 或检查权重形状 |
| 需要 NCHW 来自 NHWC | `.permute(0, 3, 1, 2)` |
| 需要 NHWC 来自 NCHW | `.permute(0, 2, 3, 1)` |
| 为线性层展平空间维度 | `.flatten(1)` 或 `.reshape(B, -1)` |
| 拆分头部：(B,T,D) 到 (B,H,T,D/H) | `.reshape(B, T, H, D//H).transpose(1, 2)` |
| 合并头部：(B,H,T,D/H) 到 (B,T,D) | `.transpose(1, 2).reshape(B, T, H*(D//H))` |
| 对非连续张量使用 .view() | `.contiguous().view(...)` 或使用 `.reshape(...)` |

**步骤 5：验证修复。**
在每一步显示结果形状。确认总元素数在任何 reshape 中保持不变。确认操作的形状约束现在已满足。

**步骤 6：检查静默错误。**
即使形状匹配，也验证：
- 广播沿预期的轴发生（而不是意外地）
- 规约在正确的维度上求和
- 批次维度（维度 0）在整个前向传播中存活
- 在维度顺序重要时使用 transpose + reshape（不仅仅是 reshape）

将你的回答格式化为：
```
操作: [哪个操作失败]
期望: [形状约束]
实际: [提供了哪些形状]
不匹配: [哪个维度，为什么]
修复: [精确代码]
结果: [修复后的形状]
```
