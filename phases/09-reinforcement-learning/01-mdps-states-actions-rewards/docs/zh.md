# MDP、状态、动作与奖励

> 马尔可夫决策过程由五个东西构成：状态、动作、转移、奖励、折扣因子。强化学习中的一切——Q学习、PPO、DPO、GRPO——都是在这个结构上进行优化。学会一次，就可以免费阅读强化学习的其余部分。

**类型：** 学习
**语言：** Python
**前置知识：** 阶段1·06（概率与分布）、阶段2·01（机器学习分类）
**时间：** ~45分钟

## 问题

你在编写一个国际象棋机器人。或者一个库存规划器。或者一个交易智能体。或者训练推理模型的PPO循环。四个不同的领域，一个令人惊讶的事实：它们都可以归结为同一个数学对象。

监督学习给你`(x, y)`对，要求你去拟合一个函数。强化学习不给你标签——只有一连串的状态、你采取的动作和一个标量奖励。这一步是否赢了棋局？这次补货决策是否节省了资金？这笔交易是否盈利？LLM刚刚生成的令牌是否导致评判者给出了更高的奖励？

在你将其形式化之前，你无法从这个流中学习。"我看到了什么"、"我做了什么"、"接下来发生了什么"、"这有多好"——每一个都必须成为一个你可以推理的对象。这种形式化就是马尔可夫决策过程。本阶段中的每一个RL算法，包括最后的RLHF和GRPO循环，都是在这个结构上进行优化。

## 概念

![马尔可夫决策过程：状态、动作、转移、奖励、折扣因子](../assets/mdp.svg)

**五个对象。**

- **状态** `S`。智能体做出决策所需的一切。在GridWorld中，是格子。在国际象棋中，是棋盘。在LLM中，是上下文窗口加上任何记忆。
- **动作** `A`。选择。上/下/左/右移动。走一步棋。发出一个令牌。
- **转移** `P(s' | s, a)`。给定状态`s`和动作`a`，下一个状态的分布。在国际象棋中是确定性的，在库存中是随机的，在LLM解码中几乎是确定性的。
- **奖励** `R(s, a, s')`。标量信号。赢 = +1，输 = -1。收入减去成本。GRPO中的对数似然比项。
- **折扣因子** `γ ∈ [0, 1)`。未来奖励相对于当前奖励的重要性。`γ = 0.99`买来约100步的视野；`γ = 0.9`买来约10步。

**马尔可夫性质** `P(s_{t+1} | s_t, a_t) = P(s_{t+1} | s_0, a_0, …, s_t, a_t)`。未来只依赖于当前状态。如果不成立，意味着状态表示不完整——不是方法的失败，而是状态的失败。

**策略和回报。** 策略`π(a | s)`将状态映射到动作分布。回报`G_t = r_t + γ r_{t+1} + γ² r_{t+2} + …`是折扣后的未来奖励之和。值`V^π(s) = E[G_t | s_t = s]`是在策略`π`下从`s`开始的期望回报。Q值`Q^π(s, a) = E[G_t | s_t = s, a_t = a]`是从特定动作开始的期望回报。每个RL算法估计这两个中的一个，然后相应地改进`π`。

**贝尔曼方程。** 本阶段中所有内容都使用的不动点方程：

`V^π(s) = Σ_a π(a|s) Σ_{s', r} P(s', r | s, a) [r + γ V^π(s')]`
`Q^π(s, a) = Σ_{s', r} P(s', r | s, a) [r + γ Σ_{a'} π(a'|s') Q^π(s', a')]`

这些方程将期望回报分解为"这一步的奖励"加上"你到达的地方的折扣值"。递归的。阶段9中的每个算法要么迭代这个方程至收敛（动态规划），要么从中采样（蒙特卡洛），要么一步自举（时序差分）。

## 构建

### 步骤1：一个微小的确定性MDP

一个4×4的GridWorld。智能体从左上角开始，终点在右下角，每步奖励-1，动作`{上, 下, 左, 右}`。参见`code/main.py`。

```python
GRID = 4
TERMINAL = (3, 3)
ACTIONS = {"up": (-1, 0), "down": (1, 0), "left": (0, -1), "right": (0, 1)}

def step(state, action):
    if state == TERMINAL:
        return state, 0.0, True
    dr, dc = ACTIONS[action]
    r, c = state
    nr = min(max(r + dr, 0), GRID - 1)
    nc = min(max(c + dc, 0), GRID - 1)
    return (nr, nc), -1.0, (nr, nc) == TERMINAL
```

五行代码。这就是整个环境。确定性转移，恒定的步长惩罚，吸收的终止状态。

### 步骤2：执行一个策略

策略是一个从状态到动作分布的函数。最简单的：均匀随机。

```python
def uniform_policy(state):
    return {a: 0.25 for a in ACTIONS}

def rollout(policy, max_steps=200):
    s, total, steps = (0, 0), 0.0, 0
    for _ in range(max_steps):
        a = sample(policy(s))
        s, r, done = step(s, a)
        total += r
        steps += 1
        if done:
            break
    return total, steps
```

运行随机策略1000次。对于这个4×4棋盘，平均回报约为-60到-80。最优回报是-6（直线路径，先向下再向右）。弥合这个差距就是阶段9的全部内容。

### 步骤3：通过贝尔曼方程精确计算`V^π`

对于小型MDP，贝尔曼方程是一个线性系统。枚举状态，应用期望，迭代直到值停止变化。

```python
def policy_evaluation(policy, gamma=0.99, tol=1e-6):
    V = {s: 0.0 for s in all_states()}
    while True:
        delta = 0.0
        for s in all_states():
            if s == TERMINAL:
                continue
            v = 0.0
            for a, pi_a in policy(s).items():
                s_next, r, _ = step(s, a)
                v += pi_a * (r + gamma * V[s_next])
            delta = max(delta, abs(v - V[s]))
            V[s] = v
        if delta < tol:
            return V
```

这是迭代策略评估。它是Sutton & Barto中的第一个算法，也是后续所有RL方法的理论基础。

### 步骤4：`γ`是一个具有物理意义的超参数

有效视野大约是`1 / (1 - γ)`。`γ = 0.9` → 10步。`γ = 0.99` → 100步。`γ = 0.999` → 1000步。

太低会导致智能体目光短浅。太高会使得信用分配变得嘈杂，因为许多早期步骤需要对遥远的未来奖励负责。LLM RLHF通常使用`γ = 1`，因为片段短且有界。控制任务使用`0.95-0.99`。长视野策略游戏使用`0.999`。

## 陷阱

- **非马尔可夫状态。** 如果你需要最近三次观察才能做出决策，那么"状态"就不仅仅是当前的观察。修复方法：堆叠帧（Atari上的DQN堆叠4帧）或使用循环状态（基于观察的LSTM/GRU）。
- **稀疏奖励。** 仅以获胜为奖励使得在大状态空间中几乎无法学习。设计奖励塑造（中间信号）或通过模仿学习进行引导（阶段9·09）。
- **奖励破解。** 优化代理奖励通常会产生病态行为。OpenAI的赛艇智能体原地打转收集能量道具，而不是完成比赛。始终根据目标结果定义奖励，而不是代理奖励。
- **折扣因子设置错误。** 在无限视野任务上`γ = 1`使每个值变为无穷大。始终通过有限视野或`γ < 1`来限制。
- **奖励尺度。** {+100, -100} vs {+1, -1}的奖励给出相同的最优策略，但梯度幅度差异巨大。在输入到PPO/DQN之前归一化到`[-1, 1]`左右。

## 使用

2026年的技术栈在接触代码之前将每个RL流水线简化为MDP：

| 场景 | 状态 | 动作 | 奖励 | γ |
|------|------|------|------|---|
| 控制（运动、操作） | 关节角度+速度 | 连续力矩 | 任务特定塑造 | 0.99 |
| 游戏（国际象棋、围棋、扑克） | 棋盘+历史 | 合法走法 | 赢=+1 / 输=-1 | 1.0（有限） |
| 库存 / 定价 | 库存+需求 | 订单数量 | 收入-成本 | 0.95 |
| LLM的RLHF | 上下文令牌 | 下一个令牌 | 末尾的奖励模型分数 | 1.0（片段约200令牌） |
| 推理的GRPO | 提示+部分回答 | 下一个令牌 | 末尾的验证器0/1 | 1.0 |

在编写任何训练循环之前先写出五个元组。大多数"RL不起作用"的bug报告，其根源可以追溯到纸上就已存在问题的MDP公式。

## 练习

1. **简单。** 在`code/main.py`中实现4×4 GridWorld和随机策略执行。运行10,000个片段。报告回报的均值和标准差。与最优回报（-6）比较。
2. **中等。** 对均匀随机策略使用`γ ∈ {0.5, 0.9, 0.99}`运行`policy_evaluation`。每个γ将`V`打印为4×4网格。解释为什么终止状态附近的状态值随着γ增大而增长更快。
3. **困难。** 将GridWorld变为随机：每个动作以概率`p = 0.1`滑向相邻方向。重新评估均匀策略。`V[start]`变好还是变差？为什么？

## 关键术语

| 术语 | 人们的说法 | 实际含义 |
|------|-----------|---------|
| MDP | "强化学习设置" | 满足马尔可夫性质的元组`(S, A, P, R, γ)`。 |
| 状态 | "智能体看到的东西" | 在所选策略类下对未来动态的充分统计量。 |
| 策略 | "智能体的行为" | 条件分布`π(a \| s)`或确定性映射`s → a`。 |
| 回报 | "总奖励" | 从当前步开始的折扣总和`Σ γ^t r_t`。 |
| 值 | "一个状态有多好" | 在`π`下从`s`开始的期望回报。 |
| Q值 | "一个动作有多好" | 在`π`下从`s`开始、第一个动作为`a`的期望回报。 |
| 贝尔曼方程 | "动态规划递归" | 将值/Q分解为单步奖励加上折扣后继值的不动点分解。 |
| 折扣因子`γ` | "未来 vs 现在" | 对遥远未来奖励的几何加权；有效视野`~1/(1-γ)`。 |
