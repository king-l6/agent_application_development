# 多智能体强化学习（Multi-Agent RL）

> 单智能体强化学习假设环境是稳定的。把两个学习智能体放在同一个世界中，这个假设就被打破了：每个智能体都是对方环境的一部分，而双方都在变化。多智能体强化学习就是当马尔可夫假设不再成立时，让学习收敛的一系列技巧。

**类型：** 构建
**语言：** Python
**前置知识：** Phase 9 · 04（Q-learning）、Phase 9 · 06（REINFORCE）、Phase 9 · 07（Actor-Critic）
**时间：** ~45分钟

## 问题

一个学会在房间中导航的机器人是单智能体强化学习问题。一支足球队不是。AlphaStar对战星际争霸对手不是。一个由竞标智能体组成的市场不是。两辆车在四向停车标志处协商不是。任何多方参与的现实世界问题都不是。

在每个多智能体环境中，从任何一个智能体的视角来看，其他智能体*就是*环境的一部分。随着它们学习并改变行为，环境变得非平稳。马尔可夫性质——"下一个状态只取决于当前状态和我的动作"——被违反了，因为下一个状态还取决于*其他*智能体选择了什么，而它们的策略是移动目标。

这破坏了表格收敛证明（Q-learning的保证假设环境是稳定的）。它也破坏了朴素的深度强化学习：智能体互相追逐形成循环，永远无法收敛到稳定策略。你需要多智能体特有的技术：集中式训练/分散式执行、反事实基线、联赛玩法、自我对弈。

2026年的应用：机器人集群、交通路由、自动驾驶车队、市场模拟器、多智能体LLM系统（Phase 16），以及任何有多个智能玩家的游戏。

## 概念

![四种多智能体强化学习范式：独立、集中式评论家、自我对弈、联赛](../assets/marl.svg)

**形式化：马尔可夫博弈。** MDP的泛化：状态 `S`，联合动作 `a = (a_1, …, a_n)`，转移 `P(s' | s, a)`，以及每个智能体的奖励 `R_i(s, a, s')`。每个智能体 `i` 在其自身策略 `π_i` 下最大化自身的回报。如果奖励相同，则为**完全合作型**。如果是零和的，则为**对抗型**。如果是混合的，则为**一般和型**。

**核心挑战：**

- **非平稳性。** 从智能体 `i` 的视角看，`P(s' | s, a_i)` 依赖于 `π_{-i}`，而后者在变化。
- **信用分配。** 在共享奖励下，是哪个智能体导致了它？
- **探索协调。** 智能体必须探索互补的策略，而不是冗余地在同一状态下探索。
- **可扩展性。** 联合动作空间随 `n` 指数级增长。
- **部分可观测性。** 每个智能体只能看到自身的观测；全局状态是隐藏的。

**四种主要范式：**

**1. 独立Q-learning / 独立PPO（IQL, IPPO）。** 每个智能体学习自己的Q值或策略，将其他智能体视为环境的一部分。简单，有时有效（尤其是当经验回放作为一种平滑的智能体建模技巧时）。理论收敛性：无。在实践中：对松散耦合的任务效果不错，对紧密耦合的任务效果差。

**2. 集中式训练，分散式执行（CTDE）。** 最常见的现代范式。每个智能体有自己的*策略* `π_i`，基于局部观测 `o_i` 进行条件化——部署时是标准的分散式执行。在*训练*期间，集中式评论家 `Q(s, a_1, …, a_n)` 基于完整的全局状态和联合动作进行条件化。示例：
- **MADDPG**（Lowe等人，2017年）：为每个智能体配备集中式评论家的DDPG。
- **COMA**（Foerster等人，2017年）：反事实基线——问"如果我选择了动作 `a'` 而不是当前动作，我的奖励会是什么？"——分离出我的贡献。
- **MAPPO** / **IPPO** 与共享评论家（Yu等人，2022年）：带有集中式价值函数的PPO。2026年在合作型MARL中占主导地位。
- **QMIX**（Rashid等人，2018年）：值分解——`Q_tot(s, a) = f(Q_1(s, a_1), …, Q_n(s, a_n))`，具有单调混合。

**3. 自我对弈。** 同一智能体的两个副本互相对战。对手的策略*就是*我过去某个快照中的策略。AlphaGo / AlphaZero / MuZero。OpenAI Five。最适合零和博弈；训练信号是对称的。

**4. 联赛玩法。** 自我对弈在一般和/对抗环境中的扩展：维护一个过去和当前策略的种群，从联赛中采样一个对手，针对它进行训练。添加利用者（专门击败当前最佳策略）和主利用者（专门击败利用者）。AlphaStar（星际争霸II）。当博弈存在"石头剪刀布"策略循环时是必要的。

**通信。** 允许智能体之间发送学习到的消息 `m_i`。在合作环境中有效。Foerster等人（2016年）证明了可微分的智能体间通信可以端到端训练。今天的基于LLM的多智能体系统（Phase 16）本质上使用自然语言进行通信。

## 动手构建

本课程使用一个6×6的GridWorld，包含两个合作智能体。它们从对角方向出发，必须到达一个共享目标。共享奖励：只要任一智能体还在移动，每步 `-1`，当两者都到达时 `+10`。见 `code/main.py`。

### 第1步：多智能体环境

```python
class CoopGridWorld:
    def __init__(self):
        self.size = 6
        self.goal = (5, 5)

    def reset(self):
        return ((0, 0), (5, 0))  # 两个智能体

    def step(self, state, actions):
        a1, a2 = state
        new1 = move(a1, actions[0])
        new2 = move(a2, actions[1])
        done = (new1 == self.goal) and (new2 == self.goal)
        reward = 10.0 if done else -1.0
        return (new1, new2), reward, done
```

*联合*动作空间为 `|A|² = 16`。全局状态是两个位置。

### 第2步：独立Q-learning

每个智能体运行自己的Q表，以联合状态为键。每一步：两者都选择 ε-贪婪动作，收集联合转移，每个智能体用共享奖励更新自己的Q值。

```python
def independent_q(env, episodes, alpha, gamma, epsilon):
    Q1, Q2 = defaultdict(default_q), defaultdict(default_q)
    for _ in range(episodes):
        s = env.reset()
        while not done:
            a1 = epsilon_greedy(Q1, s, epsilon)
            a2 = epsilon_greedy(Q2, s, epsilon)
            s_next, r, done = env.step(s, (a1, a2))
            target1 = r + gamma * max(Q1[s_next].values())
            target2 = r + gamma * max(Q2[s_next].values())
            Q1[s][a1] += alpha * (target1 - Q1[s][a1])
            Q2[s][a2] += alpha * (target2 - Q2[s][a2])
            s = s_next
```

在这个任务上有效，因为奖励密集且对齐。在紧密耦合的任务上会失败（例如，其中一个智能体必须*等待*另一个）。

### 第3步：带分解值更新的集中式Q

使用一个联合动作上的Q值 `Q(s, a_1, a_2)`。从共享奖励更新。在执行时通过边缘化来分散：`π_i(s) = argmax_{a_i} max_{a_{-i}} Q(s, a_1, a_2)`。用指数级联合动作空间换取*正确的*全局视图。

### 第4步：简单自我对弈（对抗性2智能体）

同一智能体，两个角色。训练智能体A对抗智能体B；在 `K` 个回合后，将A的权重复制给B。对称训练，持续进步。AlphaZero配方的微型版本。

## 陷阱

- **非平稳经验回放。** 使用独立智能体的经验回放比单智能体更糟糕，因为旧的转移是由现已过时的对手生成的。修复方法：重新标记或按时间远近加权。
- **信用分配模糊性。** 长时间回合后的共享奖励；无法明确判断哪个智能体做出了贡献。修复方法：反事实基线（COMA），或每智能体奖励塑形。
- **策略漂移/追逐。** 每个智能体的最佳响应随着其他智能体的更新而变化。修复方法：集中式评论家、缓慢的学习率，或一次冻结一个智能体。
- **通过协调进行奖励黑客。** 智能体找到设计者未预料到的协调性利用方法。拍卖智能体收敛到出价为零。修复方法：仔细的奖励设计、行为约束。
- **探索冗余。** 两个智能体探索相同状态-动作对。修复方法：每智能体熵奖励，或角色条件化。
- **联赛循环。** 纯粹的自我对弈可能陷入主导循环。修复方法：使用多样化对手的联赛玩法。
- **样本爆炸。** `n` 个智能体 × 状态空间 × 联合动作。使用函数逼近进行近似；分解动作空间（每个智能体一个策略输出头）。

## 应用

2026年的多智能体强化学习应用地图：

| 领域 | 方法 | 备注 |
|--------|--------|-------|
| 合作导航/操作 | MAPPO / QMIX | CTDE；共享评论家 + 分散式Actor。 |
| 双人游戏（国际象棋、围棋、扑克） | 自我对弈 + MCTS（AlphaZero） | 零和；对称训练。 |
| 复杂多人游戏（Dota、星际争霸） | 联赛玩法 + 模仿预训练 | OpenAI Five、AlphaStar。 |
| 自动驾驶车队 | CTDE MAPPO / 带注意力的PPO | 部分观测；可变的团队规模。 |
| 拍卖市场 | 博弈论均衡 + 强化学习 | 当 `n` → ∞ 时使用平均场RL。 |
| LLM多智能体系统（Phase 16） | 自然语言通信 + 角色条件化 | 在智能体规划层的强化学习循环。 |

2026年，MARL最大的增长领域是基于LLM的：语言模型智能体群进行协商、辩论、构建软件。强化学习表现为对*轨迹级别*输出的偏好优化，而非token级别（Phase 16 · 03）。

## 练习

1. **简单。** 在2智能体合作GridWorld上训练独立Q-learning。需要多少个回合直到平均回报 > 0？绘制联合学习曲线。
2. **中等。** 添加一个"协调"任务：只有当两个智能体在同一回合中踏上目标时才算到达。独立Q还能收敛吗？是什么出了问题？
3. **困难。** 为MAPPO式训练实现一个集中式评论家，并在协调任务上与独立PPO比较收敛速度。

## 关键术语

| 术语 | 人们说的 | 实际含义 |
|------|-----------------|-----------------------|
| 马尔可夫博弈 | "多智能体MDP" | `(S, A_1, …, A_n, P, R_1, …, R_n)`；每个智能体有自己的奖励。 |
| CTDE | "集中式训练，分散式执行" | 训练时的联合评论家；每个智能体的策略仅使用局部观测。 |
| IPPO | "独立PPO" | 每个智能体单独运行PPO。简单基线；常被低估。 |
| MAPPO | "多智能体PPO" | 带有基于全局状态条件化的集中式价值函数的PPO。 |
| QMIX | "单调值分解" | `Q_tot = f_monotone(Q_1, …, Q_n)` 允许分散式argmax。 |
| COMA | "反事实多智能体" | 优势 = 我的Q减去对动作进行边缘化的期望Q。 |
| 自我对弈 | "智能体对战过去的自己" | 单一智能体，两个角色；零和博弈的标准方法。 |
| 联赛玩法 | "种群训练" | 缓存过去的策略，从池中采样对手；处理策略循环。 |

## 延伸阅读

- [Lowe等人（2017年）。用于混合合作-竞争环境的多智能体Actor-Critic（MADDPG）](https://arxiv.org/abs/1706.02275) — 带集中式评论家的CTDE。
- [Foerster等人（2017年）。反事实多智能体策略梯度（COMA）](https://arxiv.org/abs/1705.08926) — 用于信用分配的反事实基线。
- [Rashid等人（2018年）。QMIX：单调值函数分解](https://arxiv.org/abs/1803.11485) — 带单调性的值分解。
- [Yu等人（2022年）。PPO在合作多智能体博弈中惊人的有效性（MAPPO）](https://arxiv.org/abs/2103.01955) — PPO在MARL中出乎意料地强大。
- [Vinyals等人（2019年）。使用多智能体强化学习达到星际争霸II大师级水平（AlphaStar）](https://www.nature.com/articles/s41586-019-1724-z) — 大规模联赛玩法。
- [Silver等人（2017年）。无需人类知识掌握围棋（AlphaGo Zero）](https://www.nature.com/articles/nature24270) — 零和博弈中的纯粹自我对弈。
- [Sutton & Barto（2018年）。第15章——神经科学 与 第17章——前沿](http://incompleteideas.net/book/RLbook2020.pdf) — 包含教材中对多智能体设置和CTDE设计要解决的非平稳性问题的简要阐述。
- [Zhang, Yang & Başar（2021年）。多智能体强化学习：选择性综述](https://arxiv.org/abs/1911.10635) — 涵盖合作型、竞争型和混合型MARL的综述，包含收敛性结果。
