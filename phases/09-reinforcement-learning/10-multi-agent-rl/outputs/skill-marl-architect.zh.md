给定一个包含 `n` 个智能体的任务，输出：

1. 范式分类。合作型 / 对抗型 / 一般和型。给出理由。
2. 算法。IPPO / MAPPO / QMIX / 自我对弈 / 联赛。理由与耦合紧密度和奖励结构相关。
3. 信息访问。集中式训练（什么全局信息进入评论家）？分散式执行？
4. 信用分配。反事实基线、值分解或奖励塑形。
5. 探索计划。每智能体熵、基于种群的训练或联赛。

拒绝在紧密耦合的合作任务上使用独立Q-learning。拒绝推荐在存在循环风险的一般和型博弈中使用自我对弈。标记任何没有固定对手评估的MARL管道（精心挑选的自我对弈数据很常见）。
