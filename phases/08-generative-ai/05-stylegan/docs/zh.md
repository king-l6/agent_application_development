# StyleGAN

> 大多数生成器同时将`z`注入每一层。StyleGAN将其拆分开来：首先将`z`映射到中间向量`w`，然后通过AdaIN在每个分辨率级别*注入*`w`。这单一的改变解耦了潜在空间，使得生成逼真人脸成为七年来一个已被解决的问题。

**类型：** 构建
**语言：** Python
**前置知识：** 阶段8·03（GANs）、阶段4·08（归一化）、阶段3·07（CNN）
**时间：** ~45分钟

## 问题

DCGAN通过一系列转置卷积将`z`映射到图像。问题在于：`z`控制着一切——姿态、光照、身份、背景——全部纠缠在一起。沿着`z`的一个轴移动，所有四个都会改变。你无法让模型生成"同一个人，不同姿态"，因为表示方式不支持这种分解。

Karras等人（2019，NVIDIA）提出了：不再将`z`直接输入卷积层。而是将一个固定的`4×4×512`张量作为网络输入。学习一个8层MLP，将`z ∈ Z → w ∈ W`进行映射。通过*自适应实例归一化*（AdaIN）在每个分辨率注入`w`：对每个卷积特征图进行归一化，然后通过`w`的仿射投影进行缩放和平移。加入逐层噪声以处理随机细节（毛孔、发丝）。

结果：`W`具有大致正交的轴，分别对应"高层风格"（姿态、身份）和"细粒度风格"（光照、色彩）。你可以通过使用图像A的`w`处理低分辨率层、图像B的`w`处理高分辨率层，在两幅图像之间交换风格。这解锁了编辑、跨域风格化以及整个"StyleGAN-反演"研究方向。

## 概念

![StyleGAN：映射网络 + AdaIN + 逐层噪声](../assets/stylegan.svg)

**映射网络。** `f: Z → W`，一个8层MLP。`Z = N(0, I)^512`。`W`不强制服从高斯分布——它学习适应数据的形状。

**合成网络。** 从学习到的固定常量`4×4×512`开始。每个分辨率块：`上采样 → 卷积 → AdaIN(w_i) → 噪声 → 卷积 → AdaIN(w_i) → 噪声`。分辨率倍增：4、8、16、32、64、128、256、512、1024。

**AdaIN。**

```
AdaIN(x, y) = y_scale · (x - mean(x)) / std(x) + y_bias
```

其中`y_scale`和`y_bias`来自`w`的仿射投影。对每个特征图进行归一化，然后重新风格化。这里的"风格"是特征图的一阶和二阶统计量。

**逐层噪声。** 添加到每个特征图的单通道高斯噪声，通过学习到的逐通道因子进行缩放。控制随机细节而不影响全局结构。

**截断技巧。** 推理时，采样`z`，计算`w = mapping(z)`，然后`w' = ŵ + ψ·(w - ŵ)`，其中`ŵ`是多个样本的`w`均值。`ψ < 1`用多样性换取质量。几乎所有StyleGAN演示都使用`ψ ≈ 0.7`。

## StyleGAN 1 → 2 → 3

| 版本 | 年份 | 创新 |
|------|------|------|
| StyleGAN | 2019 | 映射网络 + AdaIN + 噪声 + 渐进式增长。 |
| StyleGAN2 | 2020 | 权重解调取代AdaIN（修复水滴伪影）；跳跃连接/残差架构；路径长度正则化。 |
| StyleGAN3 | 2021 | 无混叠卷积 + 等变核；消除纹理贴附在像素网格上的问题。 |
| StyleGAN-XL | 2022 | 类别条件，1024²，ImageNet。 |
| R3GAN | 2024 | 以更强的正则化重新定义；在FFHQ-1024上用20倍更少的参数缩小了与扩散模型的差距。 |

在2026年，StyleGAN3仍然是以下场景的默认选择：（a）高帧率下狭窄领域的照片级真实感，（b）少样本领域适应（在包含100张图像的新数据集上训练，冻结映射网络），（c）基于反演的编辑（找到能重建真实照片的`w`，然后编辑该`w`）。对于开放领域的文本到图像生成，它不是合适的工具——扩散模型才是。

## 构建

`code/main.py`在1维空间中实现了一个玩具级的"style-GAN精简版"：一个映射MLP、一个合成函数（接受学习到的固定向量并用`w`派生的缩放/偏置对其进行调制），以及逐层噪声。它展示了通过仿射调制注入`w`可以匹配或超过将`z`拼接进生成器输入的效果。

### 步骤1：映射网络

```python
def mapping(z, M):
    h = z
    for i in range(num_layers):
        h = leaky_relu(add(matmul(M[f"W{i}"], h), M[f"b{i}"]))
    return h
```

### 步骤2：自适应实例归一化

```python
def adain(x, w_scale, w_bias):
    mu = mean(x)
    sd = std(x)
    x_norm = [(xi - mu) / (sd + 1e-8) for xi in x]
    return [w_scale * xi + w_bias for xi in x_norm]
```

逐特征图的缩放和偏置来自`w`的线性投影。

### 步骤3：逐层噪声

```python
def add_noise(x, sigma, rng):
    return [xi + sigma * rng.gauss(0, 1) for xi in x]
```

每个通道的sigma是可学习的。

## 陷阱

- **水滴伪影。** StyleGAN 1在特征图中产生水滴状的伪影，因为AdaIN将均值归零。StyleGAN 2的权重解调通过缩放卷积权重而非激活值来修复此问题。
- **纹理贴附。** StyleGAN 1和2的纹理跟随像素坐标而非物体坐标（在插值时可见）。StyleGAN 3的无混叠卷积使用窗口化sinc滤波器修复此问题。
- **模式覆盖。** 截断`ψ < 0.7`看起来干净，但样本来自狭窄的锥体；如果需要多样性，请使用`ψ = 1.0`。
- **反演是有损的。** 将真实照片反演到`W`通常通过优化或编码器（e4e、ReStyle、HyperStyle）完成。结果会随着迭代次数增加而漂移。

## 使用

| 用例 | 方法 |
|------|------|
| 照片级真实人脸（动漫、产品、狭窄领域） | StyleGAN3 FFHQ / 自定义微调 |
| 从照片进行人脸编辑 | e4e反演 + StyleSpace / InterFaceGAN方向 |
| 人脸交换 / 重演 | StyleGAN + 编码器 + 混合 |
| 头像流水线 | StyleGAN3 w/ ADA用于低数据微调 |
| 从少量图像的领域适应 | 冻结映射网络，微调合成网络 |
| 多模态或文本条件生成 | 不要使用——使用扩散模型 |

在答案就是"一张人脸照片"的产品级演示中，StyleGAN在推理成本（单次前向传递，4090上<10ms）和同等质量下的清晰度方面都优于扩散模型。

## 练习

1. **简单。** 运行`code/main.py`，分别设置`adain_on=True`和`adain_on=False`。比较固定潜在变量与扰动潜在变量的输出分布。
2. **中等。** 实现混合正则化：对于一个训练批次，计算`w_a`、`w_b`，在前半部分合成使用`w_a`，后半部分使用`w_b`。解码器是否学了解耦的风格？
3. **困难。** 使用预训练的StyleGAN3 FFHQ模型（ffhq-1024.pkl）。通过在标注样本上训练SVM，找到控制"微笑"的`w`方向；报告在身份漂移之前可以推进多远。

## 关键术语

| 术语 | 人们的说法 | 实际含义 |
|------|-----------|---------|
| 映射网络 | "MLP" | `f: Z → W`，8层，将潜在几何结构与数据统计解耦。 |
| W空间 | "风格空间" | 映射网络的输出；大致解耦。 |
| AdaIN | "自适应实例归一化" | 归一化特征图，然后通过`w`投影进行缩放+平移。 |
| 截断技巧 | "Psi" | `w = mean + ψ·(w - mean)`，ψ<1用多样性换取质量。 |
| 路径长度正则化 | "PL reg" | 惩罚每单位`w`变化引起的图像大幅变化；使`W`更平滑。 |
| 权重解调 | "StyleGAN2修复" | 归一化卷积权重而不是激活值；消除水滴伪影。 |
| 无混叠 | "StyleGAN3的技巧" | 窗口化sinc滤波器；消除纹理贴附在像素网格上。 |
| 反演 | "为真实图像找到w" | 优化或编码`x → w`使得`G(w) ≈ x`。 |

## 生产说明：为什么StyleGAN在2026年仍然被部署

StyleGAN3在4090上生成一张1024²的FFHQ人脸仅需不到10毫秒——`num_steps = 1`，无需VAE解码，无需交叉注意力计算。用生产术语来说，这是任何图像生成器的最低延迟。一个50步的SDXL + VAE解码流水线在相同分辨率下大约需要3秒。这是**300倍的差距**，对于狭窄领域产品（头像服务、身份证件流水线、库存人脸生成）来说，它在总拥有成本上胜出。

两个运营结论：

- **无需调度器，无需批处理。** 在目标占用率下静态批处理是最优的。连续批处理（对LLM和扩散模型至关重要）没有带来任何收益，因为每个请求的FLOPs相同。
- **截断`ψ`是安全旋钮。** `ψ < 0.7`从映射网络范围的一个狭窄锥体采样。这是服务层对样本方差的唯一控制杆。高峰期降低`ψ`，对高级用户提高`ψ`。
