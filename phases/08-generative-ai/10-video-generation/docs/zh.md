# 视频生成

> 图像是一个2维张量。视频是一个3维张量。理论是相同的；计算量却难了10-100倍。OpenAI的Sora（2024年2月）证明了这是可能的。到2026年，Veo 2、Kling 1.5、Runway Gen-3、Pika 2.0和WAN 2.2都能从文本生成1080p的生产级视频——而开放权重技术栈（CogVideoX、HunyuanVideo、Mochi-1、WAN 2.2）落后12个月。

**类型：** 构建
**语言：** Python
**前置知识：** 阶段8·07（潜在扩散）、阶段7·09（ViT）、阶段8·06（DDPM）
**时间：** ~45分钟

## 问题

一段10秒的1080p视频，24fps，是240帧1920×1080×3像素。每段片段约1.5 GB的原始数据。像素空间扩散不可行。你需要：

1. **时空压缩。** 一个VAE，编码视频而非帧，变成一系列时空补丁。
2. **时间连贯性。** 帧需要在数秒内共享内容、光照和物体身份。网络必须对运动进行建模。
3. **计算预算。** 在相同模型大小下，视频训练比图像贵10-100倍。
4. **条件化。** 文本、图像（首帧）、音频或另一个视频。大多数生产模型接受全部四种。

解决这个问题的架构是应用于时空补丁的**扩散Transformer（DiT）**，在巨大的（提示、字幕、视频）数据集上训练。与第06课相同的扩散损失。

## 概念

![视频扩散：分块、DiT、解码](../assets/video-generation.svg)

### 分块

使用3D VAE（学习到的时空压缩）对视频进行编码。潜在表示的形状是`[T_latent, H_latent, W_latent, C_latent]`。分割成大小为`[t_p, h_p, w_p]`的补丁。对于Sora风格的模型，`t_p = 1`（逐帧补丁）或`t_p = 2`（每两帧）。一段10秒的1080p视频压缩到约20,000-100,000个补丁。

### 时空DiT

一个Transformer处理扁平的补丁序列。每个补丁有一个3维位置嵌入（时间 + y + x）。注意力通常是分解的：

- **空间注意力** 在每帧的补丁内。
- **时间注意力** 在相同空间位置跨帧。
- **完整的3D注意力**贵16-100倍；仅用于低分辨率或在研究中。

### 文本条件化

与大型文本编码器的交叉注意力（Sora使用T5-XXL，CogVideoX-5B使用T5-XXL）。长的提示很重要——Sora的训练集有GPT生成的字幕，平均每个片段200个令牌。

### 训练

在时空潜在表示上的标准扩散损失（ε或v预测）。数据：网络视频 + 约1亿精选片段 + 合成文本字幕。计算量：即使是一个小的研究运行也需要10,000+ GPU小时；Sora级别的需要100,000+。

## 构建

`code/main.py`模拟了核心的时空DiT思想：将一个小型合成视频分块，添加每补丁位置嵌入，并用Transformer风格的跨补丁注意力对整个序列进行去噪。没有numpy；纯Python。我们展示了即使在一维空间中，当相邻帧的补丁共享去噪器和位置嵌入时，时间连贯性也会涌现。

### 步骤1：对合成1维"视频"进行分块

```python
def make_video(T_frames=8, rng=None):
    # "视频"是遵循平滑轨迹的1维值序列
    base = rng.gauss(0, 1)
    return [base + 0.3 * t + rng.gauss(0, 0.1) for t in range(T_frames)]
```

### 步骤2：每帧的位置嵌入

```python
def pos_embed(t, dim):
    return sinusoidal(t, dim)
```

### 步骤3：去噪器看到整个序列

独立地对每一帧去噪不同，我们的小型网络连接所有帧的值 + 它们的位置嵌入，并联合预测所有帧的噪声。

### 步骤4：时间连贯性测试

训练后，采样一个视频。测量帧间增量。如果模型学习了时间结构，增量会比独立采样每帧更小。

## 陷阱

- **独立逐帧采样 = 闪烁。** 如果你对每一帧单独运行图像扩散，输出会闪烁，因为每帧的噪声是独立的。视频扩散通过注意力或共享噪声耦合帧来修复这一点。
- **朴素的3D注意力 = OOM。** 在10秒1080p潜在表示上的完整3D注意力是数千亿次操作。分解为空间+时间。
- **数据字幕比大小更重要。** Sora相比先前工作的主要升级是在大约10倍更详细的字幕上训练（GPT-4重新标注的片段）。OpenAI的技术报告明确指出了这一点。
- **首帧条件化。** 大多数生产模型也接受一幅图像作为首帧。这就是"图生视频"模式；训练包括这种变体。
- **物理漂移。** 长片段（>10秒）会累积微妙的不一致性。滑动窗口生成 + 关键帧锚定有所帮助。

## 关键术语

| 术语 | 人们的说法 | 实际含义 |
|------|-----------|---------|
| 视频VAE | "3D VAE" | 将`(T, H, W, C)`压缩为时空潜在表示的编码器。 |
| 补丁 | "令牌" | 潜在表示的固定大小3D块；输入到DiT。 |

## 生产说明：视频潜在表示是一个内存带宽问题

一段10秒的1080p片段，24fps，是240帧 × 1920 × 1080 × 3 ≈ 1.5 GB的原始像素。经过4倍视频VAE压缩（`2 × 空间 × 2 × 时间`）后，潜在表示每次请求约100 MB。通过时空DiT运行30步，batch 1，你每步移动约3 GB通过HBM——瓶颈是内存带宽，而不是FLOPs。

三个生产旋钮，都直接来自生产推理文献：

- **跨DiT的张量并行（TP）。** 文生视频模型通常≥10B参数。跨4个H100的TP=4是标准；对于405B级模型使用PP=2 × TP=2。每步延迟大致随TP线性下降，直到全归约瓶颈。
- **帧批处理 = 连续批处理。** 在生成时，视频概念上是由注意力连接的一批帧。连续批处理（飞行中调度）适用：如果模型架构允许滑动窗口生成，在返回帧`t-1`的同时开始渲染帧`t+1`。
- **片段级预填充缓存。** 对于图生视频，首帧条件化类似于LLM的提示预填充：计算一次，在时间解码器传递中重用。这实际上就是视频的KV缓存。
