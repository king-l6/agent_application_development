# 生成模型 — 分类与历史

> 每一个图像模型、文本模型、视频模型和 3D 模型都归入这五个类别之一。选错类别，你将与数学搏斗数周。选对类别，过去十二年的领域进展就清晰地沉淀在你脑海中。

**类型：** 学习
**语言：** Python
**前置知识：** 阶段 2（ML 基础），阶段 3（深度学习核心），阶段 7 · 14（Transformer）
**时间：** ~45 分钟

## 问题

生成模型只做一件事：给定从某个未知分布 `p_data(x)` 中抽取的训练样本，输出看起来来自同一分布的新样本。人脸、句子、MIDI 文件、蛋白质结构——如果你眯起眼睛看，都是同样的问题。

难点在于 `p_data` 存在于数百万维度的空间中（一张 512x512 的 RGB 图像约 786k 维），样本位于该空间内的一个薄流形上，而你只有大约 1000 万个样本。暴力拟合密度是不可能的。每个生成模型都是一种折衷，将一个困难问题转化为一个稍微不那么困难的问题。

五个家族经受住了过去十二年的考验。了解每个家族所做的折衷，就能告诉你为什么它在某些任务上取胜，在另一些任务上失效。

## 概念

![五个生成模型家族——按建模对象分类](../assets/taxonomy.svg)

**1. 显式密度，可计算。** 将 `log p(x)` 写成一个可以实际评估的和。自回归模型（PixelCNN、WaveNet、GPT）将 `p(x) = ∏ p(x_i | x_<i)` 分解。归一化流（RealNVP、Glow）将 `p(x)` 构建为简单基分布的可逆变换。优点：精确的似然、干净的训练损失。缺点：自回归推理串行（长序列慢），流需要可逆架构（架构上受限）。

**2. 显式密度，近似。** 从下界（ELBO）约束 `log p(x)` 并优化这个界。VAE（Kingma 2013）使用带有变分后验的编码器-解码器。扩散模型（DDPM、Ho 2020）训练一个去噪器，隐式优化一个加权的 ELBO。扩散是 2026 年图像、视频和 3D 生成的主导骨干。

**3. 隐式密度。** 完全跳过密度；学习一个生成器 `G(z)` 来产生样本，和一个判别器 `D(x)` 来区分真假。GAN（Goodfellow 2014）。推理速度快（一次前向传播），但训练过程中以不稳定著称。即使在 2026 年，StyleGAN 1/2/3 仍然是固定领域照片级真实感（人脸、卧室）的 SOTA。

**4. 基于分数 / 连续时间。** 直接学习对数密度的梯度 `∇_x log p(x)`（分数）。Song & Ermon（2019）展示了分数匹配将扩散推广到 SDE。流匹配（Lipman 2023）是 2024-2026 年的热门：无仿真训练、更直的路径、比 DDPM 快 4-10 倍的采样。Stable Diffusion 3、Flux、AudioCraft 2 都使用流匹配。

**5. 基于离散编码的 token 自回归。** 使用 VQ-VAE 或残差量化器将高维数据压缩成离散 token 的短序列，然后使用 Transformer 对 token 序列建模。Parti、MuseNet、AudioLM、VALL-E、Sora 的补丁分词器都使用这种方法。这是类别 1 加上一个学习到的分词器。

## 简要历史

| 年份 | 模型 | 重要性 |
|------|-------|-----------------|
| 2013 | VAE（Kingma） | 第一个具有可用训练损失的深度生成模型。 |
| 2014 | GAN（Goodfellow） | 隐式密度，无似然——样本异常清晰。 |
| 2015 | DRAW、PixelCNN | 序列图像生成。 |
| 2017 | Glow、RealNVP | 可逆流；带深度的精确似然。 |
| 2017 | Progressive GAN | 第一个百万像素级人脸。 |
| 2019 | StyleGAN / StyleGAN2 | 照片级真实感人脸，在该领域仍然难以超越。 |
| 2020 | DDPM（Ho） | 扩散变得实用。 |
| 2021 | CLIP、DALL-E 1、VQGAN | 文本到图像走向主流。 |
| 2022 | Imagen、Stable Diffusion 1、DALL-E 2 | 潜在扩散 + 文本条件 = 商品化。 |
| 2022 | ControlNet、LoRA | 对预训练扩散的精细控制。 |
| 2023 | SDXL、Midjourney v5、流匹配 | 规模 + 更好的训练动态。 |
| 2024 | Sora、Stable Diffusion 3、Flux.1 | 视频扩散；流匹配胜出。 |
| 2025 | Veo 2、Kling 1.5、Runway Gen-3、Nano Banana | 生产级视频。 |
| 2026 | Consistency + Rectified Flow | 从扩散骨干实现单步采样。 |

## 五问题分诊法

当一篇新的生成模型论文出现时，在阅读方法部分之前先回答这五个问题。

1. **被建模的是什么？** 像素、潜在表示、离散 token、3D 高斯体、网格、波形？
2. **密度是显式的还是隐式的？** 他们写下了 `log p(x)` 吗？
3. **采样：一次完成还是迭代？** 迭代意味着更慢的推理；一次完成通常意味着对抗性或蒸馏。
4. **条件：无条件、类别、文本、图像、姿态？** 这决定了损失和架构支撑。
5. **评估：FID、CLIP 分数、IS、人类偏好、任务准确率？** 每个都有已知的失败模式（见第 14 课）。

在接下来的每一课中，你都会重新回答这五个问题。到最后，它们将成为本能反应。

## 构建它

本课的代码是一个轻量级可视化：使用三种玩具方法（核密度估计、离散直方图和最近邻样本"GAN 风格"生成器）从样本拟合一维高斯混合模型，从而让你在一个屏幕就能看到显式与隐式密度的区别。

运行 `code/main.py`。它从一个双峰高斯混合中抽取 2000 个样本，然后打印：

```
explicit density (histogram): p(x in [-0.5, 0.5]) ≈ 0.38
approximate density (KDE):     p(x in [-0.5, 0.5]) ≈ 0.41
implicit (nearest-sample gen): 20 new samples printed, no p(x)
```

注意：前两种让你可以问"这个点有多大可能？"第三种不能。这就是*显式 vs 隐式*的区别，它将影响未来的每一课。

## 使用它

2026 年，哪个家族适合哪个任务？

| 任务 | 最优家族 | 原因 |
|------|-------------|------|
| 照片级真实感人脸，窄领域 | StyleGAN 2/3 | 仍然最清晰，推理最快。 |
| 通用文本到图像 | 潜在扩散 + 流匹配 | SD3、Flux.1、DALL-E 3。 |
| 快速文本到图像 | Rectified flow + 蒸馏 | SDXL-Turbo、SD3-Turbo、LCM。 |
| 文本到视频 | Diffusion Transformer + 流匹配 | Sora、Veo 2、Kling。 |
| 语音 + 音乐 | 基于 token 的 AR（AudioLM、VALL-E、MusicGen）或流匹配（AudioCraft 2） | 离散 token 扩展成本低廉。 |
| 3D 场景 | 高斯泼溅拟合、扩散先验 | 3D-GS 用于重建，扩散用于新视角。 |
| 密度估计（无需采样） | 流 | 唯一具有精确 `log p(x)` 的家族。 |
| 仿真 / 物理 | 流匹配、分数 SDE | 直线路径、平滑向量场。 |

## 关键术语

| 术语 | 大家的说法 | 实际含义 |
|------|-----------------|-----------------------|
| 生成模型 | "它制造新东西" | 学习一个针对 `p_data(x)` 的采样器，可选地暴露 `log p(x)`。 |
| 显式密度 | "你可以评估它" | 模型提供封闭形式或可计算的 `log p(x)`。 |
| 隐式密度 | "GAN 风格" | 仅提供采样器——无法评估给定点的 `p(x)`。 |
| ELBO | "证据下界" | `log p(x)` 的可计算下界；VAE 和扩散模型优化它。 |
| 分数 | "对数密度的梯度" | `∇_x log p(x)`；扩散和 SDE 模型学习这个场。 |
| 流形假设 | "数据存在于表面上" | 高维数据集中在低维流形上；这就是降维有效的原因。 |
| 自回归 | "预测下一个部分" | 将联合分布分解为条件分布的乘积。 |
| 潜在表示 | "压缩编码" | 低维表示，解码器可以从该表示重建输入。 |

## 生产注意：五个家族，五种推理形态

每个家族对应不同的推理服务器成本曲线。生产推理文献将 LLM 推理框架化为预填充 + 解码；同样的分解在这里也适用：

- **自回归（类别 1 和 5）。** 串行解码主导延迟；KV 缓存、连续批处理和投机解码都直接适用。
- **VAE / 扩散 / 流匹配（类别 2 和 4）。** 没有 LLM 意义上的解码。成本 = `num_steps × step_cost`，而 `step_cost` 是完整潜在分辨率下的 Transformer 或 U-Net 前向传播。生产的旋钮是步数（DDIM / DPM-Solver / 蒸馏）、批大小和精度（bf16 / fp8 / int4）。
- **GAN（类别 3）。** 一次前向传播。无调度、无 KV 缓存。TTFT ≈ 总延迟。这就是为什么 StyleGAN 在窄领域 UX 上仍然胜出的原因。

当你在论文摘要中看到"比扩散更快"时，将其翻译为"更少的步骤 × 相同的步骤成本"或"相同的步骤 × 更低的步骤成本"。其他都是营销。
