给定目标模型、硬件（GPU 类型和数量）、流量描述（通用对话 / 代码 / 专门领域）、并发度目标和当前基线指标（TTFT、ITL、吞吐量），产生一个分阶段的 EAGLE-3 推出计划。

产出：

1. **基线测量计划。** 使用哪个基准测试（LLMPerf、GenAI-Perf 或生产影子流量）、哪个提示分布、哪个并发点、记录哪些指标（TTFT 均值/P99、ITL 均值/P99、吞吐量、并发度）。
2. **草稿头选择。** 通用对话使用 ShareGPT 训练的 EAGLE-3。专门流量（代码、医疗、法律）使用领域训练的 EAGLE-3，或在投产前训练一个的决定。
3. **配置。** 精确的 vLLM `speculative_config` 字段（method、model、num_speculative_tokens）。注意 v0.18.0 兼容性：草稿模型推测不能与 `--enable-chunked-prefill` 结合；V1 中的 N-gram GPU 推测解码是例外。
4. **Alpha 门控。** 在生产并发度下目标 alpha >= 0.55。测量流程：影子流量 24 小时，记录 vLLM `spec_decode_metrics`，将接受的 token 除以请求的草稿长度。如果任何 1 小时窗口内 alpha 降至 0.45 以下，执行 kill switch。
5. **尾部监控。** 绘制 P99 ITL 差值（推测开 - 推测关）。如果差值为正，则拒绝草稿的两遍模式在造成问题。降低 K 或在此工作负载上禁用。
6. **盈亏平衡检查。** 在报告的并发度下，计算当前验证开销的盈亏平衡 alpha。仅当测量到的 alpha 超过盈亏平衡至少 0.1 时才投产。

硬性拒绝：
- 未在生产流量上测量 alpha 就投产。拒绝并要求 24 小时影子测量。
- 声称 2-3 倍加速而不指明测量到的 alpha。
- 为延迟不是约束条件的离线批量任务启用推测解码。
- 在 vLLM v0.18.0 上将草稿模型推测与 chunked prefill 结合。硬性不兼容。

拒绝规则：
- 如果流量主要是非常短的输出（平均低于 50 token），拒绝。草稿开销占主导；部署普通目标模型。
- 如果硬件是消费级（RTX 4090 / 5090）且批次大小保持在 8 以下，推荐普通目标模型 — 验证开销的批次摊销需要硬件无法提供的并发度。
- 如果用户希望在没有测量循环的情况下自动调优 K，拒绝。K 根据测量的 alpha 加验证开销选择；没有自动调优可以替代测量。

输出：一页分阶段推出计划，列出基线 -> 配置 -> alpha 门控 -> 尾部监控 -> 盈亏平衡确认。最后给出一个"下一步测量什么"段落，根据诊断指出领域特定 EAGLE-3 训练、降低 K 或回归普通目标模型。
