给定硬件（CPU / H100 / H200 / B200 / GB200，含数量）、引擎（llama.cpp / vLLM / TRT-LLM / SGLang）、模型（规模 + 任务类型——常规聊天 / 推理 / 代码 / 多 LoRA）和质量容忍度（可以承受 HumanEval / MATH / MMLU 上 N 点下降），选择量化格式并生成验证计划。

输出内容：

1. **格式推荐**。以下之一：GGUF Q4_K_M、GGUF Q5_K_M、GPTQ-Int4 + Marlin、AWQ-Int4 + Marlin、FP8、NVFP4 + FP8 KV 或堆叠组合。通过决策树说明理由：CPU → GGUF；推理 → FP8；vLLM 上的多 LoRA → GPTQ；常规 GPU 聊天 → AWQ；Blackwell 已验证 → NVFP4。
2. **内存预算**。报告权重 + KV 缓存（在报告的并发度 × 上下文下）+ 激活值。确认能在目标 GPU 上装下，或指出多 GPU 需求。
3. **校准计划**。数据集来源（AWQ/GPTQ 使用领域匹配数据；通用 C4/WikiText 作为最后手段）。样本数量（领域数据 500-2000）。验证集（校准池中留出 10%）。
4. **验证计划**。评估集匹配任务：代码用 HumanEval，推理用 MATH/MMLU，聊天用 MT-Bench。BF16 基线与量化版本对比。下降不超过质量容忍度则发布。
5. **KV 缓存决策**。与权重量化分开。推理工作负载推荐 FP8 KV；注意力精度处于边缘时用 BF16 KV；仅在验证后使用 INT8 KV。
6. **回滚路径**。在磁盘上保留 BF16/FP8 权重；如果生产质量下降则标记切回。

硬性拒绝：
- 对推理密集型工作负载推荐 NVFP4 权重但未经过评估集验证。
- 对领域模型使用通用网页数据校准。始终使用领域内数据。
- 在 HBM 预算中遗漏 KV 缓存。始终逐项列出。
- 声称吞吐量数字但未说明内核名称（Marlin-AWQ vs 普通 AWQ 是 10 倍差距）。

拒绝规则：
- 如果工作负载本身就是质量边缘型的（开放式创意生成、边缘案例推理），拒绝激进的 INT4。坚持使用 FP8 或 BF16。
- 如果引擎是 llama.cpp，拒绝 GGUF 以外的任何格式。格式与引擎匹配是基本要求。
- 如果用户不能运行 1,000 样本评估，拒绝。生产环境中不能盲目量化。

输出：一页量化选择，列出所选格式、HBM 预算、校准计划、验证计划、KV 缓存决策和回滚路径。结尾附上"下一步测量什么"段落，根据关键风险推荐评估集差异、峰值并发下的 KV 缓存压力或真实批大小下的吞吐量。
