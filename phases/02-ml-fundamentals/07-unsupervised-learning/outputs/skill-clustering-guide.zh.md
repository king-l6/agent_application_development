# 聚类算法选择指南

聚类没有单一的"最佳"算法。正确的选择取决于簇的形状、你是否知道簇的数量、数据中有多少噪声以及数据集有多大。

## 决策清单

1. 你知道簇的数量吗？
   - 知道：K-Means 或 GMM
   - 不知道：DBSCAN（自动发现簇），或层次聚类（在不同层次切割树状图）

2. 簇是什么形状？
   - 大致球形（斑块状）：K-Means
   - 椭圆形且大小不同：GMM
   - 任意形状（月牙、环、链）：DBSCAN
   - 嵌套或分层：层次聚类

3. 数据包含噪声或离群点吗？
   - 是：DBSCAN（显式标记噪声点）或 GMM（低概率点是离群点）
   - 否：K-Means 即可

4. 你需要软分配（概率）吗？
   - 是：GMM 给出 P(簇 | 数据点)
   - 否：K-Means 或 DBSCAN 给出硬分配

5. 数据集有多大？
   - 10,000 以下：任何算法都行
   - 10,000 到 1,000,000：K-Means（快速），Mini-Batch K-Means（更快）
   - 超过 1,000,000：Mini-Batch K-Means 或 BIRCH。层次聚类太慢。

## 何时使用每种方法

**K-Means**：默认起点。快速（O(n * k * 迭代)），简单，对许多问题足够好。使用肘部法或轮廓系数选择 K。局限性：假设球形簇，对初始化敏感（使用 K-Means++ 或多次运行），不能很好地处理不同大小的簇。

**DBSCAN**：最适合发现任意形状的簇并自动检测离群点。两个参数：eps（邻域半径）和 min_samples（最小密度）。不需要指定 K。局限性：当簇具有非常不同的密度时难以处理，调优 eps 可能很棘手。使用 k-距离图估计 eps：计算每个点的第 k 个最近邻距离，排序，寻找肘部。

**层次（凝聚式）聚类**：构建合并树。当你想要在多个粒度上探索簇结构（在不同高度切割树状图）时很有用。Ward 链接最适合紧凑的簇。单链发现拉长的簇但对噪声敏感。局限性：O(n^2) 内存和 O(n^3) 时间，因此对大数据集不实用。

**GMM（高斯混合模型）**：具有概率分配的软聚类。将每个簇建模为具有自己均值和协方差的高斯分布。当簇是椭圆形或重叠时优于 K-Means。使用 BIC（贝叶斯信息准则）选择成分数量。局限性：假设高斯分布，在非凸形状上可能失败，对初始化敏感。

## 评估聚类质量（无标签）

| 度量 | 它衡量什么 | 范围 | 何时使用 |
|------|----------|------|---------|
| 轮廓系数 | 内聚 vs 分离 | -1 到 1（越高越好） | 比较 K 值或算法 |
| 惯性（簇内平方和）| 簇的紧密程度 | 0 到 inf（越低越好） | K-Means 的肘部法 |
| BIC / AIC | 带复杂度惩罚的模型拟合 | 越低越好 | 选择 GMM 成分数量 |
| Calinski-Harabasz 指数 | 组间与组内方差比 | 越高越好 | 快速比较 |
| Davies-Bouldin 指数 | 簇之间的平均相似度 | 越低越好 | 惩罚重叠簇 |

## 常见错误

- 不缩放特征就运行 K-Means（尺度较大的特征主导距离计算）
- 在 2D 中通过目测选择 K，而实际数据是高维的（使用轮廓分数）
- 在非球形簇上使用 K-Means（月牙形或环形数据需要 DBSCAN）
- 设置 DBSCAN eps 太大（所有内容在一个簇中）或太小（所有内容都是噪声）
- 将簇标签视为真实标签（聚类是探索性的；用领域知识验证）
- 在超过 20,000 个点的数据集上运行层次聚类（内存和时间爆炸）

## 快速参考

| 算法 | 簇形状 | 找到 K | 处理噪声 | 软分配 | 可扩展性 |
|------|--------|--------|---------|--------|---------|
| K-Means | 球形 | 否（你设置 K） | 否 | 否 | 数百万 |
| Mini-Batch K-Means | 球形 | 否 | 否 | 否 | 数千万 |
| DBSCAN | 任意 | 是 | 是 | 否 | 数十万 |
| 层次聚类 | 任何（依赖链接） | 灵活（切割树状图） | 取决于链接 | 否 | 2 万以下 |
| GMM | 椭圆形 | 否（你设置 K） | 部分（低概率） | 是 | 10 万以下 |
| HDBSCAN | 任意 | 是 | 是 | 部分 | 数十万 |
