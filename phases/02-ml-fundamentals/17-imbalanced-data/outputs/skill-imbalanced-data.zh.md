# 不平衡数据策略

这是一个处理不平衡分类的决策清单。按照此顺序为你的问题选择合适的方法。

## 第1步：测量不平衡程度

- 统计每个类别的样本数
- 计算不平衡比（多数类 / 少数类）
- 轻度：比率 < 3:1（例如 70/30）
- 中等：比率 3:1 到 20:1（例如 95/5）
- 严重：比率 > 20:1（例如 99/1）

## 第2步：选择正确的指标

对不平衡数据集优先使用精确率/召回率/F1而非准确率。根据你的问题选择：

| 情况 | 主要指标 | 次要指标 |
|-----------|---------------|-----------------|
| 遗漏正例代价高昂（欺诈、疾病） | 召回率 | F2分数 |
| 误报代价高昂（垃圾邮件过滤、推荐） | 精确率 | F0.5分数 |
| 两者大致同样重要 | F1分数 | MCC |
| 需要一个单一的排名指标 | AUPRC | AUC-ROC |
| 需要在数据集间进行比较 | MCC | AUPRC |

## 第3步：选择重新平衡策略

### 按不平衡严重程度

| 不平衡程度 | 首选尝试 | 第二次尝试 | 避免 |
|-----------|-----------|------------|-------|
| 轻度（< 3:1） | 类别权重 | 阈值调优 | 过采样（没必要） |
| 中等（3:1 到 20:1） | SMOTE + 类别权重 | 在此基础上阈值调优 | 欠采样（数据丢失太多） |
| 严重（> 20:1） | SMOTE + 类别权重 + 阈值 | 平衡装袋集成 | 仅欠采样 |

### 按数据集大小

| 数据集大小 | 首选策略 | 原因 |
|-------------|-------------------|--------|
| < 1,000 样本 | 过采样或SMOTE | 不能承受丢失多数类数据 |
| 1,000 - 10,000 | SMOTE + 阈值调优 | 有足够的少数类样本进行k-NN |
| > 10,000 | 类别权重或欠采样 | 快速，有足够的少数类数据 |

## 第4步：应用技术

### 类别权重（始终先尝试）
- 在sklearn中：`class_weight='balanced'`
- 无需修改数据
- 适用于任何基于损失的模型
- 在期望值上等价于过采样

### SMOTE
- 仅应用于训练数据（绝不用于测试/验证）
- 使用k=5个邻居（默认）
- 与类别权重结合使用效果最佳
- 注意边界附近可能有噪声的合成点

### 阈值调优
- 训练模型，在验证集上获取预测概率
- 在0.05到0.95之间扫描阈值
- 选择最大化所选指标的阈值
- 始终在验证数据上调优，绝不在测试数据上

## 第5步：正确验证

- 使用分层交叉验证（保持每折中的类别比例）
- 在原始（未重采样）测试集上报告指标
- 绝不在划分前应用SMOTE——仅在训练折上应用
- 与"始终预测多数类"的基线进行比较

## 第6步：避免常见错误

- 在训练/测试划分前对整个数据集应用SMOTE（数据泄露）
- 使用准确率作为评估指标
- 不先尝试类别权重（最简单的方法，通常足够）
- 过采样后进行交叉验证（合成点跨折泄露）
- 忽略阈值调优（免费的性能提升，无需重新训练）
- 在小数据集上使用随机欠采样（丢弃太多数据）

## 快速决策树

1. 不平衡比 < 3:1？-> 仅尝试类别权重
2. 数据集 > 10,000 样本？-> 类别权重 + 阈值调优
3. 数据集 < 1,000 样本？-> SMOTE + 类别权重
4. 其他情况 -> SMOTE + 类别权重 + 阈值调优
5. 仍然不够好？-> 平衡装袋集成
