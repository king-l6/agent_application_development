# 特征选择策略

这是一个快速参考，用于选择和应用正确的特征选择方法。

## 第1步：从清理开始

在应用任何方法之前，先移除明显无用的特征：

- **常数特征**：方差 = 0。移除它们。
- **近常数特征**：方差 < 0.01（或你的阈值）。移除它们。
- **重复特征**：相同的列。保留一个，丢弃其余的。
- **ID列**：每行唯一的，不携带可泛化的信息。移除它们。

这只需要几秒钟，但可以消除混乱的真实世界数据集中10-30%的特征。

## 第2步：根据你的情况选择方法

### 快速决策树

1. **少于50个特征？** 从互信息排名开始。保留前K个。
2. **50 - 500个特征？** 先使用方差阈值，如果使用线性模型则用L1（Lasso），如果使用树则用树重要性。
3. **超过500个特征？** 串联方法：方差阈值 -> 互信息过滤（前50%）-> 对剩余特征用RFE。
4. **需要可解释性？** L1正则化给出精确的零/非零。树重要性给出排名的分数。
5. **需要捕捉非线性关系？** 互信息或基于树的重要性。避免L1（仅限线性）。
6. **需要特征交互？** RFE或基于树的重要性。过滤方法会遗漏交互。

### 方法参考

| 方法 | 何时使用 | 何时避免 |
|--------|------------|---------------|
| 方差阈值 | 始终，作为第一步 | 永远不要跳过这个 |
| 互信息 | 快速排名，非线性关系 | 当需要特征交互检测时 |
| RFE | 彻底选择，中等特征数量 | 非常昂贵的模型，超过1000个特征 |
| L1 / Lasso | 线性模型，快速嵌入选择 | 非线性问题，高度相关的特征 |
| 树重要性 | 非线性关系，特征交互 | 具有高基数特征时存在偏差 |
| 排列重要性 | 与模型无关的验证，最终检查 | 对初始筛选来说太慢 |

## 第3步：验证你的选择

- 比较使用选中特征与使用全部特征的模型性能
- 使用交叉验证，而不是单次训练/测试划分
- 如果性能下降超过1-2%，你可能移除了有用的特征
- 如果性能提升，你成功移除了噪声

## 第4步：处理常见陷阱

### 相关特征
- L1会从相关组中任意选择一个并将其他置零
- 首先计算相关矩阵，决定要保留哪些相关特征
- 树重要性会在相关特征之间分配重要性

### 数据泄露
- 仅在训练数据上拟合特征选择
- 将相同的选择应用到测试数据
- 在交叉验证中，特征选择必须在每折内部进行

### 过拟合到特征选择
- 迭代次数过多的RFE可能过拟合到训练集
- 在保留数据上验证，而不是在用于选择的数据上
- 使用稳定性选择（在子样本上重复）以获得更稳健的结果

## 第5步：生产清单

- [ ] 应用方差阈值作为第一道过滤器
- [ ] 仅在训练数据上拟合特征选择
- [ ] 记录选中的特征（名称、使用的方法、分数）
- [ ] 性能对比：选中特征 vs 全部特征
- [ ] 交叉验证评估，而非单次划分
- [ ] 特征选择集成到训练流程中（非手动执行）
- [ ] 监控特征漂移（选中的特征可能会过时）
