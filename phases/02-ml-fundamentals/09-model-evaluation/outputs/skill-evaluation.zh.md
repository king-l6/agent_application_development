# 模型评估策略

正确评估任何ML模型的清单。按此顺序操作，以避免最常见的评估错误。

## 第1步：正确分割数据

- 在任何预处理（缩放、填充、编码）之前进行分割
- 对分类任务使用分层分割
- 预留一个测试集，仅在最后使用一次
- 对于小数据集，使用5折或10折交叉验证代替单次分割
- 对于时间序列，使用基于时间的分割（绝不打乱）

## 第2步：选择正确的指标

### 分类

| 场景 | 使用此指标 | 原因 |
|-----------|----------------|-----|
| 均衡类别，简单比较 | 准确率 | 易于解释，类别均衡时有意义 |
| 假正例成本高（垃圾邮件过滤、欺诈告警） | 精确率 | 衡量被标记的样本中有多少实际为正 |
| 假负例成本高（癌症筛查、安全） | 召回率 | 衡量你捕获了多少实际正样本 |
| 需要平衡精确率和召回率 | F1分数 | 调和平均数，惩罚极端不平衡 |
| 跨阈值比较模型 | AUC-ROC | 与阈值无关的排序质量 |
| 不平衡数据 | F1、AUC-ROC或PR-AUC | 准确率在不平衡类别上具有误导性 |

### 回归

| 场景 | 使用此指标 | 原因 |
|-----------|----------------|-----|
| 标准回归，可接受异常值 | RMSE | 与目标单位相同，惩罚大误差 |
| 鲁棒于异常值的评估 | MAE | 平等对待所有误差，不被异常值主导 |
| 比较不同尺度的模型 | R平方 | 归一化0-1尺度（解释的方差比例） |
| 业务需要美元金额 | MAE或RMSE | 直接解释为误差量级 |

## 第3步：建立基线

在评估模型之前，计算基线性能：
- 分类：多数类预测器（总是预测最常见类别）
- 回归：总是预测训练目标的均值
- 任何无法击败这些基线的模型都不是在学习

## 第4步：交叉验证

- 使用K折（K=5或K=10）以获得稳定估计
- 对分类使用分层K折
- 报告折间的均值和标准差
- 均值为0.85、标准差为0.02的模型比均值为0.87、标准差为0.10的更可信

## 第5步：统计比较模型

- 不要只选择平均分数最高的模型而不检查显著性
- 在交叉验证折上使用配对t检验
- 如果|t| < 2.78（对于K=5, df=4, p<0.05），差异可能来自偶然
- 当性能差异不显著时，考虑更简单的模型

## 第6步：检查常见错误

- 数据泄漏：是否有任何测试数据信息流入训练？（分割前缩放、从目标衍生的特征）
- 类别不平衡：准确率是否隐藏了差的少数类性能？
- 过拟合：训练和验证性能之间的差距是否很大？
- 评估过多：你是否多次查看了测试集？

## 第7步：报告最终性能

- 在训练+验证组合上训练
- 在保留的测试集上评估恰好一次
- 报告所选指标，尽可能附置信区间
- 说明基线对比（比随机/均值好多少）
