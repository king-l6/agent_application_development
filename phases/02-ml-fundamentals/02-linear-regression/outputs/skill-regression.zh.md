# 回归策略指南

回归预测连续值。正确的方法取决于特征与目标之间的关系、特征数量以及过拟合风险。

## 决策清单

1. 特征与目标之间的关系大致是线性的吗？
   - 是：从普通线性回归开始
   - 否：尝试多项式特征或非线性模型

2. 相对于样本数量，你有多少特征？
   - 特征少，样本多：普通线性回归效果良好
   - 特征多，样本少：使用正则化（Ridge 或 Lasso）
   - 特征比样本多：Lasso（L1）用于特征选择，或 Ridge（L2）用于收缩所有权重

3. 你需要可解释性吗？
   - 是：使用特征少的线性回归，或 Lasso 进行自动特征选择
   - 否：使用多项式特征，或转向树模型或神经网络

4. 你的数据集小吗（少于 10,000 行）？
   - 使用正规方程（闭式解）以获得速度
   - 交叉验证对于可靠的评估至关重要

5. 你的数据集大吗（数百万行）？
   - 使用随机梯度下降（SGD）或小批量梯度下降
   - 正规方程因 O(n^3) 的矩阵求逆而太慢

## 何时使用每种方法

**普通线性回归**：任何回归任务的基线。从这里开始。如果 R 平方可接受且模型简单，就到此为止。

**多项式回归**：散点图显示的是曲线而非直线。从 2 次开始。只有在验证性能证明合理时才增加次数。次数 > 5 几乎总是过拟合。

**Ridge 回归（L2）**：许多相关特征。所有权重都向零收缩，但没有一个完全为零。当你认为所有特征都有贡献时使用。

**Lasso 回归（L1）**：特征多且你怀疑只有少数特征重要。Lasso 将不相关特征的权重驱动到恰好为零，执行自动特征选择。

**Elastic Net**：结合 L1 和 L2 惩罚。当你有很多相关特征且希望进行一些特征选择时使用。

## 常见错误

- 在梯度下降前跳过特征缩放（收敛变得极其缓慢）
- 使用测试集性能来调整超参数（应使用验证集或交叉验证）
- 拟合高次多项式而不检查验证误差（训练 R^2 总是随次数增加而增加）
- 忽略残差图（如果残差显示模式，R^2 可能具有误导性）
- 将 R^2 视为唯一指标（检查残差分布、MAE 和领域特定阈值）

## 快速参考

| 方法 | 何时使用 | 正则化 | 特征选择 |
|--------|------------|---------------|-------------------|
| OLS | 基线，特征少 | 无 | 手动 |
| Ridge | 特征多，全部相关 | L2（收缩） | 否 |
| Lasso | 特征多，少数相关 | L1（归零） | 自动 |
| Elastic Net | 许多相关特征 | L1 + L2 | 部分 |
| 多项式 | 非线性关系 | 在此基础上加 Ridge/Lasso | 手动选择次数 |
