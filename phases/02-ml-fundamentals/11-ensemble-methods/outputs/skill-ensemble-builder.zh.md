# 集成方法选择指南

集成组合多个模型以产生比任何单一模型更好的预测。问题始终是：哪种类型的集成，以及何时使用？

## 决策清单

1. 你当前模型的主要问题是什么？
   - 高方差（过拟合）：使用bagging（随机森林）
   - 高偏差（欠拟合）：使用boosting（梯度提升、XGBoost）
   - 两者都有，或者你想要最大精度：使用stacking

2. 你有多少数据？
   - 1000行以下：随机森林（鲁棒，难以配置错）
   - 1000到100,000：XGBoost或LightGBM（表格数据总体最佳）
   - 超过100,000：LightGBM（最快的梯度提升，很好地处理大数据）

3. 你可以投入多少调参时间？
   - 最少：使用默认参数的随机森林（几乎总是有效）
   - 中等：XGBoost，learning_rate=0.1，用早停调n_estimators
   - 最大：LightGBM或XGBoost配合贝叶斯超参数搜索

4. 你需要可解释性吗？
   - 是：单决策树或带特征重要性的小随机森林
   - 部分：带SHAP值的梯度提升
   - 否：stacking或深度集成

5. 数据是否含有许多异常值的噪音？
   - 是：随机森林（bagging对噪音鲁棒）
   - 否：梯度提升（可以在干净数据上进一步提高精度）

## 何时使用每种方法

**随机森林（Bagging）**：你安全的首选。在自助样本上训练许多树并平均。降低方差而不增加偏差。在中等数据上几乎不可能过拟合。需要最少的调参：设置n_estimators=100-500，其余保持默认。

**AdaBoost**：带样本重新加权的顺序提升。与简单基学习器（决策桩）配合良好。对异常值和噪音标签敏感，因为它加大了错误分类点的权重。在实践中大部分已被梯度提升取代。

**梯度提升**：将每棵新树拟合到当前集成的残差。降低偏差。是表格数据最强大的方法。需要调参：learning_rate、n_estimators、max_depth、min_child_weight、subsample。

**XGBoost**：带正则化、二阶优化和系统级加速的梯度提升。原生处理缺失值。Kaggle比赛和生产ML中表格数据的默认选择。

**LightGBM**：带按叶子生长（而非按层级生长）的梯度提升。在大数据集上比XGBoost更快。使用基于直方图的分裂。最适合超过5万行的数据集。

**CatBoost**：带原生类别特征处理的梯度提升。无需独热编码。当你有许多类别特征时很好用。

**Stacking**：在多个多样基模型的预测上训练元学习器。当你需要绝对最佳精度且有计算资源时使用。始终通过交叉验证生成基模型预测以避免泄漏。

**投票**：最简单的集成。硬投票（多数类）或软投票（平均概率）。快速组合2-3个多样模型，无需元学习器。

## 常见错误

- 使用无早停的梯度提升（如果你让它运行太多轮次，它会过拟合）
- 设置学习率太高（0.3以上通常导致不稳定）
- 不调优梯度提升的max_depth（默认无限制或非常深的树会过拟合）
- Stacking中使用相同类型的模型（多样性是stacking的核心）
- 在噪音数据上使用AdaBoost（异常值每轮获得越来越高的权重）
- 期望随机森林修复欠拟合（它降低方差，而非偏差）

## 按方法排序的调参优先级

**随机森林：**
1. n_estimators: 100-500（更多很少更差，只是更慢）
2. max_depth: None（让树充分生长）或限制在10-20以提升速度
3. max_features: 分类用"sqrt"，回归用"log2"或n/3

**XGBoost / LightGBM：**
1. learning_rate: 0.01-0.3（越低越好，如果你有计算资源支持更多树）
2. n_estimators: 在验证集上用早停代替猜测
3. max_depth: 3-8（从6开始）
4. min_child_weight / min_data_in_leaf: 1-20（越高防止过拟合）
5. subsample: 0.7-1.0
6. colsample_bytree: 0.7-1.0
7. reg_alpha (L1) 和 reg_lambda (L2): 0-10

## 快速参考

| 方法 | 降低 | 速度 | 调参工作量 | 最适合 |
|--------|---------|-------|--------------|----------|
| 随机森林 | 方差 | 快 | 低 | 噪音数据、快速基线 |
| AdaBoost | 偏差 | 快 | 低 | 简单基学习器、干净数据 |
| 梯度提升 | 偏差 | 中等 | 高 | 表格数据、比赛 |
| XGBoost | 两者 | 快 | 高 | 生产表格ML |
| LightGBM | 两者 | 最快 | 高 | 大数据集（5万+行） |
| CatBoost | 两者 | 中等 | 中等 | 多类别特征 |
| Stacking | 两者 | 慢 | 高 | 最大精度、多样模型 |
| 投票 | 方差 | 快 | 无 | 快速组合2-3个模型 |
