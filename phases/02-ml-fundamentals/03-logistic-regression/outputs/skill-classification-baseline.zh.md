# 分类基线指南

在尝试复杂模型之前，先用逻辑回归建立一个基线。它秒级训练、输出概率，并且完全可解释。大量实际问题根本不需要更花哨的方法。

## 决策清单

1. 决策边界很可能是线性的吗？
   - 是：逻辑回归可能就足够了
   - 否：你仍然需要它作为基线来衡量改进程度

2. 你有多少个特征？
   - 50 以下：标准逻辑回归效果良好
   - 50 到 10,000：添加 L2 正则化（Ridge）
   - 超过 10,000（例如，TF-IDF 文本特征）：使用 L1 正则化（Lasso）或 LinearSVC

3. 数据集不平衡吗？
   - 比率低于 5:1：不调整可能也没问题
   - 5:1 到 50:1：在 sklearn 中使用 `class_weight="balanced"`
   - 超过 50:1：结合类别权重与适当的指标（精确率、召回率或 F1）

4. 特征在不同的尺度上吗？
   - 逻辑回归前始终进行标准化。它使用基于梯度的优化，未缩放的特征会减慢收敛或扭曲决策边界。

5. 有缺失值吗？
   - 在拟合之前进行填充。逻辑回归无法处理 NaN。
   - 数值列使用中位数填充，分类列使用众数填充。

## 何时逻辑回归足够好

- 特征关系基本线性的二分类问题
- 你需要概率输出（不仅仅是类别标签）
- 需要可解释性（标准化后系数指示特征重要性的方向和相对大小）
- 训练数据小（几百到几千个样本）
- 你需要快速模型用于实时服务（推理时为单个点积）
- 监管或合规要求可解释性

## 何时升级

- 准确率远低于目标且你已经尝试了特征工程
- 特征与目标之间的关系明显是非线性的（检查残差图）
- 你有大型表格数据（10k+ 行）：尝试梯度提升（XGBoost 或 LightGBM）
- 特征具有多项式特征无法捕捉的复杂交互
- 你有图像、文本或序列数据：逻辑回归在原始输入上无法工作

## 分类基线的预处理步骤

1. 首先进行**训练/测试分割**，在任何预处理之前。这可以防止数据泄露。
2. **处理缺失值**：数值列用中位数填充，分类列用众数填充。
3. **编码分类变量**：低基数（少于 10 个值）用独热编码，更高基数用目标编码。仅在训练折上拟合目标编码（使用折外编码以防止泄露）。
4. **缩放数值**：StandardScaler（零均值、单位方差）。在训练集上拟合，在两者上变换。
5. **拟合逻辑回归**，使用 `C=1.0`（默认正则化）。
6. **评估**：混淆矩阵、精确率、召回率、F1。不仅仅是准确率。
7. **调整阈值**：默认 0.5 很少是最优的。从 0.1 到 0.9 扫描，选择符合你精确率/召回率优先级的阈值。

## 常见错误

- 在不平衡数据上仅评估准确率（预测多数类的模型得分高但毫无用处）
- 忘记缩放特征（未缩放的逻辑回归训练缓慢且收敛到更差的解）
- 使用测试集来调整决策阈值（应使用验证集或交叉验证）
- 跳过基线直接跳到 XGBoost（你失去了可解释性并且没有参考点）
- 没有检查多重共线性（高度相关的特征会使系数方差膨胀）

## 快速参考

| 场景 | 模型 | 正则化 | 关键设置 |
|----------|-------|---------------|-------------|
| 特征少，可解释 | LogisticRegression | L2（默认） | C=1.0 |
| 特征多，部分无关 | LogisticRegression | L1 | penalty="l1", solver="saga" |
| 高维稀疏（文本） | SGDClassifier | L1 或 ElasticNet | loss="log_loss" |
| 不平衡类别 | LogisticRegression | L2 | class_weight="balanced" |
| 需要概率 | LogisticRegression | L2 | predict_proba() |
| 仅需类别标签 | LinearSVC | L2 | 对于大数据比 LR 更快 |
