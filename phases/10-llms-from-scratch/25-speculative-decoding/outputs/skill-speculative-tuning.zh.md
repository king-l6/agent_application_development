给定目标模型（大小、家族、tokenizer）、工作负载遥测（任务组合、提示与解码 token 比率、p50/p99 解码延迟、加速器和 HBM 余量、平均批大小、采样温度分布）以及可用的草稿检查点，输出：

1. 草稿选择。从同家族小型（Llama-70B 的 Llama-3.2-1B）、蒸馏草稿（Qwen3-0.6B-spec）、固定在目标上的 Medusa 头或"无投机解码"（如果没有草稿的 FLOP 成本比率在 30% 以内）中选择。确认 tokenizer 逐字节匹配目标；拒绝不匹配的 tokenizer。
2. 草稿长度 K。E[tokens] / (1 + K x c) 的 argmax，其中 c 是草稿与目标成本比。使用来自 5_000 个分布内 token 校准运行的测量 α，展示 K 为 2、3、4、5、6 的工作。默认 K=4 用于聊天，K=6 用于代码，K=2 用于高温度创造性写作。
3. 温度门控。设置一个温度阈值，超过该阈值禁用投机解码。默认 0.8；如果校准显示 α 更早崩溃则降至 0.6。拒绝任何依赖于每个请求检查（增加超过 50 微秒）的温度门控。
4. 树预算。如果服务栈支持树草稿，为 8 以下的批次选择小的固定树（深度 2，分支 3-2）；32 以上的批次使用平面链。说明验证器的 KV 暂存大小（字节）并确认它适合 HBM 余量。
5. 回退策略。命名指标（过去 1_000 次验证的滑动窗口测量 α）和阈值（α 低于 0.4），此时服务器为该请求流回退到普通自回归解码。包括回退决策的每个请求生命周期。

拒绝在批大小超过验证器变为计算受限的点时使用投机解码。超过该点，投机器旨在吸收的未使用 FLOPs 不再存在；吞吐量下降。拒绝任何测量 α 低于 0.4 的任务家族的投机解码；草稿开销占主导，墙钟延迟变差。拒绝未在针对目标的保留 1_000 token 样本上验证的草稿：未验证的草稿是静默的 KL 漂移。

示例输入："Llama-3.3-70B on 8xH100, 聊天工作负载, 批次 16, p50 解码 28 ms, p99 60 ms, 温度分布均值 0.4 / 最大值 1.2, 校准显示 α 在聊天上为 0.78, 代码上为 0.61。"

示例输出：
- 草稿：Llama-3.2-1B-Instruct-spec。相同的 tokenizer，相同的家族，比率 c 约 0.03。
- K：4。E[tokens/验证] = 聊天 3.4，代码 2.5。K=5 聊天增益 0.1 token，支付额外 0.03 c；拒绝。
- 温度门控：0.8。高于 0.8 时，α 在校准集上降至 0.45 以下。
- 树预算：深度 2 分支 (3, 2)。批次 16 下的 KV 暂存 480 MB 适合。
- 回退：过去 1_000 次验证的滑动窗口 α 低于 0.40 时，对该流禁用投机解码 30 秒，然后再次探测。
