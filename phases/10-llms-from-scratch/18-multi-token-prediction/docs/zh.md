# 多 token 预测（MTP）

> 从 GPT-2 到 Llama 3 的每个自回归 LLM 在每个位置上训练一个损失：预测下一个 token。DeepSeek-V3 在每个位置上增加了一个额外的损失：预测再下一个 token。额外的 14B 参数（在 671B 模型上）通过梯度流蒸馏回主模型，训练好的 MTP 头在推理时被重新用作投机解码的草稿器，接受率超过 80%。1.8 倍的生成吞吐量免费获得。本节课从 DeepSeek 技术报告中构建了顺序 MTP 模块，计算了损失和共享头参数布局，并解释了为什么 MTP 保持了因果链，而 Gloeckle 等人的原始并行 MTP 破坏了它。

**类型：** 构建
**语言：** Python（标准库）
**前置知识：** 阶段 10 · 04（预训练迷你 GPT），阶段 10 · 15（投机解码）
**时间：** ~60 分钟

## 学习目标

- 陈述 MTP 训练目标并推导跨预测深度的联合损失。
- 解释 Gloeckle 等人的并行 MTP 头（2024）与 DeepSeek-V3 的顺序 MTP 模块之间的区别，以及为什么顺序设计保留了因果链。
- 计算向预训练运行添加 MTP 模块的参数和内存开销。
- 从头实现一个 MTP 模块：共享嵌入、每个深度的 Transformer 块、投影和共享输出头。

## 问题

下一个 token 预测是标准 LLM 训练目标。每个隐藏状态被监督来预测一件事：紧随其后的 token。这是一个令人惊讶地弱的信号。序列中的大部分信息延伸超过一个 token——结构、连贯性、事实性、算术流程。模型必须通过累积数万亿 token 上的许多单 token 信号来学习这些。

MTP 提出的问题是：如果每个隐藏状态被同时监督来预测多个未来 token 会怎样？Gloeckle 等人（Meta, 2024）证明了这有帮助。他们的实现将几个独立的输出头放在骨干网络之上，每个预测不同的偏移量。并行的、简单的，但这些头看到了相同的隐藏状态而没有层次化细化——而且预测没有因果链式连接，因此它们不能用于投机解码。

DeepSeek-V3（2024 年 12 月）将 MTP 重新设计为顺序模块，在每个预测深度保持因果链。模型从 `h_i^(0)` 预测 `t+1`，然后从新的隐藏状态 `h_i^(1)` 预测 `t+2`，该状态结合了 `h_i^(0)` 和 `E(t+1)` 嵌入，依此类推。每个深度是它自己的小型 Transformer 块。共享嵌入和共享输出头保持了参数开销的适度。在 DeepSeek-V3 的规模下，MTP 模块在 671B 主模型权重之上的额外参数为 14B。这 2% 的开销带来了更密集的训练信号和在推理时一个现成的投机解码草稿器。

本节课从头构建了一个 MTP 模块和 D 深度损失。数学简洁。实现约 150 行。

## 概念

### 顺序 MTP 配方

DeepSeek-V3 在主模型之上添加了 `D` 个 MTP 模块。每个模块 `k`（`k = 1..D`）预测深度 `k` 处的 token——即给定通过位置 `i` 的前缀，预测 `t_{i+k}`。

模块 `k` 包括：

- 一个 Transformer 块 `T_k`，带有自己的注意力和 MLP。
- 一个投影矩阵 `M_k`，用于结合前一个深度的隐藏状态和下一个深度真实 token 的嵌入。
- 共享嵌入 `E`（与主模型相同）。
- 共享输出头 `Out`（与主模型相同）。

在训练中，对于通过位置 `i` 的前缀，每个深度的隐藏状态为：

```
h_i^(0) = main model backbone at position i
h_i^(k) = T_k( M_k * concat(RMSNorm(h_i^(k-1)), RMSNorm(E(t_{i+k}))) )   for k >= 1
```

每个深度的预测为：

```
logits_{i+k} = Out(h_i^(k-1))   for k = 1..D
```

每个深度的损失是对真实 `t_{i+k}` 的交叉熵：

```
L_k = CE(logits_{i+k}, t_{i+k})
```

跨深度的联合损失：

```
L_MTP = (lambda / D) * sum_{k=1..D} L_k
```

`lambda` 是一个小的加权因子——DeepSeek-V3 在训练的前 10% 使用 0.3，之后使用 0.1。总训练损失为 `L_main + L_MTP`。

### 为什么是顺序而非并行

Gloeckle 的原始并行 MTP 有 D 个输出头，每个直接应用于 `h_i^(0)`。每个头从相同的骨干隐藏状态预测 `t_{i+k}`。这训练没问题，但预测不是相互条件化的。你不能使用 `head_1` 的输出来帮助 `head_2`——头是并行触发的。

DeepSeek-V3 的顺序设计从 `h_i^(k-1)` 加上实际的下一个 token 嵌入 `E(t_{i+k})` 构建 `h_i^(k)`。这保留了因果链：为了预测 `t_{i+k+1}`，深度 `k+1` 的模块看到了 `t_{i+k}` 处的内容。这在结构上与自回归解码器消费自己输出的方式相同——使得 MTP 模块可以直接用作投机解码的草稿器。

推理时：将 `h_i^(k-1)` 和草稿 `t_{i+k}` 输入模块 `k+1`，得到 `t_{i+k+1}` 的预测。重复。这正是 EAGLE 风格的草稿，使用训练好的 MTP 模块作为草稿网络。DeepSeek-V3 报告第一个 MTP 模块的接受率超过 80%，加速约 1.8 倍。

### 参数核算

对于一个隐藏为 `h`、词汇表大小为 `V` 的模型：

- 主模型：数十亿参数，加上一个大小为 `V * h` 的输出头。
- 共享输出头：复用主模型的头。无额外参数。
- 共享嵌入：复用主模型的嵌入。无额外参数。
- 每个 MTP 模块：
  - 投影 `M_k`：`(2h) * h = 2h^2`。
  - Transformer 块 `T_k`：注意力（MHA 为 `4h^2`）加 MLP（SwiGLU 通常为 `8h^2`，比例为 8/3）。每个块约 `12h^2`。

每个模块的额外参数总计：`~14h^2`。对于 DeepSeek-V3 的 `h = 7168`，D = 1 模块：`~14 * 7168^2 = ~720M` 参数（理论上）。DeepSeek-V3 报告 14B——差异主要在于 MTP 模块中的专家层也是 MoE。

### 投机解码的回报

在预训练期间，MTP 模块使训练速度减慢约 10%（更多的前向计算，额外的损失）。回报是双重的：

1. 更密集的训练信号。每个隐藏状态看到 D+1 个监督目标。在 DeepSeek-V3 的消融实验中，MMLU、GSM8K、MATH、HumanEval 上持续改善了几个百分点。

2. 推理时免费的投机解码草稿。MTP 模块已经训练好预测接下来的几个 token。重新用作草稿网络，可提供 80% 以上的接受率。在这个水平上，N=3 或 N=5 的投机解码可提供 1.8 倍的吞吐量。10% 的训练时间成本在第一次运行推理时就已回本。

### 与 EAGLE 的关系

EAGLE 在预训练之后**单独**训练一个小型草稿模型。MTP 将草稿融入预训练。两种方法趋近于相似的接受率，但通过不同的流程：

| 维度 | EAGLE-3 | MTP（DeepSeek-V3） |
|-----------|---------|------------------|
| 训练时机 | 预训练后 | 预训练期间 |
| 与现有权重向后兼容 | 是 | 否（需要重新训练） |
| 草稿参数 | 1-2 个 Transformer 层 | 1 个 Transformer 块 + 投影 |
| 接受率 | 0.88-0.92 | 深度 1 处 0.80+ |
| 除加速外的收益 | 仅投机解码 | 更密集的训练信号 + 加速 |

## 构建

`code/main.py` 端到端构建了一个 MTP 模块：共享嵌入、投影、Transformer 块、共享输出头。然后计算在短合成序列上的每个深度交叉熵损失，并按组件打印参数计数。一个 32 个 token 的玩具词汇表使数字保持可读。

### 步骤 1：共享嵌入表

单个 `vocab_size x hidden` 表被主模型**以及**每个 MTP 模块的每个深度使用。不是第二个副本——字面意义上的同一个张量。

### 步骤 2：每个深度的组合

```python
def combine(prev_hidden, next_token_embed, M_k):
    # 沿特征维度拼接，然后投影到 hidden
    concat = rms_norm(prev_hidden) + rms_norm(next_token_embed)  # 向量加法替代
    projected = matvec(M_k, concat)
    return projected
```

真实的 DeepSeek-V3 将两个 RMSNorm 后的向量拼接为 `[2h]`，并用 `h x 2h` 矩阵投影。玩具实现为简洁起见使用向量加法。

### 步骤 3：深度 k 处的 Transformer 块

自注意力加 MLP。在玩具中，一个单层线性注意力块和一个 SwiGLU MLP 在无需 numpy 的情况下保持了结构的可见性。

### 步骤 4：共享输出头

复用主模型的输出投影。对词汇表的 logits。

### 步骤 5：每个深度的损失

softmax(logits) 对深度 `k` 处的真实 token 的交叉熵。使用 `lambda / D` 缩放因子聚合到跨深度。

### 步骤 6：参数核算

打印总参数数量、共享（嵌入、头）数量以及每个模块额外数量。显示 MTP 额外参数与主模型大小的比例。

## 使用

MTP 已集成到 DeepSeek-V3（2024 年 12 月）和 DeepSeek-R1 系列中。推理时：

- DeepSeek 自己的服务栈开箱即用地将 MTP 模块用作投机解码器。
- 截至 2026 年 4 月，vLLM 和 SGLang 已有 DeepSeek-V3 MTP 的集成路径。
- AMD 的 ROCm SGLang 教程展示了一个特定的 MTP 投机解码配置，在 V3 检查点上测得 1.8 倍加速。

何时在新的预训练运行中使用 MTP：

- 你控制完整的预训练流程，并希望获得更密集的训练信号。
- 你知道你将大规模服务该模型，并希望免费获得投机解码。
- 你的隐藏大小至少为 4096。在 1B 规模下，开销大于收益。

何时不使用：

- 微调现有的预训练密集模型。MTP 模块未经过训练。
- 研究模型中你想要一个干净的基线进行比较。MTP 改变了架构。

## 练习

1. 运行 `code/main.py`。展示每个深度的损失随着合成信号增强而单调递减。修改合成数据使用固定模式，并验证深度 1 和深度 2 的损失都收敛。

2. 计算一个密集 70B 模型（隐藏层 8192，80 层）使用 D=1 MTP 模块的参数开销。与 DeepSeek-V3 报告的 14B 开销进行比较。解释为什么 DeepSeek 的数字更高：MTP Transformer 块继承了相同的 MoE 结构，增加了每个模块的参数计数。

3. 在玩具中实现 D=2：添加第二个 MTP 模块，接收 h^(1) 并预测 `t_{i+2}`。验证联合损失和参数核算与 DeepSeek 论文的公式 19-21 一致。

4. 将玩具切换到并行 MTP（Gloeckle 风格）：在主隐藏状态之上添加 D 个输出头，每个预测不同的偏移量。测量与相同合成信号下的顺序版本相比，每个深度的损失如何。顺序版本应在 k > 1 时产生更低的深度 k 损失，因为它以中间预测为条件。

5. 将训练好的 MTP 模块用作 EAGLE 风格的草稿：调用模块 k 在推理时提出 `t_{i+k}`。测量这些草稿 token 对主模型在保留序列上的预测的接受率。如果你在玩具上达到 50%+，你就重现了 MTP 作为草稿的经验性质。

## 关键术语

| 术语 | 人们说的 | 实际含义 |
|------|----------------|------------------------|
| MTP 模块 | "额外损失块" | 一个小型 Transformer 块加投影，预测主模型前方 `k` 个位置的 token |
| 预测深度 | "哪个偏移量" | 整数 `k`，使得模块 `k` 从通过位置 `i` 的前缀预测 `t_{i+k}` |
| 并行 MTP | "Gloeckle 风格" | 在相同骨干隐藏状态上的 D 个独立头，无条件链 |
| 顺序 MTP | "DeepSeek-V3 风格" | 每个模块以前一深度的隐藏状态加下一个 token 的嵌入为条件；保留因果链 |
| 共享输出头 | "复用主头" | MTP 模块调用主模型的 LM 头，而非单独的输出投影 |
| 共享嵌入 | "复用主表" | 相同的词汇嵌入表在所有地方使用；无重复参数 |
| 投影矩阵 M_k | "结合隐藏 + 下一 token" | 一个 `h x 2h` 线性层，将前一隐藏状态和目标 token 嵌入折叠为下一深度的输入 |
| 联合损失 L_MTP | "平均额外损失" | 每个深度交叉熵损失的算术平均值，乘以 `lambda` |
| 深度 1 接受率 | "MTP 草稿正确的频率" | D=1 MTP 模块的 top-1 预测等于主模型 top-1 预测的比率；DeepSeek-V3 上为 80%+ |
| Lambda 加权 | "额外损失重要性" | 每个深度的缩放因子；DeepSeek-V3 上训练开始时为 0.3，之后为 0.1 |
