给定一个长上下文预训练运行规范（目标上下文、基础架构、可用训练 token、GPU 拓扑、部署目标），生成一个 NSA 集成计划。

生成：

1. 压缩块大小 `l`。选择 32、64 或 128。针对目标上下文进行论证：`l = 32` 用于 16k-32k，`l = 64` 用于 64k-128k，`l = 128` 用于 256k 以上。更大的 `l` 意味着更少的压缩键，但路由信号更粗粒度。
2. Top-k 选择数量。选择 8 到 32 之间。论文的默认值是 16。针对目标任务组合进行论证：推理密集型任务（数学、代码）受益于更高的 `k`，因为选择精度更重要。检索密集型任务在较低的 `k` 下也能工作。
3. 滑动窗口 `W`。选择 256、512 或 1024。默认 512。对于结构化密集的内容（代码）使用更短的窗口，局部上下文就足够；对于散文使用更长的窗口。
4. 门控 MLP。指定宽度和初始化。默认：从 `hidden` 到 3 的线性层，使用 `sigmoid` 或 `softplus` 激活。如果门控权重坍缩到偏好一个分支则发出警告——这表明 `l`、`k` 或 `W` 调整不当。
5. 内核选择。确认目标加速器上 Triton 或 CUDA 内核的可用性。拒绝在推理时回退到密集注意力（NSA 的全部意义在于节省解码计算）。如果只有前向内核而没有反向内核，拒绝预训练并推荐在现有密集检查点上进行继续训练。

硬性拒绝：
- 在没有继续预训练的情况下，将 NSA 应用于使用密集注意力预训练的模型。不能在推理时后装。
- 目标上下文低于 16k。三分支的开销占主导地位。
- 在不支持 NSA 内核的推理栈上的仅推理部署。推荐使用 MLA 或滑动窗口注意力。

拒绝规则：
- 如果长上下文评估数据（RULER、LongBench、needle-in-haystack）不可用，拒绝并请求先校准数据。
- 如果训练数据上下文分布以短序列为主，拒绝并推荐在集成 NSA 之前重新加权数据。
- 如果加速器早于 A100，拒绝——NSA 的内核优势假设 H100/H200/MI300 内存层次结构。

输出：一页集成计划，列出 `l`、`k`、`W`、门控配置、内核路径以及目标上下文下的预期计算节省量。以"成功标准"段落结束：具体的 RULER 或 LongBench 数值（与匹配的密集注意力基线的百分比差距），证明保留 NSA 的合理性。包括回滚触发条件——低于该指标阈值时，架构应回退到 MLA 或密集 GQA。
