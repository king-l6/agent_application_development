# Jamba — 混合 SSM-Transformer

> 状态空间模型（SSM）和 Transformer 追求不同的目标。Transformer 以二次成本换取注意力质量。SSM 通过循环实现线性时间推理和恒定内存，但质量落后。AI21 的 Jamba（2024 年 3 月）和 Jamba 1.5（2024 年 8 月）将两者放在同一个模型中：每 7 个 Mamba 层配 1 个 Transformer 层，每隔一个块使用 MoE，以及一个适合单个 80GB GPU 的 256k 上下文窗口。Mamba-3（ICLR 2026）使用复数值状态空间和 MIMO 投影加强了 SSM 方面。本节课端到端地阅读两种架构，并解释了为什么混合配方在纯 SSM 和纯 Transformer 长上下文尝试都失败的情况下经受住了三年的规模化考验。

**前置知识：** 阶段 10 · 14（开放模型架构），阶段 10 · 17（原生稀疏注意力）

## 问题

注意力在序列长度上是二次的。状态空间模型是线性的。这个差异会复合：在 256k token 下，一个 Transformer 注意力图每头有 65B 个条目；一个 SSM 的循环状态是固定大小的，与序列长度无关。

纯 SSM 模型（Mamba、Mamba-2）在小规模上匹配 Transformer 的困惑度，但在状态追踪任务上落后，并在某些类别的上下文检索中失败。直觉：SSM 将历史压缩为固定状态，当历史很长时，信息会泄漏。注意力精确地记住一切，但付出二次成本。

明显的修复：两者都用。在精确召回重要的地方放置 Transformer 层。在其他地方使用 SSM 层。调整比例。Jamba 是第一个大规模交付这种混合配方的生产级模型（52B 总计，12B 活跃，256k 上下文，单个 80GB GPU）。Jamba 1.5 将家族扩展到 398B 总计 / 94B 活跃。Mamba-3（ICLR 2026）是混合模型可以围绕其重建的当前最佳纯 SSM 基线。

本节课阅读所有三篇论文，并产生"选择正确比例"的心智模型。

## 概念

### 一页理解 SSM

状态空间模型通过固定大小的状态 `h` 处理序列 `x_1, ..., x_N`：

```
h_t = A h_{t-1} + B x_t
y_t = C h_t
```

每一步，状态通过线性动力学 `A` 演化，接受输入 `B x_t`，并发出输出 `C h_t`。`A, B, C` 可以学习。注意关键属性：计算 `y_t` 只需要 `h_{t-1}` 和 `x_t`，不需要任何更早的 `x`。内存是恒定的。推理是每个 token O(1)。

建模质量的诀窍在于 `A` 的结构。S4（Gu 2021）使用了一个高度结构化的矩阵，可以在训练期间作为长卷积高效评估。Mamba（Gu, Dao 2023）用数据依赖的参数替换了固定的 `A, B, C`（"选择性"部分）。Mamba-2（2024）进一步简化了结构。Mamba-3（2026）在特定位置重新增加了复杂性。

关键属性：对于解码器 LLM，SSM 层是注意力层的即插即用替代品，具有固定大小的每层状态，而不是不断增长的 KV 缓存。

### Jamba 块

Jamba 块根据两个数字交错层：

- `l`：注意力与 Mamba 的比例。Jamba 使用 `l = 8`，意味着每 7 个 Mamba 层配 1 个 Transformer 层（7 Mamba + 1 注意力 = 每组 8 层）。
- `e`：MoE 频率。Jamba 使用 `e = 2`，意味着每隔一层应用 MoE。

一个块内的层序列：

```
M  M  M  M  M  M  M  A    (7 Mamba + 1 注意力)
|  M  |  M  |  M  |  M    (| 标记应用 MoE 的位置)
```

每个 Jamba 块有 8 层。在 4 个块深（总共 32 层）时，你得到 28 个 Mamba 层和 4 个注意力层。其中 16 个使用 MoE。

### 为什么是 1:7 比例

AI21 运行了消融实验：注意力与 Mamba 的什么比例在其长上下文评估中给出最佳的每参数困惑度和上下文召回？

- 注意力过多（1:1）：质量上升但内存和速度下降。
- 注意力过少（1:15）：内存很好但上下文检索失败。
- 最佳点：1:7 或 1:8。

直觉：Transformer 层处理精确召回和状态追踪。Mamba 层处理廉价的大部分处理。

### 位置编码

Mamba 层本身是位置感知的（通过循环）。原始基于 Mamba 的混合模型中的注意力层不使用 RoPE——SSM 层提供了位置信息。Jamba 1.5 为注意力层添加了 RoPE 以进行长上下文泛化，这是基于经验性长上下文评估的事后改进。

### 内存预算

对于 Jamba-1 形状（32 层：28 Mamba + 4 注意力，隐藏层 4096，32 个注意力头）：

- KV 缓存（仅注意力层）：`2 * 4 * 32 * 128 * 256k * 2 = 8.4 GB`，256k BF16。只有 4 个注意力层贡献。
- SSM 状态：每个 token 前缀为 `28 * hidden * state_size`，但这是每层的固定大小，不随序列长度增长。典型的 Mamba 状态为每个特征 16，隐藏层 4096：`28 * 4096 * 16 * 2 = 3.7 MB` 总计。

与相同隐藏层、32 层、32 头的全 MHA 纯 Transformer 比较：`2 * 32 * 32 * 128 * 256k * 2 = 128 GB`，256k BF16。KV 缓存减少了 8 倍。即使与大多数 2024 模型使用的 GQA(8) 基线（`2 * 32 * 8 * 128 * 256k * 2 = 32 GB`）相比，Jamba 的 1:7 混合在 16 GB 下仍然小 2 倍。

这就是 AI21 所说的"单个 80GB GPU 上的 256k 上下文"。全 MHA 纯 Transformer 的 KV 缓存放不下；即使是 GQA 基线也没有为权重和激活留下空间；Jamba 做到了。

### Mamba-3：2026 年的纯 SSM 基线

Mamba-3（ICLR 2026，arXiv:2603.15569）在纯 SSM 方面引入了三项创新：

1. **指数梯形离散化。** 用更具表现力的循环替换了 Mamba-2 中的欧拉方法离散化。在核心循环内对状态输入应用类似卷积的操作，而不是作为 `x_t` 上的外部卷积。

2. **复数值状态更新。** 先前的 Mamba 将状态矩阵从复数（S4）减少到实对角（Mamba）再到缩放单位矩阵（Mamba-2）。Mamba-3 重新添加了复数值——等效于状态上的数据依赖旋转嵌入。这恢复了先前实数值简化所代价的状态追踪能力。

3. **多输入多输出（MIMO）投影。** 使用矩阵值投影代替每个特征的标量投影。提高了建模能力和推理时硬件利用率，而不增加解码延迟。

在 1.5B 参数下，Mamba-3 将平均下游准确率提高了 0.6 个百分点，优于 Gated DeltaNet；MIMO 变体额外增加了 1.2 个百分点，总共提升 1.8 个百分点。在相同状态大小下，Mamba-3 以一半的状态匹配 Mamba-2。

Mamba-3 尚未在大规模生产化混合模型中交付——但它是下一个 Jamba 类模型 SSM 方面的明显候选。

### 何时使用混合模型

混合模型在以下情况胜出：

- 上下文足够长，以至于纯 Transformer KV 缓存变得痛苦（64k+）。
- 任务混合短距离结构（适合 SSM）和长距离召回（需要 Transformer）。
- 你希望在纯 Transformer KV 缓存本身放不下的单 GPU 内存预算上部署。

混合模型在以下情况失败：

- 上下文短（低于 16k）。SSM 开销被浪费；纯 Transformer 就可以了。
- 任务需要全局注意力（深度推理、多文档交叉引用）。混合中注意力层的稀疏性会带来伤害。
- 你正在扩展到万亿参数前沿模型。纯 Transformer + MLA + MoE（DeepSeek-V3 风格）目前在能力竞赛中胜出。

### 竞争格局

| 模型 | 家族 | 规模 | 独特主张 |

## 使用

`code/main.py` 是一个混合架构的内存计算器。给定 SSM-Transformer 比例和隐藏大小/层数配置，它计算：

- 目标上下文下的 KV 缓存。
- SSM 状态内存。
- 一系列模型形状在上下文 N 下的总内存。

计算器支持：

- 纯 Transformer 基线（KV 缓存随 N 增长）。
- Jamba 风格的 1:7 混合。
- 纯 SSM（完全没有 KV 缓存）。

数字直接来自 Jamba-1 和 Jamba-1.5 论文中已发布的形状，并为假设变体外推。

真实部署的集成考虑：

- 大多数生产推理服务器（vLLM、SGLang）支持 Jamba 和 Mamba。检查特定版本。
- 在 256k 上下文下，Jamba 的内存优势体现在并发请求吞吐量上。在相同的 VRAM 上，你可以容纳比 Transformer 序列更多的 Jamba 序列。
- Mamba-3 作为独立模型尚未在生产中交付——1.5B 的研究预览。

## 关键术语

| 术语 | 人们说的 | 实际含义 |
|------|----------------|------------------------|
| 状态空间模型（SSM） | "具有固定状态的循环" | 具有学习到的循环 `h_t = A h_{t-1} + B x_t` 的层；每个 token 恒定内存 |
| 选择性 SSM | "Mamba 的诀窍" | 数据依赖的 A、B、C 参数，为模型提供类似门控的选择性，在线性时间内 |
| 注意力与 Mamba 比例 | "多少个注意力层" | 在 Jamba 中，`l = 8` 意味着每 7 个 Mamba 层配 1 个注意力层 |
| Jamba 块 | "8 层组" | 一个注意力 + 七个 Mamba + 交替位置上的 MoE |
| SSM 状态 | "隐藏缓冲区" | 替换 Mamba 层的 KV 缓存的每层固定大小状态 |
| 256k 上下文 | "Jamba 的标志性数字" | Jamba-1 适合单个 80GB GPU 的序列长度；纯 Transformer 在该大小下无法做到 |
| Mamba-3 | "2026 纯 SSM" | 当前最佳纯 SSM 架构，具有复数状态 + MIMO；混合模型围绕其重建的基线 |
| MIMO | "多输入多输出" | Mamba-3 创新，使用矩阵值投影代替每个特征的标量投影 |
| 指数梯形离散化 | "Mamba-3 的循环" | 更具表现力的循环，包含 Mamba-2 的欧拉方法离散化 |
| 混合架构 | "混合注意力和 SSM" | 任何交错 Transformer 和 SSM 层的模型；Jamba 是生产原型 |
