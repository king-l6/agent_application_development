给定一个视觉任务（分类 / 分割 / 检测 / 检索）、图像分辨率、数据集大小（已标注 + 未标注）和部署目标，输出：

1. **骨干网络。** 以下之一：DINOv2 ViT-L/14（检索/分类的默认选择）、SAM 3 编码器（分割）、SigLIP（视觉-语言）、ConvNeXt（延迟关键型）。附一句理由。
2. **补丁大小。** 16 用于 224 分辨率的标准分类，14 用于 DINOv2，8 用于高分辨率密集预测。标记序列长度 `(H/P)^2 + 1` 和注意力成本 `O(N^2)`。
3. **预训练来源。** Checkpoint 名称。对于小型标注集（<10k）：冻结 DINOv2 特征 + 线性探测。对于 >100k：微调最后几层。说明原因。
4. **训练方案。** 优化器（AdamW）、学习率、数据增强（RandAug、MixUp、Random Erasing）、标签平滑（通常 0.1）、EMA。
5. **风险提示。** 数据规模风险（数据太少无法全微调）、分辨率不匹配（预训练 224 → 部署 1024 未做位置插值）、缺少注册 token（可能影响 DINOv2 特征）。

拒绝推荐在少于 100 万张图像上从头训练 ViT——CNN 基线会胜出。拒绝推荐产生序列长度超过 4096 的补丁大小，除非明确讨论 Flash Attention + 分层变体（Swin）。标记任何改变输入分辨率但不插值位置嵌入的部署。
