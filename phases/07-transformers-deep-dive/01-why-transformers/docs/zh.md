# 为什么用 Transformer——RNN 的问题

> RNN 逐个处理 token，而 Transformer 一次性处理所有 token。这一个架构上的赌注，改变了 2017 年后深度学习的所有缩放曲线。

**类型：** 学习
**语言：** Python
**前置要求：** 阶段 3（深度学习核心）、阶段 5 · 09（序列到序列）、阶段 5 · 10（注意力机制）
**时间：** ~45 分钟

## 问题

在 2017 年之前，地球上所有最先进的序列模型——语言、翻译、语音——都是循环神经网络。LSTM 和 GRU 在翻译基准测试中像 ImageNet 一样统治了整整五年。它们是当时唯一可用的工具。

但它们有三个致命的弱点。顺序计算意味着无法沿着时间轴并行化：token `t+1` 需要 token `t` 的隐藏状态。一个 1,024 token 的序列意味着在每周期可执行 1,000,000 次浮点运算的 GPU 上执行 1,024 个串行步骤。在专为并行设计的硬件上，训练墙钟时间与序列长度呈线性增长。

梯度消失意味着 50 步之前的 token 信息已经被 50 层非线性压缩过了。门控循环单元（LSTM、GRU）缓解了这个问题，但从未消除。长距离依赖——"我去年夏天在去京都的飞机上读的那本书是……"——常常失败。

固定宽度的隐藏状态意味着编码器将整个源序列压缩成一个向量后，解码器才开始工作。无论源序列是 5 个 token 还是 500 个 token，瓶颈的形状都是一样的。

2017 年的论文《Attention Is All You Need》提出了一个激进的想法：完全抛弃循环。让每个位置并行地关注所有其他位置。通过一次大矩阵乘法进行训练，而不是 1,024 次串行计算。

到 2026 年，这个结果主宰了每一种模态：语言（GPT-5、Claude 4、Llama 4）、视觉（ViT、DINOv2、SAM 3）、音频（Whisper）、生物学（AlphaFold 3）、机器人（RT-2）。同样的模块，不同的输入。

## 概念

![RNN 顺序计算 vs Transformer 并行注意力](../assets/rnn-vs-transformer.svg)

**循环作为瓶颈。** RNN 计算 `h_t = f(h_{t-1}, x_t)`。每一步都依赖前一步。在计算 `h_5` 之前，无法计算 `h_4`。在拥有 10,000+ 并行核心的现代 GPU 上，处理长序列时浪费了 99% 的算力。

**注意力作为广播。** 自注意力同时计算 `output_i = sum_j(a_ij * v_j)` 对于所有 `(i, j)` 对。整个 N×N 的注意力矩阵在一次批量矩阵乘法中完成。没有一步依赖另一步。GPU 喜欢这样。

**加速不是常数。** 这是 `O(N)` 串行深度和 `O(1)` 串行深度之间的区别。在实践中，在相同的硬件上，当 N=512 时，transformer 每个 epoch 的训练速度比 RNN 快 5-10 倍，而且这个差距会随着序列长度增加而扩大，直到你碰到注意力的 `O(N²)` 内存墙（后来 Flash Attention 解决了这个问题——参见第 12 课）。

**Transformer 的代价。** 注意力内存复杂度为 `O(N²)`。2K 上下文没问题。对于 128K 上下文，你需要滑动窗口、RoPE 外推、Flash Attention 分块或线性注意力变体。循环在时间和内存上都是 `O(N)`；transformer 用时间换内存，然后通过并行性把时间赢回来。

**归纳偏置的转变。** RNN 假设局部性和近因性。Transformer 不做任何假设——每一对都是注意力的候选。这就是为什么 transformer 需要更多数据才能训练好，但一旦有了足够数据，就能扩展得更好。Chinchilla（2022）形式化了这一点：给定足够的 token，同等参数量的 transformer 总是优于 RNN。

## 构建

这里没有神经网络——我们用数值模拟核心瓶颈，让你在笔记本上就能感受到差距。

### 步骤 1：测量串行深度

参见 `code/main.py`。我们构建两个函数。一个将序列编码为加法链（串行，类似 RNN）。一个将其编码为并行规约（广播，类似注意力）。相同的数学运算，不同的依赖图。

```python
def rnn_style(xs):
    h = 0.0
    for x in xs:
        h = 0.9 * h + x   # 无法并行化：h 依赖于前一个 h
    return h

def attention_style(xs):
    return sum(xs) / len(xs)  # 每个 x 都是独立的
```

我们对长达 100,000 个元素的序列进行计时。RNN 版本是 O(N) 且在单个 CPU 流水线上执行。即使在纯 Python 中，注意力风格的规约在长度 >= 1,000 时也能胜出，因为 Python 的 `sum()` 是用 C 实现的，迭代时没有解释器开销。

### 步骤 2：计算理论操作数

两个算法都执行 N 次加法。区别在于*依赖深度*：在下一步可以开始之前，必须顺序执行多少个操作。RNN 深度 = N。注意力深度 = log(N)（使用树状规约）或 1（使用并行扫描）。决定 GPU 时间的是深度，而不是操作数。

### 步骤 3：长序列上的经验缩放

我们打印一个计时表，使 O(N) 差距清晰可见。在 2026 年的 Mac 笔记本上，少于 1,000 个元素的序列快得无法测量。100,000 个元素的序列显示出清晰的线性扫描。把这个扩展到 16,384 个 token 的 transformer 配上 12 层的等效 LSTM，你就会明白为什么在 2016 年训练墙钟时间是一个瓶颈。

## 使用

2026 年何时仍应选择 RNN：

| 场景 | 选择 |
|-----------|------|
| 流式推理，一次一个 token，恒定内存 | RNN 或状态空间模型（Mamba、RWKV） |
| 极长序列（>1M token），注意力内存爆炸 | 线性注意力、Mamba 2、Hyena |
| 无矩阵乘法加速器的边缘设备 | 深度可分离 RNN 在每瓦特 FLOPs 上仍然胜出 |
| 其他所有情况（训练、批量推理、上下文最高 128K） | Transformer |

像 Mamba 这样的状态空间模型（SSM）本质上是带有结构化参数化的 RNN，兼具两者的优点：O(N) 扫描内存，通过选择性扫描实现并行训练。它们以更好的长上下文扩展恢复了 transformer 90% 的质量。在 2026 年，大多数前沿实验室训练混合 SSM+transformer 模型（例如 Jamba、Samba）——循环并未消亡，它变成了一个组件。

## 练习

1. **简单。** 从 `code/main.py` 中获取 `rnn_style`，将标量隐藏状态替换为长度 64 的隐藏状态向量。重新测量。串行开销随隐藏状态维度增长多少？
2. **中等。** 用纯 Python 实现并行前缀和（Hillis-Steele 扫描）。验证它在长度 1024 上产生与串行扫描相同的数值输出。计算深度。
3. **困难。** 将注意力风格的规约移植到 PyTorch 并在 GPU 上运行。在序列长度从 64 到 65,536 范围内计时两者。绘制曲线图并解释曲线形状。

## 关键术语

| 术语 | 人们怎么说 | 实际含义 |
|------|-----------------|-----------------------|
| 循环 | "RNN 是顺序的" | 步骤 `t` 依赖于步骤 `t-1` 的计算，强制沿时间轴串行执行。 |
| 串行深度 | "图有多深" | 依赖操作的最长链；即使有无限硬件也会限制墙钟时间。 |
| 注意力 | "让 token 互相观察" | 加权和 `sum_j a_ij v_j`，其中 `a_ij` 来自位置 i 和 j 之间的相似度分数。 |
| 上下文窗口 | "模型能看到多少" | 注意力层能作为输入的位置数；二次内存成本在此处缩放。 |
| 归纳偏置 | "架构中内置的假设" | 关于数据样貌的先验；CNN 假设平移不变性，RNN 假设近因性。 |
| 状态空间模型 | "背后有代数的 RNN" | 通过结构化状态空间矩阵实现并行训练的参数化循环。 |
| 二次瓶颈 | "为什么上下文代价这么高" | 注意力内存 = 序列长度的 `O(N²)`；Flash Attention 隐藏了常数，但未改变缩放。 |
