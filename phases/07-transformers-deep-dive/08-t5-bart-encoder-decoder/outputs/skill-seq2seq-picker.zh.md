给定一个 seq2seq 任务（翻译 / 摘要 / 语音转文本 / 结构化提取 / 重写）、输入和输出长度分布以及质量与延迟优先级，输出：

1. **架构。** 以下之一：编码器-解码器（T5 / BART / Whisper 风格）、解码器专用指令微调、仅编码器 + 提示模板。附一句理由。
2. **预训练目标。** 跨度破坏（T5）、去噪（BART）、下一个 token（解码器专用）或"跳过预训练，微调现有 checkpoint。"给出 checkpoint 名称。
3. **输入格式化。** 任务前缀字符串（T5 风格）vs 系统提示（解码器专用）vs 原始 token（BART）。包括 BOS/EOS 处理。
4. **解码策略。** 束搜索宽度和长度惩罚（翻译/摘要），或核采样/min-p（类似聊天的任务）。说明针对该任务的策略。
5. **评估。** 适合任务的指标：BLEU / ROUGE / WER / F1 / 精确匹配。包括测试集大小。

拒绝为生成式输出推荐仅编码器模型。拒绝在输入已经是对话时推荐编码器-解码器——解码器专用模型天然适合对话记忆。标记任何为语音转文本选择解码器专用模型却没有提及 Whisper 作为需击败的基线的情况。
