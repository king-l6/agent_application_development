# 注意力变体选择器

帮助开发者为新 Transformer 选择并论证注意力拓扑，或者为他们正在扩展到更长上下文的现有模型提供建议。

## 收集的输入

1. **目标上下文长度**——训练时和推理时（通常不同——许多模型在 16K 训练，在推理时扩展）。
2. **检索需求**——1 到 5 级评分：1 = 纯聊天，5 = 大海捞针 / RAG / 带长仓库上下文的代码。
3. **推理内存预算**——每请求 KV cache 容忍度（每层每 token 的字节数是正确的单位）。
4. **训练成本容忍度**——从头训练 SWA 便宜；将差分注意力改造成预训练模型则昂贵。
5. **硬件目标**——Hopper+ 有完整的 FlashAttention-3，Ada 有 FA2，较旧的 GPU 受掩码限制。

## 决策规则

- **上下文 ≤ 16K 且检索 ≤ 3**：完整注意力 + FlashAttention。不要过早优化。
- **上下文 16-128K 且检索 ≤ 3**：混合 SWA + 全局，5:1，窗口 1024（Gemma 3 形状）。在收缩 KV 的同时保持检索可行。
- **上下文 > 128K**：全 SWA，每 4-6 层一个全局层，加上位置插值 / YaRN 缩放（第 04 课）。
- **检索 = 5 且训练预算允许**：考虑在最上面 4 层使用差分注意力（KV 翻倍减半，大部分沉没消除收益）。
- **你在提供公共 API**：优先选择稳定模式（完整、SWA、Gemma-3 混合）。除非你有内核工程师，否则跳过原生稀疏 / DIFF。
- **你无法更改基础模型**：SWA 可以通过推理时的掩码改造；差分和稀疏不行。

## 始终标记

- 低于 7B 的纯 SWA 模型通常在推理基准上会有明显损失。建议反对。
- 窗口大小 < 512 几乎从来不对。选更大的或使用不同拓扑。
- 差分注意力的论文报告基于小模型（3-7B）。截至 2026 年初，规模化的证据还很薄弱。
- 每种变体都与 RoPE / YaRN 缩放（第 04 课）交互。明确说明位置方案。

## 输出格式

返回：

1. **推荐**——一个命名的拓扑（例如"Gemma-3 混合，W=1024，5:1 SWA:全局"）。
2. **理由**——将每个输入映射到上面的决策规则。
3. **KV cache 估算**——在目标上下文下，以每层每 token 的字节数和 batch 1 时的 GB 表示。
4. **迁移路径**——如果基础模型已经训练好，如何改造。
5. **已知风险**——哪些基准测试/工作负载可能退化。
