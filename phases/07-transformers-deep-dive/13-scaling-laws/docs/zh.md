# 缩放定律

> 2020 年 Kaplan 论文说：模型越大，损失越低。2022 年 Hoffmann 论文说：你训练不足。算力分配到两个桶——参数和 token——而分配比例并非显而易见。

**类型：** 学习
**语言：** Python
**前置条件：** 阶段 7 · 05（完整 Transformer）、阶段 7 · 07（GPT）
**时间：** ~45 分钟

## 问题

当你拥有 C FLOPs 的训练算力并想要得到最好的模型时，你面对两个旋钮：

1. **多少参数（N）？** 模型越大，容量越高。
2. **多少训练 token（D）？** 数据越多，容量利用越充分。

FLOPs 大致按 `6 × N × D` 缩放。你可以增大 N 并减小 D，或者增大 D 并减小 N。哪个更好？

2022 年之前，答案是"大力推 N"。GPT-3（2020）是 175B 参数，在约 300B token 上训练。每参数约 1.7 个 token。Kaplan 缩放定律支持这一点。

Hoffmann 等人（2022）训练了一组名为 Chinchilla 的小型模型家族，发现了不同的结果：最优比例接近**每参数 20 个 token**。GPT-3 欠训练了 10 倍。Chinchilla（70B 参数，1.4T token）在推理成本低 2.5 倍的情况下，在每个基准测试上都击败了 GPT-3（175B，300B token）。

2026 年是 Chinchilla 的世界——但有一个重要的变化。Llama 3 8B 在 15 万亿个 token 上训练，比例达到每参数 1,875 个 token。比 Chinchilla 最优多出 94 倍。对于将被大规模使用的模型，推理成本比训练成本更重要，因此过度训练（超过 Chinchilla）以获得更小的可部署模型是 2026 年的默认做法。

## 概念

![Chinchilla 曲线：不同 N/D 比例下的损失与算力关系](../assets/scaling-laws.svg)

### Hoffmann 定律

根据 Chinchilla 论文，损失遵循：

```
L(N, D) = A / N^α + B / D^β + E
```

- `N` = 参数（非嵌入层）。
- `D` = 训练 token。
- `α ≈ 0.34`，`β ≈ 0.28`（大致对称）。
- `E ≈ 1.69`，不可约损失上限。
- `A ≈ 406`，`B ≈ 411`。

随着规模扩大，两项相互权衡。在固定算力（C = 6ND）下对 N 求导并求解：

```
N_opt ≈ 0.6 × (C/6)^0.5
D_opt ≈ 0.6 × (C/6)^0.5
D_opt / N_opt ≈ 20
```

算力最优：每参数 20 个 token。

### 为什么还是要过度训练

Chinchilla 最优最小化每训练 FLOP 的训练损失。但训练成本只付一次，推理成本却是永久的。

对于一个每月服务万亿 token 的聊天机器人，推理主导总成本。Llama 的方法：训练更小、更久。8B 在 15T token 上是深度推理优化的：

- 适合消费级 GPU。
- 延迟只是 70B Chinchilla 最优模型的一小部分。
- 在大多数任务上质量足够接近。

DeepMind 2024 年的论文（"过度训练是新的最优"）将此形式化。对于推理主导的工作负载，根据服务规模，正确的比例接近每参数 100-500 个 token。

### 涌现与平滑性

声称：某些能力（算术、多步推理、思维链遵循）在某个规模下突然"涌现"。

Schaeffer 等人（2023）认为这是一个测量伪影：涌现指标使用不连续的评分（精确匹配、阈值精度），掩盖了底层 logits 中的平滑改进。连续指标（交叉熵）显示平滑曲线。

2026 年的共识是：通过连续损失进行的预测是可靠的。基准测试的跳跃通常是评分器伪影。根据连续指标规划预算。

### 2026 年的图景

缩放定律仍然有效，但是：

| 因素 | 变化方式 |
|--------|-------------|
| 数据质量 | 整理"好" token（Phi 风格）使曲线移动超过 2 倍有效算力 |
| MoE | 总参数与活跃 FLOPs 解耦；缩放定律按每活跃 FLOP 计算 |
| 训练后 | 某些能力（指令遵循、代码）随 SFT+RLHF 的变化比预训练更大 |
| 多模态 | 图像 + 文本 token 一起缩放；每种模态有单独曲线 |
| 合成数据 | 模型生成训练数据；有效算力可以复合增长 |

Muon 优化器（Kimi Moonlight，2024）在匹配数据量下显示出相比 AdamW 约 2 倍的有效算力提升。一些 2026 年的训练运行默认使用 Muon。它改变了缩放定律的绝对常数，而不改变其形状。

## 构建

参见 `code/main.py`。我们将实现 Chinchilla 损失方程，并在几个算力预算下求解算力最优的 `(N, D)`。

### 步骤 1：Chinchilla 损失

```python
def chinchilla_loss(N, D, A=406.4, B=410.7, alpha=0.34, beta=0.28, E=1.69):
    return A / N ** alpha + B / D ** beta + E
```

在固定 `C = 6ND` 下，绘制 `L` 在 `(N, D)` 上的等高线。找到最小值。

### 步骤 2：算力最优前沿

对于从 `1e17` 到 `1e25` FLOPs 的算力预算，找到在 `6ND = C` 约束下最小化损失的 `(N, D)`。验证比例 `D/N ≈ 20`。

### 步骤 3：过度训练成本

计算训练一个 10 倍小模型（1/10 的最优 N，10 倍的最优 D）所付出的额外损失。报告作为交换的推理 FLOP 节省（与 N 成比例）。

### 步骤 4：与真实模型比较

代入 GPT-3、Chinchilla、Llama 3 8B、DeepSeek-V3（活跃参数）的已知 `(N, D)` 对，比较预测损失与报告损失。

## 使用

你不太可能自己训练一个前沿模型。但缩放定律告诉你：

1. **你的微调是否有足够的数据。** 如果你的任务特定数据低于基础模型每参数 20 个 token，预期会在某个损失下限饱和。
2. **是否选择更大的基础模型。** 如果你把所有预算都花在推理上，选择更小、训练更久的模型。
3. **收益在哪里递减。** 超过 1000 倍 Chinchilla 最优后，对数损失的变化变成噪音。

**2026 年的研究轨迹：**

- **数据受限机制。** 网络上的高质量 token 数量有限（过滤后英语约 5-10 万亿）。前沿预训练正在接近这个上限。合成数据、多语言、多模态和 RLHF 规模微调是下一个杠杆。
- **算力倍增技巧。** Muon 优化器、MoE、更好的数据整理——每个都改变绝对常数，而不是渐近线。
- **RL 的缩放定律。** 开放问题。早期证据表明 RL 样本中存在幂律关系，但指数与预训练差异很大。

## 关键术语

| 术语 | 人们说的意思 | 实际含义 |
|------|-----------------|-----------------------|
| 参数（N） | "模型大小" | 非嵌入层权重数量；决定容量。 |
| Token（D） | "训练数据" | 看到的训练 token 数量；决定参数被利用的程度。 |
| 算力（C） | "花费的 FLOPs" | 对于标准 Transformer 近似为 `6 × N × D`。 |
| Chinchilla 最优 | "D/N ≈ 20" | 最小化每预训练 FLOP 的损失的比例。 |
| 过度训练 | "超过 Chinchilla" | 花费额外训练 FLOPs 以节省推理 FLOPs；D/N >> 20。 |
| 不可约损失 | "下限" | 缩放定律中的 `E` 项；数据本身的熵。 |
| 涌现能力 | "在规模下突然跳跃" | 通常是评分器伪影；连续损失是平滑的。 |
| 有效算力 | "训练效率倍增器" | 更好的数据/优化器/架构使一 FLOP 能走更远。 |
