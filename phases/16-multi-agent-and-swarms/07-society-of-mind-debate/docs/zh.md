# 心智社会与多智能体辩论

> Minsky 1986年的前提——智能是一个专家组成的社会——每十年被重新发现一次。2023年，Du 等人将其转化为一个具体的算法：多个 LLM 实例提出答案，阅读彼此的答案，批评，然后更新。经过 N 轮，它们收敛到一个在六个推理和事实性任务上超越零样本 CoT 和反思的共识。两个发现很重要：**多个智能体**和**多个轮次**各自独立地贡献。社会胜过单智能体独白；多轮交换胜过单次投票。

**类型：** 学习 + 构建
**语言：** Python（标准库）
**前置条件：** 第16阶段 · 04（原语模型）
**时间：** ~60分钟

## 问题

自洽性——多次采样同一个模型并取多数答案——是你可以附加的最便宜的推理改进。它有效，但会快速饱和。你可以在样本数量翻倍后看不到另一个有意义的跳跃。

辩论打破了饱和。不是从一个模型得到 N 个独立样本，而是 N 个智能体阅读彼此的推理并修正。样本之间的相关性下降（它们不再是独立同分布的），收敛点通常比独立同分布投票自信地错误时更正确。

## 概念

### Du 等人 2023 算法

来自 arXiv:2305.14325（ICML 2024）：

1. N 个智能体各自产生对问题的初始答案。
2. 对于轮次 r = 2..R：每个智能体被展示其他智能体第 r-1 轮的答案，并被要求"考虑这些，给出你的更新答案"。
3. 经过 R 轮后，对最终答案进行多数投票。

该论文在 MMLU、GSM8K、传记、MATH 和事实性基准上进行了测试。辩论始终优于 CoT 和自反思。

### 两个独立的旋钮

来自同一篇论文的消融实验：

- **仅智能体数量**（1 轮，N 个的多数投票）在大多数任务上胜过单智能体，但会平台期。
- **仅轮次数量**（1 个智能体看到自己的先前推理）几乎没有帮助——反思的已知弱点。
- **两者结合**产生了大的提升。多个智能体之间的多轮交换驱动了增益。

### 为什么它有效

两个机制：

1. **暴露于分歧。** 当一个智能体看到另一个具有不同结论的推理链时，它必须要么证明自己的合理性，要么更新。无论哪种方式，第 r+1 轮的上下文都比第 r 轮更丰富。
2. **相关错误减少。** 在自洽性中，所有样本来自同一个模型，所以错误是相关的——你平均得到一个自信地错误的答案。不同模型或不同种子去相关。不同的*辩论观点*进一步去相关。

### 异构辩论

A-HMAD 和相关的后续工作对不同的智能体使用*不同的基础模型*。Llama + Claude + GPT 辩论减少了单一文化崩溃（第26课），因为一个模型家族的相关错误不被其他模型共享。

缺点：一个弱模型参与辩论可以将共识拖向其错误答案（参见 "Should we be going MAD?", arXiv:2311.17371）。

### NLSOM——129 智能体扩展

Zhuge 等人（"Mindstorms in Natural Language-Based Societies of Mind," arXiv:2305.17066）将这个想法扩展到 129 个成员的社会。结果：专门化和自组织随着规模出现，系统在视觉问答等任务上胜过单智能体。

### 故障模式

- **谄媚级联。** 所有智能体都服从于听起来最自信的智能体。辩论崩溃为最大声的声音。提示词设定对抗性角色（"一个智能体必须论证相反立场"）有帮助。
- **主题漂移。** 经过多轮的辩论会偏离原始问题。缓解措施：每轮重新注入问题。
- **计算量爆炸。** N 个智能体 × R 轮 = N·R 次 LLM 调用，每次的上下文都在增长。一个 5 智能体、5 轮的辩论是 25 次调用，且上下文在增长。每个问题的成本可能超过单次 CoT 调用的 10×。

## 动手构建

`code/main.py` 在一个数学问题上运行一个 3 智能体 × 3 轮的辩论，每个智能体从一个不同的（可能是错误的）答案开始。智能体是脚本化的——每个通过由一个脚本化置信度加权的邻居答案平均来"更新"。收敛在逐轮日志中可见。

演示显示了两个关键效果：

- 单轮交换使智能体更接近正确答案。
- 超过第 2 轮的额外轮次显示出递减的回报（与 Du 等人的平台期一致）。

运行：

```
python3 code/main.py
```

## 使用它

`outputs/skill-debate-configurator.md` 为一个新任务配置辩论：智能体数量、轮次数量、异质性（相同模型 vs 混合）、角色分配（对称 vs 一个对抗性）。它还在运行前估算 token 成本。

## 投入生产

如果你部署辩论：

- **将轮次上限设为 3。** Du 等人显示 3 轮捕获了大部分增益。更多轮次是成本，不是质量。
- **将智能体上限设为 5。** 超过 5 个，上下文膨胀和成本占主导地位。
- **默认异构。** 池中至少使用两个不同的基础模型。
- **对抗性席位。** 一个智能体被提示无论如何都要反对。打破谄媚级联。
- **记录每一轮。** 隐藏中间轮次的辩论系统无法调试或审计。

## 练习

1. 运行 `code/main.py`，然后将轮数设置为 5 并观察递减的回报。在哪个轮次之后额外的收敛停止？
2. 添加第四个具有对抗性角色的智能体：总是与当前多数派意见不同。这会破坏还是改善收敛？
3. 绘制（打印）每轮的共识分数（处于多数答案的智能体比例）。它何时达到 1.0，这等同于"正确"吗？
4. 阅读 Du 等人第 4 节消融实验。使用此代码复现"仅智能体" vs "仅轮次" vs "两者结合"的结果。
5. 阅读 "Should we be going MAD?" (arXiv:2311.17371) 并列出两种超出循环赛的辩论变体——例如，法官主导、辩论链、对抗性。

## 关键术语

| 术语 | 人们所说的 | 实际含义 |
|------|-----------|---------|
| 心智社会（Society of Mind） | "Minsky 的想法" | 智能作为互动的专家；1986 年的框架现在通过 LLM 辩论实现操作化。 |
| 多智能体辩论（Multi-agent debate） | "智能体争论" | N 个智能体提出、互相批评、经过 R 轮修正、多数投票。 |
| 共识（Consensus） | "它们同意" | 不是认知真理——只是多数答案上的比例。可能是自信地错误的。 |
| 轮次（Rounds） | "交换步骤" | 一轮 = 每个智能体阅读其他的并更新一次。 |
| 异构辩论（Heterogeneous debate） | "混合模型家族" | 使用不同的基础模型来去相关错误。 |
| 谄媚级联（Sycophancy cascade） | "每个人都同意最大声的那个" | 智能体服从于最自信的智能体而不考虑正确性的辩论失败模式。 |
| NLSOM | "129 智能体社会" | 自然语言心智社会；Zhuge 等人的规模化版本。 |
| 相关错误（Correlated error） | "相同模型，相同缺陷" | 自洽性饱和的原因；跨不同观点的辩论去相关。 |
