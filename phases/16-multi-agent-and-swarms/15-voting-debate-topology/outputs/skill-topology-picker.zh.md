给定任务描述，推荐多智能体拓扑和规模。

输出：

1. **任务指纹。** 研究（长期、开放）、快速事实（封闭式答案）、逐步优化（分阶段流水线）、或意见（无真实答案）。选择一个；如果跨越两种，选择主导形态。
2. **拓扑。** 星形、链形、树形或图形。根据指纹说明理由：
   - 研究 → 图形（任意对任意批评）
   - 快速事实 → 星形（枢纽聚合）
   - 逐步优化 → 链形（或树形如果分而治之）
   - 意见 → 以上都不是；推荐单个智能体加人类决策
3. **智能体数量 N。** 3 是最便宜的有用集成；5 是常见的甜蜜点；7+ 是特殊情况。在图形拓扑上超过 5 时，警告协调税。
4. **异质性配置。** 如果同质化是问题（研究、推理），至少一个智能体必须来自不同的基础模型族。N=5 时优先使用 3 个不同的基础模型。
5. **轮次限制。** 1 轮 = 投票。2 轮 = 一次优化。3 轮 = 从众占据主导之前的最大值。绝不允许无限制。
6. **聚合。** 简单多数（便宜）、置信度加权（课程 14 的 CP-WBFT）、几何中位数（DecentLLMs）、或评判者打分。除非成本限制要求简单多数，否则默认使用置信度加权。
7. **升级。** 低于阈值的共识 → 升级到何处？人类、另一个使用不同基础模型的集成、还是弃权？

硬性拒绝：

- 任何在图形拓扑上推荐 10+ 个智能体的建议。协调税占主导；先测量。
- 对开放研究问题推荐星形拓扑。星形失去了任意对任意批评的好处。
- 任何将相同基础模型运行 N 次并称之为多智能体的建议。那是伪装的自洽性；正确标记。
- 无限制轮次。奖励从众；辩论运行越久，智能体越会因压力而非逻辑达成一致。

拒绝规则：

- 如果任务没有真实答案（意见、综合、创意），声明投票是建议性的。推荐单个智能体加人类决策。
- 如果用户无法访问多个基础模型，标出同质化上限并推荐变温度的 self-consistency 作为替代方案。
- 如果任务简单（单个事实查询，< 100 词元的推理），推荐带 N=5 自洽性的单个智能体。

输出：一页简报。以一句话推荐开头（"图形拓扑，N=5 个来自 3 个不同基础模型的智能体，2 轮，置信度加权聚合，低于阈值时升级给人类。"），然后是上述七个部分。以预算估算结束：每次查询的预期词元数和预期延迟秒数。
