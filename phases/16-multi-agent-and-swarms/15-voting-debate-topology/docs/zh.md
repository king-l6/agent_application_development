# 投票、自洽性与辩论拓扑

> 最便宜的聚合方式：采样 N 个独立智能体，多数投票。Wang 等人 2022 年的自洽性通过单一模型采样 N 次实现了这一点。多智能体通过**异质**智能体扩展了它，以摆脱同质化 —— 不同的模型、不同的提示、不同的温度、不同的上下文。除了多数投票，辩论拓扑也很重要：MultiAgentBench（arXiv:2503.01935，ACL 2025）评估了星形/链形/树形/图形协调，发现**图形最适合研究**，并且超过约 4 个智能体后存在"协调税"。AgentVerse（ICLR 2024）记录了两种涌现模式 —— 志愿行为和从众行为 —— 而从众既是特性（寻找共识）也是风险（群体思维，课程 24）。本课程绘制了拓扑空间图，构建了每种变体，并测量了协调税。

**类型：** 学习 + 构建
**语言：** Python（标准库）
**前置知识：** 阶段 16 · 07（心智社会与辩论），阶段 16 · 14（共识与 BFT）
**时间：** ~75 分钟

## 问题

辩论可以提高准确性（Du 等人，arXiv:2305.14325）。也可能降低准确性。辩论是否有帮助取决于四个结构选择：

1. 谁和谁对话（拓扑）。
2. 多少轮次（Du 2023：轮次和智能体数量各自独立影响结果）。
3. 智能体是否异质（不同的基础模型打破同质化）。
4. 是否存在对抗性声音（强化论证 vs. 弱化论证）。

那些简单地在任务上"运行 5 个智能体然后投票"的团队，表现往往还不如单个智能体。失败不是随机的，它跟随拓扑和异质性的规律。本课程就是拓扑地图。

## 概念

### 自洽性，单模型基线

Wang 等人 2022 年（"自洽性改进思维链推理"）在同一模型在温度 > 0 下采样 N 次，然后对推理路径的答案进行多数投票。在 GSM8K 上的结果：与单次贪婪解码相比，N=40 个样本有显著提升。自洽性是单智能体到多智能体投票的前身。

限制：自洽性使用一个基础模型。错误必然相关。如果模型有系统性偏差，所有 N 个样本都共享它。

### 多智能体投票，异质扩展

用 N 个*不同*的智能体替换 N 个样本。不同的基础模型（Claude、GPT、Llama），不同的提示，不同的工具访问。好处：错误不相关。代价：不同的智能体成本不同；协调它们增加了开销。

2026 年异质辩论的标准名称是 **A-HMAD** —— 对抗性异质多智能体辩论（Adversarial Heterogeneous Multi-Agent Debate）。尚未被普遍采用，但论文用这个术语来描述"不同模型进行辩论，减少同质化崩溃带来的相关错误"。

### 四种拓扑

```
star                chain               tree                graph

    ┌─A─┐           A─B─C─D         ┌──A──┐              A───B
    │   │                           │     │              │ × │
    B   C                           B     C              D───C
    │   │                          / \   / \
    D   E                         D   E F   G           (fully connected)
```

星形：一个中心枢纽，所有其他人只与枢纽对话。相当于监督者-工作者模式，没有旁路通信。
链形：线性，每个智能体看到前一个的输出。类似流水线。
树形：层次化，用于层次化智能体系统（课程 06）。
图形：任意对任意。包括完全连接团和任意有向无环图。

### 协调税（MultiAgentBench）

MultiAgentBench（MARBLE，ACL 2025，arXiv:2503.01935）在一组包含研究、编码和规划的任务上对星形、链形、树形、图形进行了基准测试。关键测量结果：

- **图形**拓扑在研究任务上胜出。信息任意对任意流动；智能体可以相互批评。
- **星形**在快速回答事实性任务上胜出。枢纽过滤和整合。
- **链形**在逐步流水线（分阶段优化）上胜出。
- **协调税**在图形拓扑中约 4 个智能体后出现。墙钟时间和词元成本增长快于质量。

4 个智能体的上限是经验性的，不是根本性的。它反映了 2026 年大语言模型的上下文容量：每个智能体的上下文被同伴的输出填满，一旦每个人都能看到每个人，增加第 N+1 个智能体的边际价值就会下降。

### 多智能体辩论策略（"我们应该变得 MAD 吗？"）

arXiv:2311.17371 是 2023 年的 MAD 策略综述。被其他人复现的关键发现：与自洽性*结构相似*的 MAD 变体（独立采样 + 聚合），在使用相同预算时往往表现不如自洽性。MAD 在智能体真正异质且辩论具有对抗性结构（一个智能体持反对意见）时最有帮助。

### AgentVerse 涌现模式

AgentVerse（ICLR 2024，https://proceedings.iclr.cc/paper_files/paper/2024/file/578e65cdee35d00c708d4c64bce32971-Paper-Conference.pdf）记录了即使没有显式设计，多智能体辩论中也会涌现的两种行为：

- **志愿行为。** 一个智能体主动提供帮助（"我可以做下一步"）。有用：它将工作分配给子任务最擅长的智能体。
- **从众行为。** 一个智能体调整其立场以匹配批评者，即使批评者是错的。这是辩论中谄媚的等价物（课程 14）。

从众行为就是为什么"辩论直到达成一致"会奖励霸凌者。限制轮次加上独立的评判者可以缓解。

### 异质性：真正影响准确度的旋钮

2024-2026 年实践文献中的一个模式：将 N 个智能体中的一个替换为不同的基础模型，比将 N 增加 1 能带来更大的准确度提升。直觉是同质化 —— 每个新的独立错误源的价值都高于一个额外的相关样本。

在极限情况下，异质性胜过数量。在大多数有清晰真实答案的任务上，三个不同的模型胜过五个相同模型的副本。

### 陪审团方法

Sibyl 框架（在 Minsky-LLM 文献中被引用）形式化了"陪审团" —— 一小组专门化的智能体，通过在每个阶段投票来优化答案。与简单的多数投票不同，陪审团有角色：一个智能体交叉检查，一个提供上下文，一个给可信度打分。陪审团方法是简单投票（便宜，易同质化）和完整 MAD（昂贵，易从众）之间的中间点。

### 投票加辩论占优的情况

- 问题有真实答案（事实、数学、代码行为）。投票收敛是有意义的。
- 智能体可以访问不同的来源或工具（异质性可用）。
- 轮次有限（通常 2-3 轮）且存在独立的评判者或验证器。
- 预算允许 3-5 个智能体。超过 5-7 个时，在图形拓扑上协调税占主导。

### 投票加辩论有害的情况

- 问题是意见型。智能体会收敛到看起来最自信的答案，而不是最正确的答案。
- 所有智能体共享一个基础模型。同质化使共识毫无意义。
- 轮次无限制。从众每次都赢。
- 任务很简单。单个智能体 N=5 的自洽性更便宜且同样准确。

## 构建它

`code/main.py` 实现了：

- `run_star(agents, hub, question)` —— 枢纽轮询每个工作者，聚合。
- `run_chain(agents, question)` —— 顺序优化。
- `run_tree(root, children, question)` —— 层次化，深度为 2 的聚合。
- `run_graph(agents, question, rounds)` —— 全连接辩论，限制轮次。
- 一个脚本化的异质性旋钮：每个智能体有一个 `error_bias` 表示其系统性错误倾向。
- 一个测量框架，在 N=3, 5, 7 下运行每种拓扑并报告（准确度、总词元数、模拟墙钟时间）。

运行：

```
python3 code/main.py
```

预期输出：拓扑 × N →（准确度、词元、延迟）的表格。图形在研究型任务上在 N=3-5 时胜出；星形在快速事实性任务上胜出；N=7 时的图形拓扑显示了协调税（延迟增长快于准确度）。

## 使用它

`outputs/skill-topology-picker.md` 是一个技能，它读取任务描述并推荐拓扑（星形/链形/树形/图形）、N（智能体数量）、异质性配置（要使用的基础模型）和轮次限制。

## 交付它

对于任何集成：

- 从一个强基础模型的 **N=5 自洽性** 开始。这是便宜的基线。
- 如果准确度重要，升级到 **N=3 的异质投票**。测量差异。
- 只有当任务有结构（研究、多步骤）且限制轮次可行时，才升级到**辩论拓扑**。
- 始终记录少数聚类。当少数派持续正确时，你得到了一个多样性信号。
- 同时基准测试墙钟时间和词元数与准确度。"以 10 倍成本获得更高准确度"是一个商业决策。

## 练习

1. 运行 `code/main.py`。绘制图形拓扑的协调税曲线：准确度 vs N，词元数 vs N。在哪个 N 值处曲线发生转折？
2. 实现 A-HMAD：三个智能体具有故意不同的偏差。在课程 14 的同质化攻击上，全相同偏差的基线如何与 A-HMAD 比较？
3. 在图形拓扑中添加一个"评判者"角色，不参与投票，只给最终共识打分。这会改变涌现的从众行为吗？
4. 阅读 AgentVerse 论文（ICLR 2024）。识别你的实现最强烈地展示了哪种涌现行为。你能通过改变提示来引发相反的行为吗？
5. 阅读 MultiAgentBench（arXiv:2503.01935）第 4 节（拓扑实验）。使用你的框架从论文中重现一个任务上的"图形在研究方面胜出"的结果。

## 关键术语

| 术语 | 人们说的 | 实际含义 |
|------|----------|----------|
| 自洽性 | "采样 N 次，投票" | Wang 2022。单一模型，N 次温度>0 采样，对推理路径进行多数投票。 |
| 异质性 | "不同的模型" | 不同基础模型或提示族的集成。打破同质化。 |
| MAD | "多智能体辩论" | 智能体在轮次间交换批评的通用术语。见 Du 2023。 |
| A-HMAD | "对抗性异质 MAD" | 强调不同模型加对抗性结构的 MAD 变体。 |
| 拓扑 | "谁和谁对话" | 星形、链形、树形、图形。决定信息流。 |
| 协调税 | "边际收益递减" | 在图形上超过约 4 个智能体时，成本增长快于质量。 |
| 志愿行为 | "主动帮助" | AgentVerse 涌现模式：智能体主动提出进行下一步。 |
| 从众行为 | "压力下同意" | AgentVerse 涌现模式：智能体与批评者保持一致。 |
| 陪审团 | "小型专门小组" | Sibyl 风格集成，带有角色（审查员、上下文提供者、打分者）。 |
