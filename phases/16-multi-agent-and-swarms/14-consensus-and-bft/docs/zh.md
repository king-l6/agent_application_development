# 共识与拜占庭容错（BFT）智能体

> 经典分布式系统 BFT 遇上随机大语言模型。2025-2026 年出现了三个研究方向：**CP-WBFT**（arXiv:2511.10400）通过置信度探测为每张投票加权；**DecentLLMs**（arXiv:2507.14928）采用去领导者模式，并行工作器提案和几何中位数聚合；**WBFT**（arXiv:2505.05103）结合加权投票与层次结构聚类，将节点分为核心节点和边缘节点。来自"AI Agents 能达成一致吗？"（arXiv:2603.01213）的诚实经验结果是：即使是标量共识在今天也很脆弱 —— 一个欺骗性智能体就能破坏混合智能体（Mixture-of-Agents）的共识。BFT 是必要的，但还不够。本课程构建了一个最小 BFT 协议，注入了三种智能体特定攻击（拜占庭谎言、谄媚从众、相关错误同质化），并测量每种共识变体如何应对。

**类型：** 学习 + 构建
**语言：** Python（标准库）
**前置知识：** 阶段 16 · 07（心智社会与辩论），阶段 16 · 13（共享内存）
**时间：** ~75 分钟

## 问题

你有 N 个大语言模型智能体，每个都产生一个答案。它们意见不一。多数投票选错了答案，因为两个智能体相关（相同的基础模型、相同的训练数据、相同的失败模式）。第三个智能体恰好以一种新方式出错 —— 所以这个多数是虚假多数。

现在加入一个欺骗性智能体：它故意撒谎。或者一个谄媚从众的智能体：它同意最后一个发言者的观点。在经典 BFT 中，假设拜占庭节点占比 `f < n/3` 且行为任意。2026 年的现实是，大语言模型节点即使诚实也是随机的，跨模型相关，且彼此的输出相互影响。你不能把它们当作独立的伯努利投票者。

经典 BFT（PBFT，1999）并不是错的 —— 它是不完整的。它处理任意位翻转，但不处理"三个诚实智能体因为共享训练数据而共同产生幻觉"这种情况。本课程从 PBFT 的基础出发，在其上叠加了三种 2025-2026 年的适应性改进。

## 概念

### 经典 BFT 提供的保障

实用拜占庭容错（Castro & Liskov, OSDI 1999）容忍 `f < n/3` 的拜占庭节点。该协议分为三个阶段（预准备、准备、提交）和两个原语（签名消息、法定人数证书）。在 `n >= 3f + 1` 个诚实或恶意节点之间就单个值达成一致。

这些保证非常强，但假设：

1. **故障独立。** 拜占庭节点不协调行动。
2. **诚实节点是真的诚实。** 诚实输出的正确性不是问题；协议只对齐分歧。
3. **问题有真实答案。** 对错误事实达成一致仍然是共识。

大语言模型智能体违反了所有三条假设。两个运行相同基础模型的智能体共享故障。一个"诚实"的大语言模型仍然可能产生幻觉。而对于模棱两可的问题，"真相"就是智能体们决定的结果 —— 没有外部预言机。

### 三种大语言模型特定攻击

**拜占庭谎言。** 一个智能体故意输出错误答案。如果 `f < n/3`，经典 BFT 可以处理这种情况。

**谄媚从众。** 一个智能体在投票前读取其他智能体的答案，并与最后一个发言者保持一致。这不是恶意的，但使结果与声音最大的意见相关。经典 BFT 无法阻止这种情况，因为该智能体通过了所有签名检查。

**相关错误同质化。** 三个智能体共享一个基础模型。它们都产生了相同的错误答案。多数是错误的。经典 BFT 无法解决，因为所有三个"诚实地"达成了一致。

### 2025-2026 年的应对方案

**CP-WBFT**（arXiv:2511.10400）—— 置信度探测加权 BFT。每个投票者为其答案附加一个置信度探测（自报概率或独立校准模型的预测）。投票权重随置信度缩放。报告在完全连接图上实现了 +85.71% 的 BFT 改进。缓解了：谄媚从众（从众型智能体对其自身立场的置信度往往较低）。

**DecentLLMs**（arXiv:2507.14928）—— 去领导者模式。工作智能体并行提出方案，评估智能体对方案打分，最终答案是打分后位置的几何中位数。在 `f < n/2` 时具有鲁棒性。缓解了：拜占庭谎言和相关错误（几何中位数对异常值具有鲁棒性，向密集聚类靠近，而不是向有模型偏好的平均值靠近）。

**WBFT**（arXiv:2505.05103）—— 带层次结构聚类的加权 BFT。投票权重由响应质量加上从历史学习的信任得分分配。将智能体聚为核心和边缘；核心智能体必须首先达成共识，边缘智能体跟随。缓解了：可扩展性（核心共识小且快）以及部分缓解同质化（核心可以选择多样性）。

### 实证："AI Agents 能达成一致吗？"（arXiv:2603.01213）

该论文测量了跨多个前沿模型的标量一致性（大语言模型智能体就单个数值达成一致）。发现令人不安：

- 即使没有对手，大语言模型智能体在许多基准测试中在标量问题上不一致的比率超过 30%。
- 单个采用欺骗性角色的智能体可以将混合智能体共识拉离诚实基线 40 个百分点以上。
- 不一致率与模型多样性相关 —— 异质集成的不一致率高于同质集成（好处是：误差不相关），但漂移也更慢（坏处是：达成一致的时间更长）。

结论：BFT 提供了对齐输出的机制，但它不告诉你对齐后的输出是否正确。与验证（阶段 16 · 08 角色专业化）、多样性（阶段 16 · 15 辩论变体）和评估智能体（阶段 16 · 24 基准测试）结合使用。

### 核心协议，精简版

针对大语言模型智能体的最小 BFT 轮次：

```
1. 任务到达；每个智能体 i 产生答案 a_i
2. 每个智能体附加置信度探测 c_i in [0, 1]
3. 聚合器从所有 n 个智能体收集 (a_i, c_i)
4. 聚合器按语义聚类分组（等价答案）
5. 聚合器计算每个聚类 C 的权重：
     w(C) = sum_{i in C} c_i
6. 胜者 = 权重最大的聚类，如果 max > threshold * sum(c_i)
   否则：重试或升级
7. 少数聚类附带来源记录以备事后审计
```

语义聚类步骤是大语言模型特有的巧妙之处。两个答案，"研究报告 4.2%" 和 "4.2% 的改进" 属于同一个聚类。朴素的字符串相等检查会漏掉这一点。在生产环境中，使用廉价的嵌入模型或显式规范化。

### 阈值调优

`threshold` 参数决定何时接受，何时重试。太低：你接受弱多数。太高：你永远无法接受任何东西。经验范围：`n=5-7` 个智能体时为 0.5-0.67，较小的 `n` 更高。低于阈值时，升级给人类或不同的智能体集成。

### 共识无济于事的情况

- **模棱两可的问题。** 如果问题没有真实答案，共识只是一种观点。请如实说明。
- **复合问题。** "编写代码并解释" —— 两个答案。分别对每个投票。
- **对抗性多轮。** 如果智能体可以观察前几轮并模仿（Du 2023 辩论），它们会开始相互同意而不管事实。限制轮次（通常 2-3 轮）。

## 构建它

`code/main.py` 实现了：

- `AgentVoter` —— 一个脚本化策略，包含（答案，置信度）。
- `MajorityVote` —— 经典简单多数。
- `CPWBFT` —— 带语义聚类的置信度加权投票。
- `DecentLLMs` —— 基于打分提案的几何中位数聚合。
- `Scenario` —— 在三种攻击模式下运行每个聚合器。

实现的攻击模式：

1. `byzantine`：一个智能体以高置信度撒谎。
2. `sycophancy`：一个智能体复制它看到的第一个答案，并匹配其置信度。
3. `monoculture`：三个智能体以中等置信度共享一个错误答案（相关错误）。

运行：

```
python3 code/main.py
```

预期输出：一个（攻击，聚合器）-> 最终答案的表格，正确答案高亮显示。简单多数在同质化情况下失败。CPWBFT 的置信度加权缓解了谄媚。当同质化群体少于总人口的一半时，DecentLLMs 的几何中位数向诚实聚类靠拢。

## 交付它

在交付任何共识机制之前：

- **至少用上述三种模式进行攻击测试。** 你的协议应该可预测地失败，而不是静默地失败。
- **记录每个少数聚类** 及其来源。少数聚类是你相关错误的早期预警系统。
- **强制限制轮次。** 不允许"一直辩论直到达成一致"——这会奖励谄媚。
- **将一致性与正确性分开。** 共识输出发送给验证器；验证器独立于集成。
- **监控一致率。** 急剧上升意味着从众偏差；急剧下降意味着模型漂移。

## 练习

1. 运行 `code/main.py`。确认简单多数在同质化攻击中失败，但当同质化置信度低于 0.7 时，CPWBFT 部分缓解了该攻击。
2. 添加第四种攻击模式：**沉默弃权** —— 一个智能体拒绝回答（"我不知道"）。每个聚合器应该如何处理弃权？实现你的选择。
3. 将语义聚类从字符串规范化切换为嵌入相似度（使用任何开源嵌入模型）。谄媚攻击会发生什么变化？
4. 阅读 CP-WBFT（arXiv:2511.10400）。实现置信度探测校准步骤（一个独立的校准模型检查每个智能体自报的置信度）。测量在同质化场景下的准确率提升。
5. 阅读"Can AI Agents Agree?"（arXiv:2603.01213）。重现一个简化的标量一致性实验：三个智能体，一个标量问题，欺骗性角色提示。CPWBFT 或 DecentLLMs 能抓住它吗？

## 关键术语

| 术语 | 人们说的 | 实际含义 |
|------|----------|----------|
| BFT | "拜占庭容错" | Castro-Liskov 1999 协议，在 `f < n/3` 任意故障下达成共识。 |
| 拜占庭 | "任何不良行为" | 一个可以撒谎、丢弃消息、静默失败的节点 —— 除了安全崩溃以外的任何行为。 |
| 置信度探测 | "你有多确定？" | 自报或校准器预测的概率，附加在投票上。 |
| 语义聚类 | "相同答案，不同说法" | 在计票前对等价答案进行分组。 |
| 几何中位数 | "鲁棒中心" | 使到样本点的距离之和最小的点。对异常值具有鲁棒性，不同于均值。 |
| 同质化 | "相同模型，相同失败" | 智能体共享训练数据或基础模型时的相关错误。 |
| 谄媚从众 | "附和声音最大的" | 智能体的投票偏向最先/最大声发言者。 |
| 核心/边缘 | "层次化 BFT" | WBFT 的划分：核心节点先达成小范围共识，边缘节点跟随。限制延迟。 |
