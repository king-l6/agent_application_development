给定一个新的智能体协议规范，生成 FIPA-ACL 映射，以便读者能够分辨哪些部分是重新发明，哪些是真正的新结构。

生成：

1. **信封映射。** 对于规范定义的每种消息类型，指出最接近的 FIPA 行事语（`inform`、`request`、`query-if`、`query-ref`、`propose`、`accept-proposal`、`reject-proposal`、`cfp`、`subscribe`、`cancel`、`failure`、`not-understood` 或其他约二十个之一）。如果没有适合的行事语，精确描述这个差距。
2. **关联模型。** 规范如何将请求与回复关联、取消与原始请求关联、流式事件与订阅关联？与 FIPA 的 `:conversation-id` 和 `:reply-with` 字段进行比较。
3. **内容语言立场。** 规范是否强制要求内容模式（类型化产物、JSON-Schema）、接受自然语言，还是保持开放？与 FIPA 的 SL0/SL1 和本体字段进行比较。
4. **交互协议库。** 在该规范之上可以实现哪些 FIPA 交互协议：合同网、订阅-通知、请求-何时、建议-接受？命名实现每个协议所需的消息。
5. **发现模型。** 智能体如何找到交易对手和能力（MCP `listTools`、A2A Agent Card、ANP DID + 元协议）？与 FIPA 的目录促进器和黄页服务进行比较。
6. **重新发明 vs 创新。** 生成一个简短的三列表格：[FIPA 概念，现代规范等价物，变化了什么]。将每一行标记为 [重新发明] 或 [新结构]。只有当规范引入了 FIPA 没有的原语——去中心化身份、类型化多模态产物和 LLM 可解释内容是常见的候选——时才标记为"新结构"。

硬性拒绝：

- 任何声称规范是"革命性"但没有展示 FIPA 没有的原语的映射。言语行为理论 + 本体开销是失败模式，不是原语。
- 忽略发现层的框架比较。没有发现的规范是不完整的，不是新颖的。
- 像"协议 X 取代 FIPA"这样的说法，没有解决当两个智能体对内容含义有分歧时（语义漂移）会发生什么。

拒绝规则：

- 如果规范处于标准化前状态（草案不到 6 个月，没有公开实现），说明该映射是临时性的，并标记三个最可能的变化。
- 如果规范是闭源的或仅限企业的（某些 ACP 变体），映射已文档化的部分并指出缺口。
- 如果用户只提供了博客文章（没有规范文档），请在映射前要求提供规范。

输出：一页简报。以一句话摘要开头（"协议 X 是带 JSON 语法和基于 DID 的发现层的 FIPA `request`/`subscribe`"），然后是上述六个部分，最后以一个总结段落回答："这个规范将重新发现哪个旧的 FIPA 失败模式？"
