给定一个受益于并行子智能体研究的用户查询，生成一个准备连接到任何框架（LangGraph、OpenAI Agents SDK、CrewAI Hierarchical）的监督者模式设计。

生成：

1. **复杂度估算。** 这个查询是简单（1个智能体，3-10次工具调用）、中等（2-4个工作者）还是复杂（5+个工作者）？使用 Anthropic 的调整投入启发式方法，用一句话说明理由。
2. **主导者系统提示词。** 必须包括：(a) 分解指令，(b) 综合指令，(c) 主导者从不读取原始源内容、只读取工作者摘要的显式规则。
3. **工作者系统提示词。** 每个角色一个，每个命名其狭窄范围和主导者预期的输出格式。
4. **子问题分解规则。** 主导者如何拆分查询？先宽后窄，还是直接分解？什么情况会使子问题不合格（与另一个重叠、太宽泛）？
5. **综合模板。** 显式冲突处理规则：如果两个工作者返回矛盾事实，综合必须表面化分歧，而不是静默选择一个。
6. **模型配对。** 哪个模型用于主导者（推理层），哪个用于工作者（更快/更便宜层）。解释权衡。
7. **可观测性要求。** 最小跟踪点：计划、每个工作者开始/结束、综合输入、综合输出。

硬性拒绝：

- 任何主导者自己使用工具的设计。主导者只做规划和综合。
- 允许范围漂移的工作者提示词（例如，无界限的"研究与 X 相关的一切"）。
- 隐藏冲突的综合模板。

拒绝规则：

- 如果查询是简单的（估计算法总工具调用少于10次），拒绝该设计并推荐单智能体。引用 Anthropic 的 15× token 成本发现。
- 如果查询是顺序的（第2步需要第1步的输出），拒绝并推荐流水线/链条模式。
- 如果用户正在优化确定性和审计，拒绝监督者并推荐 LangGraph 静态图。

输出：一页设计简报。以复杂度估算和模式适配结论（"监督者适合"）开头。如果系统将持续运行，以彩虹部署提醒结尾。
