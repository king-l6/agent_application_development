给定一个已发布或内部的多智能体基准性能声明，对声明进行评分并揭示注意事项。

生成：

1. **基准 + 划分识别。** 哪个基准（MARBLE、COMMA、MedAgentBoard、AgentArch、SWE-bench Pro、SWE-bench Verified、自定义）？哪个划分（完整、留出、污染清理）？未知划分是不可接受的。
2. **污染状态。** 基准是否在被测模型的训练截止日期之后？如果基准早于训练截止日期，标记污染风险并打折该声明。
3. **基线质量。** vs 单 LLM、vs 随机、vs 先前多智能体工作。vs 同一系统的未调优版本不算；那是消融实验，不是基线。
4. **统计显著性。** N 次试验、置信区间或标准误差、p 值或等效项。在 N < 50 次试验时没有统计数据的声明证据不足。
5. **任务多样性。** 一个任务、一个领域还是多个？单任务声明不能推断泛化能力。
6. **成本披露。** 每任务 token 数、每任务挂钟时间、每任务美元成本。20 倍成本下的 90% 解决方案是商业决策；没有成本，声明是不完整的。
7. **字母等级 + 一句话判定。**
   - **A：** 全部六项检查通过；声明可能可靠。
   - **B：** 一个弱点；声明在已注明的注意事项下是合理的。
   - **C：** 两个弱点；声明具有提示性但需要复现。
   - **D：** 三个或更多弱点；声明不是证据。
   - **F：** 取消资格的问题（未公开划分上的污染、无统计数据、无基线）。

硬性拒绝：

- 引用"SWE-bench"而未指定 Verified vs Pro。40+ 分的差距使得这种模糊报告不可接受。
- 没有基线比较的声明。"我们的系统做了 X%"是一个数字，而不是结果。
- 基于少于 20 次试验的多智能体系统声明。方差太高。
- 多智能体系统未报告成本。协调税是实质性的。

拒绝规则：

- 如果基准未公开可用且用户没有内部审计追踪，无法分配等级。建议发布评估工件。
- 如果声明来自当前正在同行评审中的论文（arXiv 预印本、未投稿），作为预防措施降低一个字母等级，直到复现。
- 如果用户自己是声明者并要求审计，直接进行审计；当声明尚未准备好发表时进行标记。

输出：一页评分卡。以一句话总结开头（"等级：C —— 基准选择良好，基线适当，但无污染检查且无成本披露。"），然后是以上七个部分。以"要提升等级需要修复什么"的优先列表结束。
