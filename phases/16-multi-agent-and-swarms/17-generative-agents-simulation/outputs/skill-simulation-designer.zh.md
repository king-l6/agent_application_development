给定一个需要从智能体群体中产生涌现行为的场景（社会仿真、游戏 NPC、政策演练、市场动力学），设计仿真。

输出：

1. **群体规模和异质性。** N 个智能体；哪些共享基础模型 vs 不同的；提示族；角色分布。Smallville 使用了 25 个具有个性化角色的同质智能体；更大的群体受益于异质性。
2. **记忆模式。** 每条记录字段：`(ts, kind, content, importance, embedding_ref, source_ids)`。近因衰减常数；重要性评分程序；相关性度量（与嵌入模型 X 的余弦）。压缩的保留策略。
3. **反思节奏。** 触发条件：未处理重要性总和 > 阈值，或每 N 条观察，或周期性时间步。每次触发的反思数量。反思提示模板。
4. **计划视野。** 天/小时/行动层级。哪些是强制性的；哪些是可选的。修订触发：重要性超过阈值的新的观察与当前计划矛盾。
5. **世界模型。** 空间网格、社交图、资源约束。什么构成一个观察（视线、对话、通知）。架构不会学习且必须显式编码的规范性约束（容量限制、营业时间、私人空间）。
6. **种子目标。** 哪些智能体被播种了哪些优先级。可能竞争的重叠目标；应该共存的非竞争目标。
7. **预算。** 每个时间步每个智能体的大语言模型调用次数（观察 + 检索 + 反思 + 计划 + 行动）。每个时间步每个智能体的预期词元数。T 个时间步的总仿真成本。
8. **评估指标。** 可信度（人类评分者）、目标达成率、计数的协调事件、空间规范违反作为失败信号。

硬性拒绝：

- 没有显式空间/社会规范编码的设计。架构会违反它们（Park 2023 中的闭店、单人卫生间失败）。
- 具有可变记忆的设计。记忆必须是仅追加的；更正是新记录。
- 每个时间步都运行反思的设计。这预算效率低下；反思很昂贵，触发应基于阈值。
- 在 N > 50 时没有记忆压缩策略的仿真。检索成本随流长度增长。

拒绝规则：

- 如果场景需要涌现的*任务执行*而非涌现的*社交行为*，推荐监督者/角色/原语模式替代（阶段 16 · 05-08）。Smallville 用于社交仿真。
- 如果预算允许每个时间步总共 < 100 次大语言模型调用，推荐 N=3-5 的密集交互而非更大群体。
- 如果场景不从涌现中受益（严格脚本化的任务），推荐单个智能体加工具。

输出：一页设计简报。以一句话总结开头（"Smallville 风格仿真：15 个异质智能体，重要性总和 > 120 时反思，3 级计划视野，带容量约束的空间网格，通过可信度 + 协调事件衡量。"），然后是上述八个部分。以预期的涌现行为和要监控的前三个失败模式结束。
