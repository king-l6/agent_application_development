# 协商与议价

> 智能体就资源、价格、任务分配和条款进行协商。2026 年的基准测试集很明确：NegotiationArena（arXiv:2402.05863）显示大语言模型可以通过角色操纵（"绝望"）改善约 20% 的收益；"测量议价能力"（arXiv:2402.15813）显示买方比卖方更难且模型规模没有帮助 —— 他们的 **OG-Narrator**（确定性出价生成器加大语言模型叙述器）将成交率从 26.67% 提升到 88.88%；大规模自主协商竞赛（arXiv:2503.06416）进行了约 18 万次协商，发现**隐藏思维链**的智能体通过向对手隐藏推理过程而获胜；Bhattacharya 等人 2025 年在哈佛谈判项目指标上排名显示 Llama-3 最有效、Claude-3 最具攻击性、GPT-4 最公平。本课程实现了合同网协议（FIPA 的前身，课程 02），构建了一个大语言模型风格的买方/卖方，运行了 OG-Narrator 风格的任务分解，并测量了每种结构选择如何改变成交率。

**类型：** 学习 + 构建
**语言：** Python（标准库）
**前置知识：** 阶段 16 · 02（FIPA-ACL 遗产），阶段 16 · 09（并行群网络）
**时间：** ~75 分钟

## 问题

两个智能体需要就价格达成一致。2024-2026 年的大语言模型仅靠纯语言提示，成交率出奇地低（在 arXiv:2402.15813 的严格参数化议价中约 27%）。模型规模无法解决这个问题：GPT-4 在议价能力上并不比 GPT-3.5 在结构上更好；它只是在议价的*语言*上更好。

根本问题是大语言模型混淆了两个工作 —— 决定出价和叙述出价。OG-Narrator 将两者分离：一个确定性出价生成器计算数值移动；大语言模型只负责叙述。成交率跃升至约 89%。

这反映了一个经典的多智能体发现：将机制与通信层解耦能够获胜。合同网协议（FIPA, 1996; Smith, 1980）是参考任务市场机制。将大语言模型插入叙述槽位，你就得到了一个现代的大语言模型驱动的任务市场。

## 概念

### 合同网协议，一段话概括

Smith 1980 年的合同网协议：一个**管理者**广播**招标书（cfp）**；**投标者**回应带有报价的**提案（propose）**消息；管理者选择赢家并向赢家发送**接受提案（accept-proposal）**，向输家发送**拒绝提案（reject-proposal）**。赢家执行工作。可选消息：**拒绝（refuse）**（投标者拒绝提案）。FIPA 将其编码为 `fipa-contract-net` 交互协议。

### OG-Narrator 为什么赢

"测量语言模型的议价能力"（arXiv:2402.15813）观察到：

- 大语言模型经常违反议价规则（以无意义的价格出价，忽略对方的 ZOPA）。
- 它们锚定能力差（接受糟糕的首轮出价；以象征性而非战略性数额还价）。
- 仅靠模型规模无法解决这些问题。更大的模型能生成更合理的语言，但带有类似的策略性错误。

OG-Narrator 分解：

```
           ┌──────────────────┐        ┌──────────────────┐
  state  → │ 出价生成器        │ price → │  大语言模型叙述器  │ → message
           │ (确定性)         │        │  (编写人性化     │
           │                  │        │   的伴生语言)    │
           └──────────────────┘        └──────────────────┘
```

出价生成器是经典的协商策略：Rubinstein 议价模型、Zeuthen 策略、或简单的以牙还牙式价格博弈。大语言模型负责叙述。消息中包含确定性价格和自然语言框架。

成交率跃升的原因：
- 价格保持在议价区域内。
- 锚定是策略性的，而不是情绪化的。
- 大语言模型做它擅长的事：写作。

### NegotiationArena 的发现

arXiv:2402.05863 提供了标准基准测试。关键发现：

- 大语言模型可以通过采用角色（"我急于在周五前卖掉这个"）来改善约 20% 的收益 —— 角色操纵是真实的策略。
- 公平/合作的智能体会被对抗性智能体利用；防御需要明确的对抗姿态。
- 对称配对在大约 40% 的基准场景中收敛到不公平的结果。

这并不是"大语言模型是糟糕的谈判者"。而是"大语言模型谈判得太像人类了，包括那些容易被利用的部分。"

### 思维链隐藏

大规模自主协商竞赛（arXiv:2503.06416）运行了约 18 万次跨多种大语言模型策略的协商。胜者向对手隐藏了推理过程：

- 如果智能体将"我只愿意出到 75 美元；我的保留价是 70 美元"打印到公开可见的草稿区，对手会看到它。
- 胜者私下计算策略；输出通道只包含出价和最低限度的必要叙述。

这是 2026 年对经典博弈论的回响（Aumann 1976 关于理性和信息）：透露你的私人估值会损害收益。大语言模型没有这种直觉，会愉快地在推理痕迹中输入保留价，而这些痕迹会变得对对手可见。

工程要点：将私人草稿区上下文与公开消息上下文分离。这不是可选的。

### Bhattacharya 等人 2025 年 —— 模型排名

在哈佛谈判项目指标（原则性谈判、BATNA 尊重、利益互惠）上：

- **Llama-3** 在达成交易方面最有效（成交率 + 收益）。
- **Claude-3** 是最具攻击性的谈判者（高锚定、晚让步）。
- **GPT-4** 最公平（跨配对收益方差最小）。

这是 2025 年的快照。重点不是哪个模型在 2026 年 4 月胜出 —— 而是不同的基础模型具有持久的谈判风格。异质集成（课程 15）将其作为多样性来源。

### 通过合同网加大语言模型的任务分配

合同网在大语言模型多智能体中的现代重用：

1. 管理者智能体将任务分解为单位。
2. 向工作智能体广播带任务描述的 `cfp`。
3. 每个工作者返回一个报价：`(price, eta, confidence)`，其中 price 可以是词元数、计算单元或美元。
4. 管理者选择赢家（单个或多个，取决于任务）并分配。
5. 被拒绝的工作者可以自由投标其他任务。

这在超过 100 个工作者时扩展良好，因为协调是广播-回应模式，而不是同步聊天。已在生产中使用：微软 Agent Framework 的编排模式，一些 LangGraph 实现。

### 大语言模型利益相关者互动协商

NeurIPS 2024（https://proceedings.neurips.cc/paper_files/paper/2024/file/984dd3db213db2d1454a163b65b84d08-Paper-Datasets_and_Benchmarks_Track.pdf）引入了具有**秘密分数**和**最低接受阈值**的多方可评分游戏。每个利益相关者有私人效用；大语言模型必须从消息中推断它们。这是双方议价到 N 方联盟形成的泛化。对于具有异质工作能力的生产任务市场具有相关性。

### 叙述 vs 机制规则

在所有 2024-2026 年协商基准测试中，一致的工程规则是：

> 让大语言模型叙述。不要让大语言模型计算出价。

如果出价需要是数字（价格、预计时间、数量），从协商状态确定性地生成它，然后让大语言模型产生框架。如果出价需要是提案结构（任务分解、角色分配），让大语言模型起草，但在发送前根据 schema 和约束检查进行验证。

## 构建它

`code/main.py` 实现了：

- `ContractNetManager`, `ContractNetTask`, `Bid` —— 管理者 + 投标者，广播 cfp，收集提案，分配。
- `og_narrator_bargain(state, rng)` —— OG-Narrator 买方：确定性的 Zeuthen 风格向中点让步。
- `seller_response(state, rng)` —— 确定性卖方还价策略（两种风格的结构性真实基准）。
- `naive_llm_bargain(state, rng)` —— 模拟全大语言模型谈判者：以高方差选择价格，经常在 ZOPA 之外。
- 测量：1000 次试验中的成交率，每次试验采样新鲜保留价。

运行：

```
python3 code/main.py
```

预期输出：naive-LLM 成交率约 65-75%；OG-Narrator 成交率约 85-95%；15-25 个百分点的差距是将出价生成与叙述分离的结构性优势。外加一个带三个投标者和一个任务的合同网任务市场分配示例。

## 交付它

生产级协商检查清单：

- **分离草稿区。** 私人状态永远不会到达对方的上下文。这是不可谈判的。
- **确定性出价生成。** 价格、数量、预计时间：计算，不要提示。
- **验证所有收到的出价** 是否符合 schema。在协议边界拒绝超出 ZOPA 的出价。
- **限制轮次。** 最多 3-5 轮；在死锁时升级给调解者。
- **持续监控成交率和收益方差。** 成交率下降是一个症状 —— 通常是提示漂移或对方攻击。
- **记录所有被拒绝的提案** 及其确定性理由。对于合同网管理者，失败的投标者需要知道原因。

## 练习

1. 运行 `code/main.py`。确认 OG-Narrator 在成交率上击败 naive-LLM。高出多少？
2. 实现**基于角色的收益改善**（arXiv:2402.05863）—— 买方仅在叙述中采用"急于本周购买"的角色，出价生成器不变。成交率或收益会变化吗？
3. 实现思维链**隐藏**：维护一个不传给对方的私人草稿区字符串。如果你意外泄露它会发生什么（通过交换通道模拟）？
4. 将合同网扩展为带保留价的 N 投标者拍卖。当所有投标超过保留价时，管理者如何在最低价和最高质量之间决定？你选择哪个分配规则，为什么？
5. 阅读 Bhattacharya 等人 2025 年关于哈佛谈判项目指标的论文。实现两个具有不同风格（攻击型 vs 公平型）的谈判者。在对称和非对称配对下测量收益方差。

## 关键术语

| 术语 | 人们说的 | 实际含义 |
|------|----------|----------|
| 合同网 | "任务市场" | Smith 1980, FIPA 1996。cfp + propose + accept/reject。标准任务市场。 |
| ZOPA | "可能达成一致的区间" | 买方最高价和卖方最低价的重叠。在此之外的出价无法成交。 |
| BATNA | "谈判协议的最佳替代方案" | 如果这笔交易失败，你的备选方案。设定你的保留价。 |
| OG-Narrator | "出价生成器 + 叙述器" | 分解：确定性出价，大语言模型叙述。 |
| Zeuthen 策略 | "风险最小化让步" | 基于风险限制进行让步的经典出价生成器。 |
| Rubinstein 议价 | "交替出价均衡" | 带有贴现的无限期议价的博弈论模型。 |
| CoT 隐藏 | "隐藏你的推理" | arXiv:2503.06416 中胜者保持私人草稿区；公开通道只显示出价。 |
| 角色操纵 | "情绪姿态" | arXiv:2402.05863：从绝望/紧迫角色中获得约 20% 的收益提升。 |
