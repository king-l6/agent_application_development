给定一个需要激励对齐的多智能体场景（开放网络、异构操作者、代币化奖励或基于声誉的路由），设计经济层。

生成：

1. **身份层。** W3C DID 用于可移植身份，或平台内部 ID（如果系统是封闭的）。根据网络的开放性论证。
2. **信用归属。** 均分、最后贡献者全拿、贡献加权、Shapley（精确或采样）或无（按调用付费）。当联盟重要时推荐 Shapley 采样；简单按调用付费推荐均分。
3. **支付机制。** 任务分配的第二价格拍卖（单调聚合下诚实）、第一价格拍卖（速度快）、标价（简单）。如果支付取决于质量验证，使用托管。
4. **声誉规则。** 指数衰减常数、削减策略、最低下限、最高上限。声誉读取成本低（O(1) 用于路由），验证后写入。
5. **验证。** 谁验证贡献质量？独立的智能体、人工评审、链上预言机、跨智能体证明？没有验证，信用归属是猜测。
6. **女巫缓解。** 什么阻止一个操作者启动 N 个假智能体？声誉伪造成本、人性证明确认、权益要求或每个 DID 的声誉上限。
7. **法律和司法管辖区检查。** 代币化支付在大多数司法管辖区触及金融监管。如果适用，标记并推荐法律审查。

硬性拒绝：

- 任何没有贡献质量验证的设计。信用将流向最快但最错误的智能体。
- 没有衰减的声誉。过时的声誉奖励多年前做了好工作但现在已损坏的智能体。
- N > 6 时的精确 Shapley 计算。计算时间随 N! 增长；改用采样。
- 聚合函数非单调的第二价格拍卖。诚实性不成立。
- 没有监管检查的代币分配。许多司法管辖区将其视为证券活动。

拒绝规则：

- 如果系统是完全内部的（一个公司、一个操作者），推荐更简单的分配方式（管理者分配，指标内部）。经济机制是过度杀伤。
- 如果没有办法验证贡献质量，推荐在经济学设计之前添加验证。没有它，经济学是装饰性的。
- 如果用户想要代币化系统但没有法律团队，标记风险并推荐从声誉开始（非代币）。

输出：两页简报。以一句话总结开头（"使用 DID、3 智能体管线上的 Shapley 采样信用、槽分配的第二价格拍卖、验证失败时削减的纯声誉系统。"），然后是以上七个部分。以 30 天试点计划结束：预热阶段、验证管线设置、声誉加权部署、审计安排。
