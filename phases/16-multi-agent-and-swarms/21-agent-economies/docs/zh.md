# 智能体经济、代币激励与声誉

> 长时程自主智能体（METR 的 1 小时到 8 小时工作曲线）需要经济行为能力。新兴的**5 层堆栈**是：**DePIN**（物理算力）→ **身份**（W3C DID + 声誉资本）→ **认知**（RAG + MCP）→ **结算**（账户抽象）→ **治理**（智能体 DAO）。生产级智能体激励网络包括 **Bittensor**（TAO 子网奖励特定任务模型）、**Fetch.ai / ASI Alliance**（ASI-1 Mini LLM + FET 代币）和 **Gonka**（基于 Transformer 的 PoW，将算力重新分配给生产性 AI 任务）。学术工作：AAMAS 2025 的去中心化 LaMAS 使用 **Shapley 值信用归属** 来公平奖励贡献智能体；Google Research "大型语言模型的机制设计" 提出单调聚合下的第二价格代币拍卖。本课构建一个最小智能体市场，对多智能体管线应用 Shapley 值信用归属，并运行一个第二价格代币拍卖，使博弈论机制具体落地。

**类型：** 学习
**语言：** Python（标准库）
**前置条件：** Phase 16 · 16（谈判与讨价还价），Phase 16 · 09（并行群体网络）
**时间：** ~75 分钟

## 问题

当多智能体系统共同产生价值但需要分别奖励每个智能体时，事情变得复杂。经典机制 —— 均分、最后贡献者全拿 —— 要么不公平要么可被博弈。基于联盟的 Shapley 值奖励通过构造是公平的，但计算成本高昂。2025-2026 年的文献推动了有用的近似方法：Shapley 采样、单调聚合拍卖，以及基于已确认贡献累积的链上声誉。

除了信用归属，该领域已转向实际的经济智能体：Bittensor TAO 奖励提供算力来微调子网特定模型的矿工，Fetch.ai/ASI 用 FET 代币奖励 ASI-1 Mini LLM 的使用，Gonka 将 Transformer 工作量证明重新分配给生产性 AI 任务。能够自主交易的智能体今天已经存在；问题是如何对齐激励。

本课将智能体经济视为一个特定的问题族 —— 信用归属、机制设计和声誉 —— 并以最少的数学构建每个方面，使这些思想扎实落地。

## 概念

### 5 层智能体经济堆栈

1. **DePIN（物理算力）**。出租 GPU、存储、带宽的去中心化基础设施。Bittensor 子网、Render Network、Akash。非智能体专用；智能体使用它。
2. **身份**。W3C 去中心化标识符（DID）为每个智能体提供一个独立于任何平台的持久 ID。声誉累积到 DID。智能体网络协议（ANP）使用 DID 作为发现层。
3. **认知**。智能体的推理循环：LLM + RAG + MCP。这是其他阶段构建的内容。
4. **结算**。账户抽象（ERC-4337）让智能体无需持有 ETH 即可从其自身余额支付 gas。智能体可以支付服务费用、彼此之间或算力费用。
5. **治理**。智能体 DAO：人类*和*智能体对协议变更进行投票的治理结构，投票权与声誉挂钩。

并非每个生产系统都使用全部五层。Bittensor 使用 1、2、部分 3、部分 4、不包含 5。OpenAI 智能体除了 3 之外什么都不用。该堆栈是一个参考地图，而非要求。

### Bittensor、Fetch.ai、Gonka —— 实际运行的系统

**Bittensor（TAO）**。子网是专门化的任务（语言建模、图像生成、预测）。矿工提交模型输出。验证者对其进行排名；权益加权评分分配 TAO 奖励。每个子网有自己的评估。经济教训：为任务特定的输出质量付费，而非使用的算力。

**Fetch.ai / ASI Alliance**。ASI-1 Mini LLM 在 Fetch.ai 网络上运行；用户支付 FET 代币进行推理。在此，智能体即对等的叙事更为强烈：Fetch 上的一个智能体可以调用另一个来完成任务并用 FET 支付。

**Gonka**。Transformer 工作量证明："工作"是 Transformer 的前向传播。矿工通过运行具有已知正确输出（来自训练数据）的推理任务来赚取收益。资源生产性 PoW 取代基于哈希的 PoW。

截至 2026 年 4 月，三者均已达到生产级。收益分配不同。Bittensor 根据子网验证者的标准奖励质量；Fetch 奖励由付费用户衡量的效用；Gonka 奖励可验证的推理工作。

### Shapley 值信用归属

三个智能体协作完成任务。输出得分为 0.8。谁贡献了什么？

Shapley 值：满足四个公理（效率、对称性、线性、零元素）的唯一信用分配。对于智能体 `i`：

```
shapley(i) = (1/N!) * sum over all orderings O of (v(S_i_O ∪ {i}) - v(S_i_O))
```

其中 `S_i_O` 是在排序 `O` 中位于 `i` 之前的智能体集合。实践中：枚举所有排列，记录每个排列中每个智能体的边际贡献，取平均。

对于 N=3 个智能体，有 6 种排列。对于 N=10，有 360 万种 —— 因此实践中你采样排序而非枚举。

### 用于聚合的第二价格拍卖

Google Research（"大型语言模型的机制设计"）提出了用于聚合 LLM 输出的第二价格代币拍卖。设置：N 个智能体各自提出一个补全；每个智能体对被选中有一个私人估值。拍卖人选择估值最高的提案，并支付*第二高*的估值。在单调聚合下（价值取决于选择哪个提案，而非有多少投标），这是诚实的 —— 智能体出价其真实估值。

为什么这对 LLM 系统很重要：你可以将补全任务外包给多个具有不同定价的智能体；拍卖选择最佳 + 支付公平，且智能体没有误报的动机。

### 声誉资本

一个绑定 DID 的声誉分数从已确认的贡献中累积。一个简单的更新规则：

```
rep(i, t+1) = alpha * rep(i, t) + (1 - alpha) * contribution_quality(i, t)
```

衰减因子 `alpha` 接近 1。声誉：

- 读取成本低，用于路由决策（"将困难任务发送给高声誉智能体"）。
- 伪造成本高（随时间累积，绑定到 DID）。
- 可被削减：未通过验证的贡献会减少声誉。

### AAMAS 2025 去中心化 LaMAS

LaMAS 提案（AAMAS 2025）结合了：DID 身份、Shapley 值信用归属和简单的拍卖机制。其核心主张：去中心化信用归属步骤使系统可审计且免疫于单点操纵。

### 经济学在哪些方面失效

- **价格预言机操纵。** 如果信用函数可被博弈，智能体会去博弈它。每个机制都需要对抗性测试。
- **女巫攻击。** 一个操作者启动 N 个假智能体来膨胀自己的贡献。DID 可以减缓但无法阻止；伪造声誉成本是缓解措施。
- **验证成本。** 信用归属的公平性取决于验证者。如果验证成本低（小型 LLM），它可被博弈；如果成本高（人工评审），系统无法扩展。
- **监管悬而未决。** 智能体经济触及金融监管。Bittensor、Fetch 和 Gonka 截至 2026 年在某些司法管辖区均处于法律灰色地带。

### 智能体经济何时有意义

- **具有异构操作者的开放网络。** 没有单一团队控制所有智能体。
- **可验证的输出。** 没有验证，信用归属是猜测。
- **长时程工作流。** 一次性任务不会从声誉积累中受益。
- **代币化支付在法律上可行** 在你的司法管辖区。

在封闭的企业系统中，经济学让位于更简单的分配方式（管理者分配工作，指标是内部的）。经济学文献主要适用于开放网络。

`code/main.py` 实现：

- `shapley(value_fn, agents)` —— 对于小 N，通过枚举进行精确 Shapley 计算。
- `second_price_auction(bids)` —— 诚实机制；胜者支付第二高。

- 演示 2：五个智能体竞标一个任务槽；第二价格拍卖选出胜者 + 支付。
- 演示 3：100 轮任务分配给具有异构声誉的智能体；声誉加权路由优于随机。

运行：

```
python3 code/main.py
```

## 交付它

在 2026 年运行智能体经济：

- **从声誉开始，而非代币。** 声誉实现成本低且本身就有价值；代币增加了法律和经济复杂性。
- **先验证，后奖励。** 切勿在没有独立验证步骤的情况下分配信用。自我报告的质量会招致女巫游戏。
- **Shapley 采样，而非精确 Shapley。** 采样 100-1000 个排序；精确枚举不可扩展。
- **设置衰减因子下限和声誉下限。** 无限衰减会抹去合法贡献者；过慢的衰减会奖励过时的高声誉智能体。
- **对抗性审计机制。** 在开放网络前运行红队场景。每个机制都有博弈论；你要找到漏洞，而不是攻击者。

## 练习

1. 运行 `code/main.py`。确认 Shapley 值之和等于总价值（效率公理）。改变价值函数；Shapley 分配是否按预期方向变化？
2. 实现 Shapley *采样*（在 K 个排序上的蒙特卡洛）。K 如何影响近似精度？与 N=4 时的精确值进行比较。
3. 在拍卖前实现一个联盟形成步骤：智能体可以合并为团队并作为一个整体竞标。哪些联盟会形成？结果是否帕累托优于个体竞标？
4. 阅读 Google Research 机制设计博文。识别一个若被违反会破坏诚实性的假设。在 LLM 环境中该失败模式是什么样的？
5. 阅读 AAMAS 2025 去中心化 LaMAS 论文。在合成任务上对 10 个智能体实现其 Shapley 步骤。精确计算需要多长时间？100 次抽样的采样结果有多接近？

## 关键术语

| 术语 | 人们说的 | 实际含义 |
|---|---|---|
| DePIN | "去中心化物理基础设施" | 代币激励的算力/存储/带宽。Bittensor、Akash、Render。 |
| DID | "去中心化标识符" | W3C 可移植 ID 规范。智能体声誉绑定到 DID 而非平台。 |
| ERC-4337 | "账户抽象" | 可赞助 gas 的合约账户，使智能体支付成为可能。 |
| Shapley 值 | "公平信用归属" | 满足效率、对称性、线性、零元素的唯一分配。 |
| 第二价格拍卖 | "Vickrey 拍卖" | 诚实机制：胜者支付第二高出价。兼容单调聚合。 |
| 声誉资本 | "累积质量分数" | 绑定 DID 的分数，来自已确认贡献；随时间衰减。 |
| 智能体 DAO | "智能体 + 人类治理" | 智能体作为一等投票者的 DAO，投票权与声誉挂钩。 |
| TAO / FET / GPU credits | "代币面额" | Bittensor TAO、Fetch.ai FET、各类 DePIN 代币。 |
