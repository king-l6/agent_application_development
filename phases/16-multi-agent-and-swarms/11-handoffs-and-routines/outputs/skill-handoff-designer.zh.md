给定一个面向用户的任务（通常是分诊或基于技能的路由），生成一个可直接映射到 OpenAI Swarm 或 OpenAI Agents SDK 的交接拓扑。

生成内容：

1. **智能体名册。** 每个智能体：名称、一句话用途、工具以及它可以交接给哪些其他智能体。
2. **交接函数。** 每个智能体的工具签名。每个交接函数返回一个目标 Agent。
3. **上下文传递策略。** 在每个交接边上：完整历史、最近 N 条消息或摘要快照。说明理由。
4. **安全防护。** 每个智能体的输入验证（允许哪些提示触发对敏感专家的交接）、必要时的交接身份验证。
5. **循环检测。** 检测乒乓的规则（例如，"A 交接给 B；B 交接回 A"连续出现超过一次）。
6. **后备行为。** 如果交接目标缺失（智能体被移除、认证失败），哪个智能体处理会话。
7. **会话/记忆方案。** 是否使用 Agents SDK 会话、调用方管理的记忆，或完全不使用记忆。

硬拒绝：

- 任何没有循环检测的交接设计。
- 将完整历史传递给具有不同工具权限的专家的交接函数（安全风险）。
- 假设 Swarm 的无状态行为但又需要多轮记忆的设计——改用 Agents SDK 会话。

拒绝规则：

- 如果任务需要并行执行，拒绝 Swarm 并推荐监督者（第 05 课）。
- 如果任务需要确定性审计/重放，拒绝并推荐 LangGraph 静态图。
- 如果任务是一个简单的阶段有向无环图（研究 → 编码 → 审查），推荐 CrewAI 顺序模式。

输出：一页的交接简报。以安全说明结束，说明提示注入如何触发不必要的交接以及哪些安全防护可以阻止它。
