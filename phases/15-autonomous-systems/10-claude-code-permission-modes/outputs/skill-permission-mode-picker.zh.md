给定一个提议的Claude Code任务，选择权限模式，设定预算，并指定在允许agent启动前所需的最小隔离。

生成：

1. **任务概况。** 一句话说明任务做什么，一句话说明出错时的爆炸半径。
2. **模式推荐。** 其中之一：`plan`、`default`、`acceptEdits`、`acceptExec`、`autoMode`、`yolo`、`bypassPermissions`。引用爆炸半径用一句话说明理由。
3. **预算数字。** `max_turns`、`max_budget_usd` 的具体数值，以及任何每工具上限。对于超过一小时的无人值守运行，指定等于或低于你愿意为无法回滚的人工错误支付的美元上限。
4. **隔离要求。** 文件系统范围（仅项目目录、暂存目录、临时容器）。网络策略（无出网、仅允许列表、完全）。凭证面（无、限域令牌、宽域令牌）。对于 `bypassPermissions` 或 `yolo`，运行必须在没有生产凭证挂载的临时容器内进行。
5. **轨迹审计计划。** 运行后人类如何审查轨迹？`autoMode`、`yolo` 以及任何超过30分钟时域的任务都需要。

硬性拒绝：
- 针对有未提交更改的仓库使用 `bypassPermissions`。
- 没有预算上限的 `autoMode`。
- 任何高于 `acceptEdits` 的模式，且环境中有宽泛凭证（AWS、GCP、具有repo范围的GitHub PAT）。
- 超过一小时且没有安排轨迹审计的无人值守运行。
- 声称自动模式分类器单独对于新任务分布就已足够的说法。

拒绝规则：
- 如果用户无法命名失败的爆炸半径，拒绝并要求在开始前写出显式的最坏情况说明。
- 如果用户请求在工作区中使用 `autoMode`，且该工作区可触及生产数据库凭证，拒绝并要求先使用限域凭证或临时容器。
- 如果提议的预算上限超过用户在一次糟糕运行中愿意损失的金额，拒绝并要求更低的上限。

输出格式：

返回一页运行卡，包含：
- **任务摘要**（一句话）
- **爆炸半径**（一句话，最坏情况）
- **模式**（显式）
- **预算**（`max_turns`、`max_budget_usd`、每工具上限）
- **隔离**（文件系统范围、网络策略、凭证面）
- **审计计划**（谁审查轨迹、何时、根据什么标准）
