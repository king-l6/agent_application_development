# 嵌入模式

每个嵌入工作流都遵循这个约定：

```
text -> embed(text) -> vector (float array)
similarity(vector_a, vector_b) -> score (float)
```

嵌入模型和相似度指标是唯一重要的两个决策。其他都是管道工程。

## 何时使用嵌入

- 跨文档的语义搜索（查找含义，而非关键词）
- 相似项聚类（支持工单、产品评论、错误报告）
- 通过最近邻进行分类（通过与标注示例的相似性标记新项）
- 推荐系统（查找与用户喜欢的内容相似的项目）
- 去重（使用相似度阈值查找近似重复内容）

## 何时不使用嵌入

- 精确关键词匹配（使用全文搜索）
- 结构化查询（使用 SQL、过滤器）
- 手动标注更快的小数据集（<100 项）
- 可解释性比准确性更重要的任务（嵌入是不透明的）

## 模型选择

根据你的约束条件选择：

- **需要 API，最佳性价比**：OpenAI text-embedding-3-small（1536d，$0.02/100 万词元）
- **需要最大准确性**：Voyage-3（1024d，$0.06/100 万词元，最高 MTEB）
- **需要本地/私有**：BGE-M3（1024d，免费，多语言，推荐 GPU）
- **需要快速本地原型**：all-MiniLM-L6-v2（384d，免费，在 CPU 上运行）
- **需要多语言**：Cohere embed-v3（1024d）或 BGE-M3（都支持多语言）

规则：绝不要在索引和查询之间混合使用嵌入模型。来自不同模型的向量存在于不兼容的空间中。

## 分块规则

1. 目标为每个块 256-512 词元，50 词元重叠
2. 如果可以避免，绝不要在句子中间切分
3. 为每个块包含元数据（源文件、章节标题、位置）
4. 对于结构化文档（Markdown、HTML），首先在标题边界切分
5. 通过搜索已知答案并检查检索来测试块质量

## 相似度指标选择

- **余弦相似度**：默认选择，处理可变长度文本，已归一化
- **点积**：当向量已经单位归一化时使用（OpenAI 模型就是），略快
- **欧几里得距离**：用于聚类，当绝对位置重要时

当向量被归一化时，三者给出相同的排名。选择只对非归一化向量有意义。

## 存储优化

三种压缩级别，可叠加：

1. **Matryoshka 截断**：减少维度（1536 -> 256 = 6 倍节省，3-5% 精度损失）
2. **Float16 量化**：每维存储减半（2 倍节省，<1% 精度损失）
3. **二进制量化**：每维 1 比特（32 倍节省，5-10% 精度损失，配合重新评分使用）

生产模式：对整个语料库进行二进制搜索，用 float32 向量重新评分前 1000 个。

## 检索-重排序

两阶段流水线实现最佳准确性：

1. 双编码器检索前 100 个候选（快速，使用预计算嵌入）
2. 交叉编码器重排序为前 10 个（慢，处理每个查询-文档对）

这在精度指标上比单阶段检索好 10-15%。当准确性比延迟更重要时使用。

## 常见错误

- 索引和查询使用不同的嵌入模型
- 嵌入整个文档而不是分块（嵌入变成一切的平均）
- 在余弦相似度前不归一化向量（大多数模型预归一化，但请验证）
- 忽略块重叠（在边界处切分的句子丢失上下文）
- 只存储向量而不存原始文本（检索需要两者）
- 模型变更时不重新嵌入（旧向量不兼容）
- 仅根据准确性选择维度（存储和延迟与维度线性相关）

## 调试嵌入

如果搜索结果差：

1. 验证查询嵌入非零（空或空白输入产生零向量）
2. 手动检查已知相关文档的相似度分数
3. 尝试重新表述查询以匹配文档词汇
4. 检查块边界以确保相关内容未被切分到不同块
5. 跨指标（余弦、点积、欧几里得）比较 top-k 结果以发现归一化问题
6. 用简单匹配的查询（从文档中复制一个句子）测试以确认流水线工作

## 生产参数

- 块大小：256-512 词元
- 块重叠：50 词元（块大小的 10-20%）
- Top-k 检索：5-10 用于直接使用，50-100 用于重排序
- 相似度阈值：余弦相似度 0.7+（低于此结果通常无关）
- 批量嵌入：每次 API 调用处理 100-500 段文本以获得吞吐量
- 索引重建：当模型变化或文档显著更新时重新嵌入
