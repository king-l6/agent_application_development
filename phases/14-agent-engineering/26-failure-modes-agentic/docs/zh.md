# 故障模式：智能体为何失效

> MASFT（伯克利，2025）在 3 个分类中记录了 14 种多智能体故障模式。微软的分类法记录了现有 AI 故障如何在智能体环境中被放大。行业实地数据收敛到五种反复出现的模式：幻觉行为、范围蔓延、级联错误、上下文丢失、工具误用。

**类型：** 学习 + 构建
**语言：** Python（标准库）
**前置知识：** 阶段 14 · 05（自我精炼与 CRITIC），阶段 14 · 24（可观测性）
**时间：** ~60 分钟

## 学习目标

- 说出 MASFT 的三个故障类别以及每个类别中至少四种具体模式。
- 解释为什么智能体故障会放大现有的 AI 故障模式（偏见、幻觉）。
- 描述五种行业反复出现的模式及其缓解措施。
- 实现一个在标准库中为智能体追踪打上故障模式标签的检测器。

## 问题

团队交付的智能体在 90% 的追踪上工作正常。那 10% 的失败并非随机噪声——它们属于少量反复出现的类别。一旦您能够命名它们，就可以监控并修复它们。

## 概念

### MASFT（伯克利，arXiv:2503.13657）

多智能体系统故障分类法。14 种故障模式聚类为 3 个类别。标注者间 Cohen's Kappa 0.88——这些类别可以可靠地区分。

核心主张：故障是多智能体系统中的根本设计缺陷，而不是通过更好的基础模型就能修复的 LLM 局限性。

### 微软智能体 AI 系统故障模式分类法

- 现有的 AI 故障（偏见、幻觉、数据泄露）在智能体环境中被放大。
- 新的故障从自主性中涌现：大规模意外行为、工具误用、任务偏移。
- 该白皮书是智能体产品的风险登记册。

### 智能体 AI 中故障的特征刻画（arXiv:2603.06847）

- 故障源于编排、内部状态演化和环境交互。
- 不仅仅是"糟糕的代码"或"糟糕的模型输出"。

### LLM 智能体幻觉调查（arXiv:2509.18970）

两种主要表现：

1. **指令遵循偏差**——智能体不遵循系统提示。
2. **长程上下文误用**——智能体忘记或误用了早期轮次的上下文。

子意图错误：遗漏（错过步骤）、冗余（重复步骤）、无序（步骤顺序错误）。

### 五种行业反复出现的模式

Arize、Galileo、NimbleBrain 2024-2026 年实地分析收敛于：

1. **幻觉行为。** 智能体调用不存在的工具或捏造参数。
2. **范围蔓延。** 智能体将任务扩展到用户要求之外（创建额外的 PR、发送额外的邮件）。
3. **级联错误。** 一个错误的调用触发下游影响。一个幻影 SKU 幻觉触发四次 API 调用——一次多系统事故。
4. **上下文丢失。** 长周期任务忘记早期轮次的约束。
5. **工具误用。** 使用正确的工具但参数错误，或者完全使用错误的工具。

级联是最致命的。智能体无法区分"我失败了"和"任务不可能完成"，并且常常在 400 错误上幻觉出成功消息以闭合循环。

### 缓解措施：每一步设置关卡

在推理链的每一步设置自动化验证关卡，检查与环境状态的事实依据。具体来说：

- 逐步骤安全分类器（第 21 课）。
- 工具调用参数验证（第 06 课）。
- 交叉检查检索内容与已知事实（第 05 课，CRITIC）。
- 通过重新探测状态检测成功幻觉（文件是否实际创建了？）。

### 故障监控出错的地方

- **仅标记崩溃。** 大多数智能体故障产生看起来正常的输出。需要进行内容级检查。
- **没有基线。** 漂移检测需要已知良好的参照；没有它就无从判断"情况正在恶化"。
- **过度告警。** 每个故障都触发通知。需要进行聚类和限速。

## 构建

`code/main.py` 实现了一个标准库故障模式标签器：

- 覆盖五种模式的合成追踪数据集。
- 每种模式的检测函数（基于工具调用、输出、重复行为的签名模式）。
- 为每个追踪打标签并报告模式分布的标签器。

运行：

```
python3 code/main.py
```

输出：每个追踪的标签 + 聚合分布，是对 Phoenix 追踪聚类能力的低成本复现。

## 练习

1. 添加"成功幻觉"检测器：智能体返回成功但目标状态未改变。
2. 标记您构建的产品的 100 条真实追踪。哪种模式占主导？修复它的成本是多少？
3. 实现一个"级联半径"指标：给定第 N 步的故障，它影响了多少下游步骤？
4. 阅读 MASFT 的 14 种故障模式。选择适用于您产品的三种。编写检测器。
5. 将一个检测器接入 CI 任务：如果 >=5% 的追踪标记了某种模式，则使构建失败。

## 关键术语

| 术语 | 人们说的 | 实际含义 |
|------|---------|---------|
| MASFT | "多智能体故障分类" | 伯克利 14 种模式分类 |
| 级联错误 | "连锁故障" | 一个早期错误通过 N 步传播 |
| 上下文丢失 | "忘记约束" | 长周期交互丢失早期轮次的事实 |
| 工具误用 | "错误工具/错误参数" | 合法调用但执行不当 |
| 成功幻觉 | "虚假完成" | 智能体在 400 错误上声称成功；状态未变 |
| 范围蔓延 | "越界" | 智能体做了超出要求的事情 |
| 指令遵循偏差 | "不服从" | 忽略系统提示或用户约束 |
| 子意图错误 | "计划中的 bug" | 计划执行中的遗漏、冗余、无序 |
