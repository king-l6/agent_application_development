给定一个目标运行时、一个向量后端（Qdrant、pgvector、Chroma、sqlite-vec）、一个键值后端（Postgres、Redis、dict）和一个图后端（Neo4j、内存边），生成一个融合记忆系统。

生成内容：

1. 三个存储类，统一在一个 `add(text, user_id, session_id, scope, importance, tags)` 外观类后面。写入时，提取器将 `text` 分解为记录、键值三元组和图三元组。没有一个存储是可选的。
2. 一个融合评分器 `score = w_rel * relevance + w_imp * importance + w_rec * recency`。将三个权重暴露为可配置项。按产品调整，而非按每次调用调整。
3. 作用域分类：`user`、`session`、`agent`。检索必须遵守作用域。用户查询绝不能泄露另一个用户的记录。
4. 时间失效。矛盾将旧边/旧记录标记为无效；绝不删除。暴露 `search(query, as_of=timestamp)` 用于历史查询。
5. 一个提取器接口。默认可以是 LLM 驱动的；允许一个确定性的正则表达式回退用于测试。限制每次 `add()` 的图边数量以防止爆炸。

硬性拒绝：

- 被描述为"Mem0 风格"的单一存储记忆。仅向量、仅键值、仅图的产品没问题，但不是混合记忆。不要误称。
- 没有按作用域权重或显式 `scope=` 过滤器的跨作用域检索。作用域泄露是合规和隐私事件。
- 在产生矛盾时删除。应使其失效并添加时间戳。删除会掩盖 bug 并破坏审计。

拒绝规则：

- 如果用户要求"不做重要性加权"，拒绝。在一百万条记录上使用扁平相关性排序是一场即将发生的检索失败。
- 如果图后端没有冲突检测器，拒绝将结果系统称为"Mem0 风格"。降低名称等级。
- 如果产品涉及 PII（医疗、法律、人力资源），拒绝在提取器未经产品负责人审计的情况下发布。

输出：每个存储一个文件，外加 `memory.py`（外观类）、`config.py`（权重）、`README.md` 解释融合权重、作用域策略、提取器契约和失效语义。以"下一步阅读"结束，如果 Agent 需要学习新技能则指向第 10 课，如果需要在记忆操作上添加 OTel 跨度则指向第 23 课，或如果涉及检索时的不可信输入处理则指向第 27 课。
