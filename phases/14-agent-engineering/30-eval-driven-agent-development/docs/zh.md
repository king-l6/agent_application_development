# 评估驱动的智能体开发

> Anthropic 的指导："从简单的提示开始，通过全面评估优化它们，仅在需要时才添加多步骤智能体系统。"评估不是最后一步。它是驱动阶段 14 中所有其他选择的外循环。

**类型：** 学习 + 构建
**语言：** Python（标准库）
**前置知识：** 阶段 14 全部
**时间：** ~60 分钟

## 学习目标

- 说出三个评估层面——静态基准、自定义离线、在线生产——以及各自的用途。
- 解释评估器-优化器紧循环。
- 描述 2026 年最佳实践：评估与代码共存，在 CI 中运行，门控 PR 合并。
- 将阶段 14 的每节课连接到它生成的评估用例。

## 问题

智能体通过演示。它们在演示无法预测的方式下在生产中失败。基准测试回答的是"这个模型大致有能力吗？"而不是"这个智能体在为我的产品提供正确的补丁吗？"答案是：在三个层面上持续评估，每个护栏和学到的规则都映射到一个评估用例。

## 概念

### 三个评估层面

1. **静态基准**——用于代码的 SWE-bench Verified（第 19 课）、用于浏览/桌面的 WebArena/OSWorld（第 20 课）、用于通才的 GAIA（第 19 课）、用于工具使用的 BFCL V4（第 06 课）。用于跨模型比较和回归门控。数据污染是真实的：SWE-bench+ 发现 32.67% 的解决方案泄露。始终报告 Verified/+ 审核后的分数。

2. **自定义离线评估**——您的产品形态：
   - LLM 作为评判者（Langfuse、Phoenix、Opik——第 24 课）。
   - 基于执行（运行补丁，检查测试）。
   - 基于轨迹（比较动作序列与黄金标准；OSWorld-Human 显示顶级智能体是黄金标准的 1.4-2.7 倍）。

3. **在线评估**——生产环境：
   - 会话回放（Langfuse）。
   - 护栏触发的告警（第 16、21 课）。
   - 每步成本/延迟追踪（第 23 课 OTel 跨度）。

### 评估器-优化器（Anthropic）

紧循环：

1. 提议者生成输出。
2. 评估者评判。
3. 精炼直到评估者通过。

这是自我精炼（第 05 课）的泛化。任何您关心的智能体流程都可以包裹在评估器-优化器中以提高可靠性。

### 2026 年最佳实践

- 评估与代码共存。
- 在每个 PR 的 CI 中运行。
- 根据评估分数门控合并（例如，"与主分支相比无超过 5% 的回归"）。
- 每个护栏映射到一个评估用例。
- 每个学到的规则（Reflexion、专业工作流中的学习规则）映射到一个失败用例。

### 评估驱动开发失败的地方

- **没有基线。** 没有已知良好参照的评估是不可读的。存储基线。
- **没有依据的 LLM 评判者。** 评判者也会产生幻觉。CRITIC 模式（第 05 课）——评判者依据外部工具进行判断。
- **过度拟合评估。** 针对评估集优化会偏离生产实用性。轮换用例。
- **不稳定的评估。** 非确定性用例导致误报。固定种子，快照状态。

## 构建

`code/main.py` 是一个标准库评估框架：

- 带有分类（基准、自定义、在线）的用例注册表。
- 一个被测试的模拟智能体。
- 评估器-优化器循环：提议、评判、精炼直到通过或达到最大轮次。
- CI 门控：聚合通过率 + 与基线的回归。

运行：

```
python3 code/main.py
```

输出：每个用例的通过/失败、回归标志、CI 门控裁定。

## 使用

- 在与智能体代码相同的仓库中编写评估用例。
- 通过 CI 在每个 PR 上运行它们。
- 在回归时使构建失败。
- 跟踪随时间变化的通过率。
- 将每个生产故障关联到一个新用例。

## 交付

`outputs/skill-eval-suite.md` 为一个智能体产品构建一个三层评估套件，包含 CI 门控和回归追踪。

## 练习

1. 取一个您的生产故障。编写一个复现它的评估用例。您的智能体现在能通过它吗？
2. 为您的领域构建一个包含三个维度（事实准确性、语气、范围）的 LLM 评判者评分标准。对 50 个会话进行评分。
3. 将评估套件接入 CI。在 >=5% 回归时使构建失败。
4. 添加轨迹效率指标：智能体走了多少步与黄金轨迹相比？
5. 将阶段 14 中的每节课映射到您套件中的一个评估用例。有缺失吗？那是需要填补的空缺。

## 关键术语

| 术语 | 人们说的 | 实际含义 |
|------|---------|---------|
| 静态基准 | "现成的评估" | SWE-bench、GAIA、AgentBench、WebArena、OSWorld |
| 自定义离线评估 | "领域评估" | 基于 LLM 评判者/执行/轨迹，针对您的产品形态 |
| 在线评估 | "生产评估" | 会话回放、护栏告警、成本/延迟追踪 |
| 评估器-优化器 | "提议-评判-精炼" | 迭代直到评判者通过 |
| CI 门控 | "合并阻止器" | 在评估回归时使构建失败 |
| 基线 | "已知良好参照" | 用于检测回归的参考分数 |
| 轨迹效率 | "步数/黄金步数" | 智能体步数除以人类专家最小值 |
