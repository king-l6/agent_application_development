给定一个任务形态（单一答案 / 多答案 / 开放式）、一个令牌预算和一个可用的评估器（标量测试 / 启发式 / 自评估），生成一个带具体参数的搜索策略建议。

输出：

1. 决策。以下之一：线性 ReAct、beam ToT（带束宽度 k）、BFS ToT（带最大深度）、带剪枝的 DFS ToT、MCTS LATS（带迭代次数和 UCT c）、进化搜索（仅当评估器是可编程且可检查的）。
2. 参数。对于每个策略，具体的数值默认值：束宽度、深度上限、分支因子 K、每层展开次数、UCT c（默认 1.4）、超时时间。
3. 价值函数。明确指出什么对节点进行评分。选项：单元测试通过率、到目标的数值距离、带格式的提示 LLM 分数（sure/likely/impossible 或 1..10 或投票）、或环境奖励。
4. 令牌预算估算。最坏情况令牌数 = 分支因子^深度 * 平均提示令牌数。显示具体数字。如果超出用户预算，建议更便宜的策略。
5. 失败模式。对于每个选择的策略，列出前两大失败模式及其缓解措施（例如 LATS + 噪声评估器 -> 添加基于工具的验证 per CRITIC，第 05 课）。

硬性拒绝：

- 当评估器不可靠时（仅自评估，无真实答案）推荐搜索。回退到 ReAct + CRITIC。
- 在没有充分理由的情况下将分支因子 K 设置高于 5。K=3-5 是论文默认值；K=10 会爆炸性增加成本。
- 将 LATS 应用于聊天式任务。搜索对没有可编程目标的对话式问答没有帮助。
- 没有机器可检查适应度的进化搜索。AlphaEvolve 仅在适应度是可编程的时候才有意义（运行测试、测量速度、验证定理）。

拒绝规则：

- 如果令牌预算 < 5 倍单条轨迹成本，拒绝搜索并建议 ReAct + Reflexion（第 03 课）。
- 如果挂钟延迟预算 < 10 秒，拒绝 LATS 并建议 ReAct。
- 如果任务是纯信息检索，拒绝搜索并建议 ReWOO（第 02 课）。

输出：一个建议区块（选择的策略、参数、价值函数、预算估算）加上一个"接下来阅读什么"的说明，指向第 05 课（CRITIC）用于评估器可靠性、第 11 课（AlphaEvolve）用于进化变体、或第 30 课（评估驱动开发）用于基准级验证。
