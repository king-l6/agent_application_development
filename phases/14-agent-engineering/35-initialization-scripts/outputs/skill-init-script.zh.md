给定一个仓库、Agent 产品及其依赖面，生成项目特定的初始化脚本和 CI 接入。

产出：

1. `tools/init_agent.py` 包含以下探测：运行时版本、列出的依赖、测试命令可解析性、必需的环境变量、状态文件新鲜度。
2. 在脚本旁边文档化的 `init_report.json` schema。每个探测返回 `(name, status: pass|warn|fail, detail)`。
3. `.github/workflows/agent-init.yml`（或等效文件），运行脚本并在任何失败严重级别的探测上阻止 Agent 作业。
4. Agent 运行时可以在每个会话开始前调用的 `pre-task` 钩子脚本。
5. `docs/init.md` 中的文档，列出每个探测、其严重级别以及如何修复失败。

硬性拒绝：

- 没有超时就调用网络的探测。初始化必须快速且离线安全。
- 需要 LLM 调用的探测。初始化是确定性的管道。
- 被包装器吞下的非零退出码。大声失败是整个要点。
- 没有幂等性就触及状态的探测。连续两次运行必须产生相同的报告，时间戳除外。

拒绝规则：

- 如果项目没有测试命令，拒绝交付脚本。将缺口添加到工作台审计中。
- 如果环境变量列表包含脚本会打印的密钥，拒绝并强制编辑。初始化报告绝不应携带密钥。
- 如果某个探测在干运行中耗时超过三秒，在交付前展示该发现。长探测将初始化变成形式主义。

输出结构：

```
<仓库>/
├── tools/
│   ├── init_agent.py
│   └── pre_task.sh
├── docs/
│   └── init.md
└── .github/
    └── workflows/
        └── agent-init.yml
```
