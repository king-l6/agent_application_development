给定一个问题类别和准确率目标，构建一个辩论协议。

产出：

1. 具有不同提示（以及理想情况下的不同模型）的 `Debater`，以避免同质化。
2. 轮次运行器：全连接网格、星型或环型拓扑。
3. 收敛规则：多数投票、按置信度加权或超级多数加回退。
4. 第 1 轮强制分歧：尽可能让每个辩论者返回不同的提案。
5. 成本核算：每问题总批评操作数 + 令牌成本。

硬性拒绝：

- 所有辩论者使用相同的提示 AND 相同的模型。必然导致群体思维。
- 未检查成本的全连接网格 N >= 6。辩论操作按 O(N*R) 增长。
- 没有收敛规则。返回辩论者 0 的第 R 轮答案不是收敛。

拒绝规则：

- 如果产品对延迟敏感（<1s 预算），拒绝辩论。改用自我精炼（第 05 课）或并行投票（第 12 课）。
- 如果问题类别是简单的事实查询（首都、日期、定义），拒绝辩论。查询 + CRITIC（第 05 课）更便宜。
- 如果评估集上的任何问题在首轮后辩论者之间没有分歧，拒绝该协议。您需要模型/提示的多样性。

输出：`debater.py`、`topology.py`、`convergence.py`、`runner.py`、`README.md`，解释 N/R 选择、拓扑原理以及在评估集上的成本与准确率测量。以"下一步阅读"结尾，如果任务更简单则指向第 12 课（工作流模式），如需将辩论嵌入更大系统则指向第 28 课（编排模式）。
