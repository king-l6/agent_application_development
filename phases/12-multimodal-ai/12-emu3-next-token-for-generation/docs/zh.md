# Emu3：图像和视频生成的下一令牌预测

> BAAI 的 Emu3（Wang 等人，2024 年 9 月）是 2024 年的一个成果，本应结束扩散与自回归的争论。一个单一的 Llama 风格仅解码器 Transformer，仅在下一令牌预测目标上训练，使用一个统一的词汇表（文本 + VQ 图像令牌 + 3D VQ 视频令牌），在图像生成上击败了 SDXL，在感知任务上击败了 LLaVA-1.6。不需要 CLIP 损失。不需要扩散调度。在推理时使用无分类器引导来提升质量，但核心训练目标是带有教师强制的下一令牌预测。发表在《自然》期刊上。本课程将研读 Emu3 的论文——为什么更好的分词器加上规模化就是你所需要的全部——并与扩散方法进行对比。

**类型：** 学习
**语言：** Python（标准库，3D 视频分词器数学 + 自回归采样器骨架）
**前置要求：** 阶段 12 · 11（Chameleon）
**时间：** ~120 分钟

## 学习目标

- 解释为什么 Emu3 的单损失下一令牌目标能够奏效，尽管长期以来一直认为图像质量需要扩散。
- 描述 3D 视频分词器：时空 VQ 码本是什么样子的，为什么图像块跨越时间维度。
- 比较 Emu3 与 Stable Diffusion XL 在（训练计算、推理成本、质量上限）方面的差异。
- 列举同一个 Emu3 模型扮演的三个角色：Emu3-Gen（图像生成）、Emu3-Chat（感知）、Emu3-Stage2（视频生成）。

## 问题

截至 2024 年的传统观点：图像生成需要扩散。理由是：离散图像令牌在重构细节时会丢失太多信息，而自回归采样会在数千个令牌中累积误差。Stable Diffusion、DALL-E 3、Imagen、Midjourney 都使用了某种形式的扩散。Chameleon（第 12.11 课）在小规模上部分反驳了这一点，但在质量上未能达到 SDXL 的水平。

Emu3 正面迎击了这一论点。其主张是：更好的视觉分词器 + 足够的规模 + 下一令牌损失 = 在同一个也能做感知的模型中实现超越扩散的图像生成。

这个赌注在发表时是有争议的。两年后，开源统一生成家族（Emu3、Show-o、Janus-Pro、Transfusion）已成为研究领域的默认路径；生产级前沿模型似乎也在使用某种变体。

## 概念

### Emu3 分词器

关键要素是视觉分词器。Emu3 训练了一个自定义的 IBQ 类分词器（逆瓶颈量化器，SBER-MoVQGAN 家族），每个令牌实现 8x8 的分辨率缩减。一张 512x512 的图像变成 64x64 = 4096 个令牌，码本大小为 32768。

这比 Chameleon 的每张 512x512 图像 1024 个令牌（K=8192）要大，但每个令牌的代价更低（更小的码本查找，更简单的编解码器）。关键指标：重构 PSNR 为 30.5 dB，与 Stable Diffusion 的连续潜空间（32 dB）相比具有竞争力。

对于视频：一个 3D VQ 分词器将一个时空块（4x4x4 像素）编码为一个整数。一个 4 秒片段在 8 FPS 下有 32 帧；在 256x256 分辨率下，空间缩减 4 倍、时间缩减 4 倍，令牌数量为 (256/4) * (256/4) * (32/4) = 64 * 64 * 8 = 32,768 个令牌。

分词器质量是上限。Emu3 的贡献部分在于"我们训练了一个非常好的分词器"。

### 单损失训练

Emu3 使用一个目标：在共享词汇表上的下一令牌预测，涵盖文本令牌、2D 图像令牌和 3D 视频令牌。训练时根据模态特定因子对权重进行乘法调整以平衡贡献，但损失函数是相同的。

在以下混合数据上训练：
- 图像生成：`<文本描述> <image> 图像令牌 </image>`
- 图像感知：`<image> 图像令牌 </image> <问题> 文本令牌`
- 视频生成：`<文本描述> <video> 视频令牌 </video>`
- 视频感知：类似。
- 纯文本：标准 NTP。

模型从数据分布中学习何时生成图像令牌与文本令牌。生成能力源自模型在 `<image>` 标签之后预测图像令牌。

### 无分类器引导和温度

自回归图像生成在推理时使用无分类器引导（CFG）会好得多。Emu3 使用了这一技术：生成两次，一次带有完整描述，一次带有空描述，用引导权重（典型值 3.0-7.0）混合 logits。这与扩散使用的 CFG 技巧相同，被借用到自回归设置中。

温度很重要：太高会产生伪影；太低会导致模式崩溃。Emu3 的推荐温度：感知任务为 1.0，图像生成为 0.8。

### 三个角色，一个模型

Emu3 以三个功能不同的 API 形式发布，但底层是同一套权重：

- **Emu3-Gen**：图像生成。输入文本，输出图像令牌。
- **Emu3-Chat**：VQA 和描述生成。输入图像（令牌），输出文本。
- **Emu3-Stage2**：视频生成和视频 VQA。输入文本或视频，输出文本或视频。

没有任务特定的头部。只是不同的提示模板。同一个检查点。

### 基准测试

来自 Emu3 论文（2024 年 9 月）：

- 图像生成：在 MJHQ-30K FID 上击败 SDXL（5.4 vs 5.6），GenEval 总体得分（0.54 vs 0.55——统计持平），Deep-Eval 综合得分相当。
- 图像感知：在 VQAv2 上击败 LLaVA-1.6（75.1 vs 72.4），在 MMMU 上大致持平。
- 视频生成：4 秒片段质量在 FVD 上与 Sora 时代的公开基准模型具有竞争力。

这些数字并不总是胜出——Emu3 在某项上有所取舍——但"下一令牌预测就是你所需要的"这一主张在多种模态上都是可辩护的。

### 计算成本

Emu3 使用了大约 3000 亿多模态令牌和 7B 参数模型进行训练。GPU 小时数大致相当于 Llama-2-7B 预训练（在 A100 级芯片上 2000-4000 GPU 年）。像 Stable Diffusion 3 这样的扩散模型在类似的预算下训练，但需要单独的文本编码器和更复杂的流水线。

在推理方面，Emu3 每张图像比 SDXL 慢：4096 个图像令牌以 30 令牌/秒的速度计算，每张 512x512 图像大约需要 2 分钟，而 SDXL 只需要 2-5 秒。推测性解码和 KV 缓存优化缩小了差距，但并未消除。自回归图像生成计算量很大；这是当前的权衡。

### 为什么它很重要

Emu3 的深层贡献是概念性的。如果下一令牌预测能够规模化并匹配扩散在图像生成上的表现，那么统一模型路径（一个损失，一个主干，任何模态）就是可行的。未来的模型不需要单独的文本编码器、单独的扩散调度器、单独的 VAE。一个 Transformer，每种模态一个分词器，规模化。

Show-o、Janus-Pro 和 InternVL-U 都建立在这一论点之上或对其提出挑战。中国实验室（BAAI、DeepSeek）在这一方向上的发表比美国实验室更加积极，这一趋势延续到 2025 年。

## 使用它

`code/main.py` 构建了两个玩具组件：

- 一个 2D 与 3D VQ 分词器计数计算器：给定（分辨率、块大小、片段长度、FPS），计算图像与视频的令牌数量。
- 一个带有无分类器引导和温度控制的自回归图像令牌采样器。

CFG 实现与 Emu3 的方案一致——用引导权重混合条件和无条件 logits。

## 交付它

本课程生成 `outputs/skill-token-gen-cost-analyzer.md`。给定一个生成产品规格（图像或视频、目标分辨率、质量等级、延迟预算），它计算令牌数量、推理成本，并在 Emu3 系列和扩散之间进行选择。

## 练习

1. Emu3 每张 512x512 图像产生 4096 个令牌，缩减率为 8x8。计算 1024x1024 和 2048x2048 的等效值。推理延迟会发生什么变化？

2. 阅读 Emu3 第 3.3 节关于视频分词器的内容。描述 3D VQ 块形状，为什么是 4x4x4 而不是 8x8x1？

3. 无分类器引导权重 5.0 与 3.0：视觉效果有什么不同？在 `code/main.py` 中追踪其数学原理。

4. 计算 Emu3-7B 在 300B 令牌上的训练 FLOPs，并与 Stable Diffusion 3 进行比较。哪个的训练成本更高？

5. Emu3 在 FID 上击败了 SDXL，但在 VQAv2 上未能战胜专门的 VLM。解释为什么统一损失方法在不同基准测试上表现出与专门模型不同的优势。

## 关键术语

| 术语 | 人们说的意思 | 实际含义 |
|------|-------------|---------|
| 下一令牌预测 | "NTP" | 标准自回归损失：给定 token[0..i] 预测 token[i+1]；对所有模态都适用，只要进行了分词 |
| IBQ 分词器 | "逆瓶颈量化器" | 一类 VQ-VAE，具有更大的码本（32768+）和比 Chameleon 更好的重构质量 |
| 3D VQ | "时空量化器" | 按（时间、行、列）索引的码本；一个令牌覆盖一个 4x4x4 像素立方体 |
| 无分类器引导 | "CFG" | 以权重 gamma 混合条件和无条件 logits；在推理时提升图像质量 |
| 统一词汇表 | "共享令牌" | 文本 + 图像 + 视频都来自同一个整数空间；模型预测接下来出现的任何模态 |
| MJHQ-30K | "图像生成基准" | Midjourney 质量基准，包含 3 万个提示；Emu3 在此报告 FID |
