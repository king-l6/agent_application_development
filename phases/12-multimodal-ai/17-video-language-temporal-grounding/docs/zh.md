# 视频-语言模型：时间 token 与时序定位

> 视频不是一堆照片的组合。一个 5 秒的片段具有因果顺序、动作动词和事件时序，这些都是图像模型无法表示的。Video-LLaMA（Zhang 等人，2023 年 6 月）发布了第一个具有音视频定位能力的开放视频-LLM。VideoChat 和 Video-LLaVA 规模化了这个模式。到 2025 年，Qwen2.5-VL 的 TMRoPE 缩小了与前沿专有模型的差距。每个系统都以不同的方式解决了时间 token 的问题——每个片段的 Q-former、每帧的拼接池化、每个 token 的 TMRoPE。本课程解读这些模式，构建均匀 vs 动态帧采样器，并在时序定位任务上进行评估。

**类型：** 构建
**语言：** Python（标准库，帧采样器 + 时序定位评估器）
**前置知识：** 阶段 12 · 08（LLaVA-OneVision）
**时间：** ~180 分钟

## 学习目标

- 解释为什么时间位置编码独立于视觉编码器影响视频 VLM 的性能。
- 在每秒 token 数与定位准确率方面，比较均匀采样、动态 FPS 采样和事件驱动帧采样。
- 描述每个片段的 Q-former（Video-LLaMA）vs 每帧池化（Video-LLaVA）vs 每 token 的 M-RoPE（Qwen2.5-VL）设计。
- 说出四个视频基准测试：VideoMME、TempCompass、EgoSchema、Video-MMMU。

## 问题

一个 1 分钟、30 FPS 的视频有 1800 帧。以每帧 196 个视觉 token（ViT-B 在 224 分辨率下）计算，就是 35.2 万个 token——比任何 2024 年代的 LLM 上下文都大。

存在三种缩减策略：

1. 子采样帧（根据内容，1-8 FPS）。
2. 激进地池化每帧的块 token（3x3 或 4x4 双线性池化）。
3. 通过 Q-former 压缩，它接受一个 16 帧的片段并输出 64 个 token。

每种权衡都不同。子采样丢失时间细节。池化丢失空间细节。Q-former 两者都丢失一些，但节省了 token。

时间位置编码是另一个维度：模型如何知道第 5 帧在第 6 帧之前？选项包括简单的一维时间 RoPE（Video-LLaMA）、学习得到的时间嵌入（Video-LLaVA）和 TMRoPE（Qwen2.5-VL，全 3D）。

## 概念

### Video-LLaMA：每个片段的 Q-former + 音频分支

Video-LLaMA（2023）是第一个开放的视频-LLM。架构：

- 以 2 FPS 采样的 16 帧片段（即 8 秒）。
- 逐帧 ViT 特征 -> 视频 Q-former，它对所有 16 帧进行交叉注意力 -> 32 个学习到的查询 -> LLM。
- 并行音频分支：波形 -> ImageBind 音频编码器 -> 音频 Q-former -> 32 个查询 -> LLM。

优势：音视频联合推理。弱点：固定片段长度，无任意时间定位。

### VideoChat 和 Video-LLaVA

VideoChat 保留了 Video-LLaMA 的想法，但去掉了音频并进行了简化。Video-LLaVA（Lin 等人，2023）训练了一个单一的视觉编码器，同时处理图像和视频帧（"投影前的对齐"），给出了一个统一的表示。两者都是冻结 CLIP 编码器 + MLP + LLM。

两者都不能处理长视频。两者都是 8-16 帧的系统。

### Qwen2.5-VL 和 TMRoPE

Qwen2.5-VL 引入了 TMRoPE——时间-模态旋转位置嵌入。每个块 token 携带一个 (t, h, w) 位置，其中 t 是实际时间戳（而非帧索引）。

与简单时间嵌入的关键区别：

- 绝对时间，而非索引。模型看到的是"在 4.2 秒处"而非"在第 15 帧处"。
- 每个 token 的旋转，而非每个片段。每个视觉 token 根据其时间戳独立旋转。
- 与动态 FPS 兼容。如果你在这里以 2 FPS 采样，在那里以 4 FPS 采样，TMRoPE 自然处理不均匀的间隔。

TMRoPE 使"猫在第几秒跳起来的？"这样的查询成为可能。模型可以输出"在 4.2 秒处"。Video-LLaMA 只能说出"片段早期"。

### 帧采样策略

均匀采样：在持续时间内均匀采样 N 帧。简单，但错过运动峰值。

动态 FPS：基于运动强度自适应采样。光流或帧差分选择高运动段进行更密集的采样。Qwen2.5-VL 在此策略上训练。

事件驱动：运行一个轻量级检测器，在动作发生处采样更多。由 VideoAgent 使用。

关键帧 + 上下文：在镜头边界处采样，加上几个相邻帧。用于电影内容。

### 每帧池化

在 1 FPS 和每帧 576 个 token 的情况下，一个 5 分钟的片段是 172,800 个 token。在 Qwen2.5-VL-72B 的 128k 上下文中可行，但成本高。

3x3 双线性池化减少到每帧 64 个 token -> 5 分钟为 19,200 个 token。对大多数任务来说是最佳选择。

对于空间细节不太重要的智能体工作流，可以更激进地池化（6x6 -> 每帧 16 个 token）。

### 四个视频基准测试

- VideoMME：全面的视频理解，短 + 中 + 长。
- TempCompass：细粒度时序推理，"之前"/"之后"问题。
- EgoSchema：长时间跨度的第一人称视频。
- Video-MMMU：多模态多学科视频问题。

一个完整的视频 VLM 评估应该覆盖全部四个。它们强调不同的维度——TempCompass 全是关于顺序，EgoSchema 是关于 3 分钟以上的推理，VideoMME 跨越不同时长。

### 定位输出格式

时序定位的输出格式：

- 自由文本："猫在大约 4 秒标记处跳起。"易于解析但不精确。
- 结构化 JSON：`{"event": "jump", "start": 4.1, "end": 4.3}`。Qwen2.5-VL 训练这种格式。
- 基于 token：特殊的 `<time>4.1</time>` token 与答案交错。Qwen2.5-VL 的内部格式。

对于下游使用，基于 token 是最准确的。Qwen2.5-VL 的 JSON 输出格式可直接解析。

### 2026 年最佳实践

对于 2026 年的视频 VLM：

- 编码器：支持 M-RoPE 或 TMRoPE 的 SigLIP 2（Qwen2.5-VL）。
- 帧采样：动态 FPS（1-4，取决于运动），带最大帧数限制。
- 每帧池化：3x3 双线性。
- 输出：带时间和事件字段的结构化 JSON。
- 基准测试：通用使用 VideoMME + TempCompass；长时间跨度使用 EgoSchema。

## 使用它

`code/main.py` 包含：

- 均匀和动态 FPS 帧采样器。
- 一个玩具级别的时序定位评估器：给定时间 T 处的"真实"事件和模型输出，带容差地评分准确率。
- 跨 Video-LLaMA（16 帧，Q-former）、Video-LLaVA（8 帧，MLP）、Qwen2.5-VL（动态 FPS + TMRoPE）的比较。

## 交付它

本课程产出 `outputs/skill-video-vlm-frame-planner.md`。给定一个视频任务（监控、动作识别、时序定位、摘要），它选择帧采样器、池化因子、输出格式和预期的准确率等级。

## 练习

1. 对于一个 3 分钟的烹饪演示，选择均匀 vs 动态 FPS。用 token 计数证明你的选择。

2. TMRoPE 具体增加了什么简单时间嵌入表无法做到的功能？

3. 为 VLM 可以学习发出的时序定位写一个 JSON 模式。包括错误情况。

4. 阅读 Video-LLaVA 第 3 节"投影前的对齐"。为什么这比训练分开的图像和视频编码器更好？

5. 查看 VideoMME 排行榜，截至 2026 年，顶级开放模型和顶级专有模型之间的差距是多少？其中多少差距可归因于时间编码 vs 基座 LLM 的规模？

## 关键术语

| 术语 | 人们说的 | 实际含义 |
|------|---------|---------|
| 时序定位 | "时间定位的答案" | VLM 输出事件发生时的特定时间戳范围 |
| TMRoPE | "时间-多模态 RoPE" | 带绝对时间戳的 3D 旋转位置编码，由 Qwen2.5-VL 使用 |
| 动态 FPS | "运动感知采样" | 在高运动段采样更多帧，在静态段采样更少 |
| 帧池化 | "每帧空间压缩" | 在进入 LLM 之前，用双线性插值减少每帧的块数 |
| 视频 Q-former | "片段压缩器" | 将 N 帧映射到 K 个学习查询的交叉注意力瓶颈 |
| VideoMME | "视频基准" | 全面的短/中/长视频基准测试，2500+ 样本 |
