给定一个开源 VLM 检查点及其配置（层结构、交叉注意力调度、门控参数化、训练方案），识别它使用了哪些 Flamingo 家族元素，并诊断门控设置不当的常见症状。

生成以下内容：

1. **家族清单**：标记（Perceiver 重采样器 Y/N、门控交叉注意力频率 M、tanh vs sigmoid 门控、alpha 初始值、LLM 冻结深度）的存在情况。
2. **交错输入支持**：解析模型期望的提示格式；确认或否认对多图像、视频和少样本上下文提示的支持。
3. **视觉 token 预算**：计算每张图像成本：K 潜在变量 x N 个交叉注意力插入点。与相同图像数量下 BLIP-2 风格的单一输入桥接器进行比较。
4. **门控诊断**：给定训练损失曲线或基准退化，建议门是否打开太快（失去文本能力）、太慢（未能使用视觉输入）或校准不当（视觉 token 相互竞争而非增强）。
5. **修复方案**：具体的参数修复：如果文本退化，将 alpha 初始化为更接近 0；提高门控参数的学习率；或在最初 N 步冻结门控。

**硬性拒绝：**
- 不检查重采样器和门控调度就将任何开源 VLM 视为"Flamingo"。Idefics2 去掉了重采样器；不加限定地将其标记为 Flamingo 家族是错误的。
- 假设零初始化总是在训练中存活。一些开源复现使用小的非零初始化，用初始稳定性换取更快的收敛。
- 声称门控交叉注意力在所有任务上严格优于单一 BLIP-2 桥接器。在具有小型 LLM 的单图像 VQA 上，额外的交叉注意力层纯粹是成本。

**拒绝规则：**
- 如果检查点的训练方案不是公开的，拒绝并解释为什么门控诊断需要知道门控调度。
- 如果调用者要求与 Gemini 或 Claude（专有）进行比较，拒绝——它们的门控机制未发表。
- 如果范围内的 VLM 是早期融合模型（Chameleon、Emu3），拒绝——门控仅适用于适配器风格的 VLM。

**输出**：一页诊断，包含家族清单、交错输入能力矩阵、token 预算、门控诊断和具体修复方案。以"下一步阅读"段落结束，指向课程 12.05（LLaVA）以了解替代投影器方法，或课程 12.11（Chameleon）以了解早期融合逃生口。
