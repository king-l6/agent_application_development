# LLaVA-OneVision：单图像、多图像、视频合一

> 在 LLaVA-OneVision（Li 等人，2024 年 8 月）之前，开放 VLM 世界有各自独立的谱系：用于单图像的 LLaVA-1.5、用于多图像的 Mantis 和 VILA、用于视频的 Video-LLaVA 和 Video-LLaMA。每个模型在自己的基准上表现出色，但在其他基准上失败。LLaVA-OneVision 论证了一个单一的课程学习可以训练一个模型在所有三种场景上占据主导地位，并且涌现的任务迁移效应（单图像技能迁移到视频，多图像推理迁移到单图像）超过了专家的总和。其配方出奇地简单：一个在不同场景间保持恒定的视觉 token 预算，加上一个从单图像到 OneVision（多图像）再到视频的显式课程。本节课解读这个预算、课程和涌现行为。

**类型：** 构建
**语言：** Python（标准库，token 预算求解器 + 课程规划器）
**前置知识：** 第 12 阶段 · 05（LLaVA），第 12 阶段 · 06（任意分辨率）
**时间：** ~180 分钟

## 学习目标

- 设计一个在单图像、多图像和视频输入间保持恒定的视觉 token 预算。
- 安排一个训练课程，将技能从单图像迁移到视频而不发生灾难性遗忘。
- 解释为什么在课程学习做得好的情况下，单个模型在相同参数量上优于专家模型。
- 说出 LLaVA-OneVision 报告的三种涌现能力：多摄像头推理、标记集提示、iPhone 截图智能体。

## 问题

图像、多图像和视频对模型的压力各不相同。

单图像需要高分辨率 token（AnyRes，约 2880 视觉 token）来捕捉 OCR 和精细细节。每样本预算：一张图像，2880 token。

多图像需要几张中等分辨率（每张约 576 token）的图像，以便跨图像的推理能放入上下文。每样本预算：4-8 张图像，每张 576，共 2300-4600 token。

视频需要许多低分辨率帧（池化后每帧约 196 token）来捕捉时间动态。每样本预算：8-32 帧，每帧 196，共 1600-6200 token。

如果你训练单独的模型，你选择一个预算。如果你训练一个模型，你需要预算在不同场景间合理缩放而不撑爆上下文。

在 OneVision 之前，默认答案是"训练一个场景，忽略其他场景。"Video-LLaVA 通过额外的训练阶段将视频改造到图像模型上。LLaVA-NeXT 通过平铺增加了多图像支持。没有一个能干净地处理所有三种场景。

## 概念

### OneVision token 预算

LLaVA-OneVision 为每样本选择约 3000-4000 token 的统一视觉 token 预算，按场景进行不同分配：

- **单图像：** AnyRes-9（3x3 图块 + 缩略图），每个图块在 384 分辨率下有 729 个 patch，激进的 2x2 双线性池化后每图块 182 个。总计：9 * 182 + 182 = 1820 token。或者 AnyRes-4 每图块 729 = 2916 + 729。
- **多图像：** 每张图像中等分辨率（384，不平铺），729 token，无池化。预算 6 张图像 → 4374 token。
- **视频：** 32 帧，384 分辨率，激进 3x3 双线性池化 → 每帧 81 token。总计：32 * 81 = 2592 token。

这种分配使总 token 数大致保持恒定。LLM 永远不会看到一个撑爆其上下文的批次。编码器按场景产生不同的几何形状，但 LLM 消耗相同的预算。

### 三阶段课程

LLaVA-OneVision 分三个阶段训练：

1. **单图像 SFT（SI 阶段）。** 所有数据都是单图像加文本。在高分辨率 AnyRes 输入上训练。这教会感知、OCR 和细粒度理解。使用 LLaVA-NeXT 数据加上 OneVision 特定的单图像数据。
2. **OneVision SFT（OV 阶段）。** 混合单图像 + 多图像 + 视频（均匀采样帧）。在统一 token 预算上训练。这教会模型处理异构批次形状。没有权重重置——从 SI 阶段继续。
3. **任务迁移（TT 阶段）。** 继续使用目标任务组合训练，通常根据产品需求增加多图像或视频的比例。用于部署的可选微调。

关键点：课程顺序至关重要。先训练视频或多图像产生的图像性能比先训练单图像更差，即使使用相同的数据。论文明确地消融了这一点。

### 课程为何有效

单图像训练建立了感知基础。Patch token 携带细粒度视觉特征；LLM 学习将它们与文本集成。多图像和视频引入了结构挑战（哪张图像是哪张，什么先发生），如果没有强大的感知基础就很难学习。

如果从一开始就同时训练所有场景，模型会欠拟合感知（每批次单图像数据有限）并过拟合结构（大量多图像/视频数据）。结果是：一个能够进行跨图像推理但视觉上浅薄的模型。

课程排序让你从 SI 阶段获得感知强度，从 OV 阶段获得组合/时间推理能力，而不会失去任何一方。

### 涌现的跨场景技能

LLaVA-OneVision 论文报告了三种涌现能力：

1. **多摄像头推理。** 在多图像和视频上分别训练；推理时，要求对多摄像头驾驶场景进行推理。尽管从未在训练中见过这种确切格式，模型仍能正确整合多视角信息。
2. **标记集提示。** 用户用编号标记标注图像中的物体；模型推理"标记 3 相对于标记 7 在做什么"。从未在标记或标注上进行训练；从空间定位加多图像引用的组合中学习而来。
3. **iPhone 截图智能体。** 用户提供 iPhone 屏幕截图并请求规划下一次点击。在 UI 截图、用户工作流程视频和多图像前后对上进行训练。泛化到智能体用例。

这些不是训练出来的任务；它们是从课程的组合结构中涌现出来的。

### 视觉 token 池化

token 预算需要池化。OneVision 在 2D patch 网格上使用双线性插值：24x24 = 576 个 patch 变成 12x12 = 144（2 倍因子）或 8x8 = 64（3 倍因子）。池化在 patch 网格空间中进行，而不是 token 空间，以保持局部性。

每种场景的池化因子选择本身就是一个超参数。较少池化 = 更多 token = 更丰富的表示。更多池化 = 更少 token = 可以容纳更多帧/图像。

### LLaVA-OneVision-1.5

2025 年的后续工作（LLaVA-OneVision-1.5，arXiv 2509.23661）在训练数据、模型权重和代码上"完全开放"。在某些基准上缩小了与专有模型的差距，并使配方大众化。相同的课程、更多的数据、更好的基础 LLM。架构没有变化。

### 与 Qwen2.5-VL 对比

Qwen2.5-VL（第 12.09 课）做出了不同的选择。它使用 M-RoPE 和动态 FPS 而不是固定池化。其预算随输入缩放——1 分钟的视频比 5 秒的视频使用更多 token。LLaVA-OneVision 固定预算并缩放池化。两者都有效；它们在可配置性和可预测性之间进行权衡。

## 使用

`code/main.py` 是一个用于 OneVision 风格 VLM 的课程和预算规划器。给定每样本的 token 预算和目标场景组合（例如 40% 单图像、30% 多图像、30% 视频），它：

- 为每个场景分配分辨率、池化因子和帧数。
- 检查每个场景是否都适合共享预算。
- 报告预期 token 数量、LLM FLOPs 以及哪些场景 token 不足。
- 打印分阶段训练计划。

用它来规划 OneVision 微调，或作为 VLM 部署中每请求成本的合理性检查。

## 练习

1. 你的产品支持 80% 单图像、10% 多图像（2-4 张图像）、10% 视频（8-16 帧）。设计 token 预算。你会把不做大量多图像而节省的额外预算用在哪里？

2. 阅读 LLaVA-OneVision 第 4.3 节（涌现能力）。提出课程很可能解锁但论文未报告的第四种涌现技能。

3. 交换课程顺序——先训练多图像，然后单图像，最后视频。预测哪些基准会下降以及为什么。

4. 论文报告的视频基准仅使用每样本 8 帧进行训练。这能泛化到推理时的 30 秒视频吗？什么先被突破——token 预算还是时间推理？

5. 将 24x24 patch 双线性池化为 12x12 是每维 4 倍缩减。用标准库 Python 实现池化，并验证每个 2x2 块上的均值与双线性输出一致。

## 关键术语

| 术语 | 人们常说的 | 实际含义 |
|------|-----------|---------|
| OneVision 场景 | "单图像、多图像或视频" | 统一 VLM 处理的三种输入形态之一；预算在其间保持恒定 |
| Token 预算 | "每样本多少 token" | LLM 在每个训练/推理样本中看到的视觉 token 总数，通常为 3000-4000 |
| 课程 | "训练顺序" | 为涌现迁移而选择的阶段排序（单图像 → 多图像 → 视频） |
| 双线性池化 | "token 压缩" | 对 patch 网格（2D）应用双线性插值以减少 token 数量，同时保持局部性 |
| 涌现技能 | "未训练但仍然有效" | 在推理时出现的、没有匹配训练数据的能力，源于课程组合 |
| AnyRes-k | "k 图块设置" | k 个固定分辨率的子图块加一个缩略图，典型 k ∈ {4, 9} |
| 任务迁移 | "跨场景泛化" | 在单图像上学习到的技能通过共享骨干网络应用于视频（反之亦然） |
