给定一个多模态产品查询流程（查询中涉及哪些模态，语料库中涉及哪些模态），设计检索器、融合和生成。

产出：

1. 每模态检索器。CLIP / SigLIP 2 用于文本+图像，CLAP 用于文本+音频，VLM 隐藏状态用于其他任何模态。
2. 融合选择。默认分数融合；如果需要进行每查询路由则选择 MoE 融合；大规模场景下选择注意力融合。
3. 接地生成器。Qwen2.5-VL 或 Claude 4.7，使用来源标记的输出进行训练。
4. 评估。每个模态的 Recall@k + 融合 top-k 准确率 + 人工评判的端到端质量。
5. 智能体多跳。何时重新查询；触发重新查询的置信度阈值。
6. 存储估算。每模态向量数量和压缩方案。

硬性拒绝：
- 在没有共享空间（CLIP / CLAP）的情况下，跨模态使用双编码器检索。分数没有意义。
- 在没有训练数据的情况下提议 MoE 融合。MoE 需要监督才能正确路由。
- 声称分数融合权重可跨域迁移。事实并非如此。

拒绝规则：
- 如果语料库没有用于训练检索器的图像-标题配对数据，拒绝自定义微调，推荐现成的 CLIP / SigLIP 2。
- 如果查询延迟预算 < 200ms 且需要多跳，拒绝；推荐使用更好的检索器进行一次性检索。
- 如果接地引用是监管要求且没有生成器支持它们，拒绝并提议使用 Anthropic / OpenAI 引用 API 或显式的后处理引用层。

输出：一页 RAG 设计，包含检索器、融合、生成器、评估、智能体策略、存储方案。结尾引用 arXiv 2502.08826、2504.08748、2503.18016。
