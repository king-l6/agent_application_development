# Qwen-VL 家族与动态 FPS 视频

> Qwen-VL 家族——Qwen-VL（2023）、Qwen2-VL（2024）、Qwen2.5-VL（2025）、Qwen3-VL（2025）——是 2026 年最具影响力的开放视觉语言模型谱系。每一代都做出了一项决定性的架构赌注，并在 12 个月内被其他开放生态系统复制：通过 M-RoPE 实现原生动态分辨率、带绝对时间对齐的动态 FPS 采样、ViT 中的窗口注意力和结构化智能体输出格式。到 Qwen3-VL 时，配方已经稳定下来：一个具有原生宽高比输入的 2D-RoPE-ViT 编码器、一个连接到大型 Qwen3 语言基础的 MLP 投影器，以及以 OCR、定位和智能体行为作为一等目标的训练阶段。本节课按时间顺序阅读这个家族，让你理解为什么每个旋钮都在它现在的位置上。

**类型：** 学习
**语言：** Python（标准库，M-RoPE 编码器 + 动态 FPS 采样器）
**前置知识：** 第 12 阶段 · 06（patch-n'-pack）
**时间：** ~120 分钟

## 学习目标

- 计算 M-RoPE 的三轴旋转（时间、高度、宽度），并解释为什么三者都需要。
- 为视频选择动态 FPS 采样策略，并分析 token 每秒与事件检测精度之间的权衡。
- 按顺序说出 Qwen-VL 的四代升级及其各自带来的能力。
- 编写 Qwen2.5-VL 风格的 JSON 智能体输出格式，并从 VLM 响应中解析结构化工具调用。

## 问题

Qwen-VL 于 2023 年 8 月发布，直接对标 LLaVA-1.5 和 BLIP-2。Qwen 团队瞄准的差距有三个：分辨率、视频和结构化输出。

**分辨率：** LLaVA-1.5 运行在 336x336。对照片来说很好，但对中文发票或密集电子表格截图来说毫无用处。Qwen-VL 的第一个创新是 448x448 和有界框输出，让模型能够指向物体。

**视频：** Video-LLaMA 堆叠了每帧编码器并将其馈送到 LLM。它对短视频片段有效，但对时间轴本身就是信号的多分钟视频无效。Qwen 团队想要一个理解时间的单一编码器。

**结构化输出：** LLaVA 输出自由文本。智能体需要 JSON。Qwen-VL 在显式 JSON 输出格式上进行了训练，包括作为文本的边界框坐标。

每一代 Qwen-VL 都扩展了这三个轴之一。

## 概念

### Qwen-VL（2023 年 8 月）

第一代：OpenCLIP ViT-bigG/14 编码器（25 亿参数）、兼容 Llama 的 Q-Former（1 步，256 查询）、Qwen-7B 基础。贡献：

- 448x448 分辨率（当时开放 VLM 的最高水平）。
- 定位：在带有显式坐标 token 输出的图文对上训练。"猫在 <box>(112, 204), (280, 344)</box>"。
- 从一开始就支持中文和英文多语言训练。

当时的基准：在英文上与 GPT-4V 竞争，在中文上占据主导。定位监督才是真正的头条新闻。

### Qwen2-VL（2024 年 9 月）——M-RoPE 和原生分辨率

Qwen2-VL 用原生的动态分辨率 ViT 编码器取代了固定分辨率加 Q-Former 的堆栈。关键变化：

- **原生动态分辨率。** ViT 接受任何能被 28 整除的 HxW（patch 14，2x 空间合并）。一张 1120x672 的图像（40x24 合并 patch）产生 960 个视觉 token。无需缩放、无需平铺、无需缩略图。
- **M-RoPE（多模态 RoPE）。** 每个 token 携带 3D 位置（t, h, w）而不是 1D。对于图像 t=0，对于视频 t = 帧索引。RoPE 按每轴的频率旋转 query/key 向量。没有位置嵌入表。
- **MLP 投影器。** 丢弃 Q-Former；在合并的 patch token 上使用 2 层 MLP。
- **带动态 FPS 的视频。** 默认以 1-2 FPS 采样视频，但模型接受任意帧数。

结果：Qwen2-VL-7B 在多个多模态基准上匹配 GPT-4o，并在 DocVQA 上击败了它（94.5 对 88.4）。架构变革是关键一步。

### Qwen2.5-VL（2025 年 2 月）——动态 FPS 和绝对时间

Qwen2.5-VL 的重大转变是视频。动态 FPS 不仅仅是"需要时采样更多帧"。论文正式定义了：

- **绝对时间 token。** 使用实际时间戳而不是位置索引（第 0、1、2 帧）。"在 0:04，猫跳起来了。"模型会看到 `<time>0.04</time>` token 与帧 token 交错出现。
- **动态 FPS。** 慢速镜头以 1 FPS 采样，动作场景以 4+ FPS 采样。由用户或训练器选择；M-RoPE 自适应。
- **ViT 中的窗口注意力。** 空间注意力是窗口化的（在块内局部进行）以提高吞吐量；每几层进行一次全局注意力。
- **显式 JSON 输出格式。** 在工具调用数据上训练："{\"tool\": \"click\", \"coords\": [380, 220]}"。开箱即用，智能体就绪。
- **MRoPE-v2 缩放。** 位置随最大输入大小缩放，使得 10 分钟的视频不会超出频率范围。

基准：Qwen2.5-VL-72B 在大多数视频基准上击败 GPT-4o，在文档方面匹配 Gemini 2.0，并创下 GUI 定位的开放模型 SOTA（ScreenSpot：84% 准确率对比 GPT-4o 的 38%）。

### Qwen3-VL（2025 年 11 月）

Qwen3-VL 是一个增量升级，巩固而非革新：更大的 LLM 骨干（Qwen3-72B）、扩展的训练数据、改进的 OCR、通过 Qwen3 的"思考模式"实现更强的推理。ViT 和 M-RoPE 保持不变。论文侧重于数据和训练改进，而非架构。

谱系的启示：到 2025 年，Qwen-VL 架构已经稳定。后续几代扩展的是计算和数据，而非基本构件。

### M-RoPE 的数学原理

经典 RoPE 使用配对坐标通过位置 `m` 旋转维度 `d` 的查询 `q`：

```
q_rot[2i]   = q[2i]   * cos(m * theta_i) - q[2i+1] * sin(m * theta_i)
q_rot[2i+1] = q[2i]   * sin(m * theta_i) + q[2i+1] * cos(m * theta_i)
theta_i     = 10000^(-2i/d)
```

M-RoPE 将隐藏维度分成三个频带。假设 `d = 96`。分配 32 维给时间、32 维给高度、32 维给宽度。每个频带由自己的轴位置旋转。位于 (t=5, h=10, w=20) 的 patch 会对其三个频带分别应用旋转 `R_t(5)`、`R_h(10)`、`R_w(20)`。

文本 token 使用 `t = text_index, h = 0, w = 0`（或归一化的选择），保持兼容性。视频帧使用 `t = frame_time, h = row, w = col`。单张图像使用 `t = 0`。

好处：一个位置编码就能处理文本、图像和视频，无需分支代码或不同的位置表。

### 动态 FPS 采样逻辑

给定时长 `T` 秒的视频和目标 token 预算 `B`：

1. 计算你能负担的最大 FPS：`fps_max = B / (T * tokens_per_frame)`。
2. 从 `{1, 2, 4, 8}` 中选择满足 `fps <= fps_max` 的目标 FPS。
3. 如果运动量高（光流启发式或用户显式要求），选择较高 FPS。如果运动量低，选择较低。
4. 以所选 FPS 均匀采样；在帧之间插入 `<time>t</time>` token。

Qwen2.5-VL 隐式地训练这个逻辑；推理时用户通过 `fps` 参数控制。一个 60 秒的动作序列以 4 FPS、每帧 81 token 计算 = 19440 token，在 32k 上下文中是可管理的。

### 结构化智能体输出

Qwen2.5-VL 的智能体训练专门针对结构化工具调用：

```
{
  "tool": "mouse_click",
  "coords": [1024, 512],
  "button": "left",
  "modifier": null
}
```

解析是确定性的：对模型输出执行 JSON.parse。相比之下，自由格式的"click at (1024, 512)"需要正则表达式和歧义处理。这一转变解释了为什么 Qwen2.5-VL 的 ScreenSpot 分数从 Qwen2-VL 的 55% 跃升至 84%。

## 关键术语

| 术语 | 人们常说的 | 实际含义 |
|------|-----------|---------|
| M-RoPE | "多模态 RoPE" | 3D 旋转位置嵌入，在隐藏维度中具有时间、高度和宽度频带 |
| 动态 FPS | "智能采样" | 根据运动量、时长和 token 预算为每个视频选择的帧采样率 |
| 绝对时间 token | "时间戳 token" | 在序列中交错插入 `<time>t</time>`，让模型看到实际秒数而非帧索引 |
| 窗口注意力 | "局部注意力" | 空间自注意力限制在小窗口内以提高速度；周期性添加全局注意力 |
| 结构化智能体输出 | "JSON 模式" | 训练数据监督教会 VLM 输出带有坐标和工具名称的可解析 JSON |
| min_pixels / max_pixels | "分辨率边界" | 每请求的 Qwen2.5-VL 控制参数，限制总像素数从而限制 token 数 |
| 定位 | "指向它" | 以文本 token 形式输出边界框坐标；自 Qwen-VL v1 起使用 |
