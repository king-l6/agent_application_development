# LLaVA 与视觉指令微调

> LLaVA（2023年4月）是地球上被复制最多的多模态架构。它用两层 MLP 替换了 BLIP-2 的 Q-Former，用朴素的 token 拼接替换了 Flamingo 的门控交叉注意力，并在由 GPT-4 从纯文本标题生成的 158k 个视觉指令轮次上进行了训练。任何在 2023 年至 2026 年间构建 VLM 的从业者都构建了 LLaVA 的某种变体。LLaVA-1.5 增加了 AnyRes。LLaVA-NeXT 提升了分辨率。LLaVA-OneVision 将单图像、多图像和视频统一到一个方案中。本课程解读该方案，实现投影器，并解释为什么"更简单的方案胜出"。

**类型：** 构建
**语言：** Python（标准库，投影器 + 指令模板构建器）
**前置知识：** 阶段 12 · 02（CLIP），阶段 11（LLM 工程 — 指令微调）
**时间：** ~180 分钟

## 学习目标

- 构建一个两层 MLP 投影器，将 ViT 块嵌入（维度 1024）映射到 LLM 的嵌入维度（维度 4096）。
- 走通 LLaVA 两阶段方案：(1) 在 558k 个标题对上对齐投影器，(2) 在 158k 个 GPT-4 生成的轮次上进行视觉指令微调。
- 构建 LLaVA 格式的提示词，包含图像 token 占位符、系统提示词和用户/助手轮次。
- 解释为什么社区从 Q-Former 转向 MLP，尽管 Q-Former 在 token 预算上占优。

## 问题

BLIP-2 的 Q-Former（课程 12.03）将图像压缩为 32 个 token。简洁、高效、基准测试表现好。但它有两个问题。

首先，Q-Former 是可训练的，但其损失并非最终任务。阶段 1 训练 ITC+ITM+ITG。阶段 2 训练 LM 损失。查询学习到一些中间表示，LLM 随后需要解码这些表示。信息在瓶颈中丢失了。

其次，Q-Former 有 1.88 亿参数，在 LLaVA 2023 年的规模下，你必须与目标 LLM 协同设计它。换 LLM，重新训练 Q-Former。换视觉编码器，重新训练。每种组合都是一个独立的研发项目。

LLaVA 的答案简单得令人难堪：取 ViT 的 576 个块 token，每个通过一个两层 MLP（`1024 → 4096 → 4096`），并将全部 576 个 token 倾倒入 LLM 的输入序列。没有瓶颈。没有基于奇怪目标的阶段 1 预训练。只需在直接的 LM 损失上训练 MLP。

数据从哪里来？LLaVA 的第二个洞见：使用 GPT-4（纯文本）生成指令数据。将 COCO 标题和边界框数据喂给 GPT-4，让其生成对话、描述和复杂推理问题。免费获得 158k 个指令-回复对。无需人工标注。

结果：一个在 8 张 A100 上运行一天的 VLM，在 MMMU 上击败了 Flamingo，并发布了一个社区可以扩展的开放检查点。到 2023 年底，它已经产生了 50 多个分支。

## 概念

### 架构

LLaVA-1.5 在 13B 规模：
- 视觉编码器：CLIP ViT-L/14 @ 336（阶段 1 冻结，阶段 2 可选解冻）。
- 投影器：两层 MLP，GELU 激活，`1024 → 4096 → 4096`。
- LLM：Vicuna-13B（后来改为 Llama-3.1-8B）。

图像 + 文本提示词的前向传播：

```
img -> ViT -> 576 个维度为 1024 的块
patches -> MLP -> 576 个维度为 4096 的 token
prompt: system + "<image>" 占位符 + user 问题
将 <image> token 替换为 576 个投影后的 token
将完整序列输入 LLM
解码回复
```

图像在 LLM 上下文中占用 576 个 token。在 2048 上下文中，这为文本留下 1472 个 token。在 32k 上下文中，这只是一个舍入误差。

### 阶段 1：投影器对齐

冻结 ViT。冻结 LLM。仅训练两层 MLP。数据集：558k 个图像-标题对（LAION-CC-SBU）。损失：在标题上的语言建模，以投影后的图像 token 为条件。

在单轮 epoch 中，batch size 为 128，这可以在几小时内完成。投影器学会将 ViT 空间映射到 LLM 空间。无需特定任务的监督。

### 阶段 2：视觉指令微调

解冻投影器（仍可训练）。解冻 LLM（通常完全解冻，有时用 LoRA）。在 158k 个视觉指令轮次上训练。

指令数据是关键。Liu 等人通过以下方式生成：
1. 取一张 COCO 图像。
2. 提取文本描述（5 个人工标题 + 边界框列表）。
3. 发送给 GPT-4，使用三种提示模板：
   - 对话："生成关于这张图像的用户和助手之间的来回对话。"
   - 详细描述："给出一个丰富、详细的图像描述。"
   - 复杂推理："提出一个需要对图像进行推理的问题，然后回答它。"
4. 解析 GPT-4 的输出为（指令，回复）对。

这些都不直接接触图像——只接触文本描述。GPT-4 会幻觉出看似合理的图像内容。有些噪声，但它成功了：15.8 万个轮次足以解锁对话能力。

### 为什么社区复制了它

- 无需调整阶段 1 特有的损失。全程使用 LM 损失。
- 投影器训练只需数小时，而非数天。
- LLM 可以更换（LLaVA-Llama2、LLaVA-Mistral、LLaVA-Llama3），只需重新训练投影器即可。
- 视觉指令数据流水线使用 GPT-4，为新领域重新生成成本低廉。

### LLaVA-1.5 和 LLaVA-NeXT

LLaVA-1.5（2023年10月）增加了：
- 学术任务数据（VQA、OKVQA、RefCOCO）混入指令微调。
- 更好的系统提示词。
- 将 2048 上下文扩展到 32k。

LLaVA-NeXT（2024年1月）增加了：
- AnyRes：将高分辨率图像分割成 2x2 或 1x3 的 336x336 裁剪块网格，外加一个全局低分辨率缩略图。每个裁剪块变为 576 个 token；每张图像总共约 2880 个视觉 token。OCR 和图表任务性能大幅提升。
- 使用 ShareGPT4V（高质量的 GPT-4V 标题）改进了指令数据混合。
- 更强的基座 LLM（Mistral-7B、Yi-34B）。

### LLaVA-OneVision

课程 12.08 深入介绍了 OneVision。简而言之：相同的投影器，但使用一个课程式训练，覆盖单图像、多图像和视频，在一个模型中共享视觉 token 预算。

### 与 Q-Former 的对比

| | Q-Former (BLIP-2) | MLP (LLaVA) |
|---|---|---|
| 每张图像的视觉 token | 32 | 576（基础）或 2880（AnyRes） |
| 可训练参数 | 1.88亿 + LM | 4000万 + LM |
| 阶段 1 损失 | ITC+ITM+ITG | 仅 LM |
| LLM 替换 | 需重新训练 | 可替换，只需最小重新训练 |
| 多图像 | 笨拙 | 自然（拼接） |
| 视频 | 笨拙 | 自然（逐帧拼接） |
| Token 预算 | 小 | 大 |

MLP 在简洁性和 token 灵活性上胜出。Q-Former 在 token 预算上胜出。到 2023 年底，token 预算不再是约束条件（LLM 上下文已增长到 32k-128k+），简洁性占据了主导。

### 提示格式

```
A chat between a curious human and an artificial intelligence assistant. The assistant gives helpful, detailed, and polite answers to the human's questions. USER: <image> Describe this image in detail. ASSISTANT: The image shows ...
```

`<image>` 是一个占位符 token。在分词之前，它被替换为 576 个视觉 token（或使用 AnyRes 时为 2880 个）。分词器看到的是一个比它训练时略长的序列，但 LLM 能处理这种新颖输入，因为阶段 1 教会了它这样做。

### 参数经济

LLaVA-1.5-7B 的组成：
- CLIP ViT-L/14 @ 336：3.03亿（阶段 1 冻结，阶段 2 通常解冻）。
- 投影器（2 层线性）：约 2200 万可训练。
- Llama-7B：70 亿。
- 总计：73 亿参数。阶段 2 可训练：完整的 70 亿 + 2200 万投影器。

阶段 2 的训练成本：在 8 张 A100 上约 20 小时。这是关键数字——一天，一个节点，可复现。这就是 LLaVA 传播的原因。

## 使用它

`code/main.py` 实现了：

1. 两层 MLP 投影器（为玩具规模使用维度 16 → 32 → 32），纯 Python 实现。
2. 提示词构建流水线：系统提示词 + 用 N 个投影后的 token 替换的 `<image>` + 用户轮次 + 助手生成占位符。
3. 一个可视化工具，展示 576-token 视觉块在 LLM 上下文中的占比（消耗 2k / 32k / 128k 上下文的百分比）。

## 交付它

本课程产出 `outputs/skill-llava-vibes-eval.md`。给定一个 LLaVA 系列检查点，它运行一个 10 提示词的功能评估套件（3 个标题生成、3 个 VQA、2 个推理、2 个拒绝），并报告一个人工可读的记分卡。这不是基准测试；而是确认投影器和 LLM 连接良好的冒烟测试。

## 练习

1. 计算 `1024 → 4096 → 4096` 的两层 MLP 投影器的可训练参数数量。在包含 GELU 和偏置的情况下，它占 LLaVA-13B 的多少比例？

2. 为一个"拒绝"案例构建一个 LLaVA 提示词——图像包含一个私人个体。写出预期的助手回复。为什么 LLaVA 应该零样本拒绝这个请求，需要哪些训练数据来强化这种拒绝？

3. 阅读 LLaVA-NeXT 博客的 AnyRes 部分。计算 1344x672 图像在 AnyRes 下的视觉 token 数量。与 336x336 的基础 576 个 token 进行比较。

4. LLaVA 阶段 1 的投影器在标题上使用 LM 损失进行训练。如果跳过阶段 1 直接进入阶段 2（视觉指令微调），会发生什么？引用 Prismatic VLMs 的消融实验（arXiv:2402.07865）作为答案。

5. LLaVA-Instruct-150k 使用 GPT-4 和 COCO 标题来生成指令。对于一个新领域（医学 X 光片、卫星图像），描述生成领域指令的四步数据流水线。每一步可能出现什么问题？

## 关键术语

| 术语 | 人们说的 | 实际含义 |
|------|---------|---------|
| 投影器 | "MLP 桥梁" | 两层 MLP 加 GELU，将 ViT 维度映射到 LLM 维度 |
| 图像 token | "<image> 占位符" | 推理前被 N 个投影后的视觉 token 替换的提示标记 |
| 视觉指令微调 | "LLaVA 阶段 2" | 在 GPT-4 生成的（图像，指令，回复）三元组上训练 |
| 阶段 1 对齐 | "投影器预训练" | 冻结 ViT 和 LLM，在标题上使用 LM 损失训练投影器 |
| AnyRes | "多裁剪块平铺" | 将高分辨率图像分割成裁剪块网格并拼接每个裁剪块的视觉 token |
| LLaVA-Instruct | "GPT-4 生成" | 从 COCO 标题 + GPT-4 合成的 15.8 万条指令-回复对 |
| 视觉编码器冻结 | "主干锁定" | CLIP 权重在阶段 1 不更新，有时阶段 2 也不更新 |
| ShareGPT4V | "更好的标题" | 由 GPT-4V 生成的 100 万条密集标题，用于更高质量的对齐 |
| VQA | "视觉问答" | 回答关于图像的开放式问题的任务 |
| Prismatic VLMs | "设计空间论文" | Karamcheti 2024 的系统性消融实验，测试投影器和数据选择 |
