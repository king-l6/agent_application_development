# MIO 与任意到任意流式多模态模型

> GPT-4o 提供了一个大多数开放模型无法复制的产品：一个能听到语音、看到视频并实时回应的智能体。到 2024 年底，开放生态系统的答案是 MIO（Wang 等人，2024 年 9 月）。MIO 对文本、图像、语音和音乐进行分词，在一个因果 transformer 上训练交错序列，并生成任意模态到任意模态的输出。AnyGPT（Zhan 等人，2024 年 2 月）是概念验证；MIO 是规模化版本；Unified-IO 2（Allen AI，2023 年 12 月）是增加了视觉+动作落地的表亲。本课程解读任意到任意模式——四个分词器、一个 transformer、支持流式的解码。

**类型：** 学习
**语言：** Python（标准库，四模态 token 分配器 + 流式解码循环）
**前置知识：** 阶段 12 · 11（Chameleon），阶段 6（语音和音频）
**时间：** ~120 分钟

## 学习目标

- 设计一个共享词汇表，容纳文本、图像、语音和音乐 token，且无冲突。
- 比较 SEED-Tokenizer（图像）和 SpeechTokenizer 残差 VQ（语音）在压缩与重建权衡方面的差异。
- 解释构建任意到任意生成的四阶段课程式训练。
- 说出三种开放的任意到任意方案及其主要权衡：MIO、AnyGPT、Unified-IO 2。

## 问题

统一的多模态模型说起来容易，大规模构建起来难。直到 2024 年，大多数"任意到任意"系统都是流水线式的：视觉模型 → 文本表示 → 语音模型 → 音频。每一跳都会丢失信息、增加延迟并使训练复杂化。GPT-4o 的演示视频展示了一个亚秒级响应的单模型替代方案；开放系统落后了几个月。

工程挑战：

- 每个模态都必须有分词器，以足够无损的方式进行压缩以便重建，并以 transformer 可以消费的速率产生 token。
- 单个词汇表必须为文本（32k+）、图像（16k+）、语音（4k+）、音乐（8k+）分配空间。至少四万多个条目。
- 训练数据必须覆盖每种输入-输出对（文本→图像、图像→语音、语音→图像等），否则模型必须进行组合。
- 推理必须足够快速地流式输出 token，以满足对话延迟要求（首次音频字节输出 <500ms）。

## 概念

### 四种模态的四个分词器

MIO 的分词器栈：

- 文本：标准 BPE，词汇表约 32000。
- 图像：SEED-Tokenizer（2023）——带离散码本的量化 VAE，4096 个条目，每张图像 32x32 个 token。
- 语音：SpeechTokenizer 残差 VQ（2023）——将 16kHz 波形编码为 8 个分层码本；第一层是粗略内容，后面各层添加韵律和说话人身份。
- 音乐：类似的残差 VQ（Meta 的 MusicGen / Encodec 系列），4-8 个码本。

每种模态产生整数 token。这些 token 在共享词汇表中获得不相交的 ID 范围：

```
text:   0..31999
image:  32000..36095  (4096 个图像 token)
speech: 36096..40191  (4096 个语音基础 token，加上残差层)
music:  40192..48383  (8192 个音乐 token)
sep:    48384..48390  (<image>, <speech>, <music>, </...> 等)
```

总计：约 48k 词汇量。输入嵌入和输出投影覆盖全部范围。

### 流式解码

语音生成使用残差 VQ。transformer 预测基础（第 0 层）语音 token；一个并行解码的残差量化器预测后续各层。每个第 0 层 token 对应 16kHz 下约 50ms 的音频。

流式模式：

1. 用户对着麦克风说话；实时音频分词器每 50ms 发出语音 token。
2. MIO 在 token 到达时消费它们（提示词预填充 + 增量前向传播）。
3. 输出 token 在生成时流式输出；一个并行语音解码器将它们转换为音频样本，延迟约 50-150ms。
4. 首次音频字节输出时间：MIO 论文中约 300-500ms，接近 GPT-4o 的约 250ms。

Mini-Omni (arXiv:2408.16725)、GLM-4-Voice (arXiv:2412.02612) 和 Moshi (arXiv:2410.00037) 是互补的流式语音-LLM 设计。其中 Moshi 在单 GPU 上实现了 160ms 的往返延迟。

### 四阶段课程式训练

MIO 的训练课程：

1. 阶段 1 — 对齐。大规模模态对语料库：文本-图像、文本-语音、文本-音乐。每对使用自己的 token 词汇段。训练共享词汇表。
2. 阶段 2 — 交错。多模态交错文档（包含图像+视频的博客、带转录的播客等）。训练跨模态上下文。
3. 阶段 3 — 语音增强。额外的音频数据，在不损失文本能力的情况下提升语音质量。
4. 阶段 4 — SFT。跨模态的指令微调：VQA、标题生成、叙述、语音到语音对话。

缺少某个阶段会降低特定能力：跳过阶段 2 则模型失去跨模态上下文；跳过阶段 3 则语音质量差。

### 视觉思维链

MIO 引入了视觉思维链：模型发出中间图像 token 作为推理步骤。对于"猫在爬树吗？"这个问题，模型：

1. 发出 `<image>` token，渲染场景（从输入图像或草图中）。
2. 发出分析草图的文本。
3. 发出最终答案。

渲染的中间图像充当草稿纸。在空间推理任务上的基准测试有所提升。这个想法与文本推理的思维链类似。

### 任意到任意领域的竞争者

- AnyGPT (arXiv:2402.12226)：4 种模态（文本、图像、语音、音乐），类似设计。
- Unified-IO 2 (arXiv:2312.17172)：增加了视觉动作输出、深度、法线。任务多样性更高，规模较小。
- NExT-GPT (arXiv:2309.05519)：LLM + 模态特定的扩散解码器。不是单一模型方法。
- CoDi (arXiv:2305.11846)：可组合扩散；通过共享潜在空间实现任意到任意。

MIO 是最接近纯 token 任意到任意的方法。AnyGPT 是其概念上的前身。

### 延迟预算

对于一个对话产品，每个组件的延迟都很重要：

- 麦克风到音频 token：约 50ms。
- 预填充（音频 token + 历史）：在 8B 模型上约 100ms。
- 第一个输出 token：约 50ms。
- 并行残差 VQ + 语音解码器：约 100-150ms。

总首次音频字节输出时间：至少约 300ms。GPT-4o 声称约 250ms。Moshi 声称 160ms。MIO/AnyGPT 根据公开基准测试在 400-600ms 范围内。

### 为什么任意到任意仍然困难

即使在 2026 年，开放的任意到任意模型在两个维度上仍然落后于闭源模型：

- 语音质量。残差 VQ 分词器有损；对话语音听起来比 ElevenLabs 级别的语音更机械。
- 跨模态推理。要求模型"唱出你看到的"仍比纯视觉任务更容易失败。

这些是开放的研究问题。Qwen3-Omni（课程 12.20）是 2025 年最先进的开放尝试。

## 使用它

`code/main.py`：

- 定义四模态词汇分配并打印出来。
- 将多模态输入列表（文本、图像、音频片段、音乐）通过分词器路由器进行路由。
- 模拟文本到语音回复的流式解码，带延迟计数。
- 根据编码器、预填充和解码器延迟计算预期的首次音频字节输出时间。

## 交付它

本课程产出 `outputs/skill-any-to-any-pipeline-auditor.md`。给定一个对话产品规格（输入模态、输出模态、延迟目标），它审计 MIO 系列的设计选择并计算延迟预算。

## 练习

1. 你的产品接受语音输入并返回语音输出。端到端延迟预算目标是多少？列出消耗时间的组件。

2. SpeechTokenizer 残差 VQ 使用 8 个码本。提出为什么并行解码残差层级是必要的（与顺序解码相比），以及它带来了多少延迟节省。

3. 你的词汇表有 32k 文本 + 4k 图像 + 4k 语音。增加 8k 音乐和约 10 个分隔符。在隐藏维度 4096 下，嵌入矩阵的参数成本是多少？

4. 视觉思维链发出一个中间图像。哪些类型的问题受益？哪些类型因额外 token 而受损？

5. 阅读 Moshi (arXiv:2410.00037)。描述其"内心独白"技术，并与 MIO 的视觉思维链进行比较。

## 关键术语

| 术语 | 人们说的 | 实际含义 |
|------|---------|---------|
| 任意到任意 | "多模态输入/输出" | 一个单一模型，可接受和发出文本、图像、语音和音乐的任何方向组合 |
| 残差 VQ | "语音分词器栈" | 多码本分词，每层添加信息；基础层是内容，后面各层是韵律 |
| SEED-Tokenizer | "图像码" | 离散图像分词器，具有 4096 条目的码本，由 MIO 使用 |
| 视觉思维链 | "视觉草稿纸" | 模型在最终答案之前生成一个中间图像作为推理步骤 |
| 首次音频字节输出时间 | "TTFAB" | 从用户语音到首次音频输出的延迟；<500ms 以获得对话感 |
| 四阶段课程 | "训练方案" | 对齐 → 交错 → 语音增强 → SFT，按此顺序 |
