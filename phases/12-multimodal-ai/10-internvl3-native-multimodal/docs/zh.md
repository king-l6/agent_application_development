# InternVL3：原生多模态预训练

> 在 InternVL3 之前，每款开放 VLM 都遵循相同的三步配方：拿一个在数万亿文本 token 上训练过的文本 LLM，外挂一个视觉编码器，然后微调接缝。这种做法有效，但存在对齐债务——文本 LLM 将其全部预训练预算花在了纯文本上，并不同生地理解视觉 token。当你事后添加视觉能力时，LLM 必须在不忘却文本的情况下重新学习如何将视觉输入与其文本推理联系起来。InternVL3（Zhu 等人，2025 年 4 月）拒绝了事后方法：一次预训练运行，文本和多模态从第一步开始交错进行。结果在 780 亿参数开放模型上，MMMU-Pro 匹配了 Gemini 2.5 Pro。本节课阅读原生预训练的论据，以及当做出这一改变时会发生什么变化。

**类型：** 学习
**语言：** Python（标准库，训练语料混合器）
**前置知识：** 第 12 阶段 · 05，第 12 阶段 · 07（配方）
**时间：** ~120 分钟

## 学习目标

- 解释为什么事后 VLM 训练会累积对齐债务，引用三种可测量的症状（灾难性遗忘、答案漂移、视觉-文本不一致）。
- 描述 InternVL3 的原生预训练语料配比，以及为什么文本：交错：描述的比例很重要。
- 比较 V2PE（可变视觉位置编码）与 Qwen2-VL 的 M-RoPE。
- 说出 Visual Resolution Router（ViR）和 Decoupled Vision-Language（DvD）部署优化。

## 问题

事后 VLM 训练是默认做法。LLaVA、BLIP-2、Qwen-VL、Idefics——都拿一个已经预训练好的 LLM（Llama、Vicuna、Qwen、Mistral）并添加视觉能力。训练阶段通常如下：

1. 冻结 LLM + 冻结视觉编码器 + 可训练投影器，在描述对上训练以对齐嵌入。
2. 解冻 LLM，在指令数据上训练（LLaVA-Instruct、ShareGPT4V）。
3. 可选的特定任务微调。

三种对齐债务的症状会出现：

- **灾难性遗忘。** 事后 VLM 会忘记纯文本技能。GSM8K 分数下降 5-10 个百分点。Hellaswag 分数下降。纯文本智能体出现退化。
- **答案漂移。** 相同视觉问题的小幅度措辞变化会得到不同答案。视觉编码器与 LLM 的连接比 LLM 自身 token 的连接更弱。
- **视觉-文本不一致。** VLM 可以正确描述一张图像，然后回答一个与其自身描述相矛盾的问题。视觉 token 不像文本那样参与 LLM 的内部一致性检查。

这些症状有充分记录。MM1.5 第 4 节量化了它们。LLaVA-OneVision 的消融实验暗示了它们的存在。原生预训练就是答案。

## 概念

### 原生多模态预训练

InternVL3 从一个从一开始就是原生多模态的语料库开始训练。配比为：

- 40% 纯文本数据（FineWeb、Proof-Pile-2 等）
- 35% 交错图文数据（OBELICS、MMC4 风格）
- 20% 配对图像-描述数据
- 5% 视频-文本数据

视觉 token、文本 token 和跨模态交互从第一个梯度步骤开始参与相同的损失函数。没有对齐预训练，没有投影器冻结阶段，没有需要恢复的灾难性遗忘。

基础模型的训练是单阶段的。指令微调紧随其后，但基础模型已经将视觉 token 理解为一等公民。

### V2PE（可变视觉位置编码）

Qwen2-VL 使用 M-RoPE 和固定的轴分配。InternVL3 引入了 V2PE：位置编码随模态类型（文本、图像、视频）变化，使用可学习的缩放。在实践中：

- 文本 token 获得 1D 位置（文本索引）。
- 图像 patch 获得 2D 位置（行、列）。
- 视频帧获得 3D 位置（时间、行、列）。

三者共享相同的 RoPE 频率基础，但每个频带的隐藏维度分配是学习参数而非固定分割。允许在预训练期间自由权衡时间与空间频率分辨率。

V2PE 的消融结果：在相同计算量下，视频基准上比 M-RoPE 高 1-2 个百分点。不是革命性的，但更干净。

### Visual Resolution Router（ViR）

部署优化。并非所有图像都需要全分辨率编码。一张包含单个低细节物体的照片在 1280px 原生分辨率下编码时会浪费 token。ViR 是一个小型分类器，在编码之前预测回答问题所需的最低分辨率。

路由有三个层级：低分辨率（256 token）、中分辨率（576）、高分辨率（2048+）。在生产流量中，60% 的查询使用低或中分辨率就足够了。净效果：在同等质量下吞吐量提升 2-3 倍。

### Decoupled Vision-Language 部署（DvD）

当你服务一个大型 VLM 时，视觉编码器每张图像运行一次，但 LLM 对每个输出 token 进行自回归运行。这两个组件有不同的瓶颈（视觉 = 卷积和注意力的 GPU 内存带宽；LLM = KV 缓存）。DvD 将它们拆分到不同的 GPU 上，之间通过流式传输。

对于一个 8B + 400M 编码器的模型，DvD 相比同地部署大致使每节点吞吐量翻倍。

### 单阶段 vs 多阶段质量

InternVL3 的主要基准声明：在 78B 参数下，匹配 Gemini 2.5 Pro 的 MMMU-Pro。在 38B 下，匹配 GPT-4o。在 8B 下，领跑开放 8B 排行榜。全部基于单阶段预训练加指令微调的配方。

对齐债务假设是可测量的：InternVL3-8B 每单位视觉基准增益损失的文本基准点（MMLU、GSM8K）少于 Qwen2.5-VL-7B。模型更通用，因为训练是一体的而非两段的。

### InternVL3.5 和 InternVL-U

InternVL3.5（2025 年 8 月）扩展了配方。相同的原生预训练方法，更多的数据，更多的参数。MMMU 的改进是增量式的。

InternVL-U（2026）增加了统一生成——通过相同骨干上的 MMDiT 头实现图像输出。"U"代表"理解 + 生成"，追随 Transfusion 风格统一模型（第 12.13 课）。相同的原生预训练骨干支持理解和生成头。

### 原生预训练的权衡

原生预训练并非没有代价：

- **计算。** 从头训练一个新的 VLM 成本与训练一个文本 LLM 相同——数百万 GPU 小时。事后适配重用了现有的 LLM 权重，节省了大部分成本。
- **数据。** 大规模的交错图文语料库很稀缺。OBELICS 有 1.41 亿文档；MMC4 有 5.71 亿。纯文本有 15T token。多模态预训练数据的稀缺性是一个硬约束。
- **基础 LLM 复用。** 原生预训练放弃了以后更换新 LLM 的选项。事后允许你通过仅重训练适配器将 Llama-3.1 换成 Llama-4。

InternVL3 的赌注是：对齐债务比复用损失更严重。基准测试支持这一说法。生产成本阻碍了未来实验室廉价复制。事后 VLM 将继续存在，因为对大多数项目来说它们仍然更便宜。

## 使用

`code/main.py` 是一个训练语料混合器和 ViR 路由器模拟器。它：

- 接收目标语料配比（%文本、%交错、%描述、%视频），并计算每种模态的预期步骤数。
- 模拟 ViR 在一批查询上的路由（分布：50% 低细节、30% 中细节、20% 高细节），并报告平均 token 数量。
- 根据编码器与 LLM 的 FLOPs 比报告 DvD 吞吐量估算。
- 并排打印事后与原生预训练在参数、计算、数据和预期对齐债务症状方面的比较。

## 练习

1. 估计 InternVL3-8B（原生预训练）和 LLaVA-OneVision-7B（事后）之间的计算差异。GPU 小时数比例大约是多少？是什么解释了这一差距？

2. InternVL3 报告 40% 文本 / 35% 交错 / 20% 描述 / 5% 视频。如果你的目标任务是视频密集型的，提出一个新的配比，并论证为什么基础模型仍然需要大量的文本和描述数据。

3. 阅读 MM1.5 第 4 节关于遗忘的内容。说出事后训练显示出最大退化的确切基准。退化的程度是多少？

4. ViR 将 60% 的流量路由到低分辨率编码。什么样的查询会被它错误路由（发送到低分辨率但实际需要高分辨率）？提出三种路由器失败模式。

5. DvD 将视觉和 LLM 拆分到不同的 GPU 上。在什么样的流量模式下 DvD 会损害而非帮助吞吐量？

## 关键术语

| 术语 | 人们常说的 | 实际含义 |
|------|-----------|---------|
| 原生多模态预训练 | "从头开始一起训练" | 文本 + 图像 + 视频 token 从第一步就参与损失函数，而不是后来外挂的 |
| 对齐债务 | "事后惩罚" | 因将视觉外挂到冻结 LLM 上而导致的可测量的文本技能退化和答案一致性下降 |
| V2PE | "可变视觉位置编码" | 每模态可学习的位置编码分配；InternVL3 对 M-RoPE 的继承 |
| ViR | "分辨率路由器" | 小型分类器，在编码前选择每查询所需的最小分辨率，节省推理 token |
| DvD | "解耦部署" | 视觉编码器在一个 GPU 上，LLM 在另一个 GPU 上，通过流式传输交接；对于大型 VLM 吞吐量翻倍 |
| InternVL-U | "统一理解 + 生成" | 2026 年的后续工作，在原生预训练骨干上添加了图像生成头 |
| 交错语料库 | "OBELICS / MMC4" | 文本和图像按自然阅读顺序排列的文档；原生预训练的原材料 |
