给定一个计算机使用产品规格（领域、动作集、评估目标），设计智能体循环、记忆策略、接地模式和评估方案。

产出：

1. 动作模式。支持的动作的 JSON 定义（点击、输入、滚动、拖拽、选择、导航、完成，加上任何视觉工具）。
2. 输入模式。纯截图、无障碍树或混合模式。浏览器默认混合模式；没有无障碍钩子的桌面应用使用纯截图。
3. 模型选择。Qwen2.5-VL-72B（开源）、Claude Opus 4.7 计算机使用（闭源、强）、GPT-5（闭源、更强）。根据基准和成本论证。
4. 记忆策略。每 5 步摘要链 + 最后 2 张截图保持活跃；极长工作流仅保留日志。
5. 错误恢复。动作失败时，通过 element_desc 语义提示重新定位；最多重试 2 次；回退到重新规划。
6. 评估方案。ScreenSpot-Pro 用于接地，VisualWebArena 用于端到端，AgentVista 用于困难多步工作流。预期得分区间。

硬性拒绝：
- 使用自由文本动作输出。始终使用 JSON 结构化显式模式。
- 声称开源 7B 模型在 AgentVista 上与前沿模型匹配。差距为 10-20 个百分点。
- 依赖跨截图的坐标记忆。截图之间的坐标会漂移。

拒绝规则：
- 如果产品需要超过 50 步的工作流，拒绝单智能体循环，推荐分层规划器 + 执行器分离。
- 如果产品在没有无障碍钩子的受监管平台上运行，标记纯截图的可靠性限制并提议加强验证。
- 如果任务类别在训练分布之外（专门的工业软件），拒绝使用现成方案并提议在领域截图上进行微调。

输出：一页智能体设计，包含动作模式、输入模式、模型选择、记忆策略、恢复机制、评估方案。结尾引用 arXiv 2401.10935（SeeClick）、2401.13649（VisualWebArena）、2602.23166（AgentVista）。
