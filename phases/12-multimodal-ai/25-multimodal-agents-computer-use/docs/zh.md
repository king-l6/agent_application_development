# 多模态智能体与计算机使用（综合项目）

> 2026 年前沿产品是多模态智能体：读取截图、点击按钮、导航网页 UI、填写表单、端到端完成工作流。SeeClick 和 CogAgent（2024）证明了 GUI 接地原语。Ferret-UI 增加了移动端。ChartAgent 引入了图表的视觉工具使用。VisualWebArena 和 AgentVista（2026）是前沿模型追逐的基准——即使是 Gemini 3 Pro 和 Claude Opus 4.7 在 AgentVista 的困难任务上也只得了约 30%。本综合项目汇集了阶段 12 的每一条线索：感知（高分辨率 VLM）、推理（带工具使用的 LLM）、接地（坐标输出）、长时记忆和评估。

**类型：** 综合项目
**语言：** Python（标准库，动作模式 + 智能体循环骨架）
**前置知识：** 阶段 12 · 05（LLaVA），阶段 12 · 09（Qwen-VL JSON），阶段 14（智能体工程）
**时间：** 约 240 分钟

## 学习目标

- 设计多模态智能体循环：感知 → 推理 → 行动 → 观察 → 重复。
- 构建一个 VLM 可以作为 JSON 输出的 GUI 接地输出模式（点击坐标、输入文本、滚动、拖拽）。
- 比较纯截图智能体与无障碍树智能体及混合智能体。
- 在小型 VisualWebArena 切片上设置多模态智能体基准评估。

## 问题

一个订票网站工作流："帮我找 4 月 15 日去东京的航班，靠过道座位，800 美元以内，订好。"

一个多模态智能体需要：

1. 截取浏览器截图。
2. 将截图 + URL + 目标解析为计划。
3. 发出结构化动作：点击（在 x,y 处），输入"东京"（在元素 E 处），向下滚动，选择（单选按钮）。
4. 将动作应用到浏览器。
5. 观察新状态（下一张截图）。
6. 重复直到任务完成。

每一步都是多模态 VLM 调用。VLM 输出必须是可解析的 JSON。错误会跨步骤累积，因此恢复很重要。

## 概念

### GUI 接地 — 原语

GUI 接地是：给定一个截图和自然语言指令，输出要点击的 (x, y) 坐标（或其他动作）。

SeeClick (arXiv:2401.10935) 是首个大规模开放结果：在合成 + 真实 GUI 数据上微调 VLM，将坐标输出为纯文本令牌。有效。

CogAgent (arXiv:2312.08914) 增加了 1120x1120 高分辨率编码，用于密集 UI。得分：网页导航约 84%。

Ferret-UI (arXiv:2404.05719) 专注于移动 UI，与 iOS 无障碍数据集成。

输出格式通常为 JSON：

```json
{"action": "click", "x": 384, "y": 220, "element_desc": "搜索按钮"}
```

`element_desc` 有助于恢复：如果截图之间坐标漂移，语义提示让系统重新定位。

### 动作模式

典型的动作模式有 6-10 种动作类型：

- `click`：(x, y)
- `type`：(text, x?, y?)
- `scroll`：(direction, amount)
- `drag`：(x0, y0, x1, y1)
- `select`：(option_index)
- `hover`：(x, y)
- `navigate`：(url)
- `wait`：(ms)
- `done`：(success, explanation)

智能体每步发出一个动作。浏览器包装器执行并返回新状态。

### 纯截图 vs 无障碍树

两种输入模式：

- 纯截图：完整图像，无结构化信息。最通用；适用于任何应用。
- 无障碍树：结构化 DOM / iOS 无障碍信息。对于接地来说更可靠；在树可用时有效。
- 混合模式：两者结合，树作为原子动作的可靠定位器，截图作为语义上下文。

生产级智能体在可能时使用混合模式。浏览器自动化（Selenium + 无障碍）始终有树；桌面应用有时有。

### 长时记忆

一个 20 步的工作流生成 20 张截图。VLM 的上下文很快占满。三种压缩策略：

- 摘要链：每 5 步后，总结已发生的事情，丢弃旧截图。
- 跳帧：保留第一张、最后一张和每第 3 张截图。
- 工具记录日志：执行动作，保留已做事情的文字日志；不再查看旧截图。

Claude 的计算机使用 API 使用日志模式。更简单，更可靠。

### 视觉工具使用

ChartAgent (arXiv:2510.04514) 引入了用于图表理解的视觉工具使用：裁剪、缩放、OCR、调用外部检测。智能体可以输出"裁剪到区域 (100, 200, 300, 400) 然后调用 OCR"作为工具调用。工具返回文本；VLM 继续推理。

这种模式可以推广：标记提示集、区域标注和外部检测工具都适合同一个"输出工具调用，接收结构化响应"的模式。

### 2026 年基准

- ScreenSpot-Pro。约 1k 网页截图上的 GUI 接地。开源 SOTA Qwen2.5-VL-72B 约 85%。前沿约 90%。
- VisualWebArena。端到端网页任务（购物、论坛、分类信息）。开源 SOTA 约 20%。Gemini 3 Pro 约 27%。
- AgentVista (arXiv:2602.23166)。2026 年最难的基准。跨 12 个领域的真实工作流。前沿模型得分 27-40%；开源模型 10-20%。
- WebArena / WebShop。较旧基准；已被前沿模型饱和。

### 为什么仍然困难

智能体性能瓶颈：

1. 细粒度视觉定位。"点击那个小 X"在移动分辨率下经常失败。
2. 长时规划。10 步后，智能体偏离目标。
3. 错误恢复。当点击失败（错误按钮）时，检测 + 恢复很少在训练数据中。
4. 跨页上下文。在标签页或长表单之间跳转会丢失状态。

研究方向：记忆架构、显式重新规划、多模态验证（动作成功的截图匹配）。

### 综合项目构建

综合项目任务：构建一个计算机使用智能体，能够：

1. 读取订票网站模拟页面的 HTML + 截图。
2. 规划多步序列：搜索 → 选择 → 填写表单 → 提交。
3. 发出匹配动作模式的 JSON 动作。
4. 在固定的 10 个任务切片上进行评估。

本课程提供脚手架代码，易于扩展到真实浏览器。

- 10 任务微型基准（合成页面），用于测量端到端成功率。

## 关键术语

| 术语 | 人们说的 | 实际含义 |
|------|----------|----------|
| GUI 接地（GUI grounding） | "点击坐标" | 模型输出截图上指令目标的 (x,y) |
| 动作模式（Action schema） | "工具定义" | 有效动作的 JSON 描述（点击、输入、滚动、拖拽） |
| 无障碍树（Accessibility tree） | "结构化 DOM" | 来自浏览器/iOS API 的机器可读 UI 层次结构 |
| 混合智能体（Hybrid agent） | "截图 + 树" | 同时使用图像和结构化信息；比单独使用任一方式更可靠 |
| 视觉工具使用（Visual tool use） | "缩放/裁剪/检测" | 智能体在计划中调用外部视觉工具（OCR、检测） |
| 摘要链（Summary-chain） | "记忆压缩" | 定期文本摘要替代长截图历史 |
| VisualWebArena | "端到端网页基准" | 端到端网页任务的 2024 年基准 |
| AgentVista | "2026 困难基准" | 12 领域真实工作流；即使是 Gemini 3 Pro 也只得了约 30% |
