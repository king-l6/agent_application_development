# 开放权重 VLM 配方：真正重要的因素

> 2024-2026 年的开放权重 VLM 文献是一片消融实验的森林。Apple 的 MM1 测试了 13 种图像编码器、连接器和数据配比的组合。Allen AI 的 Molmo 证明了详细的人工描述优于 GPT-4V 蒸馏。Cambrian-1 进行了 20 多次编码器比较。Idefics2 正式定义了五轴设计空间。Prismatic VLMs 在受控基准上比较了 27 种训练配方。在所有这些噪声中，有一小部分结果在所有论文中保持一致：图像编码器比连接器架构更重要，数据配比比这两者都重要，详细的人工描述优于蒸馏合成数据。本节课阅读这些表格，这样你就不必亲自做了。

**类型：** 学习 + 实验
**语言：** Python（标准库，消融表解析器 + 配方选择器）
**前置知识：** 第 12 阶段 · 05（LLaVA 基线）
**时间：** ~180 分钟

## 学习目标

- 说出五轴 VLM 设计空间：图像编码器、连接器、LLM、数据配比、分辨率调度。
- 阅读 MM1 / Idefics2 / Cambrian-1 的消融表，预测哪个旋钮会影响哪个基准指标。
- 在给定计算预算和任务组合的情况下，为新 VLM 选择配方（编码器、连接器、数据、分辨率）。
- 解释为什么在相同 token 数量下，详细的人工描述优于 GPT-4V 蒸馏。

## 问题

存在数百种开放权重 VLM。"好"与"最先进"之间的差距大部分不在于架构，而在于数据、分辨率调度和编码器选择。知道当模型表现不佳时应该先转动哪个旋钮，可以避免一个 500 万 GPU 小时的错误。

2023 年这一波（LLaVA-1.5、InstructBLIP、MiniGPT-4）依赖于描述对预训练 + LLaVA-Instruct-150k。不错的基线，但在 MMMU 上最高约 35%。

2024 年这一波（MM1、Idefics2、Molmo、Cambrian-1、Prismatic VLMs）进行了详尽的消融实验。结果出人意料且实用。

## 概念

### 五轴设计空间

Idefics2（Laurencon 等人，2024）命名了这五个轴：

1. **图像编码器。** CLIP ViT-L/14、SigLIP SO400m/14、DINOv2 ViT-g/14、InternViT-6B。编码器在 patch 大小、分辨率和预训练目标上有所不同。
2. **连接器。** MLP（2-4 层）、Q-Former（32 查询 + 交叉注意力）、Perceiver Resampler（64 查询）、C-Abstractor（卷积 + 双线性池化）。
3. **语言模型。** Llama-3 8B / 70B、Mistral 7B、Phi-3、Gemma-2、Qwen2.5。LLM 大小是主要的参数量成本。
4. **训练数据。** 描述对（CC3M、LAION）、交错式（OBELICS、MMC4）、指令（LLaVA-Instruct、ShareGPT4V、PixMo、Cauldron）。
5. **分辨率调度。** 固定 224/336/448、AnyRes、原生动态。训练期间逐步提升或保持恒定。

每个生产级 VLM 都会在每个轴上做出选择。MMMU 分数的大多数方差由轴 1、4 和 5 解释——而不是你选择了哪种连接器。

### 轴 1：编码器 > 连接器

MM1 第 3.2 节表明：从 CLIP ViT-L/14 切换到 SigLIP SO400m/14 在 MMMU 上增加了 3 个以上的百分点。将连接器从 MLP 切换到 Perceiver Resampler 增加的不到 1 个百分点。Idefics2 复现了这一点：SigLIP > CLIP，在相同 token 数量下 Q-Former 约等于 MLP 约等于 Perceiver。

Cambrian-1 的 "Cambrian Vision Encoders Match-Up"（Tong 等人，2024）在视觉中心基准（CV-Bench）上运行了 20 多个编码器。排行榜前列是 DINOv2 和 SigLIP 的混合；CLIP 处于中游；ImageBind 和 ViT-MAE 较低。从 CLIP ViT-L 到 DINOv2 ViT-g/14 在 CV-Bench 上的差距约为 5-7 个百分点。

2026 年开放 VLM 的默认编码器是 SigLIP 2 SO400m/14（用于语义 + 密集特征），有时会与 DINOv2 ViT-g/14 特征拼接（Cambrian 的 "Spatial Vision Aggregator" 就是这样做的）。

### 轴 2：连接器设计差别不大

MM1、Idefics2、Prismatic 和 MM-Interleaved 都得出了相同的结论：在固定视觉 token 数量的情况下，连接器架构几乎无关紧要。在均值池化 patch 上使用 2 层 MLP 与使用 32 查询的 Q-Former 在相同 token 预算下的差距在 1 个百分点以内。

真正重要的是 token 数量。更多的视觉 token = 更多的 LLM 计算 = 更好的性能（达到某一点后收益递减）。每张图像 64 个 token 对于 OCR 来说太少。576-1024 个 token 是大多数开放 VLM 的最佳区间。2048+ 仅在处理文档和图表时才有帮助。

Q-Former 与 MLP 是一个成本问题，而非质量问题：Q-Former 将 token 上限固定为 32-64，与图像分辨率无关；MLP 则输出所有 patch token。对于高分辨率输入，Q-Former 节省 LLM 上下文；对于低分辨率输入，差异可忽略不计。

### 轴 3：LLM 大小设定上限

将 LLM 从 7B 翻倍到 13B，在所有 VLM 论文的 MMMU 上都稳定增加 2-4 个百分点。在 70B 时，大多数基准达到饱和。VLM 的多模态推理上限就是 LLM 的文本推理上限——视觉编码器只能提供输入，而不能进行推理。

这就是为什么 Qwen2.5-VL-72B 和 Claude Opus 4.7 在 MMMU-Pro 和 ScreenSpot-Pro 上表现优异：语言大脑非常庞大。一个 7B 的 VLM 无法通过巧妙的连接器设计来替代 70B 的 VLM。

### 轴 4：数据——详细人工描述优于蒸馏

Molmo + PixMo（Deitke 等人，2024）是每个人都应该阅读的 2024 年成果。Allen AI 让人类标注员通过 1-3 分钟的密集语音转文字描述来标注图像，产生了 71.2 万张密集描述的图像。训练数据中没有任何 GPT-4V 蒸馏。

Molmo-72B 在 11 个基准测试中的全部 11 个上击败了 Llama-3.2-90B-Vision。差距不在于架构——而在于描述质量。详细的人工描述每张图像包含的信息量是简短网络描述的 5-10 倍，并且在 GPT-4V 蒸馏会产生幻觉的地方保持事实准确。

ShareGPT4V（Chen 等人，2023）和 Cauldron（Idefics2）遵循相同的策略，采用人工 + GPT-4V 描述的混合方式。趋势很明显：对于 2026 年的前沿模型，描述密度 > 描述数量 > 蒸馏便利性。

### 轴 5：分辨率及其调度

Idefics2 的消融实验：384 -> 448 增加 1-2 个百分点。448 -> 980 使用图像分割（AnyRes）在 OCR 基准上再增加 3-5 个百分点。固定分辨率训练在中等精度时达到平台期；分辨率提升（从 224 开始，以 448 或原生分辨率结束）训练更快，最终精度更高。

Cambrian-1 进行了分辨率与 token 的权衡实验：在固定计算量下，你可以选择低分辨率下更多 token，或高分辨率下更少 token。高分辨率在 OCR 上胜出；低分辨率-多 token 在一般场景理解上胜出。

2026 年的生产配方：第一阶段在固定 384 分辨率下训练，第二阶段采用动态分辨率（对于 OCR 密集型任务最高可达 1280）。

### Prismatic 受控比较

Prismatic VLMs（Karamcheti 等人，2024）是控制了所有轴的那篇论文。相同的 13B LLM、相同的指令数据、相同的评估——每次只变化一个轴。结果：

- 每张图像的视觉 token 数量解释了约 60% 的方差。
- 编码器选择解释了约 20%。
- 连接器架构解释了约 5%。
- 其他所有因素（数据配比、调度器、学习率）解释了剩余的约 15%。

这是一个粗略的分解，但它是文献中对"我应该先消融哪个"这个问题最清晰的回答。

### 2026 年的选择器

基于这些证据，2026 年新项目的默认开放 VLM 配方如下：

- **编码器：** SigLIP 2 SO400m/14，原生分辨率使用 NaFlex，如果需要分割/定位，则与 DINOv2 ViT-g/14 拼接以获得密集特征。
- **连接器：** 在 patch token 上使用 2 层 MLP。除非受到 token 限制，否则跳过 Q-Former。
- **LLM：** Qwen2.5 / Llama-3.1 / Gemma 2，7B 用于成本控制，70B 用于质量，根据目标延迟选择。
- **数据：** PixMo + ShareGPT4V + Cauldron，再加上特定任务的指令数据。
- **分辨率：** 动态（长边最小 256，最大 1280 像素）。
- **调度：** 第一阶段对齐（仅投影器），第二阶段全参数微调，第三阶段特定任务微调。

这些默认选择中的每一个都可以追溯到本节课末尾引用的论文中实测的消融实验。

## 使用

`code/main.py` 是一个消融表解析器和配方选择器。它编码了 MM1 和 Idefics2 的消融表（压缩版），并允许你查询：

- "给定预算 X 和任务 Y，哪种配方最优？"
- "如果我在 7B Llama 上将 SigLIP 换成 CLIP，预期 MMMU 变化是多少？"
- "为了获得 80% 置信度的答案，我应该先消融哪个轴？"

输出是一个排序后的配方列表，包含预期的基准变化和"优先消融"建议。

## 交付

本节课生成 `outputs/skill-vlm-recipe-picker.md`。给定目标任务组合、计算预算和延迟目标，它会输出完整的配方（编码器、连接器、LLM、数据配比、分辨率调度），并附上证明每个选择的合理性的消融实验引用。避免工程师每次启动新 VLM 项目时都重新发明 Idefics2 的消融表。

## 练习

1. 阅读 MM1 第 3.2 节。对于固定 2B LLM 和 5000 万张图像的预算，哪种编码器胜出？如果换成 13B LLM，答案会改变吗？为什么？

2. Cambrian-1 发现，拼接 DINOv2 + SigLIP 在视觉中心基准上优于两者中的任何一个，但在 MMMU 上没有增加信号。预测哪些基准会提升，哪些会保持不变。

3. 你的目标是在 2B LLM 上构建一个移动 UI 智能体。选择编码器、连接器、分辨率和数据配比。用具体的消融表证明每个选择的合理性。

4. Molmo 发布了 4B 和 72B 模型。4B 模型在与封闭的 7B VLM 竞争中表现不俗；72B 模型在 11/11 的基准上击败了 Llama-3.2-90B-Vision。这告诉我们关于 LLM 大小平台期假设的什么信息？

5. 设计一个消融表，以在 7B VLM 上分离数据配比质量和编码器质量的影响。至少需要多少次训练运行？提出四个轴的设置方案。

## 关键术语

| 术语 | 人们常说的 | 实际含义 |
|------|-----------|---------|
| 消融 | "转动一个旋钮" | 多次训练运行，在恰好一个设计空间轴上有所不同，保持其他所有条件不变 |
| 连接器 | "桥接" / "投影器" | 将视觉编码器输出映射到 LLM token 空间的可训练模块（MLP、Q-Former、Perceiver） |
| 详细人工描述 | "密集描述" | 多句子的人工书写的描述（通常 80-300 token），比网络 alt 文本更丰富 |
| 蒸馏 | "GPT-4V 描述" | 由更强的专有 VLM 生成的训练数据；方便但容易继承幻觉 |
| AnyRes / 动态分辨率 | "高分辨率路径" | 通过平铺或 M-RoPE 输入大于编码器原生分辨率的图像的策略 |
| 分辨率提升 | "课程学习" | 从低分辨率开始训练，逐步提高的训练调度，加速对齐学习 |
| 视觉中心基准 | "CV-Bench / BLINK" | 强调细粒度视觉感知而非语言密集型推理的评估 |
| PixMo | "Molmo 的数据" | Allen AI 的 71.2 万张密集描述图像数据集；人工语音转录为密集描述 |
