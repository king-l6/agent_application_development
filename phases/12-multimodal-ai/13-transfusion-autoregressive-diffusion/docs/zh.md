# Transfusion：在一个 Transformer 中融合自回归文本与扩散图像

> Chameleon 和 Emu3 把一切都押注在离散令牌上。它们确实有效，但量化瓶颈是可见的——图像质量低于连续空间扩散模型。Transfusion（Meta，Zhou 等人，2024 年 8 月）采取了相反的赌注：保持图像的连续性，完全放弃 VQ-VAE，训练一个带有两个损失的 Transformer。文本令牌使用下一令牌预测。图像块使用流匹配 / 扩散损失。两个目标同时优化相同的权重。Stable Diffusion 3（MMDiT）的底层架构是近亲。本课程将研读 Transfusion 论文，构建一个玩具版的双损失训练器，并追踪让一个 Transformer 同时完成两项工作的注意力掩码。

**类型：** 构建
**语言：** Python（标准库，在 MNIST 级别的玩具上实现双损失训练器）
**前置要求：** 阶段 12 · 11（Chameleon），阶段 8（生成式 AI）
**时间：** ~180 分钟

## 学习目标

- 构建一个 Transformer，在一个主干上运行两个损失（文本令牌上的 NTP，图像块上的扩散 MSE）。
- 解释为什么图像块上的双向注意力加上文本令牌上的因果注意力是正确的掩码选择。
- 比较 Transfusion 风格（连续图像，扩散损失）与 Chameleon 风格（离散图像，NTP）在计算、质量和代码复杂度方面的差异。
- 列举 MMDiT 的贡献：每个块中的模态特定权重，残差流中的联合注意力。

## 问题

离散与连续图像令牌的争论比 LLM 的历史更久远。连续表示（原始像素、VAE 潜变量）保留了细节。离散令牌（VQ 索引）适合 Transformer 的原生词汇表，但在量化步骤中会丢失细节。

Chameleon / Emu3 选择了离散：一个损失，一个架构，但图像保真度受限于分词器质量。

扩散模型选择了连续：卓越的图像质量，但与 LLM 是分离的模型，需要复杂的噪声调度工程，并且与文本生成没有干净的集成。

Transfusion 问道：我们能两者兼得吗？保持图像的连续性，仍然训练一个模型，使用两个损失拼接成一个梯度步骤。

## 概念

### 双损失架构

一个单一的仅解码器 Transformer 处理包含以下内容的序列：

- 文本令牌（离散的，来自 BPE 词汇表）。
- 图像块（连续的，通过线性嵌入投影到隐藏维度的 16x16 像素块——与 ViT 编码器的输入层相同）。
- `<image>` 和 `</image>` 标签标记连续块所在的位置。

前向传播运行一次。损失根据每个令牌选择两个头部之一：

- 对于文本令牌：词汇表 logits 头部上的标准交叉熵损失。
- 对于图像块：连续图像块上的扩散损失——预测添加到每个块上的噪声。

梯度流经共享的 Transformer 主体。两个损失同时改进共享的权重。

### 注意力掩码：因果文本 + 双向图像

文本令牌必须是因果的——你不能让一个文本令牌关注未来的文本，否则教师强制会失效。然而，图像块代表一个快照；它们应该在同一个图像块内相互双向关注。

掩码：

```
M[i, j] = 1 如果：
  (i 是文本 且 j 是文本 且 j <= i)   # 文本的因果掩码
  或 (i 是图像 且 j 是图像 且 same_image_block(i, j))   # 图像内双向
  或 (i 是文本 且 j 是图像 且 j < i_image_end)   # 文本关注前面的图像
  或 (i 是图像 且 j 是文本 且 j < i_image_start)   # 图像关注前面的文本
```

在训练和推理时实现为块三角形掩码。

### Transformer 内部的扩散损失

扩散损失是标准的：向图像块添加噪声，让模型预测噪声（或者等价地预测干净图像块）。Transfusion 的版本使用流匹配——预测从含噪到干净的 velocity 场。

训练过程中：
1. 对于每个图像块 x0，采样一个随机时间步 t。
2. 采样噪声 ε，计算 xt = (1-t) * x0 + t * ε（流匹配的线性插值）。
3. Transformer 预测 v_theta(xt, t)；损失 = MSE(v_theta(xt, t), ε - x0)。
4. 与来自同一序列的文本 NTP 损失一起反向传播。

推理时，生成方式为：
- 文本令牌：标准自回归采样。
- 图像块：以前面的文本令牌为条件的扩散采样循环（典型为 10-30 步）。

### MMDiT：Stable Diffusion 3 的变体

Stable Diffusion 3（Esser 等人，2024 年 3 月）大约在 Transfusion 同时发布了 MMDiT（多模态扩散 Transformer）。这两个架构是姊妹关系。

MMDiT 的关键区别：

- **每个块的模态特定权重**：每个 Transformer 块对文本令牌和图像块有独立的 Q、K、V 和 MLP 权重。注意力是联合的（跨模态）；其他一切都是模态特定的。
- **修正流训练**：一种特定的流匹配变体，具有已知的采样方法和比 DDPM 更简单的数学。
- **规模**：MMDiT 是 SD3（2B 和 8B 参数变体）的主干网络。Transfusion 的论文扩展到 7B。

两者都收敛到相同的核心思想：一个 Transformer 对文本运行 NTP，对连续图像表示运行扩散。

### 为什么这优于 Chameleon 风格

连续扩散与离散 NTP 在图像生成上的质量差距是可测量的。Transfusion 论文报告：

- 在 7B 参数下，在 FID 上比同等大小的 Chameleon 风格模型好 3-5 个点。
- 不需要训练分词器——图像编码器更简单（线性投影到隐藏维度，与 ViT 的输入层相同）。
- 推理时可以并行化图像块去噪，不同于自回归图像令牌。

缺点：Transfusion 是一个双损失模型，使得训练动态更加棘手。损失权重需要调参。NTP 和扩散之间的调度不匹配可能导致一个头部主导另一个。

### 后续发展

Janus-Pro（第 12.15 课）通过解耦视觉编码器来细化 Transfusion 的思想——SigLIP 用于理解，VQ 用于生成——同时共享 Transformer 主体。Show-o（第 12.14 课）将扩散替换为离散扩散（掩码预测）。统一生成家族在 Transfusion 之后迅速分支发展。

2026 年能够生成图像的生产级 VLM——Gemini 3 Pro、GPT-5、Claude Opus 4.7 的图像生成路径——几乎肯定使用了这个家族的某个后代。具体细节是专有的。

## 使用它

`code/main.py` 在一个微型的 MNIST 级别问题上构建了一个玩具版 Transfusion：

- 文本描述是描述一个数字（0-9）的短整数序列。
- 图像是 4x4 的字节网格。
- 一对共享权重的线性投影充当 Transformer 的替代品；文本使用 NTP 损失，含噪块使用 MSE 损失。
- 训练循环交替两个损失，注意力掩码是显式构造的。
- 生成过程在一次前向传播中产生文本描述和 4x4 图像。

Transformer 是玩具级别的。双损失管道、注意力掩码构建和推理循环才是真正的产出物。

## 练习

1. 一个 Transfusion 风格模型训练时包含 70% 文本令牌和 30% 图像块。图像扩散损失的幅值大约是文本 NTP 损失的 10 倍。什么样的损失权重可以平衡它们？

2. 为序列 `[T, T, <image>, P, P, P, P, </image>, T]` 实现块三角形掩码。标记每个条目为 0 或 1。

3. MMDiT 有模态特定的 QKV 权重。与 Transfusion 的完全共享 Transformer 相比，这增加了多少参数开销？在 7B 参数规模下，这值得吗？

4. 生成过程：给定一个文本提示，模型运行 NTP 生成 50 个令牌，然后遇到 `<image>`，之后在 256 个块上运行 20 步去噪扩散。总共需要多少次前向传播？

5. 阅读 SD3 论文第 3 节。描述修正流，并解释为什么它比 DDPM 收敛所需的推理步骤更少。

## 关键术语

| 术语 | 人们说的意思 | 实际含义 |
|------|-------------|---------|
| 双损失训练 | "NTP + 扩散" | 同一个 Transformer 在同一个梯度步骤中优化文本令牌上的交叉熵和连续图像块上的 MSE |
| 流匹配 | "修正流" | 扩散变体，预测从噪声到干净数据的 velocity 场；数学比 DDPM 更简单 |
| MMDiT | "多模态 DiT" | Stable Diffusion 3 的架构：联合注意力，模态特定的 MLP 和归一化层 |
| 块三角形掩码 | "因果文本 + 双向图像" | 在文本上因果但在图像区域内双向的注意力掩码 |
| 连续图像表示 | "无 VQ" | 图像块作为实值向量，而不是整数码本索引 |
| 速度预测 | "v-参数化" | 网络输出是噪声和数据之间的 velocity 场，而不是噪声本身 |
