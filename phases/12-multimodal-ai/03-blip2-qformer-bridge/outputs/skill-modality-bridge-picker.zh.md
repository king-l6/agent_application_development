给定视觉编码器每张图像的 token 数、LLM 的上下文预算、每个提示的目标图像数以及训练计算预算，推荐使用哪种模态桥接器并用参数量和 token 经济性进行合理性证明。

生成以下内容：

1. **Token 预算审计**：报告视觉编码器中每张图像的原始 token 数、每种桥接器选项后每张图像的 token 数，以及在声明的每提示图像数下消耗的 LLM 上下文比例。
2. **桥接器比较**：对于 Q-Former（32 token，约 188M 参数）、MLP 投影器（所有 patches，约 20M 参数）和 Perceiver 重采样器（通过 N 层交叉注意力的 K 个可学习查询，可变）中的每一个，给出参数、质量代理和训练成本大致范围。
3. **推荐**：针对声明的约束条件的单一最佳选择，附一行理由。当约束条件相互矛盾时（高质量 + 紧 token 预算 + 低训练计算）进行标记。
4. **两阶段训练追踪**：如果选择 Q-Former，概述阶段 1 的 ITC + ITM + ITG 损失和阶段 2 的 LM 损失。为每个阶段命名一个代表性数据集（COCO、LAION、Visual Genome）。
5. **消融检查清单**：调用者在锁定桥接器之前应进行的五个实验（查询数量、两阶段 vs 单阶段、投影器深度、冻结调度、微调子集）。

**硬性拒绝：**
- 任何忽略 token 预算的推荐。在 4k 上下文中，每个图像 576 token 的"使用 MLP"在 10 张图像时失败。
- 声称 Q-Former 严格支配 MLP。在具有无限上下文的单图像高质量任务中，MLP 胜出。
- 将 Perceiver 重采样器视为与 Q-Former 等同。Flamingo 在每个 LLM 层应用它；BLIP-2 只应用一次。

**拒绝规则：**
- 如果调用者要求一个能处理视频的桥接器但没有指定帧数和帧率，拒绝——视频桥接器与单图像桥接器的区别在于规格，而不仅仅是规模。
- 如果范围内的 LLM 是与视觉塔一起从头训练的（早期融合，Chameleon 风格），拒绝——课程 12.11 单独涵盖这种情况。
- 如果没有说明训练计算量，拒绝并询问调用者是否负担得起 BLIP-2 的阶段 2（约几百 A100 小时）或仅投影器训练。

**输出**：一页桥接器推荐，包含 token 数学、参数量、推荐架构、训练大纲和消融清单。以"下一步阅读"段落结束，指向课程 12.04（Flamingo）以了解处处交叉注意力、课程 12.05（LLaVA）以了解仅 MLP，或课程 12.07（消融）以了解数据与架构的权衡。
