# 从 CLIP 到 BLIP-2 —— Q-Former 作为模态桥接器

> CLIP 对齐了图像和文本，但不能生成标题、回答问题或进行对话。BLIP-2（Salesforce，2023 年）用一个小的可训练桥接器解决了这个问题：32 个可学习查询向量通过交叉注意力关注冻结 ViT 的特征，然后直接嵌入到冻结 LLM 的输入流中。1.88 亿参数的桥接器连接了一个 11B 的 LLM 和一个 ViT-g/14。一直到 2026 年，每个基于适配器的 VLM——MiniGPT-4、InstructBLIP、LLaVA 的"表亲们"——都是它的后代。本节课解读 Q-Former 的架构，解释其两阶段训练，并构建一个将视觉 token 输入到冻结文本解码器的玩具版本。

**类型：** 构建
**语言：** Python（标准库，交叉注意力 + 可学习查询演示）
**前置知识：** 阶段 12 · 02（CLIP），阶段 7（Transformer）
**时间：** ~180 分钟

## 学习目标

- 解释为什么在冻结的视觉编码器和冻结的 LLM 之间设置一个可训练的瓶颈在成本和稳定性上优于端到端微调。
- 实现一个交叉注意力块，其中一组固定的可学习查询关注外部的图像特征。
- 讲解 BLIP-2 的两阶段预训练：表示学习（ITC + ITM + ITG）然后是生成式学习（使用冻结解码器的 LM 损失）。
- 比较 Q-Former 与 LLaVA 中使用的更简单的 MLP 投影器，并论证每种选择的优势场景。

## 问题

你有一个冻结的 ViT，每张图像产生 256 个维度为 1408 的 patch token。你有一个冻结的 7B LLM，期望 token 嵌入的维度为 4096。明显的桥接器——一个从 1408 到 4096 的线性层——是有效的，但将所有 256 个 patch token 输入 LLM 的上下文每张图像就要消耗 256 个额外的 token。在一个 32 张图像的批次中，仅视觉模态就消耗了 8192 个 token。

BLIP-2 的问题是：你能将 256 token 的图像表示压缩到更少的 token（比如 32 个）中，同时保留足够的信息供 LLM 进行标题生成、问题回答和图像推理吗？而且你能在不触及冻结骨干网络的情况下训练这个桥接器，将训练成本限制在仅桥接器的参数上吗？

答案是：Q-Former。32 个可学习的"查询"向量，交叉关注 ViT 的 patch token，产生一个 32 token 的视觉摘要供 LLM 消费。总参数 1.88 亿。在接触 LLM 之前，用对比、匹配和生成目标进行训练。

## 概念

### 可学习查询

Q-Former 的核心技巧：不让 LLM 的文本 token 关注图像 patch，而是引入一组新的 32 个可学习查询向量 `Q`，让*它们*关注图像 patch。查询是模型的参数——它们在训练期间学习，并且相同的 32 个查询用于每张图像。

在交叉注意力之后，每个查询持有图像的一个压缩摘要——"描述主要物体"、"描述背景"、"计算物体数量"等。查询并非严格按语义标签特化；它们学习任何使下游损失下降的编码方式。

### 架构

Q-Former 是一个小型 transformer（12 层，约 1 亿参数），有两条路径：

1. 查询路径：32 个查询向量通过自注意力（在它们自己之间），然后交叉注意力作用于冻结 ViT 的 patch token，然后是 FFN。
2. 文本路径：一个类似 BERT 的文本编码器与查询路径共享自注意力和 FFN 权重。文本路径禁用交叉注意力。

在训练时，两条路径都运行。查询和文本通过共享的自注意力交互，这意味着对于需要它的任务（ITM、ITG），查询可以以文本为条件。在 VLM 传递的推理时，只有查询通过，产生 32 个视觉 token。

### 两阶段训练

BLIP-2 分两个阶段进行预训练：

阶段 1：表示学习（无 LLM）。三种损失：
- ITC（图像-文本对比）：CLIP 风格的对比，在池化查询 token 和文本 CLS token 之间。
- ITM（图像-文本匹配）：二元分类器——这对（图像、文本）是否匹配？经过难负例挖掘。
- ITG（图像引导的文本生成）：文本上的因果 LM 头，以查询为条件。迫使查询编码可生成文本的内容。

只有 Q-Former 训练。ViT 是冻结的。没有 LLM 参与。

阶段 2：生成式学习。附加一个冻结的 LLM（OPT-2.7B 或 Flan-T5-XL 等）。通过一个小型线性层将 32 个查询输出投影到 LLM 的嵌入维度。将它们前置到文本提示之前。只在线性投影和 Q-Former 上训练 LM 损失，在整个连接后的提示 + 图像 + 标题序列上。

阶段 2 之后，Q-Former + 投影就是完整的视觉适配器。推理时：图像 -> ViT -> Q-Former -> 线性投影 -> 前置到文本输入 -> 冻结 LLM 产生输出。

### 参数经济性

BLIP-2 使用了 ViT-g/14（1.1B，冻结）+ OPT-6.7B（6.7B，冻结）+ Q-Former（1.88 亿，可训练）= 总计 8B，1.88 亿可训练。Q-Former 本身约为完整栈参数的 2.4%。训练成本反映了这一点：在几个 A100 上花费数天，而非端到端的数周。

质量：BLIP-2 在零样本 VQA 上匹配或超越 Flamingo-80B，同时体积小 50 倍。桥接器有效。

### InstructBLIP 和指令感知的 Q-Former

InstructBLIP（2023 年）用一个额外的输入扩展了 Q-Former：指令文本本身。在交叉注意力时，查询现在既能访问图像 patch 也能访问指令。查询可以按指令特化（"计算汽车数量"、"描述氛围"），而不是学习一个单一的固定摘要。在保留任务上的基准测试中取得了提升。

### MiniGPT-4 和仅投影器方法

MiniGPT-4 保留了 Q-Former，但只训练输出线性投影，冻结其他一切。成本低，但代价是质量——查询是 BLIP-2 的，不是你自己的。适合快速迭代，但不是最佳架构。

### 为什么 LLaVA 选择了更简单的方案

LLaVA（2023 年，课程 12.05）用简单的 2 层 MLP 替换了 Q-Former，将每个 ViT patch token 投影到 LLM 空间——24x24 网格每张图像 576 个 token，全部输入 LLM。压缩效果更差，但让 LLM 可以关注原始 patch。当时这很有争议；到 2023 年底它占据主导地位，因为视觉指令数据（LLaVA-Instruct-150k）证明了 MLP 可以训练以保留足够的信号。权衡：LLaVA 的上下文填充更快，但它自然地扩展到多图像和视频。

到 2026 年，该领域分裂了：Q-Former 在 token 预算很重要时存活（长视频、多图像）；MLP 投影器在每 token 原始质量优先时占主导。

### 门控交叉注意力：Flamingo，祖先

Flamingo（课程 12.04）先于 BLIP-2，使用了相同的交叉注意力思想，但在每个冻结的 LLM 层，而不是作为一个单一的桥接器。BLIP-2 表明你可以压缩到仅输入层并且仍然有效。Gemini 和 Idefics 结合了两种方式：交错输入 token 加上可选的门控交叉注意力用于上下文中的少样本学习。

### 2026 年的后代

- Q-Former：BLIP-2、InstructBLIP、MiniGPT-4 以及大多数视频-语言模型（出于 token 预算原因）。
- Perceiver 重采样器：Flamingo 的变体（课程 12.04）；Idefics 家族、Eagle、OmniMAE。
- MLP 投影器：LLaVA、LLaVA-NeXT、LLaVA-OneVision、Cambrian-1。
- 注意力池化：VILA、PaliGemma。

所有四种都有效。决定性问题是：你是在 token 预算上受限，还是在每 token 质量上受限？

## 使用它

`code/main.py` 构建了一个标准库 Q-Former 风格的交叉注意力：

1. 模拟 256 个图像 patch token（维度 128）。
2. 实例化 32 个可学习查询（维度 128）。
3. 运行缩放点积交叉注意力（Q 来自查询，K/V 来自 patches）。
4. 通过线性层投影到 LLM 维度（512）。
5. 输出 32 个 LLM 就绪的视觉 token。

所有数学计算都是纯 Python（向量上的嵌套循环）。玩具级但形状正确。注意力权重矩阵会被打印出来，以便你看到每个查询从哪些 patch 中提取了信息。

## 练习

1. 在 PyTorch 中实现交叉注意力块。验证使用 32 个查询和 256 个键/值时，注意力权重矩阵是 32 x 256，且 softmax 后每行和为 1。

2. 在 BLIP-2 阶段 1 中，Q-Former 同时运行三个损失：ITC、ITM、ITG。用伪代码写出每个损失的前向签名。哪个需要文本编码器路径是激活的？

3. 比较参数量：Q-Former（12 层，768 隐藏维度）vs 2 层 MLP 投影器（1408 -> 4096，两层）。在什么样的 LLM 规模下，1.88 亿参数的 Q-Former 成本才能在训练效率上回本？

4. 阅读 BLIP-2 论文（arXiv:2301.12597）第 3.2 节关于 Q-Former 初始化方式的内容。解释为什么从 BERT-base（而非随机）初始化能加速收敛。

5. 对于一个 10 分钟、1 FPS 采样到 60 帧的视频，计算每帧的 token 成本：（Q-Former -> 每帧 32 token）vs （MLP 投影器 -> 每帧 576 token）。哪个能适应 128k token 的 LLM 上下文窗口？

## 关键术语

| 术语 | 人们说的 | 实际含义 |
|------|---------|---------|
| Q-Former | "查询 transformer" | 小型 transformer，带有 32 个可学习查询向量，交叉关注冻结的 ViT 特征 |
| 可学习查询（Learnable queries） | "视觉的软提示" | 一组固定参数，作为交叉注意力的查询端；按模型学习，跨所有输入共享 |
| 交叉注意力（Cross-attention） | "Q 来自这里，K/V 来自那里" | 查询、键和值来自不同来源的注意力；查询如何从 ViT patches 中提取信息 |
| ITC | "图像-文本对比" | CLIP 风格损失，应用于 Q-Former 池化查询 vs 文本 CLS |
| ITM | "图像-文本匹配" | 在难负例挖掘对上的二元分类器；迫使查询区分细粒度不匹配 |
| ITG | "图像引导的文本生成" | 因果 LM 损失，其中文本以查询为条件生成；迫使查询编码可文本解码的内容 |
| 两阶段预训练 | "表示学习然后生成式学习" | 阶段 1 单独训练 Q-Former（ITC/ITM/ITG）；阶段 2 附加冻结的 LLM，仅训练投影 + Q-Former |
| 冻结骨干（Frozen backbone） | "不微调" | 视觉编码器和 LLM 权重固定；仅桥接器训练 |
| 投影头（Projection head） | "线性到 LLM 维度" | 将 Q-Former 输出映射到 LLM 嵌入维度的最终线性层 |
| Perceiver 重采样器 | "Flamingo 的版本" | 类似的可学习查询交叉注意力，Flamingo 在每一层使用，而非作为单一桥接器 |
