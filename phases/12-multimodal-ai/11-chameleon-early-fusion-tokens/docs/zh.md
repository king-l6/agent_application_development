# Chameleon 与 Early-Fusion 纯令牌多模态模型

> 我们目前看到的所有 VLM 都是将图像和文本分开处理的。视觉令牌来自视觉编码器，经过投影器，然后在 LLM 内部与文本汇合。视觉词汇表和文本词汇表从未重叠。Chameleon（Meta，2024 年 5 月）提出了一个问题：如果它们重叠会怎样？训练一个 VQ-VAE，将图像转换为来自共享词汇表的离散令牌序列。每个多模态文档现在都是一个序列——文本令牌和图像令牌交错排列，使用单一的自回归损失。副作用：模型可以在单次推理调用中生成混合模态的输出——交替生成文本和图像令牌。本节课程将研读这篇 early-fusion 论文，并端到端地构建一个玩具版本。

**类型：** 构建
**语言：** Python（标准库，VQ-VAE 分词器 + 交错解码器）
**前置要求：** 阶段 12 · 05，阶段 8（生成式 AI）
**时间：** ~180 分钟

## 学习目标

- 解释为什么共享词汇表 + 单一损失会改变模型的能力。
- 描述 VQ-VAE 如何将图像分词为与 Transformer 的下一个令牌预测目标兼容的离散序列。
- 列举 Chameleon 的训练稳定性技巧：QK-Norm、Dropout 放置、LayerNorm 顺序。
- 比较 Chameleon 与 BLIP-2 的 Q-Former 方法，并描述各自适用的场景。

## 问题

基于适配器的 VLM（LLaVA、BLIP-2、Qwen-VL）将文本和图像视为两种不同的东西。文本令牌经过 `embed(text_token)`；图像经过 `visual_encoder(image) → projector → ... pseudo_tokens`。模型有两个输入路径，在中途合并。

三个后果：

1. LLM 只能消费图像，不能生成图像。输出仅为文本。
2. 混合模态文档（如文章中的段落和图像交错排列）处理起来很笨拙——要么在模型外部解析多模态输入，要么串联多次生成。
3. 分布不匹配。视觉令牌和文本令牌位于隐藏空间的不同区域，会产生微妙的对齐问题。

Chameleon 拒绝了这一前提：图像只是来自共享词汇表的离散令牌序列。在交错文档上训练模型，一个损失，一个自回归解码器，你就可以免费获得混合模态生成能力。

## 概念

### VQ-VAE 作为图像分词器

分词器是一个向量量化变分自编码器。其架构如下：

- 编码器：CNN + ViT，将图像映射为空间特征图，例如 32x32 个维度为 256 的特征。
- 码本：一个包含 K 个向量的学习词汇表（Chameleon 使用 8192），维度也是 256。
- 量化：对于每个空间特征，通过 L2 距离查找最近的码本条目。用整数索引替换连续特征。
- 解码器：CNN，将量化后的特征还原为像素。

训练：VAE 重构损失 + 承诺损失 + 码本损失。码本索引构成了图像的离散字母表。

对于 Chameleon：一张图像变成 32*32 = 1024 个令牌，来自一个大小为 8192 的词汇表。与文本令牌（来自 LLM 的 BPE 词汇表，约 32000）拼接。最终词汇表大小：40192。Transformer 看到一个序列，一个损失。

### 共享词汇表

Chameleon 的词汇表结合了文本令牌、图像令牌和模态分隔符。每个令牌都有一个唯一的 ID。输入嵌入层将每个 ID 映射为 D 维隐藏向量。输出投影将隐藏向量映射回词汇表 logits。Softmax 选择下一个令牌，无论其模态。

分隔符很重要：`<image>` 和 `</image>` 标签将图像令牌序列括起来。在生成时，如果模型生成了 `<image>`，下游软件就知道接下来的 1024 个令牌是 VQ 索引，需要发送给解码器进行像素渲染。

### 混合模态生成

推理是在共享词汇表上的下一个令牌预测。示例提示："画一只猫并描述它。" Chameleon 生成：

```
<image> 4821 1029 2891 ... (1024 个图像令牌) </image>
这只猫是橙色的，坐在窗台上...
```

模型自主决定顺序——它可能先生成图像再生成文本，或者先文本后图像，或者交错生成。相同的解码器，相同的损失。

与适配器 VLM 相比，后者的生成仅限于文本。Chameleon 重新开启了模型输出模态的问题。

### 训练稳定性——QK-Norm、Dropout、LayerNorm 顺序

Early-fusion 训练在大规模下不稳定。Chameleon 的论文记录了三个技巧：

- **QK-Norm**：在注意力机制内部，点积之前，对查询和键的投影应用 LayerNorm。防止深度下 logit 幅值爆炸。多个 2024 年之后的大型模型都使用了这一技巧。
- **Dropout 放置**：在每个残差加法之后都放置 Dropout，而不仅仅是在注意力和 MLP 之后。当图像令牌的梯度可能占主导时，需要更多的正则化。
- **LayerNorm 顺序**：残差分支上的 Pre-LN（标准做法），加上最后一个块的跳跃连接上的额外 LN。稳定了最后一层的梯度流。

如果没有这些技巧，34B 参数的 Chameleon 训练在多个检查点处会发散。有了它们，训练就能收敛。训练方案与架构本身同等重要。

### 分词器的重构上限

VQ-VAE 是有损的。对于 8192 个码本条目和每张 512x512 图像 1024 个令牌的情况，重构 PSNR 大约在 26-28 dB 左右。这对于可识别的图像生成是足够的，但明显差于连续空间扩散（Stable Diffusion 3 达到 32+ dB）。

分词器是瓶颈。更好的分词器（MAGVIT-v2、IBQ、SBER-MoVQGAN）可以提升上限。Emu3（第 12.12 课）仅通过更好的分词器就实现了 SDXL 级别的生成质量。

### Chameleon 与 BLIP-2 / LLaVA 的对比

Chameleon（early fusion，共享词汇表）：
- 一个损失，一个解码器。
- 可生成混合模态输出。
- 分词器是质量上限。
- 代价高：每次生成图像都需要在推理路径上运行 VQ-VAE 解码器。

BLIP-2 / LLaVA（late fusion，分离式塔结构）：
- 仅图像输入，文本输出。
- 复用预训练的 LLM。
- 理解任务没有分词器瓶颈。
- 代价低：单次前向传播。

根据任务选择。如果你需要图像生成，选择 Chameleon 系列。如果你只需要理解能力，适配器 VLM 更简单且能复用更多预训练计算。

### Fuyu 和 AnyGPT

Fuyu（Adept，2023）是一种相关的方法：跳过独立的视觉编码器，将原始图像块通过 LLM 的输入投影直接送入，就像是令牌一样，不需要分词器。比 Chameleon 更简单，但失去了共享词汇表的输出生成能力。

AnyGPT（Zhan 等人，2024）将 Chameleon 扩展到四种模态：文本、图像、语音、音乐。每种模态都使用相同的 VQ-VAE 技巧，共享一个 Transformer。任意到任意的生成。在第 12.16 课中有更详细的介绍。

## 使用它

`code/main.py` 构建了一个玩具版的端到端 early-fusion 模型：

- 一个微型的 VQ-VAE 风格量化器，将 8x8 的图像块映射为码本索引（K=16）。
- 共享词汇表包含（文本 ID 0..31）+（图像 ID 32..47）+（分隔符 48, 49）。
- 一个玩具自回归解码器（bigram 表），在合成的描述 + 图像令牌序列上训练。
- 采样循环，在给定提示后交替生成文本 + 图像令牌。

代码有意将 Transformer 保持得很小（bigram 级别），这样你可以端到端地追踪信号流。

## 练习

1. Chameleon 使用 K=8192 个码本条目，每张 512x512 图像使用 1024 个令牌。估算相对于 24 位 RGB 图像的压缩比。它是有损的吗？损失有多大？

2. 一张 4K 图像（3840x2160）在相同的 VQ-VAE 密度下会产生多少个图像令牌？Chameleon 风格的模型能在一次推理调用中生成 4K 图像吗？什么首先会成为瓶颈——上下文长度、分词器质量还是 KV 缓存？

3. 用纯 Python 实现 QK-Norm。给定一个 64 维的查询和键，展示 LayerNorm 前后的点积结果。为什么在深度网络中幅值控制很重要？

4. 阅读 Chameleon 论文第 2.3 节关于训练稳定性的内容。描述论文在 34B 规模下没有 QK-Norm 时观察到的确切失败模式。"范数爆炸"的特征是什么？

5. 扩展玩具解码器，使其能够根据纯文本提示生成混合模态响应。在训练数据分布为 60% 文本优先 / 40% 图像优先的情况下，测量模型选择图像优先与文本优先的频率。

## 关键术语

| 术语 | 人们说的意思 | 实际含义 |
|------|-------------|---------|
| Early fusion | "统一令牌" | 图像从一开始就被转换为与 Transformer 共享词汇表的离散令牌 |
| VQ-VAE | "图像分词器" | CNN + ViT + 码本，将图像映射为 Transformer 可以预测的整数索引 |
| 共享词汇表 | "一个字典" | 单一的令牌 ID 空间，覆盖文本 + 图像 + 模态分隔符 |
| QK-Norm | "注意力稳定器" | 在查询和键的点积之前对其应用 LayerNorm，防止范数爆炸 |
| 混合模态生成 | "文本 + 图像输出" | 推理时自主生成交错文本和图像令牌的能力 |
| 码本大小 | "K 个条目" | VQ-VAE 可以量化到的离散向量数量；在压缩和保真度之间权衡 |
| 分词器上限 | "重构限制" | 解码 VQ 令牌所能达到的最佳 PSNR；限制了模型的图像质量 |
