# LLM 的差分隐私

> DP-SGD 仍然是标准——注入噪声的梯度更新提供正式的 (ε, δ) 保证。计算、内存和效用方面的开销很大；参数高效的差分隐私微调（LoRA + DP-SGD）是常见的 2025 配置（ACM 2025）。两组证据存在张力：基于金丝雀的成员推断 (Duan 等人 2024) 报告对语言模型成功有限；训练数据提取 (Carlini 等人 2021; Nasr 等人 2025) 恢复了大量逐字记忆。解决方案 (arXiv:2503.06808, 2025 年 3 月)：差距在于测量的内容——插入的金丝雀 vs "最可提取的"数据。新的金丝雀设计实现了无需影子模型的基于损失的 MIA，并首次对在真实数据上以现实 DP 保证训练的 LLM 进行了非平凡 DP 审计。替代方案：PMixED (arXiv:2403.15638)——通过混合专家在下一个词元分布上进行推理时私有预测；DP 合成数据生成 (Google Research 2024)。新兴攻击：通过 LLM 反馈的差分隐私逆转——置信度分数泄露。

## 学习目标

- 定义 (ε, δ)-差分隐私并阐述 DP-SGD 的步骤。
- 解释 2024-2025 年的张力：金丝雀 MIA 与训练数据提取给出不同的图景。
- 描述 PMixED 以及为什么推理时私有预测是 DP 训练的替代方案。
- 描述通过 LLM 反馈的差分隐私逆转攻击。

## 问题

LLM 会记忆。Carlini 等人 2021 表明生产型语言模型会根据要求逐字复现训练文本。DP 是正式防御：训练使得输出在证明上不依赖于任何单个训练样本。2024-2025 年的证据表明 DP-SGD 是必要的，但部署的 ε 值可能与威胁模型不匹配。

## 概念

### (ε, δ)-差分隐私

如果对于任何差异一个样本的两个数据集和任何事件 S，随机算法 M 是 (ε, δ)-DP：
P(M(D) ∈ S) <= e^ε * P(M(D') ∈ S) + δ。

解释：输出分布足够接近（由 ε 参数化），以至于任何个体贡献不能被可靠推断，除了概率 δ。

### DP-SGD

Abadi 等人 2016。标准步骤：
1. 采样一个小批次。
2. 计算每个样本的梯度。
3. 将每个样本的梯度裁剪到阈值 C。
4. 求和裁剪后的梯度并添加标准差为 σ * C 的高斯噪声。
5. 使用噪声和更新参数。

隐私成本由记帐员（矩记帐员、Rényi DP 记帐员）追踪。LLM 文献中报告的 ε 值因威胁模型、数据敏感性和效用目标而有很大差异；没有普遍"安全"的默认 ε。已发表的示例大致涵盖 ε ≈ 1–10 的一些 LLM 训练设置，但这些是指示性的——不是推荐的默认值。较低的 ε 通常需要更多噪声并可能导致效用损失增加。

### LoRA + DP-SGD

前沿模型的完整 DP-SGD 是禁止性的。LoRA (Hu 等人 2022) 将梯度更新限制在一个小适配器上，减少了每个样本的梯度存储。LoRA + DP-SGD 是常见的 2025 配置。DP 保证适用于适配器；基础模型保持固定。

### 2024-2025 年的张力

两行证据：

- **金丝雀 MIA (Duan 等人 2024)。** 将唯一的金丝雀插入训练数据，测量成员推断攻击者是否能识别它们。报告对语言模型成功有限。表明 MIA 是困难的。
- **训练数据提取 (Carlini 2021, Nasr 等人 2025)。** 用前缀提示模型；测量是否能恢复训练中的逐字文本。报告大量记忆。表明 MIA 在相关意义上很容易。

2025 年 3 月的解决方案 (arXiv:2503.06808)：两者测量不同事物。MIA 在插入的金丝雀上问"样本 e 在 D 中吗？"。提取问"我能恢复 D 的什么？""最可提取的"样本才是隐私相关的；金丝雀低估了这一点，因为它们不是为了可提取性而优化的。

新的金丝雀设计。无需影子模型的基于损失的 MIA。首次对在真实数据上以现实 DP 保证训练的 LLM 进行非平凡 DP 审计。

### DP 训练的替代方案

- **PMixED (arXiv:2403.15638)。** 推理时的私有预测。基于下一个词元分布的混合专家；每个专家看到训练数据的一个分片；聚合加入噪声以实现 DP。完全避免 DP 训练。
- **DP 合成数据生成 (Google Research 2024)。** 使用 DP-SGD 进行 LoRA 微调，采样合成数据，在合成数据上训练下游分类器。

两者都以不同的威胁模型为代价，绕过了完整 DP 训练的效用成本。

### 通过 LLM 反馈的差分隐私逆转

2025 年新兴攻击。使用 DP 训练模型的置信度分数作为预言机来重新识别个体。即使输出不泄露，置信度分布也可以。

防御：不暴露置信度，或在暴露前进行截断/量化。这是 (ε, δ)-DP 训练之外的额外要求。

## 使用它

`code/main.py` 在玩具二分类数据集上模拟 DP-SGD。你可以扫描噪声倍数 σ 和裁剪范数 C，并追踪 (ε, δ) 预算和准确性成本。"金丝雀攻击"插入一个独特的训练样本，并测量对数损失测试在 DP 前后能否检测到它。

本课产出 `outputs/skill-dp-audit.md`。给定语言模型部署上的 DP 声明，它审计：(ε, δ) 值、使用的记帐员、MIA 评估协议以及是否评估了置信度暴露向量。

## 练习

1. 运行 `code/main.py`。扫描 σ ∈ {0.5, 1.0, 2.0} 并报告 (ε, δ)-准确性权衡。确定效用崩溃的点。

2. 实现金丝雀插入和对数损失测试。在 σ = 1.0 时测量 DP-SGD 前后的检测率。

3. 阅读 Nasr 等人 2025 关于训练数据提取的论文。为什么提取成功不会在中等 ε 下崩溃？这对作为评估的 MIA 意味着什么？

4. 设计一个完全在推理时运行的 PMixED (arXiv:2403.15638) 部署。PMixED 解决的、DP-SGD 没有解决的威胁模型是什么？

5. 勾勒 DP 通过 LLM 反馈逆转攻击。设计一个限制置信度分数泄露的对策并估计其部署成本。

## 关键术语

| 术语 | 人们说的 | 实际含义 |
|------|----------|----------|
| DP | "(ε, δ)-差分隐私" | 正式隐私：邻近数据集变化下的输出分布接近 |
| DP-SGD | "注入噪声的 SGD" | 梯度裁剪 + 高斯噪声添加；标准 DP 训练 |
| LoRA + DP-SGD | "高效的私有微调" | 低秩适配器上的 DP-SGD；标准 2025 配置 |
| MIA | "成员推断" | 确定样本是否在训练数据中的攻击 |
| 金丝雀 | "插入的水印样本" | 用于测量 DP 泄露的唯一训练样本 |
| PMixED | "私有推理混合" | 通过下一个词元分布上的混合专家实现推理时 DP |
| DP 逆转 | "置信度泄露攻击" | 使用模型置信度作为预言机进行重新识别的攻击 |
