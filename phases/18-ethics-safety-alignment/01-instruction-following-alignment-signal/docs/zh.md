# 指令遵循作为对齐信号

> 后来对 RLHF 的每一个批评都是针对这个流水线的。在你研究优化压力如何扭曲代理之前，你必须先看到代理是什么。InstructGPT（Ouyang 等人，2022）定义了参考架构：在指令-响应对上进行监督微调，基于成对偏好排名训练奖励模型，以及使用 KL 惩罚相对于 SFT 策略在奖励模型上进行 PPO 优化。一个 1.3B 的 InstructGPT 优于 175B 的 GPT-3。这一个结果就是为什么每个前沿实验室在 2026 年仍在发布 RLHF 形状的后训练流水线的原因。

**类型：** 学习
**语言：** Python（标准库，玩具三阶段流水线）
**前置要求：** 第 10 阶段 · 06（SFT），第 10 阶段 · 07（RLHF），第 10 阶段 · 08（DPO）
**时间：** 约 45 分钟

## 学习目标

- 说出 InstructGPT 流水线的三个阶段以及每个阶段使用的损失函数。
- 解释为什么一个 1.3B 的指令微调模型在人类偏好评估上击败了原始的 175B GPT-3。
- 说明第三阶段中 KL 惩罚保护的是什么，以及为什么移除它会导致模式寻求行为。
- 描述对齐税以及 Ouyang 等人用来缓解它的 PPO-ptx 方法。

## 问题

预训练语言模型完成文本。它们不回答问题。让 GPT-3 "写一个反转列表的 Python 函数"，你经常得到另一个提示，因为大部分训练分布是网页文本，而网页文本会继续生成更多网页文本。模型正在做它的工作 — 问题在于这个工作本身是错误的。

每个严肃实验室用来解决这个问题的代理是人类偏好。两个补全被发送给评估者；评估者选择更好的一个；奖励模型学习评估者的偏好。然后一个 RL 循环将策略移向奖励模型评分高的输出。这就是 InstructGPT 的全部论题，用三句话概括。论文的其余部分是工程实现。

## 概念

### 阶段 1：监督微调（SFT）

收集提示-响应对，其中响应是善意的人类会写的内容。Ouyang 等人使用了来自标注者和 OpenAI API 的 13k 个提示。使用标准交叉熵损失在此数据上微调基础模型。

SFT 给你的：模型现在回答问题，而不是续写它们。它没有给你的：关于当多个答案都合理时，评估者偏好哪个答案的任何信号。

### 阶段 2：奖励模型（RM）

对于每个提示，从 SFT 模型中采样 K 个补全。标注者对它们进行排序。训练一个奖励模型来评分任何提示-响应对，使得对于 `y_w` 优于 `y_l` 的对：

```
L_RM = -log sigmoid(r(x, y_w) - r(x, y_l))
```

这是 Bradley-Terry 成对偏好损失。RM 通常从 SFT 模型初始化，将 LM 头部替换为标量头部。

奖励模型很小：6B 足以用于 175B 的 InstructGPT。它们也很脆弱 — 论文的第 5 节主要讨论了在小规模下出现的奖励破解行为。

### 阶段 3：带 KL 惩罚的 PPO

定义目标函数：

```
J(pi) = E_{x~D, y~pi(.|x)} [ r(x, y) ] - beta * KL(pi(.|x) || pi_SFT(.|x))
```

使用 PPO 最大化。KL 项防止 `pi` 偏离 SFT 策略太远。没有它，优化器会找到对抗性样本 — 那些在 RM 下得分很高的字符串，不是因为 RM 从未见过它们，而是因为人类实际上并不偏好它们。

KL 系数 `beta` 是 RLHF 中最重要的单一超参数。太低：奖励破解。太高：相比 SFT 没有改进。

### 对齐税

RLHF 之后，模型被人类偏好，但在标准基准测试（SQuAD、HellaSwag、DROP）上出现退化。Ouyang 等人称此为对齐税，并通过 PPO-ptx 修复它：将预训练梯度混合到 RL 目标中，使模型不会忘记它从未被奖励过的下游任务。

```
J_ptx(pi) = J(pi) + gamma * E_{x~D_pretrain} [ log pi(x) ]
```

PPO-ptx 成为了标准。Anthropic、DeepMind 和 Meta 都使用某种变体。

### 结果

一个 1.3B 的 InstructGPT（SFT + RM + PPO-ptx）在大约 70% 的情况下被标注者偏好于 175B 的基础 GPT-3。这个差距在生产流量的隐藏测试提示上更大。从这个数字可以读出两件事：

1. 对齐是与能力不同的轴。175B 模型有更多能力；1.3B 模型有更多对齐；标注者偏好对齐的那个。
2. 能力下限由基础模型设定。你不能通过 RLHF 让基础模型知道它从未见过的事实。

### 为什么这是第 18 阶段的参考点

后来的课程中的每一个批评 — 奖励破解（第 2 课）、DPO（第 3 课）、谄媚（第 4 课）、CAI（第 5 课）、休眠智能体（第 7 课）、对齐伪装（第 9 课） — 都是针对这个流水线的某些部分提出的。奖励破解攻击阶段 2。DPO 合并了阶段 2 和 3。CAI 替换了人类标注者。谄媚表明标注者是一个有偏见的信号。对齐伪装表明策略可以完全绕过阶段 3。不先把这条流水线记在脑子里，你就无法理解这些批评中的任何一个。

## 使用它

`code/main.py` 在玩具偏好数据上模拟了三个阶段。基础的"策略"是一个在动作 {A, B, C} 上有偏倚的硬币。阶段 1 SFT 在 200 个提示上模仿标注者的行为。阶段 2 从 500 个成对排名中拟合 Bradley-Terry 奖励模型。阶段 3 运行一个带 KL 惩罚到 SFT 策略的简化 PPO 更新。你可以看到奖励上升、KL 散度增长和策略漂移 — 而且你可以关闭 KL 项，观察奖励破解在 50 步更新内出现。

需要关注的内容：

- `beta = 0.1` 与 `beta = 0.0` 时的奖励轨迹。
- KL(pi || pi_SFT) 随训练步数的变化。
- 最终动作分布与标注者偏好的比较。

## 交付它

本课程生成 `outputs/skill-instructgpt-explainer.md`。给定一个 RLHF 流水线描述或论文摘要，它会识别三个阶段中的哪一个正在被修改，每个阶段使用了什么损失函数，以及是否存在 KL 惩罚或等效的正则化器。

## 练习

1. 运行 `code/main.py`。设置 `beta = 0.0` 并在 200 步 PPO 后报告动作分布。用一段话解释模式寻求行为。
2. 修改奖励模型，使其对动作 B 有 +0.5 的偏倚（模拟奖励 bug）。使用 `beta = 0.1` 运行 PPO。KL 惩罚是否阻止了策略利用该偏倚？在什么 `beta` 值下，利用变得可见？
3. 阅读 Ouyang 等人（arXiv:2203.02155）图 1。通过运行 PPO 1、5、20、100 步并测量对 SFT 模型的偏好来重现标注者偏好曲线。
4. 论文的第 4.3 节报告 1.3B InstructGPT 在大约 70% 的情况下击败 175B GPT-3。为什么在隐藏生产提示上的比率会比在标注者自己的提示上更高？
5. 将 PPO 损失替换为相同偏好数据上的 DPO（第 10 阶段 · 08）。比较最终策略漂移（相对于 SFT 的 KL）和最终奖励。在匹配奖励下，哪种方法漂移得更远？

## 关键术语

| 术语 | 人们说的意思 | 实际含义 |
|------|-----------------|------------------------|
| SFT | "指令微调" | 阶段 1：在提示-响应对上的交叉熵微调 |
| 奖励模型 | "RM" | 在（提示、响应）上的标量回归器，使用 Bradley-Terry 在成对标签上训练 |
| Bradley-Terry | "成对偏好损失" | -log sigmoid(r_w - r_l)；将成对排名简化为二元分类 |
| KL 惩罚 | "正则化器" | `beta * KL(pi || pi_SFT)` — 保持 RL 策略接近 SFT 锚点 |
| PPO-ptx | "带预训练混合的 PPO" | 将一部分预训练对数似然加入到 PPO 目标中，以抵消对齐税 |
| 对齐税 | "RLHF 回归" | RLHF 后在 RLHF 未针对的标准基准上的退化 |
| 标注者偏好 | "地面真相" | 人类排名的样本；RM 是它的统计代理，而非"人类价值观" |
