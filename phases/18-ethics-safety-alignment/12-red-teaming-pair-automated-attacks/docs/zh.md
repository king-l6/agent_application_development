# 红队：PAIR 与自动化攻击

> Chao、Robey、Dobriban、Hassani、Pappas、Wong（NeurIPS 2023，arXiv:2310.08419）。PAIR——提示自动化迭代精炼——是经典的黑盒自动化越狱方法。一个带有红队系统提示词的攻击者 LLM 迭代式地为目标 LLM 提出越狱方案，将尝试和响应的累积作为上下文反馈存入自身聊天历史中。PAIR 通常在 20 次查询内成功，比 GCG（Zou 等人的令牌级梯度搜索）效率高出数个数量级，且无需白盒访问。PAIR 现在与 GCG、AutoDAN、TAP 和说服性对抗提示一起，成为 JailbreakBench（arXiv:2404.01318）和 HarmBench 的标准基线。

**类型：** 构建
**语言：** Python（标准库，针对玩具目标的模拟 PAIR 循环）
**前置知识：** 阶段 18 · 01（指令遵循），阶段 14（智能体工程）
**时间：** 约 75 分钟

## 学习目标

- 描述 PAIR 算法：攻击者系统提示词、迭代精炼、上下文反馈。
- 解释为什么在目标为黑盒时 PAIR 严格比 GCG 更高效。
- 说出其他四种自动化攻击基线（GCG、AutoDAN、TAP、PAP）的名称，并说明每种的一个区别性特征。
- 描述 JailbreakBench 和 HarmBench 的评估协议，以及"攻击成功率"在每个协议中的含义。

## 问题

红队过去是手动活动。少数专家测试者构造对抗性提示词并追踪哪些有效。这种方法不可扩展：攻击成功率需要统计样本，而目标随着每次模型发布而变化。PAIR 将红队操作化为以黑盒目标为对象的优化问题。

## 概念

### PAIR 算法

输入：
- 目标 LLM T（我们正在攻击的模型）。
- 裁判 LLM J（为响应是否属于越狱评分）。
- 攻击者 LLM A（红队优化器）。
- 目标字符串 G："响应 [有害指令]。"
- 预算 K（通常为 20 次查询）。

循环，k 从 1 到 K：
1. 向 A 提供目标 G 和迄今的（提示词，响应）历史。
2. A 发出新的提示词 p_k。
3. 将 p_k 提交给 T；接收响应 r_k。
4. J 对 (p_k, r_k) 在目标上的表现评分。
5. 如果分数 >= 阈值，停止——找到越狱。
6. 否则，将 (p_k, r_k) 追加到 A 的历史中；继续。

实证结果（NeurIPS 2023）：对 GPT-3.5-turbo、Llama-2-7B-chat 的攻击成功率超过 50%；成功所需的平均查询数在 10-20 之间。

### 为什么 PAIR 高效

GCG（Zou 等人 2023）通过梯度搜索对抗性令牌后缀；它需要白盒模型访问并产生不可读的后缀。PAIR 是黑盒的，产生可在模型之间迁移的自然语言攻击。PAIR 的上下文反馈让攻击者每次被拒绝都能学习；GCG 没有等效机制（每次令牌更新都必须重新发现先前的进展）。

### 相关自动化攻击

- **GCG（Zou 等人 2023，arXiv:2307.15043）。** 对抗性后缀的令牌级梯度搜索。白盒，可迁移，产生不可读字符串。
- **AutoDAN（Liu 等人 2023）。** 在层次化目标引导下的提示词进化搜索。
- **TAP（Mehrotra 等人 2024）。** 攻击树与剪枝——分支多个 PAIR 风格的展开。
- **PAP（Zeng 等人 2024）。** 说服性对抗提示——将人类说服技巧编码为提示词模板。

### JailbreakBench 和 HarmBench

两者（2024）标准化了评估：

- JailbreakBench（arXiv:2404.01318）。跨越 10 个 OpenAI 政策类别的 100 个有害行为。攻击成功率（ASR）作为主要指标。需要裁判（GPT-4-turbo、Llama Guard 或 StrongREJECT）。
- HarmBench（Mazeika 等人 2024）。跨越 7 个类别的 510 个行为，包含语义和功能危害测试。比较 18 种攻击对 33 个模型的表现。

ASR 通常在固定查询预算下报告。比较攻击需要匹配预算；在 200 次查询时 90% 的 ASR 与在 20 次查询时 85% 的 ASR 不可比。

### 对 2026 年部署的重要性

每个前沿实验室现在在发布前都会对生产模型运行 PAIR 和 TAP。ASR 轨迹出现在模型卡（第 26 课）和安全论证附录（第 18 课）中。这种攻击并不陌生——它是标准基础设施。

### 在阶段 18 中的位置

第 12 课是自动化攻击基础。第 13 课（多射越狱）是互补的长度利用。第 14 课（ASCII 艺术/视觉）是编码攻击。第 15 课（间接提示注入）是 2026 年的生产攻击面。第 16 课涵盖防御工具对应物（Llama Guard、Garak、PyRIT）。

## 使用它

`code/main.py` 构建了一个玩具 PAIR 循环。目标是一个模拟分类器，拒绝"明显"的有害提示词（关键词过滤）。攻击者是一个基于规则的精炼器，尝试 paraphrasing、角色扮演框架和编码。裁判对响应评分。你可以观察攻击者在约 5-15 次迭代内突破关键词过滤器，但在语义过滤器上失败。

## 关键术语

| 术语 | 人们说 | 实际含义 |
|------|--------|----------|
| PAIR | "自动化越狱" | 提示自动化迭代精炼；攻击者 LLM + 裁判 LLM 循环 |
| GCG | "梯度越狱" | 白盒令牌级对抗性后缀的梯度搜索 |
| 攻击成功率（ASR） | "k 次查询时的越狱百分比" | 主要指标；必须与查询预算和裁判身份一起报告 |
| 裁判 LLM | "评分器" | 对响应是否满足有害目标进行评分的 LLM |
| JailbreakBench | "评估基准" | 带标签类别的标准化有害行为集 |
| HarmBench | "更广泛的基准" | 510 个行为，功能+语义危害测试 |
| TAP | "攻击树" | PAIR 带分支+剪枝；更高计算量下更好的 ASR |
