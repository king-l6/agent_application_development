给定一个种子想法、一个狭窄领域和 30 美元计算预算，构建一个运行实验树搜索、编写可评审的 LaTeX 论文并生成可复现性包的 Agent。

构建计划：

1. 文献检索：Semantic Scholar Graph API + OpenAlex；在 FAISS 中缓存摘要；生成 1 页领域概览。
2. 树搜索：实现实验节点上的最佳优先扩展，使用 `expand(node) -> children`（每个子节点一个配置编辑）和 `score(node) = novelty*0.4 + quality*0.5 + budget*0.1`。
3. 每节点沙盒：每个实验运行 `docker run --network=none --memory=8g --cpus=2 --pids-limit=256 --read-only` 或等效 E2B 配置；确定性种子；资源上限强制执行。
4. 计划-执行-验证：验证步骤检查损失是否收敛、基线是否运行、消融是否隔离了声明。
5. 编写器：生成 LaTeX，编译为 PDF，将 PDF 送入 Claude Opus 4.7 视觉模式进行布局和声明-证据对齐评审，最多迭代 3 次。
6. 评审集成：五个评判（Opus 4.7、GPT-5.4、Gemini 3 Pro、DeepSeek R1、Qwen3-Max）使用 NeurIPS 评分标准评分（新颖性、严谨性、清晰度、可复现性、影响力）；均值 < 4.0 返回编写器。
7. 红队：集成对抗性任务（fork 炸弹、文件系统逃逸、LLM 编写的网络调用）。确认所有都被阻止。生成 `red_team.md`。
8. 可复现性包：paper.pdf + review.md + 树搜索追踪 JSON + 种子 + W&B 运行链接 + 沙盒配置 + 一行命令复现。

评估标准：

| 权重 | 标准 | 衡量方式 |
|:-:|---|---|
| 25 | 论文质量 | 与同一种子主题的已发表研讨会论文对比的盲审评分 |
| 20 | 实验严谨性 | 基线、种子、消融；每个声明由结果表中的单元格支持 |
| 20 | 成本和计算纪律 | 每篇论文 30 美元上限强制执行，Langfuse 追踪 |
| 20 | 安全性 | 沙盒红队通过；网络策略和终止开关通过记录的尝试验证 |
| 15 | 可复现性 | 一行命令使用相同种子复现论文 |

硬性拒绝：

- 在沙盒外运行的实验。这个综合项目的全部论点是执行是被隔离的。
- 不重新读取编译后 PDF 的编写器步骤（视觉评审是关键载荷）。
- 没有基线、种子或消融部分的论文。
- 仅作为事后警告而非硬上限强制执行的成本预算。

拒绝规则：

- 拒绝在没有明确人工覆盖的情况下发表评审均值低于 4.0/5 的论文。
- 拒绝在需要从沙盒内进行网络访问的种子想法上运行。改为添加单独的只读数据集卷。
- 拒绝复现尚未执行和记录红队测试的论文。

输出：一个包含树搜索引擎、沙盒策略、编写器/评审器循环、三个示例运行（带可复现性包）、红队报告、成本分类账 CSV 以及一份报告的仓库。报告命名你复现了哪些 Sakana v2 失败模式以及缓解措施是如何工作的。
