# 流水线并行与气泡分析

> 张量并行将矩阵乘法拆分到多个 rank 上。流水线并行将模型拆分到多个 rank 上，每个 rank 一个阶段。微批次流过流水线。开始和结束时的空闲时间是气泡；最小化它就是整个工艺。

**类型：** 构建
**语言：** Python
**前置知识：** 第 19 阶段 Track C 课程 42-49
**时间：** ~90 分钟

## 学习目标

- 将一个顺序模型拆分成 N 个阶段，并在 N 个 rank 上模拟前向流水线。
- 使用 GPipe 调度（先全部前向填充，然后反向）调度 M 个微批次通过流水线，并计算气泡比例。
- 将气泡与 Megatron-LM 和 PipeDream 中使用的交错 1F1B 调度进行比较。
- 论证阶段分配：每阶段计算量相等比每阶段参数数量相等更重要。

## 问题

一个 70B 参数的 fp16 模型仅参数就需要 140 GB。没有消费级 GPU 能装下。ZeRO-3 跨 rank 分片参数，但仍然需要每个 rank 为每个前向步骤 allgather 完整层，每层付出 log(N) 跳。流水线并行采用不同的路线：将模型切割成 N 个阶段，每个阶段放在一个 rank 上。第 1 层的前向在 rank 0 上完成并将激活张量交给 rank 1；rank 1 运行第 2 层并交给 rank 2；依此类推。反向以相反方向流动。内存线性下降，因为每个 rank 只持有一个阶段；计算是顺序的，这就是气泡问题。

气泡是在流水线开始时的空闲时间（等待第一个微批次到达最后一个阶段）和结束时（等待最后一个微批次反向流回）。对于 M 个微批次和 N 个阶段，每个阶段的气泡比例为 (N-1)/(M+N-1)。在 M=8、N=4 时是 27%。在 M=64、N=4 时是 4.5%。当每步的微批次很多时气泡缩小，这意味着每个微批次的批量较小，这就是驱动微批次设计的约束。

## 概念

```mermaid
flowchart LR
  R0[rank 0: stage 0 / layer 0] --> R1[rank 1: stage 1 / layer 1]
  R1 --> R2[rank 2: stage 2 / layer 2]
  R2 --> R3[rank 3: stage 3 / loss]
  R3 -.backward.-> R2
  R2 -.backward.-> R1
  R1 -.backward.-> R0
```

### GPipe 调度

在开始任何反向之前，用所有 M 个微批次前向填充流水线；然后反向排空。每个微批次的激活值必须保持到其反向，因此内存随 M 线性增长。前向需要 M+N-1 个周期，反向再需要 M+N-1 个周期。每个阶段的有用工作是 2M 个周期；每个阶段的气泡是 2(N-1) 个周期。当每个前向和反向各占一个单位时间时，气泡比例为 (N-1)/(M+N-1)。选择远大于 N 的 M 可以隐藏气泡。

### 1F1B 调度

交错执行：一旦微批次的前向到达最后一个阶段，立即开始其反向并让它流回。调度在每个阶段交替一个前向和一个反向。气泡仍然是 N-1，但激活内存受限于流水线深度，而不是微批次数量。生产流水线使用 1F1B（Megatron、PipeDream）。本课程首先实现 GPipe，因为它更简单，1F1B 作为练习。

### 为什么每阶段计算量相等很重要

如果阶段 0 需要 50 毫秒而阶段 1 需要 100 毫秒，每个周期都受阶段 1 的限制。其他阶段每个周期空闲 50 毫秒等待阶段 1 释放。参数数量相等是错误的轴：Transformer 的计算由每层的注意力加 MLP 主导，而嵌入层参数很多但计算很少。阶段分配应该平衡每阶段的 FLOPs，而不是每阶段的权重。

### 微批次与批次

流水线运行 M 个大小为 B 的微批次。有效批量大小为 M*B。流水线步骤结束时的梯度是组合的 M*B 个样本上的梯度。气泡比例取决于 M；优化器看到的是 M*B。调整 M 意味着在气泡（高 M 时更低）和每微批次内存（GPipe 的高 M 时更高激活内存）之间权衡。

`code/main.py` 实现：

## 生产环境中的模式

三种模式将流水线并行加固到可以交付的程度。

**激活检查点与流水线配对。** 当 M 个微批次在 GPipe 上飞行时，激活内存是 M 倍一个微批次的激活。激活检查点在反向时重新计算前向，用计算换内存；这种组合使流水线对长序列变得可行。

**阶段平衡是通过测量而不是假设的。** 生产团队运行一个性能分析过程，在实际硬件上测量每层实际计算（FLOPs 和挂钟时间），然后根据该测量进行分区。Megatron-LM 的 `--num-layers-per-stage` 标志接受一个列表，以允许在每层成本不同时具有不均匀的层数。

**发送-接收调度必须避免死锁。** 一个让每个阶段先发送后接收的流水线会在线路上死锁。标准修复是交错：偶数 rank 阶段先发送后接收，奇数 rank 阶段先接收后发送。本课程显式调度 rank，使模式可见。

## 使用

生产模式：

- **Megatron-LM。** 大规模流水线并行的参考。使用 1F1B 并支持张量 + 流水线 + 数据并行组合。
- **DeepSpeed Pipeline。** 与 ZeRO 集成；ZeRO-1 加流水线是最大开源模型的常见组合。
- **PyTorch Pipe。** PyTorch 原生流水线包装器，基于 `torch.distributed.pipeline.sync.Pipe`。

## 关键术语

| 术语 | 人们说的 | 实际含义 |
|------|----------------|------------------------|
| 流水线 | "沿深度的模型并行" | 每个 rank 一个阶段，激活值逐阶段流动 |
| 气泡 | "流水线空闲时间" | 开始和结束时的 (N-1) 步，某些阶段没有工作 |
| 微批次 | "批次的切片" | 一个前向/反向单元；气泡随 M 增长而缩小 |
| GPipe | "先填充后排空" | 在任何反向之前执行所有 M 个前向；高激活内存 |
| 1F1B | "交错调度" | 每个阶段一个前向一个反向；有界激活内存 |
