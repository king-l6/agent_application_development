# 综合项目 17 — 个人 AI 导师（自适应、多模态、带记忆）

> Khanmigo（Khan Academy）、Duolingo Max、Google LearnLM / Gemini for Education、Quizlet Q-Chat 和 Synthesis Tutor 在 2026 年都大规模提供了自适应多模态辅导。共同的形态是苏格拉底式策略（绝不直接给出答案）、每次交互后更新的学习者模型（贝叶斯知识追踪风格）、语音 + 文本 + 拍照数学输入、课程图谱检索、间隔重复调度，以及针对年龄适宜内容的严格安全过滤。这个综合项目是交付一个特定科目的导师（K-12 代数或 Python 入门），与 10 名学习者进行为期两周的效果研究，并通过对内容安全的审计。

**类型：** 综合项目
**语言：** Python（后端、学习者模型）、TypeScript（Web 应用）、SQL（通过 Postgres + Neo4j 的课程图谱）
**前置知识：** 阶段 5（NLP）、阶段 6（语音）、阶段 11（LLM 工程）、阶段 12（多模态）、阶段 14（智能体）、阶段 17（基础设施）、阶段 18（安全）
**涉及阶段：** P5 · P6 · P11 · P12 · P14 · P17 · P18
**时间：** 30 小时

## 问题

自适应辅导曾经是教育技术的研究利基。到 2026 年，它已成为消费产品。Khanmigo 部署在大多数美国学区。Duolingo Max 达到了数千万月活用户。Google 的 LearnLM / Gemini for Education 在 Google Classroom 中驱动辅导。Quizlet Q-Chat 与闪卡并肩工作。Synthesis Tutor 因面向好奇孩子的导师而爆红。共同要素：多模态输入（打字、说话、拍摄方程）、苏格拉底式教学法（先提问、后解释）、每次交互后更新的学习者模型，以及严格的年龄适宜安全措施。

你将为一个特定群体构建一个这样的系统。衡量标准是一项实际的效果研究：与 10 名学习者进行两周的前测和后测。语音循环必须感觉自然（综合项目 03 的子栈）。记忆必须尊重隐私。安全过滤器必须针对 K-12 通过 COPPA 感知的红队测试。

## 概念

四个组件。**导师策略**是苏格拉底式循环：当学习者直接询问答案时，策略提出引导性问题；当他们答对时，进入下一个概念；当他们卡住时，提供有支架的提示。**学习者模型**是贝叶斯知识追踪（或简单变体），每次交互后更新每个课程节点的掌握概率。**课程图谱**是一个 Neo4j，包含概念及其先决条件边；策略遍历图谱选择下一个概念。**记忆**是一个情景 + 语义存储（agentmemory 风格），保存过去的交互、错误和偏好。

UX 是多模态的。文本输入用于打字答案。通过 LiveKit + Whisper 的语音输入（重用综合项目 03）。通过 dots.ocr 或 PaliGemma 2 的拍照输入用于数学问题。通过 Cartesia Sonic-2 的语音输出。安全使用 Llama Guard 4 加一个年龄适宜过滤器（阻止成人内容、暴力、自残）和一个 COPPA 感知的记忆保留策略。

效果研究是交付产出。10 名学习者，前测和后测，两周时间。报告学习增益差异和置信区间。与非自适应基线（相同内容以线性方式交付，不采用导师策略）进行比较。

## 架构

```
学习者设备
  |
  +-- 文本         -> Web 应用
  +-- 语音         -> LiveKit Agents (ASR + TTS)
  +-- 拍照数学     -> dots.ocr / PaliGemma 2
       |
       v
  导师策略 (LangGraph)
       - 苏格拉底式决策头
       - 下一个概念选择器 (课程图谱遍历)
       - 提示支架器
       - 掌握度更新
       |
       v
  学习者模型 (BKT / 项目反应理论)
       - 每个概念的掌握概率
       - 间隔重复调度器 (SM-2 或 FSRS)
       |
       v
  记忆 (agentmemory 风格)
       - 情景: 每次交互
       - 语义: 学到的错误, 偏好
       - 保留策略: COPPA / GDPR 感知
       |
       v
  课程图谱 (Neo4j)

       - 关联的 OER 内容
       |
       v

    Llama Guard 4 + 年龄适宜过滤器
    按学习者 ID 作用域保护记忆访问
```

## 技术栈

- 科目选择：K-12 代数或 Python 入门（选择一个深入学习）
- 导师策略：基于 Claude Sonnet 4.7 的 LangGraph（带提示缓存）
- 学习者模型：贝叶斯知识追踪（经典）或用于间隔的 FSRS
- 课程图谱：Neo4j，包含概念 + 先决条件边 + OER 内容
- 记忆：agentmemory 风格的持久向量 + 情景 + 语义存储
- 语音：LiveKit Agents 1.0 + Cartesia Sonic-2（重用综合项目 03 子栈）
- 拍照数学：dots.ocr 或 PaliGemma 2 用于方程识别
- 安全：Llama Guard 4 + 自定义年龄适宜过滤器
- 评估：Bloom 级别问题生成、前/后测工具、效果研究工具

## 构建步骤

1. **课程图谱。** 构建 50-150 个概念节点的 Neo4j（例如，K-12 代数从"数轴"到"二次公式"），带有先决条件边。为每个节点附加 OER 内容（Open Textbook、OpenStax）。

2. **学习者模型。** 初始化贝叶斯知识追踪，带先验：猜测、失误、学习率。每次交互后更新每个概念的掌握度。为每个学习者持久化。

3. **导师策略。** LangGraph 节点：`read_signal`（学习者的答案是否正确/部分/卡住？）、`select_concept`（遍历课程图谱选择最高优先级概念）、`scaffold`（苏格拉底式提示）、`update_mastery`。

4. **记忆。** 每次交互写入情景存储。错误和偏好提升到语义记忆。COPPA 感知的保留策略：1 年后自动删除，家长可访问。

5. **语音路径。** 附加到导师策略的 LiveKit Agents 工作线程。ASR 通过 Whisper-v3-turbo。TTS 通过 Cartesia Sonic-2。支持打断（重用综合项目 03 机制）。

6. **拍照数学路径。** 上传或拍摄图像；运行 dots.ocr 或 PaliGemma 2 识别方程；作为结构化输入输入给导师。

7. **安全。** 每个模型输出通过 Llama Guard 4 + 年龄适宜过滤器（阻止自残、成人内容、暴力）。按学习者 ID 限定记忆访问范围；提供家长删除界面。

8. **效果研究。** 10 名学习者，前测（标准化 30 题基线），两周导师交互（每周 3 次），后测。与相同内容的非自适应基线队列（10 名学习者）进行比较。

9. **每周进度报告。** 每个学习者自动生成 PDF 摘要，涵盖已探索的主题、掌握轨迹和推荐下一步。

## 使用方式

```
学习者: "我不明白为什么 3x + 6 = 12 意味着 x = 2"
[signal]   卡住
[concept]  '分离变量' (先决条件: 加减法等式)
[scaffold] "你从两边减去什么数字来开始？"
学习者: "6"
[signal]   正确
[mastery]  加减法等式: 0.62 -> 0.77
[concept]  继续 '分离变量'
[scaffold] "很好。那么 3x / 3 等于什么？"
```

## 练习

1. 有和没有自适应学习者模型（随机概念顺序）的情况下进行效果研究。报告差异。预期自适应获胜，但差异的大小是有趣的数字。

2. 添加多模态探针：相同概念问题以文本、语音和照片形式交付。衡量学习者是否以他们偏好的模态更快地收敛。

3. 构建家长仪表盘：已练习的主题、掌握轨迹、即将学习的概念、安全事件（任何护栏命中）。符合 COPPA。

4. 添加语言切换模式：导师接受西班牙语输入并用西班牙语教学。衡量 X-Guard 覆盖。

5. 加压记忆隐私：验证学习者 A 即使通过语音片段重放攻击也无法看到学习者 B 的数据。记录尝试的访问并发出告警。

## 关键术语

| 术语 | 人们说的 | 实际含义 |
|------|---------|---------|
| 苏格拉底式策略 | "提问，不倾倒" | 导师提出引导性问题而不是直接给出答案 |
| 贝叶斯知识追踪 | "BKT" | 每个概念掌握概率的经典学习者模型方程 |
| FSRS | "自由间隔重复调度器" | 2024 年间隔重复调度器，优于 SM-2 |
| 课程图谱 | "概念 DAG" | 带有先决条件边的 Neo4j 概念图 |
| 情景记忆 | "每次交互日志" | 存储的每次交互，供后续检索 |
| 语义记忆 | "学习模式存储" | 从情景提升的压缩错误和偏好 |
| COPPA | "儿童隐私法" | 美国限制从 13 岁以下儿童收集数据的法律 |
