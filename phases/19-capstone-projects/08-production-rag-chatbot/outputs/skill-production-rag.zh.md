给定一个受监管领域的语料库（法律合同、临床试验方案、保险政策或类似），部署一个带有可验证引用、尊重角色和管辖区域访问策略并监控漂移的聊天机器人。

构建计划：

1. 使用 docling 或 Unstructured 解析语料库；将视觉丰富的文档通过 ColPali 路由。生成带有角色和管辖区域标签的块。
2. 稠密嵌入（Voyage-3 或 Nomic-embed-v2）到 pgvector + pgvectorscale；通过 Tantivy 的稀疏 BM25。
3. 接入 LangGraph 对话 Agent：检索（按角色+管辖区域过滤、混合稠密+BM25、倒数排名融合），重排序（bge-reranker-v2-gemma-2b 或 Voyage rerank-2），综合（带提示缓存的 Claude Sonnet 4.7）。
4. 组装带有稳定前缀的提示：系统前言 -> 策略块 -> 重排序上下文 -> 用户查询。目标 60-80% 提示缓存命中率。
5. 防护栏：Llama Guard 4 在输入和输出上，NeMo Guardrails v0.12 规则用于领域外和策略禁止的问题，Presidio PII 擦除在输出上，引用强制后置过滤器。
6. 构建 200 个专家标记的黄金集问答对（答案、引用）。在精确引用匹配、答案正确性、RAGAS 忠实度上评分。
7. 构建 50 个提示的红队（PAIR、TAP、PII 提取、领域外、跨管辖区域探测）。
8. Arize Phoenix 漂移仪表盘跟踪每周检索 nDCG 和引用忠实度；在 5% 下降时发出警报。
9. Langfuse 成本报告：提示缓存命中率、每次查询 token、按阶段的 $/query 分解。

评估标准：

| 权重 | 标准 | 衡量方式 |
|:-:|---|---|
| 25 | RAGAS 忠实度 + 答案相关性 | 200 问题黄金集上的在线得分 |
| 20 | 引用正确性 | 具有可验证源锚点的答案比例 |
| 20 | 防护栏覆盖率 | Llama Guard 4 通过率 + 越狱套件结果 |
| 20 | 成本/延迟工程 | 提示缓存命中率、p95 延迟、$/query |
| 15 | 漂移监控仪表盘 | 带每周检索质量趋势的 Phoenix 实时仪表盘 |

硬性拒绝：

- 任何泄露跨管辖区域数据的聊天机器人。角色+管辖区域过滤必须在检索前强制，而不是检索后。
- 破坏缓存前缀（在系统和上下文之间重新排序策略）的综合提示。将摧毁缓存经济性。
- 没有记录红队运行的防护配置。
- 没有引用的答案；没有可验证锚点的引用。

拒绝规则：

- 拒绝在没有对每个块进行管辖区域标签的情况下部署到受监管领域。
- 拒绝在专家标记的黄金集问题上训练检索。污染会破坏评估可信度。
- 拒绝在 README 中没有明确 SOC2/HIPAA/GDPR 适用性矩阵的情况下声称"合规"。

输出：一个包含摄入管道、LangGraph 对话 Agent、200 问题黄金集、50 提示红队、Phoenix 漂移仪表盘、Langfuse 成本仪表盘以及一份报告的仓库。报告命名你观察到的三大引用断裂模式以及每个的检索或提示修复。
