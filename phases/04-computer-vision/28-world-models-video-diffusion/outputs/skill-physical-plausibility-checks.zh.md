# 物理合理性检查

生成视频的生产部署需要自动化护栏。人工审核无法规模化；物理检查能捕获经典的失败模式。

## 何时使用

- 任何从文本或图像提示生成视频的产品。
- 自动化视频生成 API 端点的 QA。
- 监控视频模型在微调或基础模型更新后的质量漂移。

## 输入

- `video`：张量 `(T, H, W, 3)` 或 mp4 文件路径。
- 可选的参考信息：期望的物体数量、初始场景描述。

## 检查项

### 1. 物体恒存性
使用 SAM 3.1 Object Multiplex 跨帧追踪每个检测。当稳定轨迹在 <=3 帧内消失然后重新出现时标记——模型暂时丢失了物体。当物体在帧中心附近（而非边缘）消失时硬性失败；在边缘时软性失败。

### 2. 运动平滑度
连续帧之间的光流应大致连续。突然的逐像素流峰值表示瞬移。使用 RAFT 计算光流；标记第 99 百分位流幅值超过中位数 10 倍的帧。

### 3. 重力 / 支撑
对于检测为固体（食物、球、工具）的物体，检查在无提升动作的情况下其垂直位置是否非递增。除非在物体附近检测到"抓取的手"，否则标记向上的漂移。

### 4. 身份一致性
对于人或角色，使用跨帧的人脸识别嵌入。余弦相似度应在 5 帧窗口内保持 > 0.8 以实现持续身份。低于阈值意味着角色发生了变形。

### 5. 手和肢体
运行姿态估计器（第 21 课）。标记手指可见数量 > 5 或 < 4 的帧；手臂长度在帧间翻倍的帧；肢体穿过表面与身体交叉的帧。

### 6. 文本渲染（如果提示要求文本）
如果用户提示包含引号中的字符串，对生成帧进行 OCR 并计算与请求字符串之间的 CER。标记 > 20% CER 的情况。

## 报告

```
[plausibility]
  video frames:           <T>
  permanence violations:  <N>
  smoothness violations:  <N>
  gravity violations:     <N>
  identity drift:         <N 个 5 帧窗口>
  limb anomalies:         <N>
  OCR CER vs requested:   <浮点数>

[verdict]
  ship | hold | reject

[samples for review]
  每个失败发生的帧范围
```

## 规则

- 不要在任何单个检查上硬阻断；聚合得分并在总异常超过阈值时将视频搁置供审核。
- 对身份漂移和恒存性违规赋予最高权重——用户首先注意到它们。
- 随时间记录每个检查的失败率；上升趋势通常意味着基础模型已更新或提示分布发生了变化。
- 绝不要删除被标记的视频；保留它用于模型调试和事后分析。
- 对于敏感内容（人物、儿童、公众人物），无论得分如何都要求对每个视频进行人工审核。
