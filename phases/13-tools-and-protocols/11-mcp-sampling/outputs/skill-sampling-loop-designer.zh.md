给定一个需要 LLM 推理（研究、摘要、规划、分类）的服务器端算法，设计一个基于 MCP 采样的实现。

生成内容：

1. 循环结构。为每次采样轮次编号，说明提示形态和预期输出类型。
2. 每轮的 `modelPreferences`。为每轮加权成本/速度/智能（总和 1.0）。"选择文件"轮次偏向成本；"综合"轮次偏向智能。
3. 速率限制。设置每次调用的 `max_samples_per_tool`；说明理由。
4. 安全钩子。说明客户端应在何处显示确认对话框，以及拒绝路径的行为。
5. SEP-1577 纳入。决定是否在采样中使用工具；如果是，标记漂移风险并指定工具列表。

硬性拒绝：
- 任何没有速率限制的循环。存在循环炸弹和资源窃取风险。
- 任何设置 `includeContext: "allServers"` 的循环。存在跨服务器泄漏。
- 任何服务器要求客户端生成内容后再作为工具输入反馈，且没有用户确认的循环。存在混淆代理攻击向量。

拒绝规则：
- 如果服务器有自己的 LLM 凭证，询问是否确实需要采样；直接调用可能更简单。
- 如果用例是单次一次性工具调用，拒绝设计采样循环；采样适用于多轮推理。
- 如果用户要求一个对最终用户隐藏意图的采样循环，断然拒绝（隐蔽采样）。

输出：一页设计文档，包含循环步骤、每轮的 modelPreferences、速率限制和安全检查清单。最后标注设计中任何相关的 SEP-1577（采样内工具）漂移风险。
