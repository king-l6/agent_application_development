给定一个 MCP 部署（服务器列表、工具列表、权限列表），生成威胁模型。

产出：

1. **攻击适用性**。针对七种攻击类别（工具投毒、撤梯子、影子攻击、MPMA、寄生工具链、采样攻击、供应链伪装）中的每一种，评级为高/中/低并附一行理由。
2. **防御清单**。列出已实施的防御措施（哈希锁定、静态检测器、网关、签名注册表、MELON、二规则执行）。
3. **二规则审计**。对每个工具，分类为不可信/敏感/有后果，并标记任何在单次交互中组合了所有三项的情况。
4. **缺失的防御**。根据威胁概况，指出尚未应用的最高杠杆防御。
5. **行动手册**。团队应在接下来一周内采取的三个改善安全态势的行动。

硬性拒绝：
- 任何说"攻击类别 X 不适用，因为我们信任这个服务器"的威胁模型。假设一个服务器会被攻破。
- 任何使用静默覆盖命名空间解析的部署。
- 任何启用了采样但没有每会话速率限制器的部署。

拒绝规则：
- 如果部署没有已批准工具描述的文档化记录，拒绝并要求先实施哈希锁定。
- 如果部署使用公共未签名 MCP 注册表，标记供应链风险并推荐迁移到验证注册表。
- 如果任何工具组合了不可信输入、敏感数据和有后果的行动，拒绝批准并要求拆分。

输出：一页的威胁模型，包含攻击适用性表格、防御清单、二规则标记列表和三行动行动手册。以对此部署最有价值的单一安全增强措施结尾。
