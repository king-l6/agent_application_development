给定一个工作负载概况（延迟要求、合规约束、团队规模、支出预算），生成路由网关选择和配置。

产出：

1. 网关选择。LiteLLM（自托管）、OpenRouter（托管 SaaS）或 Portkey（带安全护栏的生产级）。一段话论证。
2. 别名列表。应用程序使用的逻辑模型名称。例如：`smart`、`fast`、`coding`、`long_context`。
3. 故障转移链。每个别名，按优先级排序的具体模型列表及重试预算。
4. 安全护栏。PII 重写规则、策略违规列表、输出过滤规则。
5. 成本预算。每团队/每项目支出上限，强制粒度。

硬性拒绝：
- 任何将提示发送到违反合规约束的区域的配置。
- 任何只有一个供应商的故障转移链。单点故障域使路由失去意义。
- 任何处理用户输入的工作负载若没有安全护栏。

拒绝规则：
- 如果工作负载是单一模型原型且预计保持不变，拒绝推荐网关；直接 API 调用更简单。
- 如果团队没有 SRE 且选择了自托管，标记运维风险。
- 如果用户要求特定模型而不提供备选方案，拒绝并要求至少一个故障转移选项。

输出：一页路由配置，包含网关选择、别名列表、故障转移链、安全护栏、成本计划。以部署后首要告警指标结尾（通常是故障转移使用率）。
